# CLAUDE.md

このプロジェクトは iOS/SwiftUI の 48週×1年 漫才師コンビ育成SLG の MVP。

## 絶対ルール

1. ゲームロジックは SwiftUI 非依存の純 Swift（GameCore）に置き、UI と完全分離する
2. 乱数はシード固定可能なものを注入する
3. バランス数値は全て GameConfig に集約し、全て仮とする
4. 大会・芸人・コンビ・事務所・審査員などの固有名は全て架空にする
5. 数式を変えたら tools 内の Python 検証機も同期する

## 実装順

週送り → パラメータ → 稽古 → バイト・休む → 生活費・知名度・オファー → 大会 → 突発イベント → リザルト・デバッグ

## 進め方

- 質問は1論点ずつ
- 選択肢には推奨理由を添える
- 後戻り困難な判断は【設計上の懸念】と明記する

## セッション運用ルール（共通・2026-07-05確定）

デスクトップ版／CLI／スマホのClaudeアプリを併用するため、ブランチ衝突（fetch first・rebase多発）を防ぐ共通ルール。全セッション必ず従う。

1. **アクティブに編集・pushするセッションは常に1つだけ**。作業ブランチの「持ち主」を1セッションに限定する。
2. **作業開始前に `git pull --rebase`**、**区切りごとに `git push`**（こまめに）。
3. **セッションを切り替える前に、必ず今のセッションでpushして手を離す**。切替先はpullしてから始める。
4. 並行させたい作業は**別ブランチ**を切り、後でまとめる（同一ブランチの同時編集は禁止）。

## 開発規律（2026-07-05確定・全セッション必須）

### A. 正典同期（最重要・壊すと全崩れ）
1. 数式や乱数消費順を変えたら、**①`tools/*.py`（正典）→②`GameCore`（Swift）→③`cd tools && python3 gen_golden.py` でgolden再生成→④`cd GameCore && swift test` green** を**1コミットで揃える**（ルール5の実務手順）。
2. **乱数の「消費順」も数式と同格の正典**。順序が1箇所ズレるだけで3年ビット一致（golden）が壊れる。ロジック追加時は消費順を runYear/WeekRunner と一致させる。
3. **goldenの期待値を手で書き換えない**（必ず再生成器から）。**`swift test` が赤いままコミットしない**。

### B. バランス数値
4. バランス数値は `GameConfig`／`sim_career.py` にのみ置く（散らさない）。変更時は**根拠（simの実測値 or docの§）をコミットに添える**。全て【仮】。
5. **体感の問題を数値で直さない**（ui_design §7-B）。まず演出・頻度・テキストで解き、バランスは最後。

### C. ドキュメント
6. 設計を上書きする時は古い記述を消さず**「正典vN移行」バナー**で履歴を残す。数値の正本は `canonical_vN_spec.md` 一箇所。
7. 新しい設計判断はdocを1枚足し、`docs/START_HERE.md` の索引に1行追加。
8. **オーナー判断待ち**（業態・裏天井など）はセッション側で勝手に確定しない（START_HERE の該当節）。

### D. アプリ（UI・GameCore外）
9. アプリに新規Swiftファイルを足したら `cd ManzaiGame && xcodegen generate`（`project.yml`が正本・`pbxproj`は手編集しない）。
10. UI変更は `swift test` では検証できない。**simulatorでビルド→起動→目視**まで確認して「完了」とする。

### E. コミット／秘密
11. 1コミット=1目的。メッセージは「何を・なぜ（§参照）」。末尾に Co-Authored トレーラ。
12. トークン等の秘密はチャット/スクショに貼らない。ターミナルの入力プロンプトにのみ入れる。露出したら即revoke。
