# jinro_ai — 人狼プレイヤーAIの土台

人狼（人狼ジャッジメント系ルール）を**AI自身が役職に応じて判断し、勝ちにいく**ための
土台。補助ツールではなく「プレイヤーとして指す頭脳」を目指す。強さは self-play の反復と
定石リサーチで上げていく。

> 固有名・大会名は使わない（このリポジトリ方針）。プレイヤーは `P0, P1, …` の匿名。

## 今できること（MVP・動作確認済み）

- 決定論エンジン（seed 固定で完全再現。self-play 可能）
- 標準6役: 村人 / 占い師 / 霊媒師 / 狩人 / 人狼 / 狂人
- 進行: 初日占い → 昼(議論=CO・占霊申告・投票 → 処刑) → 夜(占い・護衛・襲撃) → 勝敗判定
- 差し替え可能な頭脳（`Agent`）: `RandomAgent`（基準線）/ `HeuristicAgent`（定石ベース）
- self-play で陣営別勝率を計測

```
python3 -m jinro_ai.selfplay
```

### 計測例（N=2000, 標準9人 = 占1霊1狩1村3狼2狂1）

| マッチアップ | 村勝率 | 狼勝率 |
|---|---|---|
| 村=random / 狼=random | 28.6% | 71.4% |
| 村=heuristic / 狼=random | 55.6% | 44.4% |
| 村=random / 狼=heuristic | 20.8% | 79.2% |
| 村=heuristic / 狼=heuristic | 20.9% | 79.0% |

片側を固定してもう片側の頭脳を上げると勝率が動く＝**エンジンが強さを測れている**。
現状は両者定石だと狼優位。村側の推論を鍛えるのが次の伸びしろ。

## 構成

```
jinro_ai/
  roles.py     役職メタデータ（陣営・占い/霊媒結果・夜行動）。拡張はここに1行足す
  game.py      進行エンジン（決定論・UI/自然言語 非依存）。合法手と情報の流れだけ司る
  agents.py    頭脳の差し替え口（day/divine/guard/attack）。Random と Heuristic
  selfplay.py  陣営別マッチアップで勝率計測
```

設計上の分離（このリポジトリの GameCore 流儀を踏襲）:
- **エンジン = 純ロジック・決定論**。乱数は `random.Random(seed)` を注入。
- **頭脳 = 差し替え可能**。エンジンは「口(Agent)」だけ知り、中身の賢さに依存しない。
- **自然言語の発言生成は未実装**。昼は今 `{CO, 占霊申告, 投票}` の抽象議論。発言テキストは
  後段（下記②のブレイン層）で生成する。この分離により、頭脳を替えても進行は壊れない。

## これから（ロードマップ）

1. **信念/確率エンジン** … 公開情報（CO・占霊結果・投票・死亡）から「あり得る配役の世界」を
   数え、各人の人狼/第三陣営っぽさを数値化。村側頭脳の中核。
2. **強い村側ポリシー** … 占い騙りの真偽詰め、グレラン、確定白の活用。両者定石で村が勝てる所まで。
3. **頭脳の主軸を決める（下記オープン論点）**。
4. **自然言語の発言生成**（「返答を自動で入力」の“返答”本体）。
5. 役職拡張（妖狐・共有者・大狼・囁き狂人 …／第三陣営と勝利条件）。
6. （後回し）実アプリへの入力連携。

## オープン論点（オーナー判断待ち）

### 頭脳の主軸（②で確定する）
- **①純アルゴリズム**（確率＋定石）: オフラインで大量 self-play → 数値で強くする。決定論で
  テストしやすく、API不要・高速。発言は定型寄りになりがち。
- **②Claude ブレイン**: ロジックコアで地固めしつつ、推論と自然な発言を Claude が担う。
  発言が自然で柔軟。非決定・APIコスト・golden化しにくい。
- 現実解は**ハイブリッド**（コアで確率を出し、発言/最終判断を②が担う）。どちらでも土台
  （エンジン＋確率コア）は共通なので、コアを先に完成させてから確定する。

### 自動入力（後回し・注意点）
オンライン実対戦への自動投稿は規約違反・他プレイヤーに不公平になりやすい。当面は
「AIが返答を生成 → 人が使う」＋ self-play/対CPU で強くする方向に限定する。
