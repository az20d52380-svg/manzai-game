# jinro_ai — 人狼プレイヤーAIの土台

人狼（人狼ジャッジメント系ルール）を**AI自身が役職に応じて判断し、勝ちにいく**ための
土台。補助ツールではなく「プレイヤーとして指す頭脳」を目指す。強さは self-play の反復と
定石リサーチで上げていく。

> 固有名・大会名は使わない（このリポジトリ方針）。プレイヤーは `P0, P1, …` の匿名。

## 今できること（MVP・動作確認済み）

- 決定論エンジン（seed 固定で完全再現。self-play 可能）
- 標準6役: 村人 / 占い師 / 霊媒師 / 狩人 / 人狼 / 狂人
- 進行: 初日占い → 昼(議論=CO・占霊申告・投票 → 処刑) → 夜(占い・護衛・襲撃) → 勝敗判定
- 昼は2フェーズ（①発言=CO/占霊申告 → ②全発言が出そろってから投票）。実戦の議論→投票順
- 差し替え可能な頭脳（`Agent`）:
  - `RandomAgent` … 合法ランダム（基準線）
  - `HeuristicAgent` … 定石ベース（占いの黒を吊る／占い師護衛／狼は騙り＆占い噛み）
  - `BeliefAgent` … 村側の投票を**信念/確率エンジン**（あり得る配役の数え上げ）で強化
  - `LLMAgent` … **Claude ブレイン**（CICERO流ハイブリッド）。信念エンジンで地固めしつつ
    推論と自然な日本語の発言/騙り/投票を Claude が担う。キー無しは BeliefAgent＋テンプレ発言で動く
- 昼に**自然言語の発言(talk)チャンネル**（相手も投票時に読める）
- self-play で陣営別勝率を計測

```
python3 -m jinro_ai.selfplay          # 陣営別勝率
python3 -m jinro_ai.learn             # self-play学習ループ（回して強くする）
python3 -m jinro_ai.demo_llm          # LLMブレインのデモ（発言つき実況＋プロンプト実例）
# 本物のClaude発言:  ANTHROPIC_API_KEY=sk-... python3 -m jinro_ai.demo_llm --api
```

### 「やってくうちに強くなる」を実測（`learn.py`）
村側の投票ポリシーを数個のθ（黒への上乗せ・確率を信じる閾値…）で表し、勝率を目的に
(1+λ)進化戦略で自己対戦最適化。狼は固定・train/held-out を分けて汎化を確認。

| | train 250戦 | **held-out 500戦** |
|---|---|---|
| baseline θ | 41.6% | 40.0% |
| **学習後 θ** | 51.6% | **51.8%** |

**held-out で +11.8pt**（過適合でなく汎化）。しかも「黒指定への上乗せ `black_prior` は 0 が最適」と
自己対戦が自動発見——信念確率が既に騙りを織り込むので手動の上乗せは二重計上、という**手で当てるより
強い**発見。これが「回すほど強くなる」本体。

### 計測例（N=2000, 標準9人 = 占1霊1狩1村3狼2狂1）

| マッチアップ | 村勝率 | 狼勝率 |
|---|---|---|
| 村=random / 狼=random | 28.6% | 71.4% |
| 村=heuristic / 狼=random | 62.4% | 37.6% |
| 村=random / 狼=heuristic | 19.5% | 80.5% |
| 村=heuristic / 狼=heuristic | 34.6% | 65.4% |
| **村=belief / 狼=heuristic** | **40.2%** | 59.8% |
| 村=belief攻め / 狼=heuristic | 58.4% | 41.6% |

片側を固定してもう片側の頭脳を上げると勝率が動く＝**エンジンが“強さ”を測れている**（＝
self-play が評価関数として機能）。信念エンジンは定石に対し**対random狼で無劣化・対騙り狼で
改善**（既定は安全側）。「攻めモード」は騙り狼に大きく勝つが正直狼に落ちる——この閾値の
最適点を手で当てるのは不毛で、**self-play 学習でチューニングすべき対象**（次フェーズ）。

## 構成

```
jinro_ai/
  roles.py     役職メタデータ（陣営・占い/霊媒結果・夜行動）。拡張はここに1行足す
  game.py      進行エンジン（決定論・UI/自然言語 非依存）。合法手と情報の流れだけ司る
  belief.py    信念/確率エンジン（公開情報→各人の人狼確率）。純関数・数え上げ
  agents.py    頭脳の差し替え口（statement/vote/divine/guard/attack）＋Random/Heuristic/Belief
  llm_agent.py Claudeブレイン（信念エンジンで地固め・構造化出力・キー無しはスタブで動く）
  selfplay.py  陣営別マッチアップで勝率計測
  learn.py     self-play学習ループ（村側θを進化戦略で最適化・train/held-out検証）
  demo_llm.py  1試合を発言つきで実況＋Claudeに渡すプロンプト実例を表示
```

設計上の分離（このリポジトリの GameCore 流儀を踏襲）:
- **エンジン = 純ロジック・決定論**。乱数は `random.Random(seed)` を注入。数式に依存せずテスト可能。
- **頭脳 = 差し替え可能**。エンジンは「口(Agent)」だけ知り、中身の賢さに依存しない。
- **ハイブリッド**（CICERO流）: 決定論の信念エンジンが人狼確率で地固め（計算ミス防止）し、
  その上で LLMAgent の Claude が自然言語の推論・発言・騙りを担う。ロジック核は決定論のまま、
  非決定なのはブレイン層だけ——頭脳を替えても進行は壊れない。

## 決定した設計判断

### 頭脳の主軸（確定）
**ハイブリッド**に確定（オーナー方針「とにかく最強・将棋DL的・課金OK」を受けて）。決定論の
信念エンジンで地固めし、Claude が推論・自然発言・騙りを担う。純AlphaZero（完全情報MCTS）は
人狼（不完全情報＋隠れ役職＋自然言語の説得）に直接は効かず、CICERO系（戦略エンジン＋LLM対話）
が実証済みの最強レシピ。self-play は"強さの評価関数"として全フェーズで使う。

### 自動入力（後回し・注意点）
オンライン実対戦への自動投稿は規約違反・他プレイヤーに不公平になりやすい。当面は
「AIが返答を生成 → 人が使う」＋ self-play/対CPU で強くする方向に限定する。
