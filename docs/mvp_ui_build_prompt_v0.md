# MVP UI実装＋コンテンツ配線 指示文 v0 — CLI(Mac)向け・コピペ即用

> 使い方: Mac上のClaude Code（CLI）セッションで、下の三連バッククォート内をそのまま最初のメッセージに貼る。ブランチは `claude/manzai-slg-foundation-wprqlu`。目的＝1年MVPを「実際に遊んで楽しいレベル」まで引き上げる（パワプロ サクセス風UI＋谷口の会話＋大会＋講評の配線）。
>
> 前提: 見た目の正典は `docs/ui_mockup_pawapuro_v2.html`（ブラウザで動く仮モック）。中身の実テキストは `dialogue_design`/`judge`/`finals_direction` にある。数値表示は「出す派（パワプロ式）」で確定（旧「隠す」を意図的にひっくり返し）。

```
あなたはMac上のClaude Code（CLI）。iOS/SwiftUIの漫才師コンビ育成SLG「manzai-game」の【1年MVPを"実際に遊んで楽しいレベル"まで引き上げる】のが今回の任務です。バランスsimは完成済み（swift test 17 green・Python⇔Swiftビット一致）ですが、UIは骨組みだけでコンテンツが1文字も載っておらず「表計算」状態。これを【パワプロ サクセス風の育成画面 ＋ 谷口の会話 ＋ 大会演出 ＋ 紙講評】を配線して"漫才師の1年"にしてください。ゴールは検証①「タップだけの週48手が単調にならず、初見の人が10分触って"ゲームだ"と感じる」を通すこと。

■最初に必ず読む（この順で）
1. `docs/ui_mockup_pawapuro_v2.html` … 【今回の見た目の正典】。`open docs/ui_mockup_pawapuro_v2.html` でブラウザで見て、この配色・丸ゴシック・レイアウト・4画面（育成メイン/大会/講評/決勝）を正としてSwiftUIで再現する。
2. `docs/ui_design_v0.md` … 画面遷移・週の3タップ・成長上限の可視化。
3. `docs/dialogue_design_v0.md` … 谷口の声・相性帯4段の掛け合い・週次フレーバー（【会話の実テキストはここ】）。
4. `docs/judge_design_v0.md` と `docs/judge_comments_v1.md` … 審査員の紙講評（軸・帯別コメント）。
5. `docs/finals_direction_v0.md` … 大会/決勝の演出順序（笑い波形→通過敗退→講評→順位/優勝者指名）。
6. `docs/master_spec_v2.md` §1/§6 … 全体像・能力（センス/発想/表現/華/メンタル ＋ 体力/相性/知名度/所持金）。
7. 現状コード: `ManzaiGame/Sources/`（UI骨組み・`GameSession.swift`=year1固定/MVP1年完結）、`GameCore/Sources/GameCore/`（数値エンジン・WeekRunner/GameConfig）。

■絶対ルール（CLAUDE.md準拠・破ると全崩れ）
1. 作業前に `git pull --rebase origin claude/manzai-slg-foundation-wprqlu`。区切りごと `git push`。このブランチで作業。
2. ゲームロジックはGameCore（純Swift・SwiftUI非依存）、UIはManzaiGame。【分離を崩さない】＝UIはGameCoreを呼ぶだけ。
3. 【golden/数式/乱数消費順に触らない】（GameCoreの成長・スコア・run系の中核／`gen_golden.py`／`tools/`のsim中核）。今回はUI＋コンテンツ配線のみ＝goldenは無傷のはず。もし数式・乱数消費順に触れる必要が出たら【止めて相談】（規律A：正典同期は tools→GameCore→gen_golden→swift test を1コミットで揃える手順が要る）。UIに必要な値がGameCoreから取れない時は【読み取り専用のgetter/公開プロパティを足すのは可】（数式・draw回数は変えない）。
4. 新規Swiftファイルを足したら `cd ManzaiGame && xcodegen generate`（`project.yml`が正本・`pbxproj`は手編集しない）。
5. 固有名は全て架空（大会名・審査員・ライバル・相方）。バランス数値は全て【仮】でGameConfigに集約。既存の架空名（谷口・音羽 巴・頂グランプリ・静物画/金字塔 等）を踏襲。
6. 【UIは`swift test`では検証できない】。simulatorでビルド→起動→目視→"1年を通しで遊べる"まで確認して初めて「完了」（規律D-10）。ビルドは `cd ManzaiGame && xcodegen generate && xcodebuild -scheme ManzaiGame -destination 'platform=iOS Simulator,name=iPhone 15' build` 等、起動して実際に何週かプレイする。
7. コミットは1目的・メッセージは「何を・なぜ（§参照）」・末尾にCo-Authoredトレーラ。

■作るもの（見た目＝mockup / 中身＝docs）
● 画面（mockupの SCREEN 01-04 を正とする）
- 育成メイン：上＝年月週・やる気の顔・体力バー・目標／中＝キャラ（立ち絵は仮でよい＝シルエット/簡易図形/SF Symbols可）＋【谷口のアドバイスコメント】／能力＝6つ（センス/発想/表現/華/メンタル/相性）を色分けバー＋ランク常時表示／下＝練習コマンドのタイル（各コマンドがどの能力を伸ばすか色ドット＋体力/金コスト）＋「つぎへ」ボタン。
- 大会本番：笑い波形メーター（アニメ可）→通過/敗退スタンプ。
- 紙講評：半紙カードに審査員の1行講評＋星（能力帯連動）。
- 決勝/年末結果：順位ボード＋（負けたら）誰に負けたか＋才能解放の灯り。※1年MVPで決勝到達は稀なので【年末の結果画面】として実装（到達段階に応じて。決勝まで行けたら決勝演出、行けなければ到達回戦の結果でよい）。

● 数値表示の方針【確定】
- 能力値は常時表示。各稽古で伸びる能力も【色ドット＋おおよその量を表示】（パワプロ サクセス式）。※これは旧「効果を隠す」設計を意図的にひっくり返した確定判断（オーナー承認済み）。ミステリーは数値でなく【イベント・やる気・運】に持たせる。

● コンテンツ配線（"楽しめる"の肝）
- 谷口の会話：`dialogue_design_v0.md` の実テキストを使う。①相性帯（0-7/8-14/15-20/21+）で稽古後の掛け合いが変わる ②各行動（ネタ作り/ネタ合わせ/舞台稽古/バイト/休む/相方と過ごす）に専用フレーバー各2-3種ローテ ③条件差し替え（所持金<5万で「腹減った」系、体力<20で専用）。アドバイスコメント欄に出す。1画面2-4行・スキップ可・義務にしない（会話はご褒美）。
- 審査員講評：大会後に能力帯に応じた1行講評（`judge_comments_v1.md` の帯別）。褒めるだけにせず必ず"次の一点"を指す。
- 大会：笑い波形→通過/敗退→紙講評 の順（`finals_direction_v0.md`）。判定ロジック（絶対評価ライン）は変えず表示だけ。
- 1周内で反復に見えないよう各カテゴリ最低3種はローテ（docに在庫あり）。テキストはSwiftのデータ構造（配列/enum）に落とし、GameConfig的な1箇所にまとめる。

■スコープ（1年MVP・盛りすぎない）
- やる：48週の週送り、上記4画面、谷口会話・大会・講評の配線、年末結果、セーブは無くてよい（1周完結）。
- やらない（今回は）：トロフィー/相方ガチャ/10年周回/事務所移籍/イベント38種フル/コンプ図鑑/裏天井/ラストイヤー緩和。※MVPの目的は「1年の手触りが楽しいか」の検証（検証①）に絞る。
- キャラ立ち絵・アイコンは仮でよい。凝りすぎない（素材づくりに沈まない）。

■完了の定義（"楽しめるレベル"のバー・simulatorで実際に遊んで満たす）
1. ビルド成功・クラッシュなし・1年48週を最後まで通しでプレイできる。
2. 見た目がmockup（パワプロ風）に一致（丸ゴシック・ポップ配色・練習タイル・キャラ＋アドバイス・つぎへ）。
3. 【行動を選ぶたび谷口が違うことを喋る】（相性帯でも変わる）＝「会話がない」が解消。
4. 少なくとも1回の大会で 笑い波形→結果→紙講評 が流れる。
5. 初見の人が10分触って「表計算」でなく「ゲーム」と感じる手触り（自分で目視判断）。
6. スクショを数枚撮って残し、commit＋push。

■進め方・詰まったら
- 大きく作らず、①育成メインを見た目通り → ②谷口会話を配線 → ③大会＋講評 → ④年末結果、の順で各段階ビルドして目視。
- 素材が無い所（立ち絵等）は仮で埋めTODOコメントを残す。
- 数式・乱数に触れる必要が出たら【必ず止めて相談】（golden破壊は期限内に検証不能）。

まず `docs/ui_mockup_pawapuro_v2.html` をブラウザで見て見た目を掴み、`ManzaiGame/Sources` の現状を読んでから着手。1年を通しで楽しく遊べる状態を simulator 目視で確認して「完了」とすること。時間が足りなければ、育成メイン＋谷口会話＋1大会＋講評（＝完了定義1〜4）を最優先で通すこと。
```
