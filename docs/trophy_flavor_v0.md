# トロフィー獲得演出・フレーバー文 v0 — 全22個＋実績デザイン則

- 日付: 2026-07-04
- 位置づけ: `trophy_design_v1.md`（配点・条件の正本）の全トロフィーに、獲得瞬間の演出文（1〜2行）と楽屋名鑑の図鑑説明文を付ける。固有名は架空、数値は全て【仮】。

## 0. リサーチ — 実績デザインの傾向から採る規則

インディー開発者のコミュニティ調査記事・実績文化の分析記事・秀逸実績のまとめを調査。採用する設計則:

1. **進行で自然に解除される実績が最良**（「チャプター1クリア」型）——本作の回戦到達系はこれ。逆に「作業を強いる収集系」は嫌われる → 「稽古100回」等は普通に遊べば届く水準に置く（届かない収集ノルマは作らない）
2. **解除条件は説明文で分かるように**（秘密実績はネタバレ回避目的に限定）→ 本作は§2の「?」表示は条件ヒントを併記する
3. **失敗・事故系のユーモア実績が愛される**（クラッシュで解除される「不本意な品質保証」、不戦勝の「ハートブレイカー」等）→ 本作のバッドエンド枠（十年選手・雨天決行）はこの系譜。**負けの記念品こそ名文にする**
4. 名前は「短い日本語＋掛詞」を基本に、説明文で泣かせる二段構え（名前で笑わせ、説明で刺す）

## 1. 演出の型

- 獲得瞬間: 画面下からトロフィーカードがスッと出る（0.8秒・SE1種・ハプティクス1回）。**カード表: 名称＋一言（下表の「獲得文」）**
- 楽屋名鑑の実績ページ: カード裏に**図鑑文**（獲得後にいつでも読める・未獲得は条件ヒントのみ）
- 才能ptの加算は周回開始時にまとめて演出（trophy_design §4-4と整合）:「二人の限界が、少し先へ動いた」

## 3. 称号（0pt・15種）の扱い

称号側（trophy v0）は「獲得文なし・名鑑に静かに追加」——トロフィーとの格差演出（カードが出るのはptつきだけ）。図鑑文はv0の条件文をそのまま使い、本ドキュメントの改訂対象外とする【仮】。

## 4. 実装メモ

- 文言はUI層リソース（GameCoreは獲得判定IDのみ）。judge/dialogueと同じ分離
- 「?」未獲得表示の条件ヒント例: #21→「12月の夜、諦めなかった者へ」（直接条件を書かず方向だけ示す。§0-2とネタバレ回避の折衷）
- 獲得文はリザルト画面ではなく**獲得の瞬間**に出す（週送りを止めない0.8秒）。同週複数獲得時はカードを重ねて1回で見せる

### 5-E. 実装メモ（本節ぶん）

- S-系列も §4 と同様、文言はUI層リソース。GameCoreは達成判定IDのみ保持（判定は `AchievementTracker` のフラグ蓄積に相乗り）。
- 称号はカード演出なし＝名鑑への静かな追記のみ（§3の格差演出を維持）。未達成時は条件ヒントを名鑑に併記（§0-2）。
- S-04 は谷口の一言つき。S-20 は谷口を主題にした**主人公の地の文**（谷口クオートではない・語尾は標準語寄り）。いずれも `dialogue` リソースと重複させず名鑑内クオートとして持つ。谷口の一言は関西弁・柔らかいトーンを維持。
- 【設計上の懸念】称号を将来DLC等で増やす場合も本S-系列に続けて採番し、30pt枠（トロフィー）側は不変（`trophy_design_v1.md`§4-1）。
