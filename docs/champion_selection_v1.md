## 0. 用語 —— 役柄テンプレと表示名の分離（方針5の核）

**役柄テンプレ（Role）は不変の識別子。表示名（Name）は周回ごとに割り当てる使い捨てのラベル。** プレイヤーは表示名しか見ない。役柄がどれかは内部情報。

> 以降で使う「金字塔／ミラーボール／静物画／夜行列車」は**本書内での役柄の通称**（テンプレを指すコードネーム）であり、**ゲーム内の表示名ではない**。ゲーム内の表示名は毎周 `name_generator_v0.md` の `genDuoName()` が `runSeed` から生成する（§3）。旧固定名は今後ゲームに出ない。

### 0-1. 役柄テンプレ（3つの絆色＋背景1＋モブ）

**オーナーの関係タイプは 同期／可愛がってた後輩／飲みに行く先輩／知らない組 の4つ。うち会話で絆を育てる"色付き"は3つ**（同期・後輩・先輩）。旧4組はこの3色＋背景1に割り付ける（4組すべて役柄テンプレ化＝性格・物語・年表・関係イベントは存続、名前だけ毎周ランダム）。

| Role ID | 通称（旧由来） | 関係色 | 声（§0-3） | 物語（存続・rival §1） | GP優勝の資格年（§4-2） |
|---|---|---|---|---|---|
| `.douki` | 先に行く同期（旧金字塔） | **同期** | 標準語・慇懃で上から、根は熱い | 嫌味→会釈→本音の3段（rival §1①） | **4・5・6年目** |
| `.kohai` | 可愛がってた後輩（旧ミラーボール＋育てた後輩） | **可愛がってた後輩** | 敬語ベースの砕け・軽薄そうで底に本音 | ネタ見せに来て育つ→一度バズって天狗→消える→頭を下げて劇場へ帰還→返り咲き（rival §1③＋`batch3`後輩を統合） | **7年目** |
| `.senpaiGenius` | 飲みに行く先輩（旧静物画・準決の魔物） | **飲みに行く先輩** | 関西弁・枯れた苦労人・一言が重い | 準決の壁に何年も愛される→型を変え初決勝→頂点／奢ってくれる先輩（rival §1④＋奢り先輩を統合） | **9・10年目** |
| `.veteran` | 終電のベテラン（旧夜行列車） | **（背景のみ・無色）** | 関西弁・枯れ | 資格ラストで敗者復活散→ベテラン大会へ移籍（rival §1②） | **なし（GP恒久除外）** |

- **「資格年」**＝その役柄が**GP決勝の優勝者として表示され得る年**。窓の外では優勝者候補に入らない（谷の年扱い＝モブ or 他役柄）。窓の中でも当選は確率的（§4）。旧v0「年別に優勝を決め打ち」は廃止。
- **`.veteran`（夜行列車）はGP優勝に出ない**【2026-07-05・A2オーナー確定】。物語（終電で散る苦労人）がGP優勝と噛み合わないため、GP優勝は同じ「飲みに行く先輩」色の `.senpaiGenius`（技巧派）が担う。`.veteran` は年末の業界ニュース欄で**ベテラン大会**の優勝者として別途1行だけ登場（`rival_design §2`「今年の業界ニュース」）。関係イベント（絆）は持たない。

### 0-2c. キャラクター多様性の原則（オーナー確定・2026-07-05）

- **登場人物（相方・ライバル・審査員・モブ）の性別は自然に多様に混ぜる**（＝男性だけにしない）。役柄テンプレ（同期/後輩/先輩/知らない組）は**どれも性別を問わない**（男性・女性・男女コンビのいずれもあり得る）。
- **【禁止】性別を勝敗・悔しさ・煽りの演出装置にしない。** 「女に負けた」「女性コンビに追い抜かれた」のような**性別を強調する敗北フレーミングは作らない**（オーナー指示「今の時代まずい」）。
- ゆえに §4-5 フレーバー・§4-6 反応セリフ・`relationship_champions §4` は**すべて性別非依存**（勝者が女性でも男性でも同じ言葉が成立する）。コンビ名（`name_generator`）も性別を示さない。役柄の声（§0-3）はキャラ性であって性別記号ではない。
- 実装: 役柄・モブに性別属性を持たせて立ち絵/ボイスを分けるのは可。ただし**テキスト（優勝時セリフ・反応）は性別分岐させない**（分岐すると①煽り化リスク②文量倍増）。混在は立ち絵の絵面で見せる。

### 0-3. キャラ別の声（方針7）—— 谷口＝関西弁は"谷口のキャラ"であって全員ではない

オーナー指摘（関西弁固定だとキャラによって微妙）への回答。**声は役柄ごとに固定**（周回で不変）、**主人公側だけ相方選択で差し替え**。

| 話者 | 声プロファイル | 例 |
|---|---|---|
| 主人公の相方（谷口） | 関西弁・柔らかい・ツッコミ | 「……効くわ。来年、隣の教室から抜くで」 |
| 主人公の相方（早乙女／二階堂／八雲） | `partner_characters_v0.md` の各声（**要差し替え**・§7-3） | ——（周回の相方で切替） |
| `.douki` 同期 | 標準語・慇懃で少し上から、根は熱い | 「君ら、今月ネタ何本書いた? ……僕らは倍」 |
| `.kohai` 後輩 | 敬語ベースの砕け・軽薄そうで底に本音 | 「あの……ネタ、見てもらえません? 5分でいいんで」 |
| `.senpaiGenius` 先輩 | 関西弁・枯れた苦労人・一言が重い | 「間を半拍だけ足せ。そこにお前らの色がある」 |
| `.veteran` ベテラン | 関西弁・枯れ（ニュース欄のみ） | ——（GP優勝には出ない） |
| 知らない組 | **台詞なし**（名前と点数だけ） | —— |

- **行種別ごとの声の帰属（実装者向け・重要）**:
  - **フレーバー行**（§4-5「優勝は——〈N〉。…」）＝**中立ナレーター**（三人称・標準語・MC/実況の声）。**相方差し替えの対象外**（谷口周回でも関西弁化しない）。
  - **袖の一言**（「谷口（袖）:…」）＋**一人称の地の文**（`relationship §4` の案文の一人称部分・「俺たちは…」）＝**主人公の相方の声**。谷口周回＝関西弁、早乙女／二階堂／八雲周回は各声に**差し替え**（§7-3・`partner_characters`）。文例は谷口版が基準。
- 役柄（同期・後輩・先輩）の声は本表が**既定（デフォルト変種）**。**`relationship_archetype_variants_v0.md` により runSeed でアーキタイプが選ばれ、口調は周回ごとに上書きされ得る**（今回の後輩は天然、今回の先輩は寡黙な職人…）。周回で不変なのは**名前ではなく「関係の色」**（同期/後輩/先輩）。`.senpaiGenius`＝関西弁枯れは**"苦労人"変種の声であって固定ではない**（他変種は標準語/老人語）。相方差し替えとは別軸。

## 2. この機構が確定するもの

プレイヤーが**決勝で敗れた年**（内部スコア < 決勝ライン・初回の17〜28%が見る画面・`finals_direction §2-3` 敗北）に:

1. **優勝者**（TOP3 の1枠・表示名＋関係色＋固有1行）
2. **TOP3 の残り2枠**（優勝者を除いた2組・関係色つき）
3. **プレイヤー反応**（関係タイプ × プレイヤーの優勝経験の2軸マトリクス・ノーボケ決意トーン）

を、**その周回・その年で不変（save/めくり整合）**に確定する。「年」はプレイヤー経過年（1〜10年目）で、`rival_design §2`「プレイヤー成績は年表に干渉しない」前提を踏襲。

## 3. 周回開始時の名前割当（方針5・`name_generator` で決定的）

`runSeed` から役柄テンプレ各々へ表示名を生成する（`name_generator_v0.md §3` の `genDuoName`）。**1周内は固定**（セーブ復帰・めくり演出で同じ名前を出すため）、**周回をまたぐと別**（暗記不可）。

```swift
// 役柄の「割当順」は正典（消費順を変えると周回再現性が壊れる。§6-2／name_generator §5-2）
let ROLE_ASSIGN_ORDER: [Role] = [.douki, .kohai, .senpaiGenius, .veteran]

// slot は周回通し番号。役柄名を先に採番（slot 0..3）、以後モブは §4 で slot を継続。
func assignRoleNames(_ runSeed: UInt64, cfg: NameGenConfig) -> [Role: String] {
    var used: Set<String> = []
    var out: [Role: String] = [:]
    for (slot, role) in ROLE_ASSIGN_ORDER.enumerated() {
        let nm = genDuoName(runSeed: runSeed, slot: slot, used: used, cfg: cfg) // name_generator §3
        out[role] = nm; used.insert(nm)                                          // 非復元
    }
    return out                          // 4役柄に別々の表示名（重複なし・毎周ランダム）
}
```

- `genDuoName` は組み合わせ生成で衝突ほぼゼロ＋NGフィルタ済み（`name_generator §4/§6`）。旧 `MOB_POOL`/`CHAMP_ELIGIBLE` のindex除外は不要化（生成名はNGフィルタで常にクリーン・§5）。
- 割り当てた4名は、その周回の間 `used` に入りモブ優勝者プールから除外される（§4）。→ 役柄と同名のモブが同じ画面に出ない。
- `.veteran` も表示名を持つ（GP優勝には出ないが、ベテラン大会ニュースで表示するため）。
- 名前は**周回定数**（`genDuoName` は year=0 固定で呼ぶ・毎年同じ名前）。

## 4. 決勝敗北時の優勝者＆TOP3 抽選（方針1・フルランダム）

### 4-1. 決定的ハッシュ（v0 §4-1 を継承・機種非依存・`name_generator` と共有）

```swift
// 周回シード × 年 × 用途salt → 64bit 決定値（浮動小数を使わない・&*/&+ でオーバーフロー）
func champHash(_ runSeed: UInt64, _ year: Int, _ salt: UInt64) -> UInt64 {
    var x = runSeed &+ 0x9E3779B97F4A7C15
    x ^= (UInt64(bitPattern: Int64(year)) &* 0xBF58476D1CE4E5B9)
    x ^= (salt &* 0x94D049BB133111EB)
    x = (x ^ (x >> 30)) &* 0xBF58476D1CE4E5B9   // splitmix64 finalizer
    x = (x ^ (x >> 27)) &* 0x94D049BB133111EB
    return x ^ (x >> 31)
}

enum Salt {                              // 用途ごとに固定（衝突回避・name_generator は 0x…20-22）
    static let roleVsMob:   UInt64 = 0x5A17_0000_0000_0011   // 役柄が獲るか/モブが獲るか
    static let champPick:   UInt64 = 0x5A17_0000_0000_0012   // 優勝者インスタンス選択
    static let top3Fill:    UInt64 = 0x5A17_0000_0000_0013   // TOP3補充
    static let archetype:   UInt64 = 0x5A17_0000_0000_0030   // 関係役柄のアーキタイプ選択（relationship_archetype_variants・roleIndex 0..2 を加算）
    // 名前生成の salt は name_generator §3（nameGenPattern/WordA/WordB）
}
```

### 4-2. 役柄の資格判定（優勝者になり得る年）

```swift
func isChampEligible(_ role: Role, year: Int) -> Bool {
    switch role {
    case .douki:        return (4...6).contains(year)   // 「先に行かれる」帯
    case .kohai:        return year == 7                // 返り咲きの年のみ
    case .senpaiGenius: return (9...10).contains(year)  // 初決勝〜資格ラスト（旧shizuY10分岐を包含）
    case .veteran:      return false                    // GP恒久除外（§0-1・A2確定）
    }
}
```

- 1〜3・8年目は資格役柄ゼロ → **必ずモブが優勝**（＝「知らない組」に負ける年）。序盤の敗北が知らない組色になるのは意図（4色を序盤から見せる）。
- 旧 `shizubutsugaY10WinRate`（v0 §3）は、`.senpaiGenius` の資格年（9・10）で `roleChampPermille`（§4-3）が引くか否かに**一般化**して吸収。専用configは廃止。

### 4-3. 優勝者インスタンスの決定（2段抽選・フルランダム）

**段1**: 資格役柄がいる年は「役柄が獲るか／モブ（知らない組）が獲るか」を確率 `roleChampPermille` で分岐。**段2**: 役柄側なら資格役柄から等確率、モブ側なら `genDuoName` で新規モブ名を生成。全て `champHash`／`genDuoName` の決定値で、**同一周回×同一年なら毎回同じ**（save/めくり整合＝§6-3）。周回が変われば `runSeed` が変わり結果もシャッフル（暗記不可）。

```swift
struct GameConfig_champion {                 // 演出層・§6-1でgolden対象外を保証
    var roleChampPermille: Int = 450         // 【仮】資格役柄がその年に獲る確率(‰・0=常に知らない組／1000=常に役柄)
    // 整数のみ（浮動小数を使わない＝機種非依存・§4-1原則と一致）。体感調整はまず演出・頻度で（規律B-5）。非ゲームバランス。
}

enum Contestant { case role(Role); case mob(name: String) }   // モブは生成名を保持

func pickChampion(year: Int, runSeed: UInt64,
                  roleNames: [Role: String], playerName: String,
                  used: inout Set<String>, cfg: GameConfig_champion,
                  ngCfg: NameGenConfig) -> Contestant {
    let eligRoles = ROLE_ASSIGN_ORDER.filter { isChampEligible($0, year: year) }

    // 段1: 役柄 or モブ
    let takeRole = !eligRoles.isEmpty &&
        Int(champHash(runSeed, year, Salt.roleVsMob) % 1000) < cfg.roleChampPermille  // 整数のみ

    if takeRole {
        let h = champHash(runSeed, year, Salt.champPick)
        return .role(eligRoles[Int(h % UInt64(eligRoles.count))])
    } else {
        // モブ優勝: その年専用の slot で新規生成（役柄名・プレイヤー名は used で回避）
        let slot = 100 + year                                   // 役柄0..3/他年と衝突しない帯
        let nm = genDuoName(runSeed: runSeed, slot: slot, used: used, cfg: ngCfg)
        used.insert(nm)
        return .mob(name: nm)
    }
}
```

> **暗記不可の担保（方針1）**: 仮に `roleChampPermille` を高くして「資格年は役柄がよく獲る」偏りを付けても、**表示名は毎周 §3/`name_generator` でシャッフルされる**ので、プレイヤーが「この名前が○年目に勝つ」と暗記することは原理的に不可能。周回間で覚えられるのは「関係の色」「物語のリズム」だけで、**名前と結果の対応は毎周崩れる**。これが方針1・方針5の合わせ技。

### 4-4. TOP3 他2枠の補充（関係色つき・非復元）

優勝者を除いた2枠を、**資格役柄→モブ**の順で非復元に埋める（役柄が余っていれば優先的にTOP3へ入れ、色を画面に増やす）。優勝者・プレイヤー・既割当は `used` で必ず除外。

```swift
struct FinalsCast {
    let champion: Contestant
    let championName: String
    let championRel: Relationship          // 同期/後輩/先輩/知らない（色分け・方針2）
    let championFlavor: String             // 役柄フレーバー1行（名前差込済み・§4-5）
    let playerReaction: String             // 関係×優勝経験 マトリクス（§4-6・ノーボケ）
    let top3Others: [(name: String, rel: Relationship)]   // 2組
}

func buildFinalsCast(year: Int, runSeed: UInt64, roleNames: [Role: String],
                     playerName: String, playerHasWonBefore: Bool,
                     cfg: GameConfig_champion, ngCfg: NameGenConfig) -> FinalsCast {
    var used = Set(roleNames.values).union([playerName])       // 役柄名＋プレイヤー名を予約
    let champ = pickChampion(year: year, runSeed: runSeed, roleNames: roleNames,
                             playerName: playerName, used: &used, cfg: cfg, ngCfg: ngCfg)
    let (champName, champRel) = resolve(champ, roleNames: roleNames)   // §4-5表引き
    used.insert(champName)                                      // 優勝者名を確実に除外

    // --- TOP3補充: 優勝者を除いた資格役柄を先に、足りない分をモブで ---
    var others: [(String, Relationship)] = []
    let otherRoles = ROLE_ASSIGN_ORDER
        .filter { isChampEligible($0, year: year) && roleNames[$0] != champName }
    for r in otherRoles where others.count < 2 {
        let nm = roleNames[r]!
        if !used.contains(nm) { others.append((nm, relationship(of: r))); used.insert(nm) }
    }
    // 残り枠をモブ（知らない組）で補充。slot を Salt.top3Fill 帯で継続（決定的・非復元）
    var i = 0
    while others.count < 2 {
        let nm = genDuoName(runSeed: runSeed, slot: 200 + year * 4 + i, used: used, cfg: ngCfg)
        others.append((nm, .stranger)); used.insert(nm); i += 1
    }

    return FinalsCast(champion: champ, championName: champName, championRel: champRel,
                      championFlavor: flavorLine(champ, name: champName),
                      playerReaction: reactionLine(rel: champRel, won: playerHasWonBefore),
                      top3Others: others)
}
```

> 実装要点: `used` は「役柄名→優勝者→TOP3補充」の順で成長する単一集合。優勝者名を入れてから補充するので、TOP3補充モブと優勝者が被らない。モブ名は `genDuoName` が `used` を見て非復元生成（`name_generator §4`）。

### 4-5. 役柄フレーバー1行（名前差込・ノーボケ・存続する物語の要約）

`〈N〉` に表示名を差し込む。**役柄の物語**を淡々と語る行（感情は §4-6 に分離）。**決勝敗北の瞬間なのでボケは入れない**（方針6）。

| 対象 | フレーバー1行（`〈N〉`＝表示名・ノーボケ） |
|---|---|
| `.douki`（同期） | 「優勝は——〈N〉。あの頃、隣の席で同じネタに笑ってた二人が、先に頂点に立った。」 |
| `.senpaiGenius`（先輩・技巧派） | 「優勝は——〈N〉。準決勝の壁に何年も愛された男らが、今夜その壁を越えた。」 |
| `.kohai`（後輩） | 「優勝は——〈N〉。一度バズって、一度消えて——劇場から獲りに帰ってきた。」 |
| モブ（知らない組） | 「優勝は——〈N〉。楽屋で一度も会わんかった二人が、頂点に立っていた。」 |

- **モブ行の変更（オーナー指摘反映）**: 旧「今年の主役は隣の楽屋の二人やった」（軽い）を廃し、**事実を淡々と述べる**行に。決意（来年は絶対）は §4-6 のプレイヤー反応が担う（＝「今年の優勝〇〇。来年は絶対」の構造）。
- （`.veteran` はGP優勝に出ないためフレーバー不要。ベテラン大会優勝は業界ニュース欄で別途1行。）

### 4-6. プレイヤー反応の2軸マトリクス（方針4・6の中核・ノーボケ決意）

**縦=関係タイプ（優勝者の色）／横=プレイヤーの優勝経験。** `〈N〉`は差込。**「先を越された／先に行かれた」は未優勝列のみ**（方針4）。優勝済列は 後輩＝誇り／先輩＝恩／同期＝並走で、**禁止2語を1文字も含めない**。全文A/B/C案は `relationship_champions_v0.md §4`（本表は代表1行＋規則）。

| 関係＼優勝経験 | **未優勝**（悔しさ系・「先を越された」可） | **優勝済**（別の言葉・禁止2語ゼロ） |
|---|---|---|
| **同期** | 「同期に、先を越された。」／地の文「隣を走ってたはずやのに、気づいたら背中が遠かった。」 | **並走**「同期が、獲った。」／「順番が回ってきただけや。去年はこっち、今年はあっち。まだ、隣を走ってる。」 |
| **可愛がってた後輩** | 「可愛がってた後輩に、先を越された。」／「嬉しさと悔しさが同時に来て、どっちの顔したらええか分からんかった。」 | **誇り**「あの後輩が、獲りよった。」／「悔しさより先に、誇らしさが来た。それでええ。」 |
| **飲みに行く先輩** | 「あの先輩に、先に行かれた。」／「先に行かれるのは当たり前や。当たり前が、こんなに悔しいとは。」 | **恩**「あの先輩が、ついに。」／「さんざん飲ませてもろた恩、まだ返せてへん。返す相手が、頂点におる。」 |
| **知らない組** | 「知らん二人に、持っていかれた。」／決意「名前も知らんかった。来年は、向こうにこっちの名前を覚えさせたる。」 | 「知らん二人。ええ漫才やった。」／「今年の主役は向こうや。……来年、飲みに誘てみよか。」 |

- **禁止語ゼロの担保（方針4）**: 優勝済列（右）に「先を越された」「先に行かれた」を**1文字も含めない**。上表・`relationship_champions §4` の優勝済セル（全A/B/C案）を、リソース検証で**禁止語スキャン＝0件**にする（`relationship_champions §5-2`）。
- **ノーボケの担保（方針6）**: 決勝敗北の結果発表（フレーバー〜反応〜袖）は**真剣な決意トーン**。ボケ・軽口は入れない（笑いを取りに行く一言を禁止）。ボケは絆イベント（`relationship_champions §1-3`）側でのみ許可。**「事実→感情→来年への決意」は1画面（フレーバー→反応→袖）全体で構成し、決意は袖行が担う**（優勝済の反応主文は感情の着地でよく、決意を無理に付けない。未優勝の知らない組主文のみ決意を主文に置く＝§4-6知らない組・未優勝セル）。
- 先輩の未優勝セルは「先を越された」でなく「先に行かれた」に留める（先輩が先に行くのは自然）。ただし**未優勝列に属する**ので悔しさ帯は維持。
- **絆レベル連動（方針3・任意・§7）**: 会話イベントで育てた絆が高い相手が獲ると、より本音寄りのC案へ差し替え（`relationship_champions §4` の案C）。MVPは8セル＋汎用で成立。絆連動は【仮】拡張。

## 6. 非バランス・周回安定・save/めくり整合（方針eの明示）

### 6-1. バランス・golden 不変（最重要）
- 判定は絶対ライン方式のまま（`finals_direction §4-D`）。**判定ロジック変更ゼロ**。
- 優勝者選出・名前生成は **gameplay の注入RNGを1消費もしない**。専用の `champHash`／`genDuoName`（`runSeed`＋salt）だけで閉じる。→ 週送り/稽古/大会の**乱数消費順に触れない＝3年golden ビット一致は不変**（開発規律A）。`tools/*.py` 正典・`swift test` に影響なし。
- `GameConfig_champion.roleChampPermille`・`NameGenConfig` は**演出層の定数**。`sim_career.py`／バランスgoldenの対象外。全て【仮】。

### 6-2. 消費順＝正典（周回再現性）
- 名前 slot の採番は「役柄割当（slot 0..3）→ モブ優勝（100+year）→ TOP3補充（200+year*4+i）」の順（§3/§4）。この順を変えると同一 `runSeed` でも別名になる。**順序は数式と同格の正典**（規律A-2・`name_generator §5-2`）。
- 優勝者/TOP3 は年ごとに独立ハッシュ（`champPick`/`roleVsMob`/`top3Fill`）で、他年・他用途と衝突しない（salt分離）。

### 6-3. save/めくり整合
- `FinalsCast(year, runSeed, roleNames(runSeed), playerHasWonBefore, cfg)` は**純関数**。セーブに必要な永続情報は **`runSeed` のみ**（roleNamesもモブ名もrunSeedから再導出可能）。
- ゆえに**決勝画面でセーブ→ロードしても同じ優勝者・同じ名前・同じTOP3**。`finals_direction §4-C` の「めくり」は同じ札を再現。早送り可・カット不可とも両立。
- `playerHasWonBefore` は反応セリフ（§4-6）だけに効き、**キャスト（優勝者・TOP3）には影響しない**（優勝経験でキャストが変わるとロード再現やネタバレ整合が崩れるため）。方針4は「言葉を変える」であって「相手を変える」ではない。
