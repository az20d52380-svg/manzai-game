# ガチャ経済検証レポート v0 — 名鑑コンプ速度と採用レート

- 日付: 2026-07-04
- 方法: `tools/sim_gacha.py`。B案確定（1日1回無料＋ミッション＋人脈Pチケット）の無料プレイヤーを2000試行モンテカルロ。名鑑16体（N6/R5/SR3/SSR2）・気になるリスト（未所持最高レアを常に指名）・ダブり→人脈P還流込み。数値は全て【仮】。

## 1. 採用値

| 項目 | 採用【仮】 | 根拠 |
|---|---|---|
| **SSR排出率 1.5%**（当初案2%から引き下げ） | 全プロファイルで全16体コンプ中央値**4ヶ月**・下振れ90%タイル7〜9ヶ月 | 2%だと3ヶ月で目標帯(3〜5ヶ月)の下限に張り付き、名鑑の寿命が不足 |
| チケット50P・ダブりP（N5/R10/SR20/SSR40） | 据え置き | 下げてもコンプ速度がほぼ動かない（§2） |

## 2. 最重要の発見 —「プレイ量はコンプ速度をほぼ動かせない」

ライト（月1周）〜ヘビー（月7周）でコンプ中央値の差は**1ヶ月未満**。1日1回無料（月30連）が支配項で、人脈Pチケット（月0.7〜5連）は誤差にしかならないため。レート・チケット価格を4通り振っても構造は変わらなかった。

**設計への転換**: これは欠陥ではなく仕様にする。
- 「毎日1回だけの人も、4ヶ月後には必ず全員に会える」——コンプ欲設計として誠実で、ライト層を置いていかない
- **プレイ量の差はコンプ速度でなく「人脈Pの余剰」で報いる**: ヘビー層はPが余る→定番コスメ交換・周回開始ボーナスが早い。コスメ2層制（monetization §7）の定番層は、まさにこの余剰の受け皿として機能する
- 有料が買うものは変わらず「早さ」と「指名権」のみ

## 3. 決勝人気補正の実装と再調整（同時実施）

judge_design §10-F の「機微±1.5点」を実装した結果、決勝進出者はほぼ全員が知名度高（補正-1.2〜-1.5点）のため初回優勝率が1.5〜2%→3.0〜4.5%に上振れ。**決勝ラインを91→93に引き上げて相殺**し、最終値:

> **決勝ライン93 ＋ 人気補正1.5 → 初回優勝率 1.2〜1.4%**（「普通にやったら1%ぐらい」に最接近）

- 意味も綺麗になった: 話題のコンビ（知名度100）は実効91.5、無名の決勝進出者は93——「世間の勢いがわずかに味方する」
- 王者編への影響は実質ゼロ（防衛時は知名度100で補正-1.5 ≒ 旧ライン91+5×連覇と同等）
- Python・gen_golden・GameCore（`fameFinalBonus`）の3箇所同期済み。goldenの数値行は不変を確認

## 3-B. 【2026-07-04追記】日次無料の持ち越し無しルール（オーナー確定）

「放置で溜まる」を廃止（ストック救済なし・その日限り）。実測: コンプ中央値は皆勤4ヶ月／週5ペース5ヶ月／週3ペース7ヶ月／週末だけ10ヶ月。**§2で失われていた「通い」の差がここで生まれる**——周回量ではなく登板頻度が名鑑を埋める速度になり、「毎日1回だけでも劇場に顔を出す人が一番早い」という本作の主題（下積みの継続）と綺麗に一致する。
