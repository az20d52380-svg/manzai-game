# game_reality_check_v0 — オーナー向け実態把握ブリーフィング（中立・忖度なし）

作成: 2026-07-05。目的: 深いバランス/sim作業に潜って「結局これがどんなゲームなのか」を見失ったオーナーが、**この1枚で現在地を掴み直し、本を買うべきかも判断できる**ようにする。褒めも過度な悲観もなし、事実ベース。数値は全て【仮】。

---

## 0. 一言で言うと

**架空の漫才賞レースを舞台にした、メニュー選択式ターン制のコンビ育成シム。** タップだけで480週（10年）を回し、「ほぼ優勝できない」ことを数値で仕様化したうえで、負けて得たトロフィー（才能解放）と相方ガチャで少しずつ優勝率を上げていく**周回前提の育成SLG**。ただしこれは*設計されているゲーム*であって、*実際に動くゲーム*は「1年48週・単一シード・メタなし・会話やイベントのテキストはコードに1文字も無い」計算エンジンとプレースホルダUIの断片。**バランスだけが完成し、ゲーム本体はまだ立ち上がっていない。**

## 1. 実態スナップショット

| 項目 | 実態 |
|---|---|
| **ジャンル** | 育成SLG（期間圧縮型）が核。パワプロ・サクセスのDNAそのもの。周回メタはローグライト/競走馬育成型、ガチャはコレクション層。放置系ではない（480週全て手動選択）。 |
| **コアループ**（実装済み） | 週送り3タップ。自由週=11択（稽古5・バイト3・休む3）から1つ選ぶ／効果量は非表示。大会週=遠征手段選択→笑い波形メーター→通過敗退→紙の1行講評。GP回戦週=自動判定を見るだけ。手触りは**アクション性ゼロの数値管理ゲーム**、テキスト進行ADVに近い軽さ。 |

| **最も近い実在ゲーム** | **パワプロ・サクセス**（育成核が同一。ただし本作は480手で桁違いに長く、初回は勝てないことを実測で仕様化）／**ウマ娘**（周回＋ガチャは近いが、ウマ娘はガチャが戦力の本体＝本作は逆でトロフィーが本体）／**ダビスタ・競走馬育成**（負けた個体も次周の資産という等式が近い。ただし本作は確率継承でなく確定継承）。周回思想の近縁は**Hades**だが、ランの長さと非アクション性で手触りは別物。 |

## 2. 中立の強み（事実ベースで本当に強い3つ）

1. **「初回は勝てない」の潔さを数字で保証している。** 優勝率をsimで実測（初回0.3〜2%）し、「あと一歩の悔しさ→周回」を口先でなく数値で裏打ち。負けキャリア自体を収集対象にする設計まで一貫している。育成SLGとして珍しい、はっきりした背骨。

2. **決勝の照明（`finals_direction_v0`）が突出して作り込まれている。** 出囃子籤→数字ドラム→暫定ボード→TOP3めくり→優勝者指名（誰に負けたか必ず与える）→敗北の夜に才能解放を灯す、という順序を、判定ロジックを一切変えず演出だけで山を作る割り切り。**本作で一番「これは面白そうだ」と言える部分**（ただし1文字も未実装）。

3. **バランス検証の誠実さと再現性が本物。** Python⇔Swiftの乱数ビット一致（3年golden）・swift test 17 green・縮退戦略の破産保証・数値を看板に正直開示。個人開発の水準を超えて堅い。**ただしこれが証明しているのは「計算が正しい」ことであって「面白い」ことではない**点は次章と表裏。

---

## 3. 中立の弱み/リスク（辛口外部の最大リスク含む・4つ）

1. **最大リスク: 「設計の完成度」と「完成までの距離」を取り違えたまま燃え尽きる。** doc約10,000行に対し動くSwiftは約1,500行（GameCoreロジック935＋アプリ559）、しかもその精密さは全て*未実装のバランス*に注がれている。魅力を担う数万字のテキスト・演出・キャラはコードに0本。個人開発で最も枯渇するのは「コンテンツを実装しきる体力」で、本作の勝ち筋（テキストの質）はまさにそこに賭かっている。

2. **面白さの核が一度も動いていない。** オーナー自身が最優先と定めた**検証①「タップだけの週48手が単調にならないか」が未検証**。これは手触りの問題でsimでは答えが出ない。加えて内部レビュー（`fun_flow_review_v0`）が**4〜7年目の「停滞レジーム」**を自認——初回主役層（分散63%/のんびり74%【仮】）は9年目まで決勝に届かず、その5年を語る言葉もなくタップさせる＝開発チーム自身が「検証①の不合格基準そのもの」と書いている。フックが効くのは両端（負け方チュートリアルと決勝）だけで、体験時間の大半を占める中腹が最も薄い。

3. **収益モデルが未確定のまま、有力候補（F2P相方ガチャ）の課金トリガーが構造的に発火しない。** 「無料皆勤4ヶ月で名鑑コンプ・天井1/3の石を毎月配布」を自ら実測保証しており、売り物が無料で即充足。さらに相方は2周目（6〜8時間後）まで使えず**D1〜D7課金がほぼゼロ**。red teamの初年度売上見立ては悲観30〜100万円【仮】、買い切り（A案）の方が中央値2〜4倍という自チーム結論が出ているのに業態は凍結。`monetization_decision`内でA案推奨と確定B案が緊張したまま。**ゲーム性以前の事業の穴。**

4. **機構的な新規性は薄く、独自性はテーマと演出層に集中している。** 中核ループ（週1行動→数値が伸びる→大会判定）は最も枯れた育成SLGの形。ライン方式・審査員コメント・返り咲きライバルといった独自要素は判定に干渉しない演出/フレーバー層で、機構の深さは足していない。「架空漫才師」という皮は独自だが、骨は育成SLG＋ガチャ周回の王道。加えてお笑い題材ゲームの商業実績が事実上なく、実在大会名は使えない（寄せれば法務・離せば集客ゼロ）制約もある。

## 4. 実装の現在地（設計済み vs 実装済み・一番のボトルネック）

システム系統は本編で約12。**コードに存在するのは最初の5系統（週送り〜借金/ケガ制）まで。残り7系統（トロフィー/相方ガチャ/事務所移籍/審査員講評/イベント/ライバル/コンプ図鑑）は設計のみ。**

| レンズ | 設計 | 実装 | 未実装率（体感） |
|---|---|---|---|
| システム系統 | 12系統 | 5系統 | **約60%が設計止まり** |
| バランス/数式 | v2正典確定 | golden一致・swift test 17 green | **ほぼ実装済み（ここだけ突出）** |
| コンテンツ本数 | 数千項目執筆済 | **コードに0本配線** | **ほぼ100%が設計止まり** |
| UI/演出 | ui_design＋finals等 詳細 | 5画面のプレースホルダ | 約85%が設計止まり |
| 収益（ガチャ/名鑑/課金） | B案フル設計 | 0 | 100%が設計止まり |

**一番のボトルネックは「バランス/sim」ではなく、書いた肉をコードに配線する工数。** 数千項目の文章は書けていても、それを審査員システム・イベント発火・雑談トリガ・ファンレポ生成として実装し状態フックに繋ぐ仕事がまるごと残っている（文章執筆より配線が重い）。次点は相方ガチャF2Pの運営コスト、その次がUI目視検証がまだ始まっていないこと。

**総合体感: 本編スコープの7〜8割が設計止まり。** コードが追いついているのは「1年MVPのバランス計算」という一番細い柱だけ。正確に言えば**「設計は完成、実装は着工直後」**。

補足: 王者編（10連覇・殿堂・最長25年）は2026-07-05に廃止済み＝スコープ的に正しい縮小。ただし`content_inventory_plan_v0`の枯渇分析に「25年やり込み」列が残り、廃止と整合していない箇所がある（例: 決勝B帯コメントの枯渇根拠が王者編前提）。廃止で最悪ケースの深刻度は下がっている。

## 5. 本の要否と推薦

**結論: 大半のカテゴリで「今は本は要らない」。** このプロジェクトの病は知識不足ではなく「設計は過剰、実装とプレイテストがゼロ」という入力過多・出力不足。ここに設計書を足すのは、今犯している失敗（遊ぶ前に読み込む）をもう一冊分繰り返すこと。**最重要課題（検証①・配線・燃え尽き）ほど本で解けない**という非対称を直視するのが先。

効く弱み → 効かない弱みの仕分け:

| 弱み | 本で埋まるか |
|---|---|
| 検証①未検証 / コンテンツ配線工数 | ❌ 実機ビルド＋プレイテストと労働でしか解けない |
| 燃え尽き / 中だるみ / juice欠如 | △ 概念は増えるが直すのは実装＋テスト |
| F2P収益トリガー | △ 先に業態を決めてからでないと死蔵 |
| 絆が相性1スカラーの文章化止まり | ○ ナラティブ/キャラ設計の専門書で深まる |
| **漫才・笑いの技術（＝唯一のmoat）** | ◎ 専門クラフト。私（Claude）にも一般ゲーム設計書にも代替できない |

**優先2〜3冊（買うなら）:**

1. **塙宣之『言い訳 関東芸人はなぜM-1で勝てないのか』（新潮新書）** — 賞レース漫才の構成・ツカミ・緩急・審査で見られる点を実作者が言語化。審査員講評×パラメータ連動、決勝演出、ネタ資産の中身を書くときの一次教本。安く和書で題材直撃。**唯一「今すぐ買っていい」本。**
2. **Richard Lemarchand『A Playful Production Process』** — 「バーティカルスライスを先に作れ」という出荷規律の本。「設計は完成・実装は着工直後」という本作の病に処方箋が直撃。設計本を積む逆をやる本なので読む価値のある例外。
3.（任意）**Jesse Schell『ゲームデザインバイブル（The Art of Game Design: Lenses）』** — 進行/feel/収益/心理をレンズで横断。狭い設計本を何冊も買う代わりの1冊。ただし多くは私に聞けば足りるので優先度は落ちる。

補助的に効く領域: 笑いの量産（Helitzer『Comedy Writing Secrets』）、笑いの構造理論（Kaplan『The Hidden Tools of Comedy』）、システムで物語る（Skolnick『Video Game Storytelling』）、F2P収益（Luton『Free-to-Play』＝ただし業態をF2Pと確定してからのみ）。

**本より先にやること（これらは本で前倒しできない）:**
1. 1年MVPを実機でビルド→起動→自分で10分遊ぶ（既存の会話・審査員・決勝テキストを最小でも配線して）。**このプロジェクトで初めて「面白いか」に触れる瞬間で、まだ一度も来ていない。**
2. 他人に初見で10分触らせ「退屈」と言われないか見る（＝検証①）。simの小数点より先にこれ。
3. 業態（買い切り/F2P）を決める。未確定だと収益系の設計も本も宙に浮く（内部分析は買い切り寄り）。

## 6. 中立の結論: いま何に注力すべきか（優先順1つ）

**最優先は「実機で遊ぶ」。** 本でも、コンテンツ増産でも、バランス再測でもない。

このプロジェクトは「バランスが不安なゲーム」ではなく、**バランスだけが完成し、ゲームがまだ存在しないプロジェクト**。設計知識の在庫はすでに過剰にあり、次に測るべきは優勝率の小数点ではなく「今の空のビルドを初見の人に10分触らせて退屈と言われないか」＝検証①で、それが一度も実測されていない。

具体的な順序:
1. **既存テキスト（会話・審査員・決勝演出）を1年MVPに最小配線し、実機ビルド→起動→10分プレイ**（＝面白さに初めて触れる）。
2. **他人に初見プレイさせ、検証①を通す/落とす**を判定。
3. その結果を見て**業態（買い切り/F2P）を確定**。
4. 検証①が通ってから、必要なら塙『言い訳』だけを手に取り、moat（笑いのクラフト）の執筆に向かう。

一言: **このプロジェクトに一番効く「本」は、実は本ではなく、まだ誰も遊んでいない自分のMVPを一度遊ぶこと。** 在庫にないのは笑いのクラフト（塙）と、在庫を出荷に変える規律（Lemarchand）だけ。それ以外を積むのは、遊ぶのを先延ばしにする新しい言い訳になりやすい。
