# 10年キャリア検証レポート v1（成長逓減の導入実験）

> **【正典v2移行 2026-07-05】** 本書のライン・優勝率・D（才能解放）等の数値は**旧正典**の実測値です。
> 現行の正典は `canonical_v2_spec.md`（年齢カーブ上限・ハマった夜・生活ルール・準決74/決勝80・トロフィー=上限加算）。
> 本書の**設計結論・構造・演出**は有効。数値だけ読み替えること。


- 日付: 2026-07-03
- 方法: `tools/exp_decay.py`。v0（`docs/career_report_v0.md`）で最優先の修正候補とした「稽古の成長逓減」を実装せずに注入（`balance_sim.add` をモンキーパッチ、両シミュレータは無変更）し、初優勝年が仕様想定の6〜9年目に入る強さを探索。各設定2000キャリア、シード20260704、ボット・カレンダー・グランプリ構造はv0と同一。
- 逓減式【仮】: 能力5種の上昇量 × (1 − 現在値/D)。Dが小さいほど高能力での伸びが鈍る。体力・知名度・相性は対象外。

## 1. 実験D: 逓減の強さ（基準設定: 相性5・成長あり・初期能力10）

| 逓減 | バランス型 中央値 / 優勝なし率 | 調整型 中央値 / 優勝なし率 |
|---|---|---|
| なし(v0) | 4年目 / 0.0% | 4年目 / 0.0% |
| D=150 | 6年目 / 0.0% | 6年目 / 0.0% |
| **D=120** | **7年目 / 0.0%** | **6年目 / 0.0%** |
| D=100 | >10年 / **96.3%** | >10年 / **98.5%** |

**D=120を推奨**。バランス型の分布が 5年:9% → 6年:31% → 7年:34% → 8年:20% → 9年:5% → 10年:0.6% と5〜10年目に自然に広がり、「早い人は5年、普通は6〜8年、遅いと9〜10年」の体験になる。D=150はやや早め（5〜7年に密集）。**D=100は崖**（ほぼ全員が10年で届かない）——決勝ライン85に対し能力の実質上限が不足するため。調整時にDを下げすぎると難易度が急変する点は要注意。

## 2. 実験E: D=120の下での相方ガチャ・周回ボーナスの効き直し

### 相性 固定値（成長なし）＝相方ガチャ相当

| 相性 | バランス型 中央値 / 優勝なし率 | 調整型 中央値 / 優勝なし率 |
|---|---|---|
| 5 | 10年目 / 30.8% | 9年目 / 3.6% |
| 10 | 9年目 / 8.2% | 8年目 / 0.0% |
| 15 | 8年目 / 1.5% | 7年目 / 0.0% |
| 20 | 7年目 / 0.1% | 6年目 / 0.0% |

**逓減下では相性の価値が約3倍化**: 相性+15で中央値3年短縮（v0では1年）。相性+5ごとにほぼ1年縮む線形な効きで、ガチャのレアリティ段階（+5刻み）として使いやすい。一方、最低値（相性5固定）はバランス型で優勝なし31%——「ハズレ相方では10年で届かないことがある」という緊張感が生まれるが、救済（相方変更・相性成長）の設計が必須になる。

### 初期能力 一律加算（相性5・成長あり）＝周回ボーナス相当

| 初期能力 | バランス型 中央値 | 調整型 中央値 |
|---|---|---|
| 10（基準） | 7年目 | 6年目 |
| 15 | 6年目 | 6年目 |
| 20 | 6年目 | 6年目 |
| 25 | 6年目 | 6年目 |

**逓減下では初期能力の効きが大幅に弱まる**（+15でも中央値1年未満の短縮）。逓減式は低能力ほど伸びが速いため、初期差が数年で吸収される。つまり周回ボーナス（初期能力加算）は「序盤が楽になるが優勝時期はほぼ変えない」快適性ボーナスとして安全に配れる。優勝時期を動かしたい周回報酬は相性側（または逓減Dの緩和）に置くべき。

- v0からの逆転に注意: 逓減なしでは初期能力＞相性だったが、**逓減ありでは相性≫初期能力**。未設計システムの強さ配分はこの序列を前提にする。
- 1年目優勝は全36設定・累計7.2万キャリアで0件のまま。「新人の奇跡」を演出したいならブレ幅・ライン側の別実験が必要（v0 §4-(2)参照）。

## 3. 実験F: D=120のお金の推移（シンク規模の再確認）

| 年末 | バランス型 中央値 | 調整型 中央値 |
|---|---|---|
| 2年目 | +38万 | +18万 |
| 4年目 | +686万 | +443万 |
| 6年目 | +1,782万 | +1,547万 |
| 8年目 | +3,068万 | +2,802万 |
| 10年目 | +4,250万 | —（ほぼ全員勇退済） |

キャリアが伸びたぶんv0より余剰が拡大（4年目以降は年+500〜650万ペース）。シンクの規模感は「1〜2年目: 数万〜数十万円帯 / 中盤以降: 年200〜600万円を吸える高額帯」の2層が目安。

## 4. 結論と次アクション

1. **成長逓減 ×(1−現在値/120) を採用候補として確定**（数値は【仮】）。初優勝中央値6〜7年・分布5〜10年・優勝なし率ほぼ0%で、仕様想定6〜9年に収まる。
2. 【設計上の懸念】逓減はGameCoreの成長式そのものに入るため、Swift実装前に採否を決めたい（後から入れると全バランスが再計測になる）。本レポートはD=120で進める前提を採る。採用時は `balance_sim.py` の成長式にも正式に組み込み、Python検証機とSwiftを同期する（CLAUDE.mdルール5。現状はパッチ注入のため本体は未変更のまま）。
3. 相方ガチャは相性+5刻み（5〜20）でレアリティ設計、ハズレ救済とセットで。周回ボーナスは初期能力+5〜15を快適性報酬として自由に。
4. アイテム等のシンクは2層構成（序盤: 〜数十万 / 中盤以降: 年200〜600万）。

## 使い方

```
python3 tools/exp_decay.py        # 2000キャリア/設定（約60秒）
python3 tools/exp_decay.py 500    # 試行数を変える
```

逓減候補は `exp_decay.py` 冒頭の `DECAY_CANDIDATES`、実験E/Fの採用値は `D_PICK`。
