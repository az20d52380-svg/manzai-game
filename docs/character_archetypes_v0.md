# キャラアーキタイプ図鑑 v0 — 口調（役割語）×得意能力で相方・後輩・先輩を増やす

- 日付: 2026-07-05
- 位置づけ: オーナー指示「アニメ/漫画のキャラ類型をリサーチして、男女いろんなキャラの相方・後輩・先輩を増やす。**口調も変わるし、それに応じた能力アップにしたい**」の設計。キャラの類型（アーキタイプ）ごとに **①口調（役割語）②得意能力（育成の偏り）③二面性（アークで開花する裏能力）** を定義し、量産の型にする。
- **リサーチ根拠**: 「役割語」（言語学者・金水敏の概念＝特定の言葉遣いで特定の人物像が想起される話し方。日本語は一人称・文末詞が豊富で特に発達）。アニメ/漫画のキャラ類型（ツンデレ/クーデレ/天然/体育会系…）の話し方パターンを抽出。**特定の実在キャラは模倣せず、一般的な類型だけを型として使う**（固有名は全て架空・CLAUDE.md ルール4）。
- **性別非依存の原則を継承**（`champion_selection_v1 §0-2c`）: 類型は男女どちらにも割り当てる。**性別を勝敗・煽りの装置にしない**。口調・得意能力は性別記号ではなくキャラ性。
- **数値は全て【仮】・バランス影響あり → sim検証が要る**（§6）。本書は設計のみで golden/コードには触れない。

## 0. 得意能力＝ゲームの実パラメータへの写像

ゲームの能力（`GameCore/GameConfig.swift` `Ability`）は **センス／発想／表現／華／メンタル** の5種＋**体力**。リサーチの「漫才師の得意な力」を実パラメータへ写像する:

| リサーチの力 | ゲームの能力 | 意味 |
|---|---|---|
| 構成・理論／アドリブ・瞬発力 | **センス** | 間・構成・ツッコミの切れ（何が面白いかの勘） |
| 発想の奇抜さ | **発想** | ボケの独創性・引き出し |
| 表現・演技 | **表現** | 伝える力・芝居・間の芝居 |
| 客席掌握 | **華** | スター性・客席を掴む力 |
| 安定感／度胸(本番の強さ) | **メンタル** | 本番でブレない・恒常ブレを狭める |
| 体力・フィジカル | **体力** | 勢い・稽古量を積める土台 |

## 1. 口調→能力アップの機構（オーナー要望の core）

### 1-1. 3点セット（各アーキタイプが持つ）
1. **口調（役割語）**: 一人称・語尾・口癖。セリフ生成・差し替えの型（相方は `partner_finals_reactions` 方式で袖反応を差し替え）。
2. **得意能力（表）**: そのキャラと組む／関わると**その能力の稽古効率が上がる**（【仮】+15%〜+25%・§2）。＝「口調に応じた能力アップ」。
3. **裏能力（二面性・アークで開花）**: 専用アーク（相方）や絆イベント（後輩/先輩）を進めると、二面性の"裏の顔"に対応する**第2の能力ボーナスが解放**される（【仮】+10%）。表と別軸に置くと育成に深みが出る。

### 1-2. 既存ルールとの整合（改訂・重要）
`partner_characters §4-4` は「性能はティア表のみ・会話/アークは性能に影響しない（魅力と強さを分離）」だった。本書はこれを**改訂**する:

- **ティア（R/SR/SSR）＝生の強さ（相性上限など）** は不変。ガチャの"引きの強さ"は従来どおりティアが担う。
- **アーキタイプ＝育成の方向性（得意能力の稽古効率の偏り）** を新たに担う。これは**横方向の個性**（プレイスタイル差）であって縦の強さではない。**全アーキタイプが総合力で等価になるよう調整**（sideways・§6 でsim保証）。
- **アーク／絆＝裏能力の開花**という小さな機械的報酬を持つ（魅力が育成に少しだけ効く）。ただし開花分も横方向で、必須化しない。
- ⇒ 「魅力≠生の強さ（ティア）」は維持しつつ、「魅力・口調＝育成の色（得意能力）」を足す、という改訂。【設計上の懸念】これはバランスに触れるので §6 の sim 検証を通すまで**数値は全て仮・実装はGameConfigフラグでoff可**にする。

## 2. 得意能力ボーナスの設計（GameConfig・全て【仮】）

```swift
// 新規GameConfig（演出でなくバランス側。sim_career と1対1・§6）。全て【仮】。
struct ArchetypeSpec {
    var name: String                 // 例「毒舌・皮肉屋」
    var voiceId: String              // 役割語テンプレID（口調）
    var primaryAbility: Ability      // 表の得意能力（稽古効率+）
    var secondaryAbility: Ability    // 裏能力（アーク/絆で開花）
}
struct BalanceConfig_archetype {
    var primaryTrainingBonus  = 0.20 // 【仮】得意能力の稽古効果 +20%
    var secondaryTrainingBonus = 0.10// 【仮】裏能力（開花後）+10%
    var enabled = false              // sim検証が通るまで既定off（golden不変を保証・§6）
}
```

- ボーナスは**稽古（Training）で対象能力が伸びるときに乗る**係数。生の上限（`abilityCap`）やティア（相性上限）は変えない。
- `enabled=false` が既定＝**現行のbalance/goldenは一切変わらない**。onにする前に §6 の再測定を必須イベント化。

## 5. 後輩・先輩（NPC・絆）への適用＝周回で変わる

`champion_selection_v1`／`relationship_champions` の関係役柄（同期`.douki`／可愛がってた後輩`.kohai`／飲みに行く先輩`.senpaiGenius`）に**アーキタイプを runSeed で割り当てる**と、周回ごとに「今回の後輩は天然」「今回の先輩は寡黙な職人」と**性格が変わる**（名前が毎周変わるのと同じ層・演出・非バランス）。

### 5-1. 関係役柄のアーキタイプ・プール（例・男女混在）
| 役柄 | 割り当て候補アーキタイプ（runSeedで1つ） | 備考 |
|---|---|---|
| 同期 `.douki` | 委員長／熱血／毒舌／クール系／ムードメーカー | 「先に行く」緊張感に合う気の強い型中心・男女両方 |
| 可愛がってた後輩 `.kohai` | ダンデレ／天然／体育会系／不思議ちゃん／妹系／三枚目 | 「育てたくなる」未熟さ・素直さの型・男女両方 |
| 飲みに行く先輩 `.senpaiGenius` | 職人気質／博士／姉御肌／クール系／お嬢様(苦労人化) | 「奢ってくれる」包容・一言の重み・男女両方 |

- アーキタイプで**絆イベントの口調と、優勝時セリフの声**が変わる（`relationship_champions §4` を役割語で差し替え）。**優勝時セリフ本文は性別・アーキタイプ非依存の骨**を保ち、口調の味付けだけ変える（煽り化・文量爆発を防ぐ・§0原則）。
- **能力アップの供給**: 後輩/先輩の絆イベントは**プレイヤーに小さな稽古ボーナス**を渡せる（例: 先輩がネタを見てくれる→センス系の稽古効率+一時的／後輩に教える→自分の表現が整理される等）。効果は【仮】最小・`EVENT_FIRE_CAP` 予算内（`dialogue_batch3 §8`）・§6のsim対象。

### 5-2. 楽屋NPCの多様化
図鑑22類型は楽屋のモブNPC・业界ニュースの脇役にも流用（男女混在の景色を作る）。台詞は口調テンプレ（役割語ID）で量産。

## 6. バランス規律（最重要・仮/sim/golden）

1. **全て【仮】・バランス影響あり**。得意能力ボーナス（§2）は稽古効率＝成長に効くので優勝率に影響する。→ **`GameConfig`／`sim_career.py` に集約し、on にする前に sim で全アーキタイプが目標優勝率帯（`canonical §2`）に収まる＝どの型を選んでも壊れない/死なない（sideways）ことを実測**（開発規律B・A）。
2. **既定 off で golden 不変**（§2 `enabled=false`）。実装時は「得意能力ボーナス」を新規rng非消費の決定的係数として `sim_career.run_year` と `gen_golden` に同期追加し、golden再生成＋`swift test` green を1コミットで揃える（規律A-1）。**それまでは設計のみ・現行の3年ビット一致は不変**。
3. **裏能力開花（§1-3）は表示分岐フラグ＋係数**で、アーク/絆の進行度から決定的に決まる（セーブ再現）。
4. **数値の初期案（要sim）**: 得意+20%／裏+10%／総合力は型間で±差が出ないよう、得意で伸びる分どこかが緩やかになる設計（例: 得意1能力+20%なら他4能力は据え置き＝合計の伸びは型で不変に近づける）。最終値は Fable の sim 較正待ち。
