# 予約仕事とバックれ・遅刻システム v0

- 日付: 2026-07-04
- 位置づけ: オーナー発案「仕事が入ってる時に別の行動をしたらバックれ・遅刻になる」の設計。既存のオファー（即時完結）に**先約という時間の重み**を足す。数値は全て【仮】。

## 1. 仕組み

### 予約オファー【新設】

- 既存オファーの一部（知名度20以上で発生分の約1/3【仮】）が「**○週後の仕事**」として来る（例:「3週後の第24週、地方局の収録です」）
- 受けるか断るかをその場で選択。受けると**カレンダーに先約が入る**（週送り画面に印がつく＝リマインドはUI表示のみ、モーダルで再警告はしない）
- 報酬は即時オファーの1.3倍【仮】（拘束の対価）

### 先約週の行動選択

先約週が来ると、行動メニューの先頭に「**仕事に行く（先約）**」が出る。ただし**他の行動も選べる**——ここが本体。

| 選択 | 結果 |
|---|---|
| 仕事に行く | 報酬受領（通常処理）。体力-20 |
| **別の行動を選ぶ＝バックれ** | 報酬なし・**知名度-3・以後8週オファー発生率半減**【仮】（業界は狭い）。翌週に事務所マネージャーの電話イベント（§3） |
| 体力20未満で仕事に行く＝**遅刻・ボロボロ出演** | 報酬半額・知名度-1。「行っただけ偉い」の中間結果 |

### 設計意図

1. **金と時間の約束が初めてゲーム内で衝突する**。大会前週に先約が入っていたら？——「3週後の自分の体力」を予測して受ける判断が生まれる（予約時点が本当の意思決定）
2. バックれは**選べる自由として残す**（禁止しない）。大会直前週なら合理的なバックれもあり得る——ペナルティを払ってでも稽古する判断はプレイスタイル
3. ペナルティは金でなく**信用（オファー率）**で払う——「業界は狭い」の世界観。8週で回復するので詰みはしない

## 2. 実装とバランス

- GameCore: `bookings: [週: OfferSpec]` を状態に追加。週処理順序は「大会 → GP → **先約判定** → 自由行動」（先約週に大会が重なる場合は大会優先で**自動キャンセル・ペナルティなし**＝大会は業界公認の欠席理由。予約時に大会週はそもそも指定されない設計でも可）
- 乱数消費が変わるため **gen_golden・CareerGoldenTests の再生成が必須**（イベント層と同時に実施）
- sim検証計画: ボットに「先約は必ず守る」を実装して初回優勝率1.2〜1.4%の維持を確認。予約オファーは実質「報酬1.3倍・週固定のオファー」なので収入は微増、稽古週が数週奪われて微減——概ね中立の見込みだが要実測

## 3. 会話・演出（抜粋。全量は dialogue_batch2_v0.md）

- **受諾時**: 谷口「3週後な。……俺、その日バイトやったわ。代わってもらお」／自分のカレンダーに書き、相方のシフトも動く——先約の重みを台詞で教える
- **前週のさりげない伏線**（リマインドの代わり）: 谷口「明日ちゃうで、来週やで、例の収録。……俺が覚えてるの、逆に不安やろ」
- **バックれ翌週（マネージャーの電話）**: 「……先方、カンカンです。今回は私が頭を下げました。**次はもう、電話が鳴りません**」／谷口は何も言わない。何も言わないのが一番効く
- **遅刻強行**: 現場で汗だくの二人。ディレクター「……まあ、来ただけ良しとしましょう。次はちゃんとした顔で来てください」（報酬半額の明細演出）
