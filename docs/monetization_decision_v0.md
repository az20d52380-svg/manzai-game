- 日付: 2026-07-04
- 位置づけ: `roadmap_v1.md` §2最上段の後戻り困難点について、オーナーが決めるための1枚。**推奨案つき**。ここでの決定がUI・経済・法務・開発工数の全てを分岐させる。

## 1. 両案の構造比較

| | A. 買い切り＋DLC＋コスメ（セッション設計の現行前提） | B. F2P＋相方ガチャ（拡張アイデア集v1.2原案） |
|---|---|---|
| 収益源 | 本体（数百〜千数百円）＋相方/シナリオ/審査員DLC＋コスメ | ガチャ課金（キャラ魅力型）＋広告＋広告カット |
| 収益の形 | 販売本数×単価（初速依存・上限あり） | DAU×継続率×ARPU（上限なし・運営依存） |
| 市場実態 | モバイルはF2P支配（国内アプリ収益1.6兆円/年）だが、**プレミアムはインディー市場の主柱**（インディー収益の61%・市場は年14.5%成長）。買い切りランキング文化も定着 | 国内モバイルの主流。ただし上位は大資本の運営型が寡占 |
| 開発追加コスト | 小（現設計のまま。IAPはDLC/コスメのみ） | **大**: ガチャ確率表示・天井・課金通貨の二重管理・ログボ/ミッション・イベント運営・サーバ・カスタマーサポート・運営人員 |
| 法務/審査リスク | 小（App Store 3.1.1のIAP準拠のみ） | 中〜大: ガチャ確率表示義務・未成年課金配慮・景表法（コンプガチャ回避）・運営中の炎上リスク |
| 本作の主題との相性 | ◎「下積みの時間の重み」と買い切りの誠実さが一致。「クリアは全員可能」を製品仕様にできる | △ 相方ガチャは人脈P設計（実力で相方に出会う世界観）と正面衝突。時間圧縮売りは「10年の重み」を毀損 |
| 設計済み資産との整合 | そのまま使える（item_design三大原則・partner_gachaオーディション制） | **人脈P経済・オーディション制・王者編ゲートの再設計が必要**（強さと課金の分離を保つ二重経済の再構築） |
| コンプ欲の受け皿 | キャラDLC＋コンプ図鑑＋称号（メモの「クリア無課金/コンプ課金」原則はこの形でも成立） | ガチャそのもの（最大出力） |
| 失敗時の下振れ | 売れなくても損失は開発費のみ。作品は残る | 運営費が恒常負担。DAUが立たないと即赤字。サ終でユーザー資産消滅 |

## 2. シミュレーションが示した固有の論点

- 本作の強さの主役は**才能ポイント（トロフィー）＝プレイでしか手に入らない設計**で全バランスが検証済み。相方は「決勝到達率を買う」補助レバー（SSRでも初回優勝3%未満）。**つまりB案でガチャに入れられる魅力的な商品＝相方は、幸い性能的にP2Wにならないことが実測済み**——B案が絶対に不可能というわけではない
- ただしB案では「人脈Pでオーディション」の物語装置を捨てるか、ガチャと二重にするかの選択になり、二重にすると経済が濁る（無料で出会える相方と引く相方の格差説明）

## 3. 推奨: **A案（買い切り＋DLC＋コスメ）で出す。ただしB転換の退路を残す**

理由（優先順）:
1. **一人開発の現実**: B案は開発より運営の商売。イベント運営・CS・サーバの恒常コストは個人開発の最頻死因
2. **主題との一致が最大の差別化**: 「初回は勝てない」を実測保証する潔さと買い切りの誠実さはセットで語れる。SNS時代のインディーは「切り抜きたくなる瞬間」×「作家性」で伸びる——本作なら決勝敗退の演出・審査員コメント・静物画の物語がそれ
3. **設計資産の温存**: 現行の全設計（人脈P・王者編・アイテム三原則）が無修正で使える
4. 収益上限の低さは**DLCの継続リリース**（相方パック・シナリオ・審査員・大会）で補う。コンプ図鑑が「次のDLCを買う理由」を常時生成する

### 退路の設計（今から仕込むこと）

- 相方データを最初から「入手経路と分離」して実装（オーディション/DLC/将来のガチャ、どれでも配れる形）
- 課金通貨を今は作らない。ただし人脈Pの獲得ログを取り、将来F2P化する場合の通貨バランスの実データにする
- スイッチ判断の基準を予め決めておく:【売上が開発費の回収ラインを6ヶ月で越えない ＆ DAU継続率が高い（遊ばれてるのに売れてない）】場合のみF2P化を検討——「売れてないから無料化」ではなく「遊ばれてるのに売れてない」だけが転換のシグナル

## 5. 決めてもらうこと（1論点）

**「A案で進める」の一言で、以降の全実装がこの前提で確定します。** B案に振る場合は、人脈P経済の再設計（1〜2週間相当の設計やり直し）から始めるため、7/8以降の軽作業では収まらないことだけ留意してください。
