## 0. 結論 — 可能か

**条件つきで yes。しかも狙いは正しい方向を向いている。**

「ボケを書かない」判断は逃げではなく、**Disco Elysium が世界的評価まで持っていった"最も再現性のある笑いの作り方"**（真剣な場面を、頭の中の内声のウィットでズラす）と構造的に一致する。下積みの反復を魅力化する路線も、天穂のサクナヒメ（稲作の反復）やパワプロ サクセス（指示通りにならない試行錯誤）が実証済みの王道。

条件は1つに集約される: **紙講評と俺の心の声の"筆致"の質**。ここが平凡なら普通の育成ゲーム止まり、ここが飛び抜ければ「これ面白い」と貼られる。コメディを"書かない"分、地の文の質が唯一にして最大の差別化点になる【設計上の懸念＝ここが弱いと企画全体が沈む】。

## 1. "大衆に面白いと思われる"の条件

口コミが生まれた育成/物語系ヒットに繰り返し現れる共通項と、本作の勝ち筋:

| 共通項（実在ヒットの観察） | 本作でやること |
|---|---|
| **最初の5〜10分で"今"面白い**（Day1離脱の最大要因は最初の5分。「いずれ面白くなる」では去る） | オープニングは**結成の瞬間＋初舞台の滑りまでを5分以内**。長い世界観語り・説明は禁物。掴みは「谷口という人物のおかしみ」と「俺の心の声のキレ」で取る |
| **感情の山＝勝敗より「投資を失う痛み→次に活きる報酬」**（Hades型） | 敗退＝ただの失敗でなく「講評で弱点が言語化され→次の稽古の選択が変わる」痛みと前進をセットに |
| **反復に物語を織り込む**（勝っても負けても毎回キャラの物語が進む） | 週送りの反復で谷口との関係・審査員との因縁が少しずつ進む。**同じ稽古でも心の声/地の文が変わる** |
| **共有したくなる瞬間（スクショ映え）** | **貼りたくなる結果画面を最優先UIに**。笑い波形＋紙講評＋通過/敗退を1枚で。実況者と一般が"これ見て"と貼れる形が口コミエンジン |
| **配信者・実況で伸びる**（インディー口コミの起点） | 実況者が講評や谷口の言動に**ツッコミを入れられる余白**を残す |

**先行事例の勝因**: Disco Elysium（内声のウィットで笑わせる＝本作の直接の手本）／天穂のサクナヒメ（地味な反復の魅力化）／NEEDY GIRL OVERDOSE（キャラ1点突破＋クリップ拡散）／8番出口（実況映えする"一点"）。

**最大リスク（直視）**:
1. **筆致が唯一の窓**。笑いの窓が紙講評しかない以上、それが Disco Elysium 級に上手いことが生命線。→ **毎回必ず出る講評・心の声を"資産"として技法タグ付きで量産・在庫管理**する。
2. **テキスト重視は万人向けではない**。ライト層の一部は文章量で離脱。→ 波形/UIで"一瞬で分かる"層と、筆致を味わう層の**二層に届く設計**が要る。
3. **筆致の可笑しみも外す人はいる**。→ "笑い"を保証するのでなく、**"下積みドラマとして沁みる"を主軸、可笑しみは薬味**にすると外れにくい（Disco Elysium も本質は tragedy）。

## 2. "ちょうどいいボケ"の筆致 — 具体技法カタログ

**背骨（全技法の下敷き）**:
- **具体性(comic specificity)** — 抽象語より固有の具体で笑う。「大きい」でなく「墓石ほどの前歯」(Sedaris)。**数字・固有名詞・変な具体物を1つ入れる**。
- **落差(incongruity / バソス)** — 高尚↔卑近、深刻↔些末の急降下（Alexander Pope が1727年に bathos と命名。Monty Python の定番）。**大げさに振りかぶって、しょうもない着地**。

原則は「面白いことを書く」でなく「**正確なことを、少しズレた角度から書く**」。以下、平凡→ウィットの書き換えで7例。

### 例1 — 過小表現（結果描写）｜英国式 deadpan / Wodehouse
- 平凡:「大爆死した」
- ウィット:「客席から笑いは、一度も、来なかった。強いて言えば、二列目の男性が一回だけ鼻で息を強く吐いた。今日の最高記録だ。」
- 効き所: **大惨事をさらっと軽く言う**と読者が脳内で落差を埋める。心配性で計算屋の俺と相性が最高。上品で外しにくく、**基調に据える主武器**。

### 例2 — 誇張（心の声）｜overstatement / Sedaris
- 平凡:「緊張していた」
- ウィット:「心臓が、出番を代わってくれと本気で交渉してきた。」
- 効き所: **膨らませる対象は小さいほど良い**（些事を宇宙規模に）。俺の心配性の内心に。※誇張は低頻度のアクセント。

### 例3 — 意外な直喩（審査講評）｜comic simile / Wodehouse
- 平凡:「テンポが悪い、と講評にあった」
- ウィット:「講評の字は丁寧だった。『間が、エレベーターの閉ボタンを押した後の時間ほど、長い』。褒めていないのに、この人は俺たちの舞台をちゃんと見ていた。」
- 効き所: 例える先(B)は**具体的・生活臭い・少し失礼**なものを。素材は「**温度・気象・水回り・小動物・役所/病院/葬式**」の引き出しから拾うと品よく滑稽になる（オーナー例の「凍る」もこの系統）。

### 例4 — 三段オチ / コミック・トリプル（審査講評の"品のある毒"）｜rule of three
- 平凡:「技術はあるが面白くなかった」
- ウィット:「技術・構成・熱量、いずれも申し分ない。あとは笑いがあれば、なお良かった。」
- 効き所: **2つは真っ当、3つ目で裏切る**。褒める・褒める・そっと足りないと言う——**丁寧な三段オチが講評の毒**になる。「敗退」を痛みにも笑いにもできる。

### 例5 — バソス／急転直下（大会の地の文の"抜き穴"）｜Pope / Monty Python / Douglas Adams
- 平凡:「二人は覚悟を決めて舞台に上がった」
- ウィット:「十年分の稽古、二人の意地、事務所の未来——そのすべてが、この4分間に懸かっていた。谷口の腹が、盛大に鳴った。」
- 効き所: **荘厳に振りかぶって、しょうもなく着地**。深刻を書き続けると読者が疲れる。緊張の連続に1回抜き穴を開ける。

### 例6 — ズレた冷静さ・自虐（心の声）｜計算屋の視点 / Sedaris
- 平凡:「不安だった」
- ウィット:「冷静に勝率を見積もった。三割。切り上げても三割。俺は残りの七割のことを考えないことにした。」
- 効き所: **渦中で場違いに冷静な分析をする**とズレて可笑しい。感情を数値・確率・損益で語る癖を俺の筆致に固定。

### 例7 — 擬人化・感情の物への転移（結果描写）｜Wodehouse / Pratchett
- 平凡:「気まずい沈黙が続いた」
- ウィット:「マイクだけが、俺たちの代わりに一生懸命ハウリングしていた。」
- 効き所: 沈黙・照明・マイク・時間などの**モノに、人間の情けない感情を持たせる**。舞台装置が心情を代弁。

### （補）例8 — 温度差の実況＋言外（谷口の描写）｜二人組の黄金配置 / サブテキスト
- 平凡:「谷口は自信ありげだった」
- ウィット:「谷口は台本を三秒眺め、『いけるわ』と言った。俺には、どこがどういけるのか、最後まで分からなかった。」
- 効き所: **俺=分析的で保険的／谷口=根拠なき断定**の温度差を、俺の一人称が淡々と観測・報告するだけで可笑しくなる。谷口の繊細な本音は**言語化させず**、事実を1〜2個だけ置き判断は読者に渡す（時々"下から情"がにじむ）。

### 【一線】ボケ(オチ)は書かない
上のどの例も**ネタの中身（オチ・パンチライン）は一切書いていない**。書いているのは「反応・観測・言い回し」だけ。俺=ツッコミ語り部の構造的優位はここにある——**ボケの中身を伏せ、俺がどう凍り/どう救われたかだけ書けば可笑しみは立つ**。ネタ本体は笑い波形・通過/敗退・審査講評が担い、会話劇と地の文は"人間味と筆致のウィット"を担う、と役割を分離する。

### テキスト班への運用ルール（外さないための安全弁）
1. **1テキスト＝技法1つ**。盛ると「面白がってる感」が出て寒い（最大の禁物）。
2. **具体物を1個だけ差す**（墓石/回転寿司/エレベーター級）。
3. **比喩の素材は温度・水回り・小動物・役所/病院/葬式**から。
4. **人物ごとに技法を固定**: 俺=誇張＆冷静な自虐、谷口=過小＆楽観、審査員=人格ある文体。
5. **講評の毒は丁寧な三段オチ**で刺す。
6. **基調は過小申告＋温度差実況**（万人に外しにくい）。誇張比喩（「凍る」型）は**低頻度のアクセント**。
7. **天丼（口癖・小物の再登場）**は状況を変えて回す。「いけるわ」「今日の最高記録」等。1回目が効いていないと効かない／3回目が最大。**同一ギャグの最短再登場間隔は GameConfig で管理【仮】**（近すぎると寒い・遠すぎると忘れられる。simと目視で詰める）。
8. **間は句点・短文・改ページ位置で作る**。可笑しい語は文末に。タップ送り＝手動の"間"というメディア的強みを使う。

## 3. 芸人もの先行作から学べること（ネタを見せず世界を描く）

- **又吉直樹『火花』**（芥川賞・354万部超） — **ネタの"内容"より、ネタを巡る"論・人・言い回し"を書いた**。文体自体にウィットを仕込み、コメディの performance でなく writing sense で可笑しみを出す。＝**本作の設計は文芸的に実証済みの正道**。選考は賛否両論だった＝**手法に正解は一つでない**ことの証左でもある。最優先で再読推奨。
- **『The Marvelous Mrs. Maisel』** — 見せ場は"完成品"でなく**"良くなっていく差分"**（改稿の過程）。→ **稽古イベントは完成ネタを描かず、「昨日より少しマシになった一点」を俺の視点で拾う**。成長を差分で体感させる。
- **対照群『べしゃり暮らし』**（ネタを全部見せた）／『コントが始まる』（ネタ=メタファーで人間関係にリンク） — 一般に「映画中のネタは**俳優**が演じるとリアリティが消え、**実際の芸人**だと出る」。→ **だから本作が"ネタを文字にしない"のが安全**。プレイヤーの想像に具体ネタを演じさせない判断は、この失敗リスクを回避している。
- **書き方への直結**: 講評は「面白い」と評さず**客席・審査員の身体反応を show**（例「前列の審査員が一度もペンを置かなかった。それは感心とは違う種類の集中だった」）。**審査員ごとに言外の欲求(サブテキスト)**を持たせる（古典派は"新しさ"を恐れている等）と、講評文が審査員のキャラも同時に立てる。台詞=建前／心の声=本音の**二層**でサブテキストは機械的に作れる。

## 4. 今できること（優先順）

**最優先の規律**: 書籍は"効き目の裏取り"であって、質を決めるのは実装と検証。順序を守る。

1. **実機で遊ぶ／プレイテストを回す**（最優先）。筆致の良し悪しは机上でなく「**タップして送った時に可笑しいか**」でしか判定できない。短いサイクルで **書く→simで送る→目視** を回す（CLAUDE.md §10・§7-B と一致）。まず S2週メインから触れるビルドを作り、講評・心の声を実際に送って読む。
2. **筆致を"資産"として在庫管理する仕組みを作る**。毎回必ず出る講評・心の声を、**技法タグ付き**（過小/誇張/直喩/三段/バソス/擬人化/自虐/温度差）で量産・管理。天丼の再登場間隔は GameConfig に持つ【仮】。これが企画の生命線なので、最も投資すべき箇所。
3. **貼りたくなる結果画面を最優先UIに**。笑い波形＋紙講評＋通過/敗退を1枚で。実況の余白（ツッコミどころ）を残す。口コミの起点はここと「谷口/心の声というキャラ愛」の二点に集約投資。
4. **オープニングを"最初の1本まで5分以内"に削る**。結成〜初舞台の滑りまで最短。説明・世界観語りを後ろに送る。
5. **反復の飽きは数値でなく筆致で解く**（§7-B）。同じ稽古でも心の声・谷口の一言・審査員の因縁が進む設計を先に。
6. **失敗を痛くする**。敗退講評で弱点を言語化し、翌週の選択が変わるループを作る。
7. **書く力を磨く具体行動**: (a) 上の技法カタログを"人物ごとに固定"して当てる練習を実データ（既存 dialogue/judge_comments バッチ）でやる、(b) まず Disco Elysium と Netflix『火花』を"体験で"触って型を掴む（購入前で足りる）、(c) Tier 0 の1冊（塙『言い訳』）だけ手に取る。
