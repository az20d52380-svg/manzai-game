# Fable投入計画 v0 — いつ・何に使うか（ultracode監査で確定）

- 日付: 2026-07-05
- 位置づけ: 「Fable（残りわずかな高性能モデル）をいつ使うべきか」を、3スカウト（Fable案件棚卸し／sim受け皿の準備状況／設計のlocked-moving）＋合成＋敵対的批評のワークフローで判定した結論。数値は全て【仮】。
- 前提: Fableは**構造的バランス/sim較正の判断だけに温存**。測定・実装・doc・量産は4.8/CLI。1論点ずつ呼ぶと枯れるので、相互連動する論点を1セッションに束ねて撃つ。

---

## 0. 結論（おすすめ）

**今すぐは撃たない。** 受け皿（sim側で測れる状態）がまだ大半 missing/partial で、方式が未確定の設計に投げると測り直しになる。

**発動タイミング＝下ごしらえ（§3）が揃い、オーナー判断2件（§4）が確定した瞬間に、1セッションだけ。** そしてFableに投げるのは**5問のスイープではなく、たった1つの構造判断**（§2）。機械測定は全部4.8へ、値のフォークはオーナーへ切り分ける——これが批評で最も強く是正された点。

**動いている設計（今Fableを投げると無駄になる真のmoving論点）は3つ**:
- **得意能力ボーナスの"方式"**（稽古効率+% / 上限+ / 初期値+ の3案未決）＝方式で測定対象そのものが変わる。最優先の障壁。

- ネタ資産フェーズ2（審査員相性を判定に効かせるか）＝judge §0の格上げでオーナー裁定が親。

（逆に**確定済み**: アーキタイプoverlay機構・王者の特権機構本体・ネタ補正係数0.04・champion抽選機構＝これらはFable対象外で既に決着。）

## 2. Fableに投げるのは"1問"だけ（批評の核心）

STEP2/3の選択・SSR cap22〜24・K/PEN/DELTAのグリッド・H3のON/OFF差分——これらは**全部「どの数値がどの帯に当たるか」の閾値探索＝4.8の作業**。目標帯（やり込み41.5%/のんびり23.1%、10連覇の看板圏、SSR=贅沢品）は既に確定しているので、Fableに生スイープを投げるのは層の取り違え。

**Fableに投げる唯一の構造判断**:
> 「王者特権dynasty × SSR再価格 × デバフ × 裏天井の帯が、**同時に**成立する解は存在するか。無ければ、どの帯を犠牲にするか（難易度トポロジの裁定）。」

4.8が各レバーの閾値表（十分なシード数＋信頼区間つき）を先に作り、Fableはその表を見て**連動する帯の同時充足を裁く**。これで残り少ないFableを最大効率で使う。

## 3. Fable前の下ごしらえ（順序・批評反映済み）

0. **【オーナー先決】§4の2件を確定**（特権解除値・裏天井効果式）。測定対象の定義なので配線前に。
1. **concerns#51: gen_golden/sim_careerの二重実装を解消**しrun_yearを単一正典化（4.8）。裏天井・王者特権がrun_yearを触るため最優先。
2. 王者の特権を sim_meta重複ロジックから **sim_career.run_yearへ共通化**、canon_v2.CHAMPION_GROWTH_END（dead constant）を実配線（4.8）。
3. exp_dynasty_matrix/exp_renpa/exp_v2_meta の run_dynasty を「優勝時にGROWTH_END_YEAR解除」へ更新し、特権あり10連覇率テーブルを再生成（4.8）。
4. **gen_golden再生成→swift test green→既定OFFでビット一致確認**（規律A・run_yearを触るので必須）。
5. exp_v2_metaのTIERSをSSR cap30→cap24へ反映しcareer_scan/dynasty再測（4.8）。※capが出荷run_yearに波及するなら§4同様のgolden工程を付す。
6. exp_challengerの目標帯をv2実測帯へ更新＋K/PEN/DELTAを多値スイープ化（cli-mac）。**concerns#51(手順1)の後に直列**（run_yearを書き換えるため並行不可）。
7. 上記を**PSpread/PCasual2等の強ボット＋十分シード数＋CI**で走らせ、Fableに渡す閾値表・マトリクス・ヒストグラムに整形。
8.（裏天井track・手順1の後段）2案→単一効果式へ統合し、exp_pity.py新設（s._dry_finalカウンタ＋PITY_ONフラグ＋perform経由BURST_P上書き）＋決勝初到達年ヒストグラム。**規律Aのgolden再生成＋消費順不変アサート必須**。

## 4. Fable前にオーナーが決める2件（測定対象の定義）

1. **王者の特権の成長期限解除値**: `25年`か`MAX_YEARS（実質恒久）`か。dynasty連覇率が変わる設計フォーク。現状 sim_metaは恒久・canon_v2定数は25で不一致。**どちらで配線するかを先に確定**（決めずに再配線するとdynasty再測ごとやり直し）。
2. **裏天井の効果式の骨**: 2案（例「8年目以降ハマ率+5%/年累積」/「3年ハマなしで翌年率2倍」）を単一化する方向。式が決まらないとsimが式ごと無駄になる。

## 5. Fableに投げない（4.8/CLI/オーナーへ切り分け）

- STEP2/3・cap22〜24・K/PEN/DELTAグリッド・H3差分＝**4.8の閾値探索**（CI付きで）。
- 得意能力ボーナス／絆ボーナス＝**方式がlockするまで着手不可**（missing＋moving）。
- ネタ補正係数0.04・roleChampPermille・content量産＝Fable対象外（実測済 or 演出/量産）。
- STEP2/3の最終選択・審査員相性の格上げ＝**オーナー裁定**。

## 6. コスト感

- Fable本体: **1セッション**（構造判断1問＋派生確認）。裏天井が統合済なら同便、外れれば効果式統合後に第2便。
- 前段の下ごしらえ: 4.8-cloud（新規機構・再配線 数件）＋cli-mac（デバフ再計測）で**数日規模**。
- **一番の浪費＝この受け皿投資を惜しんで、生データ無しにFableを撃つこと。**
