# ネタ資産システム プロトタイプ仕様 v0（周回版の目玉候補）

- 日付: 2026-07-03
- 位置づけ: `improvement_proposals_v0.md` §2-1/§7-3 の起案。「ネタ作り」の成果を抽象値でなく**資産としてのネタ**にする。MVP範囲外・周回版の第1候補。固有名は架空、数値は全て【仮】。
- 一言で: **「どのネタを、どの大会で、誰の前で掛けるか」**という漫才師の本当の戦略をゲーム化する。

## 1. なぜ入れるか（体験の欠落）

現状の「ネタ作り」は発想+3の抽象行動で、48週×10年やっても**自分たちの代表作が存在しない**。実在の漫才師の1年は「今年の勝負ネタをどこで下ろし、どこで擦り、決勝までに何本仕上げるか」で回っている（実在の決勝は2本必要）。ここがゲーム化されると:

1. 「ネタ作り」週に固有名の目標ができる（「秋までに回収型を仕上げる」）
2. 大会週に**出す/出さないの意思決定**が生まれる（新ネタを賞レースで下ろすか、営業で温めるか）
3. 審査員設計（judge_design_v0.md）の7軸と噛み合い、**相手を見てネタを選ぶ**メタ読みが生まれる

## 2. データ構造【仮】

```
ネタ {
  名前:     プレイヤー命名 or 自動生成（「コンビニ強盗」「留守番電話」等の題材名）
  型:       つかみ型 / 後半山型 / 伏線回収型 / 王道しゃべくり / 飛び道具  の5種
  完成度:   0〜100（稽古で上がる）
  鮮度:     100〜0（掛けるたびに減る。時間でも微減）
  下ろし済: この大会系統で使ったことがあるか（大会ごとのフラグ）
}
持ちネタ枠: 最大4本（5本目を作るには1本捨てる=「封印」）
```

## 3. 行動との接続（既存行動の意味を書き換える）

| 行動 | 現行 | ネタシステム後 |
|---|---|---|
| ネタ作り | 発想+3 | 新ネタ作成（完成度30・型を選択）or 既存ネタ改稿（完成度+8【仮】）。能力効果は従来どおり併存 |
| 舞台稽古 | 表現+6 | 指定ネタを劇場で試す: 完成度+12・鮮度-2【仮】＋従来の能力効果 |
| 営業場数 | 華+3 | 指定ネタの鮮度+5（客前で寝かせて練る）＋従来効果 |
| コンビ練習 | センス+3 | ネタに依存しない基礎値（従来どおり）——「ネタより二人の呼吸」の受け皿として残す |

## 4. 大会での効果【仮・要シミュ検証】

出場時に持ちネタから1本選択（決勝のみ2本→後述）。

```
ネタ補正 = (完成度 − 50) × 0.06        # 完成度100で+3.0、30で-1.2
         + (鮮度 − 50) × 0.02          # 掛け倒したネタは最大-1.0
         + 審査員相性ボーナス（決勝のみ、下表）
内部スコアに加算。合計補正は ±5 にクランプ
```

- **±5クランプが生命線**: endgame検証よりスコア+5≒相性+5≒初優勝1年前倒し相当。ネタ運用の上手さは「1年ぶんの差」までに制限し、才能解放（D）の主役を食わせない
- 同一大会系統に同じネタで再挑戦: 完成度補正を半減（審査員は覚えている）。**「新ネタを間に合わせる」動機の源泉**

### 審査員相性（決勝・judge_design_v0.md の7軸と接続）

| ネタ型 | 好む審査員（+1.0） | 苦手な審査員（-0.5） |
|---|---|---|
| つかみ型 | 音羽ルリ | 目白慧 |
| 後半山型 | 白波剛 | 音羽ルリ |
| 伏線回収型 | 目白慧・卯月走太 | 神楽坂とんぼ（「回収は道具や、主役やない」） |
| 王道しゃべくり | 天堂寺銀郎・花園千代 | 卯月走太 |
| 飛び道具 | 神楽坂とんぼ（ハマれば） | 天堂寺銀郎 |

※決勝の審査員は7人固定なので合計補正は型ごとにほぼ定数（+0.5〜+1.5の差）。「どの型でも勝てるが、型と審査員の物語が生まれる」に留める。

### 決勝の2本制【仮】

- 決勝進出時、持ちネタから2本目も自動要求。2本目の完成度が60未満なら決勝の内部スコアに-3【仮】
- 実在の「2本目問題」（1本目のエースネタで上がった後、2本目の弱さで敗れる）を再現
- **初回プレイの新しい敗因**が増える: 「決勝には行けたが、2本目がなかった」——次周の年間計画（2本仕上げる）を教える敗北

## 5. UI（ネタ帳の実体化）

- メニューの「ネタ帳」タブに持ちネタ4枠を手書き風カードで表示（improvement_proposals §4「ネタ帳UI」と統合）
- ネタごとに戦績スタンプ（「3回戦で使用・通過」）が溜まる → ネタが育つ実感＝疑似的なキャラクター
- 封印したネタは巻末の「お蔵入りページ」に残る（テキストのみ・供養）

## 6. シミュレーション検証計画（実装前に必ず回す）

1. `sim_career.py` に簡易ネタ層を追加: ボットは「最高完成度のネタを選ぶ／鮮度50未満なら次点」の2行ルール
2. 計測A: ネタ補正ありなしで初優勝分布がどう動くか（目標: 中央値±0.5年以内。超えるなら係数0.06を下げる）
3. 計測B: 2本制ペナルティ(-3)による決勝勝率の低下幅（目標: 初回の決勝到達17〜28%は維持したまま、決勝突破だけを絞る方向に働くこと）
4. 計測C: ボットのネタ管理を「ランダム選択」に落とした場合の差（=プレイヤースキルの寄与幅。±5クランプの妥当性確認）

## 7. 段階導入案

- **フェーズ1（周回版）**: 持ちネタ・完成度・鮮度・大会でのネタ選択のみ（審査員相性なし）
- **フェーズ2**: 決勝2本制＋審査員相性
- **フェーズ3**: ネタの型×イベントの接続（「切り抜きバズ(#23)は飛び道具型のとき発生率2倍」等）

## 8. 【設計上の懸念】

1. 週次行動が「ネタ指定」の1タップ増える。§4「3秒ルール」を守るため、指定のデフォルト（前回と同じネタ）を必ず用意
2. 4枠×鮮度管理は初見に重い。フェーズ1では鮮度の減衰を大会使用時のみにし、時間減衰は入れない
3. ネタ名の自動生成は题材リストの管理が必要（実在ネタの題材と偶然一致しても問題ない一般名詞に限定: 「コンビニ」「家電量販店」「結婚式の余興」等）
