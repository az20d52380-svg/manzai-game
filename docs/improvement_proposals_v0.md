# ゲーム改善・修正提案 v0 — 優先度付き32項目

- 日付: 2026-07-03
- 位置づけ: シミュレーション検証（career_report_v0/v1・endgame_design_v0）と設計4本（trophy/judge/event/dialogue）を踏まえた横断提案。◎=MVP必須 / ○=MVP推奨 / △=周回版以降。数値は全て【仮】。

## 1. リザルト・演出（初回100%優勝なし設計の生命線）

| 優先 | 提案 | 内容 | 根拠 |
|---|---|---|---|
| ◎ | 到達段階リザルト | 年次リザルトの主役を「所持金・能力」ではなく**「今年どの回戦まで行けたか」の縦の絵**にする（回戦の階段を登るビジュアル。去年の到達点にマーカー） | 初回は優勝できない設計のため、進捗の可視化が離脱防止の生命線（endgame§3-3） |
| ◎ | 敗退週の翌週テキスト | 敗退の翌週に必ず短い立ち直り会話を入れる（dialogue_design§4）。「負けたのに世界が無言」を絶対に作らない | 週送りゲームは敗北後の1週が最も離脱しやすい |
| ◎ | 才能解放の因果演出 | トロフィー獲得時に「D+1」を数字で見せず「limit break」的な演出＋「来年の成長が少し軽くなる気がする」の一文で伝える | 数式の露出は世界観を壊すが、因果は伝えないと周回動機にならない |
| ○ | 決勝の審査員採点演出 | 内部スコアを7人の表示点に分解（judge_design§0）し、1人ずつ開示。優勝ライン到達感をテレビ中継の文法で | 判定自体は不変なので実装コスト小・体験価値大 |
| ○ | 10年通しの年表画面 | 勇退/完走時に10年の出来事年表（大会成績・イベント・所持金推移）を1枚で出す→スクリーンショット共有導線 | 実績研究より「プロフィール公開欲」は強い動機 |
| △ | 敗者復活の生中継演出 | 通過率5%の狭き門である事実を逆手に、演出だけ最大級にする（奇跡が起きた時の記憶を作る） | endgame§5-3の「飾り」問題への演出的回答 |

## 2. システム深化（MVP後の本命候補）

| 優先 | 提案 | 内容 | 根拠 |
|---|---|---|---|
| ○ | ネタ資産システム | 「ネタ作り」の成果を抽象値でなく**ネタ（最大4本の持ちネタ、各に鮮度と完成度）**として持つ。大会でどのネタを掛けるか選択。同じネタは審査員に飽きられる（鮮度減衰） | 実在賞レースの「2本目問題」「ネタの寿命」を再現。ネタ作り週に固有名詞の目標が生まれる |
| ○ | つかみ・構成のネタ属性 | ネタに「つかみ型/後半山型/伏線回収型」の属性を付け、審査員の軸（judge_design）と相性を持たせる | 「目白は回収型に+、音羽はつかみ型に+」→大会前の戦略が生まれる |
| △ | 劇場ランク | 出演劇場を「地下→中箱→大劇場」の3段階にし、営業場数の効果と知名度上限を劇場ランクに連動 | 下積み→昇格の空間的な実感 |
| △ | ファン数の分離 | 知名度（世間）とファン数（濃さ）を分け、単独ライブ（event#34）はファン数を伸ばす | 「バズ(#23)は知名度だけ、単独はファン」の対比で現代お笑い論を機構化 |
| △ | 相方の状態変数 | 谷口にも体力/機嫌の簡易変数を持たせ、酷使すると稽古効果が落ちる | 「相方も生きている」（dialogue設計原則）の機構化。ただし管理負担増に注意 |

## 3. 経済・バランス（シミュレーション由来の宿題）

| 優先 | 提案 | 内容 | 根拠 |
|---|---|---|---|
| ◎ | 中盤の金シンク第1弾 | 3年目以降に「稽古場の月極契約（月20万で全稽古効果+1）」を解禁 | 実験C/F: 3年目以降の年+500〜650万の余剰の受け皿。恒久スコア加点ではなく成長効率なのでD設計と干渉しない |
| ◎ | イベント実装後の再検証 | イベント38種の期待値がプラス寄りのため、実装後に `sim_career.py` へイベント層を足して優勝なし率を再計測 | event_design§5【設計上の懸念】 |
| ○ | 序盤の金シンク | 数万円帯の消耗品（ネタ見せ会参加費・衣装クリーニング・録画機材）を「気分転換」の派生選択肢に | 1〜2年目の+12万〜38万の可処分に合わせた価格帯（実験F） |
| ○ | オファーの第3種 | 「地方泊まり営業」（+40万・体力-40・知名度+2）を知名度50以上で追加 | オファー2種の単調さ解消＋高知名度帯の収入カーブ用 |
| ○ | ライン微調整の余地確保 | 準決85/決勝95はボット基準。人間プレイテストで2〜3点下げる判断を最初のプレイテスト議題に固定 | endgame§3-2 |
| △ | 生活費のライフステージ | 6年目以降、生活費を10万→12万に（付き合い・後輩への奢り）。テキストで「先輩の焼肉(#4)の払う側になった」を回収 | 恩送りの円環を経済で語る |

## 4. UX・QoL

| 優先 | 提案 | 内容 |
|---|---|---|
| ◎ | 週送りの3秒ルール | 1週の操作（状況確認→行動選択→結果）を無操作3タップ・3秒以内に。48週×10年=480手の基礎リズム |
| ◎ | 大会週の自動ハイライト | 大会週はカレンダー上で発光＋2週前からカウントダウン。エントリー忘れという概念は作らない（仕様の「2週前確認」は演出のみにする） |
| ○ | 行動のリコメンド禁止 | 「おすすめ行動」表示はしない。体力・所持金の警告色までに留める（最適化の提示は考える楽しみを殺す） |
| ○ | 倍速・オート週送り | 方針固定（例:「体力40切るまで稽古」）の連続実行。周回プレイヤー向け。ただし大会週・イベント週で必ず停止 |
| ○ | ネタ帳UI | メニュー画面を「ネタ帳」の見た目にする（ステータス=ネタ帳の書き込み、実績=ページの隅の落書き） |
| △ | 実況ヒートメーター | 大会の判定演出中、客席の笑い声の波形を見せてから結果を出す（結果の1秒前が最も面白い） |

## 5. 周回UX（エンドゲーム設計の実装面）
