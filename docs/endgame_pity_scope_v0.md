## 1. 解く問題（数字で確定済み）

「初回プレイの約8割が決勝（山場）を見ずに終わる」を **CLI(4.8) が実測して裏取りした**。

**現状ベースライン**（`exp_v2_anchor.py 1000`・正典v2・現行ライン **準決74/決勝80**・10年キャリア・メタ無し＝初回相当）:

| ボット（プレイスタイル） | 優勝率 | 決勝到達率（10年で一度でも） |
|---|---|---|
| 分散稽古型（上手い） | 2.70% | 41.5% |
| バランス型（普通） | 0.20% | 8.4% |
| のんびり改（初心者寄り） | 0.30% | 23.1% |

→ **普通/初心者は10年やっても決勝を 8〜23% しか見ない**。ボットは人間より上手い（endgame_design §64）ので**実プレイヤーはさらに低い**。優勝が稀なのは設計意図どおり（「初回は優勝なしが基本」endgame_design §44）。**問題は優勝率ではなく「山場そのものへの到達率」**。

裏天井のゴール: **決勝未到達で晩年を迎えたプレイヤーに、キャリアで最低一度は決勝の舞台を踏ませる**。ただし**優勝率1〜2%帯を壊さない**（＝「保証するのは"到達"であって"優勝"ではない」）。

## 2. 動かせるノブ（既存実装済み）

| ノブ | 現在値 | 意味・出典 |
|---|---|---|
| `BURST_P`（balance_sim.py:177） | 0.10 | 「ハマった夜」発生率。勝負ごと独立・メンタル非依存。**A案で決勝到達を特別体験に戻すレバー** |
| `BURST_BONUS`（balance_sim.py:178） | 12.0 | ハマった夜のスコア加点（一発ライン超えの原資） |
| `EVENTS_ON`（sim_career.py:107） | False | イベント層（golden対象外・既定OFF） |
| `EVENT_FIRE_CAP`（sim_career.py:110） | None | 効果イベントのキャリア通算発火上限。会話増産時の総量予算（現行採用値15＝dialogue_batch3 §8） |
| GPライン（Calendar/`LINE_SETS`） | 準決74/決勝80/復活76 | `exp_endgame.py` の感度表がそのまま使える |

**裏天井の有力な形（START_HERE の記述）**: 「**決勝未到達のまま晩年でハマ率（BURST_P）を累積**」。つまり *未到達フラグ×経過年* に応じて BURST_P を漸増させ、晩年に一度は「ハマった夜」で準決/決勝ラインを越えさせる保証弁。**恒久能力を配らない**（優勝率が暴騰する: event_design §134 で無制限再抽選が0.9%→7.8%の前例）ので、一発性のスコア補正で「到達だけ」を担保するのが筋。

## 3. 守るべき制約（越えたらバランス崩壊）

1. **優勝率は1〜2%帯**（endgame_design §8-9・イベント層込みで初回1.5〜2.05%が正典）。裏天井投入後もこの帯を維持。
2. **キャリア1回制/総量予算**（event_design §134・§137）。効果の無制限再発火は禁止。裏天井も「保証は生涯1回の到達」に閉じる。
3. **1年目優勝は0%**（endgame_design §44）を保つ。裏天井は"晩年"に効く設計で、序盤を歪めない。
4. **絶対ライン判定は不変**（finals_direction §4-D）。表示の相対化はしても、内部は score vs line のまま。

## 5. Fableに委ねる設計判断（本タスクの核心）

1. **裏天井の発火条件**: 「決勝未到達 × 結成N年目以降」の N は？ 累積の形は BURST_P 漸増か、しきい値到達で確定発火か。
2. **保証の強さ**: 「生涯必ず1回決勝到達」を100%保証するか、"ほぼ"（例: 未到達勢の決勝到達を晩年で+40〜60ptして実質到達率90%台）に留めるか。**100%保証は優勝率を押し上げやすい**ので、到達率と優勝率のトレードオフを sim で確定。
3. **優勝率の再確認**: 裏天井ON設定で `exp_v2_anchor`/`exp_events` を回し、3ボットとも優勝**1〜2%帯**に収まるかを実測（=採否の合否基準）。
4. **業態判断への接続**: 到達率がどこまで上がると「初回体験の核心（山場を見せる）」が成立するか。オーナー判断①（B案 vs ハイブリッド）の材料として、到達率↑が事前登録・課金意欲にどう効くかの所感。
5. **実装インターフェイス**: GameCore 側は `GameConfig` に何を足すか（例: `endgamePityStartYear` `pityBurstGain`）。sim(sim_career.py) と1対1で追加し golden 影響の有無を確認（イベント層と同じく golden 既定OFFにするのが安全）。

## 付記: このMVP(1年版)との関係
検証①ビルド（`ManzaiGame/`）は**1年48週**で「飽きないか」を見るもの。裏天井は**フルキャリア(10年)の晩年**に効く別レイヤーで、両者は独立。1年MVPのプレイテストで裏天井は発火しない（＝検証①の邪魔をしない）。
