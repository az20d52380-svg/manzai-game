## 1. 機構確定（論点A/B/C——選び切り）

### 論点A：二区画中間で確定

**仕様（形＝確定・水準＝全て【仮】）**：

- 稽古が発行する経験点は二区画に分かれる：**同色ロック粒**（1−ρ）＋**共通枠粒**（ρ）。`expFreeShare`＝ρ＝**0.25【仮】**（`GameConfig`）。
- 共通枠は**2つ**で、写像は稽古の系と一致させる：**ネタ**（→センス・発想。ネタ作り/ネタ合わせの系）／**舞台**（→表現・華。ネタ見せ会/フリーライブの系）。各枠の射程は**2能力に限定**（安全条件(2)充足。完全多対多は作らない）。
- **メンタルはどの枠にも属さない**＝共通粒はメンタルに注げない・メンタル粒はメンタルにしか注げない（安全条件(3)：予算②を通らない能力へ新しい供給路を作らない）。
- 予算は現行どおり**総量プール単一**（`budget/w`・`add()`②の位置不変＝安全条件(1)充足）。発行ゲートには一切戻らない。
- データ実体：`GameState` に **`expセンス/exp発想/exp表現/exp華/expメンタル`（ロック5本）＋`expネタ/exp舞台`（共通2本）**、全て `Double`・初期値0（§2）。辞書でなく名前付きフィールド＝既存の能力5本と同じ持ち方・Codableの辞書順序非決定を構造で回避。

**理由（3行）**：(i) 危険（照準力の格差）が**ρ一つのツマミ**に落ち、「形はFable・水準はMac」に載る。(ii) ρ=0で同色1:1へ**コード改修なしで縮退**できる＝退避先を機構内に内蔵。(iii) 写像を「稽古の系＝ネタ/舞台」に重ねたので、判読性が**説明でなく枠のかたち**で立つ（§2のUI文法。共通粒チップは枠ヘッダに1つ置かれ、枠内の2行だけがそれを引ける——"なぜこの粒は選べるのか"は位置が言う）。

**断言しないこと（プロンプトの釘・正確に留保）**：「自由度が上がっても総量は器が固定する」は**総量に限って構造的に真**（`growthUsed += amt×w` の単一帳簿は不変）。**到達率分布（41.5/23.1/8.4の帯）に対しては未証明**——共通枠は「大会前に要求へ寄せる」照準力を素朴帯とやり込み帯で不均等に伸ばしうる。よって**sim実測ゲート必須**（§4の合格条件）。ρ=0.25は出発点であって答えではない。

**反証条件**：ρ∈{0.15, 0.25} のsimで帯の開き（やり込み−のんびり）が単一供給スケールで復元不能なほど割れたら（発行ゲート壊滅と同型の兆候）、ρを半減→それでも割れるなら **ρ=0（同色1:1）へ縮退**（config変更のみ・UIは共通チップが0表示になるだけで動く）。逆にρ≤0.25で帯が動かず、実機で配分の決定が薄いと出たら、枠の統合（共通1本化）や射程3化**ではなく**まずρ52を開ける——写像の拡張は照準力を段違いに上げるので、新しいspikeを1本立ててから。

### 論点B：タイミングは決定として立てない（即時が既定・銀行は中立バッファ）

中間案では**配分（どの能力へ）が主役**なので、プロンプトの指示どおりタイミングは即時に畳む。ただし「自動で注ぐ」のではなく**銀行は常に経由**する（機構のパスを人とボットで分岐させない）：稽古→粒が貯まる→「のばす」タイルに残粒バッジ→開けば「おすすめ」1タップで仮置き→「注ぐ」。素朴プレイヤーの自然な動線が即時パスになる。貯めることは**できる**（余剰許容の帰結）が、論点Cのガードにより貯めても**得はしない**（むしろ僅かに損＝逓減の複利）——タイミングは「罠でも技でもない、ただのバッファ」に落ちる。spikeが示した非対称な痩せ方（上級者は即時一択・初級者は無自覚上振れ）は、上振れ側を構造で殺したので消える（sim確認は§4ゲート5）。

**反証条件**：実機で「毎週開いて注ぐだけ」が作業感として出たら、畳む方向（週末に自動おすすめ注ぎのオプション）を足す——機構は不変・UI/設定の追加のみで対応できる形にしてある。

### 論点C：(b) 段刻みループを正典に確定——ただし「全経路統一」まで踏み込む

- **注入の最小単位＝`allocationStep`（1.0経験点【仮】・GameConfig）**。+1タップも・おすすめも・sim/goldenボットも、**全てこの刻みで `pourStep` を回す**。単位を跨いだ一括評価は機構のどこにも存在しない→貯め込みの1点評価上振れ（spike実測+3.6〜4.7pt）は**構造的に不能**（`AllocationTests.testStepLoopNeverExceedsSingleShot` が「段刻み≤一括」を単体で固定）。
- **見積もり＝確定の単一化は関数共有より強い形にした**：UIプレビューは「タップ列をstateのコピーに`pourStep`で再生」、確定は「同じタップ列を権威stateに`pourStep`で再生」——**同じ関数を同じ順で呼ぶので、丸め・逓減の食い違いが構造的に起きない**（`testReplayDeterminism`）。純関数 `projectedGain`（`add()`の①→②→③と同一式・副作用なし）は pourStep のゼロ利得判定とsim較正が共有し、`add()`との**厳密一致を全分岐グリッドの単体テストで固定**した（`testProjectedGainMatchesAdd`——逓減あり/なし×予算nil/満杯/残りわずか/十分×cap際×メンタルの全組合せでビット一致）。
- **ゼロ利得段は粒を消費しない**（器満ち・上限・逓減死では残高がそのまま残る＝余剰許容の実体。`testPourStepRefusesWhenBudgetFull`）。
- **現行との系統差を正確に言う（ここは誤魔化さない）**：現行は「稽古→イベント量（+3等）を1回のadd()」だった。段刻み正典では同じ量が1.0刻みで注がれ、**各段でvが上がるぶん逓減が複利で効き、イベント一括より僅かに下振れる**（一様・全帯共通）。これは供給スケールの再照準（Mac側sim較正の本業）で吸収する。**完全ロールバック経路**も config のみで在る：`expFreeShare=0`＋`allocationStep`を発行量以上（例999）にすれば「週内同色まとめ注ぎ＝単発add()」に代数的に一致する（唯一の端＝器が尽きる境界週だけ、色間の食い合い順が現行の稽古内main→sub順と入れ替わりうる。帯には効かない量だがgolden再生成時に消える差として認識しておくこと）。

### 2-3. AllocationView の設計（判読性の解き方＝中間案を採る条件の履行）

- **粒の文法**：塗りドット＝行き先が固定の粒（同色ロック・チップは各行の右に）／**輪郭ドット＝まだ色が決まっていない粒**（共通・チップは**枠ヘッダに1つ**）。
- **枠のかたちが二区画を言う**：ネタ枠カードの中にセンス行と発想行、ヘッダに共通チップ。**共通粒がその2行からしか引けないことを、説明でなくカードの包含関係が言う**。片方の行で共通粒を使うと、同じチップの数字が減る（枠内共有が操作1回で体感される）。補助線として器カード直下に一行だけ：「色の粒は、その色の項へ。共通の粒は、同じ枠のどちらへも。」（serif・inkDim）。メンタルは枠なしの単独カード＋「器を使わない。」（inkFaint・予算外を所作で言う）。
- **操作は「タップで即」文法**：＋で1段仮置き（`appearQuick` 0.18s・バーに`color.opacity(0.35)`のゴースト・チップの数字が`numericText`で減る）／−で最後の1段を戻す／「おすすめ」＝`recommendedPlan`を仮置きに展開（確定はしない）／「もどす」＝全解除。**仮置きはタップ順の列（`taps: [Ability]`）として持ち、プレビューも確定も同じ列を同じ順で再生**——枠内で2行が共通粒を取り合っても、見えていた見積もりと確定結果が食い違わない。
- **押せない時**：沈まず横ブレ（`ShakeEffect`±3pt×2往復0.15s）＋トースト1.4s・振動なし（閲覧扱い）。文言は原因で出し分け：「注げる粒がない。」／「この年の器は、満ちた。」／「ここは、上限まで来ている。」
- **確定**：「注ぐ（N粒）」（verm・タップ＝`Haptics.confirm()`）→**先に権威stateへ確定**（途中で閉じても状態は正しい）→リビールは後追い表示：溜め0.25s→伸びた行を上から1行ずつ`emphSpring`で立ち上げ（0.34s間隔・値は`numericText`で繰り上がり・「+N」は既存の非対称トランジション規格）→余韻0.5s。+Nの整数はWeekMainViewの規約と同じ「丸め差分」（`Int(after.rounded())−Int(before.rounded())`）、丸めて0だが実伸びありは既存の「伸びわずか」表記を流用（追補⑥のA/B/C判断が出たらここも従う）。
- **照準の道具（0040指示「次大会の要求帯はMVP同梱必須」への応答）**：「つぎの本番」カード＝大会名・N週後・**地力バー（実力値＋相性・ink）に仮置きゴースト＋朱の縦線＝要求ライン**・生数値（「じぶん 34」「要求 30」）。本作の判定は能力別要求帯でなく総合ライン（`perform`）なので、帯でなくライン1本が正直な形。共通粒を「どちらに注げばラインに届くか」で選ぶ——照準の自由（論点A）がここで初めて意味を持つ。
- **器カード**：金バー（used/budget・数値なし＝growthRoomの文法）＋仮置きぶんの薄い金の先食い＋地の文（既存2種）＋右肩に「粒 のこり N」。

## 4. 申し送り（Mac側：会計移設・golden・sim較正）

### 4-1. 会計移設（規律A第1段・1コミット）

- **発行側の差し込み位置**：`applyTraining`（main/subの`.ability`正加算）を「`add()`直結」から「二区画クレジット（ロックに`(1−ρ)`・所属枠に`ρ`）」へ。`applyRest`のメンタルボーナス・`applyOffer`の`ability`加算も同様に経済へ入れるか、当面`add()`直結のまま残すかは**移設スコープのオーナー確認1点**（推奨：稽古のみ第1段・rest/offerは第2段。負の加算＝生活苦/体調ダウンのメンタルhitと、トロフィー直接アップ・イベント効果は**経済外＝`add()`直結のまま**が正典）。
