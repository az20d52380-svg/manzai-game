■効くレバーの正体（ここを外すと丸ごと空回りする・最重要）
- 効く唯一の正しいレバー: 自前の年ループ内で、**最終年(year==YEARS==10)の run_year を呼ぶ直前に `s._yg` を X だけ下げて伸びしろを+X付与する**（＝火事場の頭金）。`add` の予算は `YEAR_GROWTH_CAP − s._yg` なので、run_yearが `B.YEAR_GROWTH_CAP` を再計算しても `s._yg` は温存され、+X がその年“だけ”生きる。state は run 毎に new_state(`B.S()`) で作り直すので run 間汚染は無い。
- 罠その1（A草案が踏みやすい）: run_year を呼ぶ“前に”外から `B.YEAR_GROWTH_CAP` へ +X を代入しても効かない。run_yearが年頭で累計予算に上書きして消す（`sim_career.py:359`）。プリセットは無意味。
- 罠その2（単純に cap を上げる誤り）: `CAP_CURVE` の一時lift で代用しようとするな。年間上限は `Σ_{k=1..year} max(2.0, 6.0−0.4×(k−1))` の累計で、base を+Xすると全10年の各項が+Xされ最終年上限は約+10X＝「最終年だけ+X」にならない（構造が違う）。
- RNG非干渉の担保: `s._yg` を下げる操作は draw を1回も消費しない＝乱数消費順は不変。だからこのレバーはgolden非干渉。**グローバル(BURST_P/CAP_CURVE/COMPAT_CAP等)を一時的に触った場合は必ず finally で完全復元**する（exp_pity3流儀）。Mac本実装時も「乱数消費順不変＝draw回数不変」が必須（規律A）で、このレバーはその制約を最初から満たす形になっている。
- 用量反応の意味付け（【仮】の目安）: X=+2.0 ≈「最終年に基準1年ぶん(≈6.0の1/3)の追い成長」、X=+4.0 ≈「約1年分の伸びしろ復活」。0/+1/+2/+3/+4 でスキャンし用量反応曲線を出す。

■このセッションの唯一の最優先＝ラストイヤーの奇跡の較正 — 今すぐ着手・全時間をここへ
狙い（良いゲーム観点）: 本編10年の最終年(10年目)に「決勝で勝てないまま来た停滞コンビ」へ“最後の一花”を数学的に蘇らせる。ただし確実な必勝ボタンにはしない——最終年“単体”の決勝到達/優勝が“意味ある幅で”上がればよく、確実に勝てるようになってはいけない。「もう後がない、でもまだ伸びる」という感情のピークを実パワーに乗せることが目的で、優勝を配ることは目的ではない。

入力（全て【仮】・再確認不要）:
- 死んでいる構造: 成長は `CAREER_BUDGET` の累計予算制。年間上限=`max(2.0, 6.0−0.4×(年−1))` を1〜10年で足し込んだ累計が天井。消費済みは `s._yg`（初期能力ボーナスは前借り）。累計制ゆえ10年目には予算がほぼ尽き、最終年の伸びしろが枯れる＝「後がない芸人がまだ伸びる」演出が実パワーに乗らない。これが§1「奇跡が数学的に死んでいる」の正体。
- 緩和候補: アーク②「火事場のクソ力」を「最終年だけ 伸びしろ +X」に読み替える（X∈{0,+1,+2,+3,+4}【仮】、0=baseline。+1.0は仮の出発点で、幅は実測で決める）。
- 併存する別レイヤー: 裏天井（TH3/FLOOR8/BP0.30/決勝除外/初到達で恒久解除）は“初回体験の保証弁”＝準決以下のハマ率up。8〜10年目のまさにこの停滞層に既に発動している＝母集団が重なる。今回の最終年緩和は“実力の最終加速”＝別レイヤー。二重救済で優勝を配らないよう合成を必ず測る。

■合格条件（探索“前”に、数値許容上限まで登録してから回す・全て【仮】・後付け禁止）
以下の許容±を回す前に確定して宣言すること。探索後に基準を緩めない。
- 不変G1（0pt初回・のんびり優勝）: 基準 ≈0.62%。許容上限 ＿＿%（例: +0.15pt以内＝0.77%を超えたら違反、と数値で先に置く）。
- 不変G2（0pt初回・やり込み優勝）: 基準 ≈3.50%。許容上限 ＿＿%（例: +0.5pt以内）。
- 不変G3（30ptトロコン・やり込み優勝）: 基準 ≈62.1%（決勝到達≈98.9%）。許容上限 ＿＿%（例: +1.0pt以内＝“集めた強コンビの複利”に漏れない証明）。
- 蘇生G4: 「9年目終了時 ever_final==False で最終年に来た停滞コンビ」の最終年“単体”の決勝到達/優勝が、意味ある幅（下限を数値で先に置く）で上がる。
- 選択則: G1〜G3を割らずにG4が最も立つXを採る。両立しないなら、どの帯を犠牲にするかを明示して裁定（難易度トポロジの判断＝人間モデルに投げる価値がある核心）。

思考の型（この順で結論を出す）:
1. 診断ファースト: `exp_lastyear.py`(exp_pity3の年ループ流用)を起こし、まず緩和OFFで「10年目の残成長予算 ≈0」を1本の実測で示す。次いで「最終年に決勝未勝で来た停滞コンビの、最終年単体の決勝到達/優勝が実際どれだけ枯れているか」を実測。奇跡が死んでいる度合いを“数字で”確定する。ここを外すと緩和が的を外す。
2. 停滞コンビ母集団を測定用に切り出す。全体平均では奇跡は埋もれて見えない（＝ここが判断の肝）。既存の `run_one` は“全キャリアの won”を返すだけで、9年目終了時 ever_final==False で絞った上での“最終年だけの決勝到達/優勝”を分離しない。**台をこの目的で拡張してよい**（run_yearには触れない・experiment-onlyの集計拡張のみ）。「ever_final追跡は既存にある」と過信せず、最終年単体の条件付き集計を自分で足す。
3. 用量反応: X を 0→+4 で振り、各Xで (a) 停滞コンビ最終年の到達/優勝（蘇り度）(b) 不変帯G1/G2/G3（漏れ）を同一ハーネスで実測表に（台・シード・n付き）。
4. 二重救済チェック（この問いの肝・必須の独立ステップ）: 裏天井ON × 最終年+X の合成セルを測り、「立たせる(裏天井＝準決以下ハマ)」と「伸ばす(最終年＝実力の最終加速)」が別レイヤーとして加算的に効くだけで、**決勝の“条件付き優勝率”を跳ねさせないこと**を確認する。跳ねるなら**どのXで漏れ始めるかを特定**する。1セル/1点の目視で済ませない。
5. 必勝ボタン化の回避を機構で言語化する。「+Xしても最終年単体で優勝が確実にはならない（＝一花であって花束ではない）」を、据え置く力学として言葉にする——実力ライン下の停滞コンビは伸びても決勝の素ハマ10%の夜を越えにくい／裏天井との合成でも条件付き優勝が積み上がらない、等。予算トポロジのどこを緩め、なぜ強コンビ/決勝条件付き優勝率へ漏れないかを言語化する。

■余力があれば（優先度順・軽く）
- 緩和の“形”の弁別: 「最終年だけ+X一括」vs「9年目から段階的に緩める」vs「下限floor=2.0を最終年だけ引き上げる」——実測で弁別が作れるならどの骨が“最後の一花”の手触りに最も合うか1メモ。作れないなら「一括+Xで十分・config増やさない」と明記。
- 温存の誠実な出口: 実測で「緩和が自明に安全（帯無風で停滞コンビだけ静かに蘇る）」または「自明に無害でない（どの幅でもG1〜G3を割る）」と分かり構造判断が軽いなら、机上メモと実測表だけ残して“確定フリップを撃たない”選択をオーナーに提示してよい。**ただし撃たない場合でも**、「奇跡が死ぬ原因＝累積予算トポロジ」と「緩和が角の複利／裏天井合成に漏れないか」の裁定は、必ず“数字で”残すこと（空手形の温存を禁ずる）。
- 演出への申し送り: 蘇生が“見えない救済”にならないよう、最終年の緩和が発動する年の非明示フレーバー（谷口の「今年が最後や、まだ伸びるぞ」等）を1行、finals/dialogue側TODOとして残す（演出側・非バランス）。

■出力の形（結論は `docs/lastyear_calibration_v0.md` に【仮】で・pity_calibrationの型を踏襲）
1. 推奨: 緩和機構＋緩和幅X＋対象母集団＋発動条件（全て【仮】）。または“撃たない”提案＋その根拠。両立しない場合の犠牲帯の裁定を明記。
2. 根拠の実測: 回した台＝`exp_lastyear.py`・シード・n・用量反応表（OFF→+Xスキャン）・停滞コンビ条件付き表・**裏天井ON×最終年+Xの合成セル**（どのXで漏れ始めるか）を貼る。
3. 機構的説明＋確信度と残リスク: なぜ不変帯を壊さないかの“据え置く力学”、特に角の複利／裏天井合成へ漏れないかの見立て。
4. Fable後のTODO:
   - Mac＝run_yearへ最終年緩和を本実装（最終年のみ `s._yg` を−X相当で伸びしろ+X・**乱数消費順を変えない＝draw回数不変が必須**・規律A／`gen_golden` diff空＋`swift test` green を1コミット）。
   - 4.8＝正典ハーネスで再測し canonical / red_team §1 の該当行を確定値へ。
   - オーナー＝緩和幅Xの体感ダイヤル（“最後の一花”が強すぎ/弱すぎの最終選択）と発動年の非明示演出の方向性。

時間配分はこの1問に全振り。時間が尽きても、診断(手順1)＋「不変帯G1〜G3を割らずに停滞コンビ最終年を蘇らせるX候補1本」を実測付きで出せれば成功。残りは箇条書きの申し送りにせよ。
