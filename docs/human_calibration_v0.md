# 人間プレイヤー補正の実測と「分散稽古」問題 v0 —【設計上の懸念・最重要】

- 日付: 2026-07-04
- 位置づけ: 検証台帳の残項目「人間プレイヤー補正」に回答するため、下手な・偏ったプレイスタイルのボット4種を追加して初回キャリアを実測した（`tools/exp_human.py`）。その過程で**現行バランスの土台を覆す構造穴**を発見。修正候補を5系統実測した（`exp_human_fix.py` / `exp_human_fix2.py`）。
- 結論の要旨: **「初回優勝1.5%」はプレイスタイル依存の局所解だった。稽古週率を上げて能力を分散で伸ばすだけで優勝100%になる。** 借金ペナルティでは直らず、確実に効くのは「年間成長上限」だが全ラインの再調整（再アンカー）が必要——オーナー判断待ち。

## 1. 発見（プレイスタイル別の初回10年・イベント込み正典・2000キャリア）

| プレイスタイル | 優勝 | 決勝到達 | 10年目実力値 | 稽古週率の目安 |
|---|---|---|---|---|
| バランス型（従来の基準ボット） | 1.55% | 34.8% | 64.5 | 約5割 |
| 調整型（従来の基準ボット） | 1.50% | 25.3% | 64.6 | 約5割 |
| のんびり型（初見の人間近似） | 2.25% | 39.1% | 65.2 | 約5割 |
| **ランダム型** | **66.5%** | 95.3% | 78.0 | 約4.5割（ただし5種に分散） |
| **バイト特化** | 0.00% | 0.0%（1回戦すら1%） | 17.0 | 0割 |
| **稽古特化（無料3種を回すだけ）** | **100.0%** | 100.0% | 81.7 | 約8.5割 |

- 従来の「ボットは人間より上手い＝実測値は上限」という前提は**誤り**。従来ボットは (a)稽古を金持ち時に1種（ネタ見せ会）へ集中する (b)金策週が多い——の2点で実力値64で頭打ちになっていただけ
- 真因は2つの掛け算:
  1. **能力ごとの成長逓減 ×(1−現在値/D)** は、1能力への集中を罰し**複数能力への分散を構造的に最強**にする（ランダムですら強いのはこのため）
  2. **マイナス所持金にペナルティがない**ため金策が実質不要で、稽古週率を8割超まで上げられる
- 実力値78〜82＋相性20 ≒ 98〜102 は、決勝の実効ライン92.5（94−人気補正1.5）を安定して超える
- **MVP（1年版）は無傷**: 1年では実力値30前後にしかならず、この穴は表面化しない。これは10年ループ（本編）の問題

## 2. 修正候補の実測① 借金・金圧レバー（全て不発）

| 修正 | 稽古特化 | 分散稽古型※ | 判定 |
|---|---|---|---|
| A 借金中は稽古効果半減 | 100% | 100% | ×（賞金収入で借金を回避できる） |
| B 借金中は稽古効果ゼロ | 0% | 100% | ×（踏み倒し型だけ死に、金管理型は素通り） |
| C 生活費未払いで生活苦（体力-15/メンタル-5） | 100% | 100% | × |
| P1 無料稽古の有料化（2〜3万の稽古場代） | 0% | **99.5%** | ×（バイト1回で稽古2〜3週が賄える） |
| P2 逓減を実力値基準に変更 | 100% | 100% | ×（逆効果: 集中型まで解放されバランス型も98.5%に）|

※分散稽古型 = 新設の「正解プレイ」近似ボット（残高150k維持・5能力の最低を埋める・`exp_human_fix.PSpread`）

**教訓: 金をいくら重くしても、バイト1週=稽古2〜3週ぶんの収入がある限り稽古週率は下がらない。逓減の向きを変えると別の壁が崩れる。**

## 3. 修正候補の実測② 年間成長上限（効く・ただし再アンカー必須）

演技系4能力の**実力値換算の年間成長量に上限**を設ける（`balance_sim.YEAR_GROWTH_CAP`・年初リセット）:

| 年間上限 | 稽古特化 | 分散稽古型 | バランス型 | のんびり型 |
|---|---|---|---|---|
| なし（現行） | 実力81.7・優勝100% | 78.7・100% | 64.5・1.3% | 65.1・1.9% |
| 7.0 | 73.0・98% | 76.3・50% | 55.9・0% | 57.5・0% |
| 6.0 | 70.0・17% | 69.9・6% | 54.0・0% | 55.8・0% |
| **4.5** | **55.0** | **55.0** | **50.3** | **51.3**（優勝は全員0%＝ライン再調整前） |

- 上限4.5〜5.0で**プレイスタイル間の実力差が28pt→2〜5ptまで収束**する（「誰でも上限に届くが、上限までしか行けない」）
- この帯なら決勝ライン（例: 75〜78程度＝相性20込みで実力55前後が挑戦圏）を引き直すことで「初回1%・準決ほぼ全員」を**プレイスタイルに依存せず**成立させられる
- 副産物として筋が良い点: **トロフィー（才能解放）を「D加算」から「年間上限+α」に置き換えられる**。Dは±1で優勝率が激変する剃刀レバーだったが、年間上限は線形で予測可能。「実績が成長の天井を押し上げる」という物語も同じ
- プレイの腕は「上限を何週で使い切るか」→残り週を金・ネタ・知名度・体力管理に回す配分ゲームへ移る（週の使い道の多様化＝ネタ資産システムとも好相性）

## 4. 推奨と影響範囲【オーナー判断待ち】

1. **推奨: 年間成長上限方式への移行**（上限4.5〜5.5帯で再アンカー）。移行時に引き直すもの: GP全ライン・道中大会ライン・トロフィー効果（D→上限+α）・王者編STEP・事務所/相方の効き——**実測ドキュメントの数値は全て旧アンカー前提になる**（設計結論〈複数ルート・初回勝てない・飽きられ等〉は再現可能）
2. 次善: 現行Dのまま「稽古の連続実行にソフト減衰（同カテゴリ連打で効率低下）」等の行動側ナーフ——未実測・小手先の恐れ
3. 実装状況: 全レバーは既定OFFのフラグで `balance_sim.py` / `sim_career.py` に組み込み済み（goldenバイト一致を確認済み）。採用判断が出れば `exp_human_fix2.py` の枠組みでライン再調整に即入れる
4. 【重要】この決定まで、**既存ドキュメントの「初回優勝1.5%」等の数値は「従来ボット帯（稽古週率5割・集中育成）での値」と読み替える**こと

## 5. 【2026-07-04追記】正典v2移行の決定と第1次アンカースキャン

**オーナー委任により移行を決定**（master_spec 未決8→確定）。生活ルール2種（借金3段・ケガ制＝rule_holes_v0）と年間成長上限をセットで正典v2とする。第1次スキャン（上限4.5・下位ライン0.55倍圧縮・準決52〜60×決勝72〜80の格子・500キャリア）の結果:

- 決勝72で のんびり1.0〜1.6%・バランス1.8〜2.2%（目標帯）だが、分散稽古型が54%
- 決勝80で分散型6%まで絞れるが、のんびり帯は0%（決勝の壁が高すぎ）
- **原因を特定**: 固定上限では、カジュアル帯が**晩年に失速**する（逓減×少ない稽古週で8年目以降の自然成長が2.0/年未満に落ち、上限4.5に届かない）。上級帯は毎年上限を使い切るため、差が晩年に蓄積して10〜12点開く

### 第2次設計方針: 年齢カーブ型上限

`上限(year) = 6.0 − 0.4×(year−1)`【仮】（1年目6.0→10年目2.4）。若手ほど伸び、ベテランは頭打ち——実在の芸人の成長曲線とも、王者編の「飽きられ」とも整合する。若年時は誰でも上限に到達（差がつかない）、晩年は上限自体が小さく差の蓄積が止まる。

### 5-B. 第2次スキャン結果【2026-07-04・年齢カーブ(6.0, -0.4/年, 下限2.0)＋生活ルール一式＋下位ライン0.55倍】

- **収束成功**: 10年目実力値の帯が のんびり47.5〜分散型52.0（差4.5pt。旧方式は28pt差）
- **優勝率アンカー確定候補: 準決52・決勝73（人気補正1.5込み）**（n=1000）:
  - のんびり型 1.60%・バランス型 1.30% ——「普通にやったら1%」の目標帯
  - 分散稽古型（ほぼ最適プレイ）30.3% ——上手いプレイヤーは初回でも勝ち得るが、王者編は18ptゲートで封印されたまま
  - 鉄人型（金策放棄）は破産100%で退場
- **トロフィー効果は「上限カーブ+0.02×pt/年」で機能**: 0pt=1.6% → 10pt=2.2% → 20pt=3.4% → 30pt=5.0%（N相方・素の腕のまま）。相方ティア（相性上限+10）がここに乗る
- **【残課題・最重要】決勝到達が97%になり旧体験（25〜35%だけが決勝で敗れる）から変質する**。原因: 成長が収束・早期平坦化したため、ブレ幅による準決通過が10年で累積する（毎年15〜45%の通過チャンス×多年）
- 【第3次スキャン・不発】終盤ピーク型（上限3.0+0.3×年・逓減廃止）も検証したが、逓減を切るとメンタルが即カンストして**ブレ幅が±5に痩せ、上位ラインが数学的に閉じる**（全設定で優勝0%）。ブレ幅（メンタル式）は「勝負の運」の供給源であり、成長制御と独立に設計し直す必要がある——という構造理解を得た
- 残る対策候補: (i)決勝進出の相対選抜化 (ii)ブレ幅の2層化 (iii)「決勝常連なのに勝てない」への体験再定義 → **【2026-07-05オーナー決定: A案（決勝は特別な場所のまま）】**、対策(ii)で実装する

### 5-C. A案の実装と実証【2層ブレ「ハマった夜」】

- 仕組み: 恒常ブレ（メンタル依存±5〜20＝安定感）に加え、本番ごとに独立な**「ハマった夜」抽選（P=10%・スコア+12【仮】、`balance_sim.BURST_P/BURST_BONUS`・既定OFF・goldenバイト一致確認済み）**。準決ラインを恒常ブレの届かない高さに置くことで、決勝進出は「ハマった夜」にだけ起こる特別な出来事に戻る
- 実証（準決76・決勝82・n=1000・生活ルール一式＋カーブ上限）: 分散稽古型=優勝0.70%・決勝到達28.7%／のんびり改（大会前週に休むだけの初心者）=0.20%・13.9%／**決勝到達が「2〜3割の特別体験」に復帰し、かつ最適プレイでも優勝1%未満**——旧v2候補（分散型30%優勝）の歪みが解消
- のんびり「改」の意味: 素ののんびり型（大会前も休まない）は到達5%。ヘッダの大会バッジを見て前週に休む——UIが教える最低限——だけで13.9%に上がる。**「準備する」がプレイの腕として正しく効く**
- 最終値の確定: 準決74〜76×決勝80〜82×ハマ+10〜12の最終グリッド1回で「普通にやったら1%」に着地させる（次バッチ）。その後トロフィー・相方・王者編の全再スキャン
