## 2. 反証で崩れた/但し書きが付いた点（正直な列挙）

**全主張が独立反証を生き延びたわけではない。** Fableを立てる必要はないので正確さ優先で列挙する。confirmedの3点（B・E・G後半・H）は§1の通り但し書きは軽微（値の絶対水準のシード依存性のみ）で、載荷主張は無傷。以下は覆し/要修正の5点。

### 2-A【覆し・最重要】X=2の「両側一意」は単一シードのナイフエッジ

Fableは「整数ダイヤルの解は1つしかない」と述べたが、これは**元シード20260704の偶然の配置**。8シード族で合格X集合を再測すると `{2}` は元シード1族のみ、7族が崩れる。二つの独立破綻モード:

1. **X=3のG2硬上限が非ロバスト**: 元シードでは G2(X=3)=7.38% が7.00線を辛くも超えるが、他6族中5族で **4.88〜6.88%と7.00%内に収まりX=3が共解**になる。
2. **X=2のG4下限が非ロバスト**: 3族（50M/70M/80M）で停滞層優勝Δが +1.21/+1.41/+1.40 と登録下限+2.0未達となり、**X=2自体が失格**しX=3が唯一解に転落。

いずれの拘束（7.38 vs 7.00線／優勝Δ≈3.95 vs +2.0）も**シードノイズの射程内**（停滞層n≈500・優勝率2〜6%でモンテカルロ幅大）。→**単一シードでX=2を確定するのは危険。§3・§4で正典ハーネス複数シード平均での再選定を必須化。**

### 2-C【要修正】G3の「≤65.6%固定天井」は独立シードで突破される

構造（ゲートがG3上昇を停滞層シェアの自己蘇生に有界化・飽和・テール漏れゼロ）はX=50まで全シードで堅牢。だが**絶対値65.6%はbaseの高い独立シード20360704で65.875%（X≥6）に突破**。入力パラメータ「シェア5.4%」「基準優勝34.88%」もn=800単一標本のアーティファクト。→**Cは `base_G3 + stagnant_share×(1−base_st_win)` というconfig固有の式として述べ、65.6%を固定上限として提示しない。**

### 2-D【要緩和】Dの但し書き2本（差≤1.3pt・符号全負）はシード局所値

本質（正の乗算項なし＝生存）は確証。だが「符号全負」は既定シードX0/X1で正の反例、「差≤1.3pt」はseed999983で全崩壊（−2.48〜−3.33）。→**閾値表現を「符号は概ね負・OFF/ON同等オーダー・有意な正の相乗なし」へ緩める。** 負方向拡大はむしろ反相乗機構を強化する。

### 2-F【要修正】Fの「後段−0.12pt完全飽和」は基準シード専用

「角は必勝化しない(<90%)」は圧倒マージンで確証（崩れない）。崩れたのは補助主張の数値のみ: クロスシードの後段tailは**≒0でなく+1〜2ptの浅い正勾配**（飽和の向きは正しいがフラット/負ではない）。n2000で2/16が>+3ptに触れるが高nで消えるノイズ。→**「後段は≒0のプラトー」でなく「後段は+1〜2ptの浅い正・角<90%は不変」と記述。** なお**のんびり角（壁2）では後段tailが全シード+5.05〜+6.85ptと明確に非飽和**——「完全飽和プラトー」を一般命題として読むとのんびり側で破れる（ただしF規範の90%はやり込み角限定なので載荷主張は無傷）。

### 2-G【要修正】絆は「壁を破る侵食ベクトル」ではない（到達は押すが優勝は飽和）

leave-one-out首位（+4.62pt）は確証。だが「侵食ベクトル＝増幅で壁を破る力」は否定: 絆をcompat_start+8から+40まで盛っても**優勝は+16以降6.75%で完全ビット不変（ハード飽和）**、増幅後は得意A×2.0が追い抜く。**絆が押すのは到達（76.5→85.0%）であって優勝ではない。** 壁はcompat clampでなく**のんびりの実力/到達構造**に由来。→「絆が最有力"侵食"」は「絆は到達レバーとして最大だが優勝天井は割らない」へ。将来レバーの角チェック第一問（「稽古量を要求しない直付与か」）は有効。

## 3. Macへの申し送りの補強（反証で見えた本実装時の注意）

反証は攻撃であると同時にMac本実装への**配線チェックリスト**になった。

1. **ゲート発火判定にrng不使用（B・E で構造確認）**: 停滞層ゲートの発火（ever_final==False判定）は**乱数を消費してはならない**。attack_Bで非停滞層のbit不変（誤適用0）を確認済み＝この構造保証がgolden一致とG3漏れゼロの両方を担保する。run_year本実装で`_yg`を触るのは停滞層のみ・判定は決定的にすること。**乱数消費順は runYear/WeekRunner と一致**（規律A-2）。

2. **`ever_final`（≒ever_final/9年目終了時決勝到達フラグ）の配線が全ての土台**: A・B・C・G4の全判定がこのフラグの正しさに乗る。Mac実装で `ever_final` の集計タイミング（year9終了時点）がsimと1週でもズレると停滞層シェアが動き、A/C/Dの数値が全て変わる。**このフラグ配線を最優先で golden 突き合わせ。**

3. **15pt常設（＝X常時開放でなく停滞層限定ゲート）の維持**: Eでのんびりが純スラック＝CAP非到達と確認。ゲートを外して全員にXを配ると（Fable §4のゲート無し G3+13.4pt漏れ）非停滞層に漏れる。**ゲート必須は反証でも揺らがない設計の核。**

4. **絆の見張り線「のんびり角優勝≤15%」を常設アサートに（G後半）**: 本攻撃で天井が構造的（絆+40でも≤~10%・多シード最大10.2%）と判明。Mac本実装後の正典ハーネスに**このセルを回帰テストとして常設**し、15%超えを設計崩れの検知線にする。

5. **X=2は"単一シードで確定しない"（A）**: 本実装後の正典ハーネス再測（§4）では**Xを複数シード平均で選ぶ**。単一シードだとX=2/X=3が入れ替わる。合格X集合が族間で `{2}`〜`{3}` を揺れるので、**平均G2・平均優勝Δでの再選定**を必須手順にする。

6. **角レバーの配線依存（F・G前半の壁1）**: 得意A/絆が予算クランプ・compat上限24の**内側**に実装されていることが飽和の前提。Mac実装でこれらが直付与（クランプ外）になると壁1が消える。実装後に**角セル（全部ON・やり込み/のんびり両型）を正典ハーネスで再測**。

## 4. 残る未確定（本実装後の正典再測でしか閉じない点）

以下は本反証（Python台・experiment-only・bot/PSpread最適近似）では原理的に閉じない。**Mac の run_year 本実装＋4.8の正典ハーネス再測**を待つ。
