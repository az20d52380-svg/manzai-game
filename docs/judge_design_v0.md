# 審査員設計 v0 — 7人の個性とスコア帯別コメント集

- 日付: 2026-07-03
- 位置づけ: グランプリ決勝（＋準決勝の講評）の審査員キャラクターと、点数・状態に応じたコメントのマスターデータ案。人名・来歴は全て架空。数値は全て【仮】。
- リサーチ根拠: 実在賞レースの審査講評で繰り返し評価される観点を分解すると「間・話術」「構成・伏線回収」「演技（喋っていない側の芝居）」「人物像・関係性」「会場の掌握・つかみ」「熱量・声」「新規性・手数」に大別される。これを7人に1軸ずつ配し、**ゲーム内パラメータと審査観点を1対1で対応**させる（＝プレイヤーの育て方が講評の返り方を変える）。

## 0. 仕組み（実装仕様案）

- 内部スコア（実力値＋相性＋ブレ＋体力補正）から各審査員の**表示点**を生成:
  `表示点 = 50 + 内部スコア×0.45 + 審査員バイアス + 審査員ノイズ(±1〜3)`、50〜99にクランプ【仮】
  - 内部スコア95（決勝ライン）→ 表示点93前後。「90点台の攻防」という実在の絵に寄る
- **【2026-07-04オーナー決定】審査は全員が総合点（内部スコア）で行う。** 審査員ごとの違いは (a)口調とコメント (b)表示点の固定バイアス (c)自分が重視する能力（華・技術など）が高い/低いときの表示点±1〜2の味付け——の3つだけで、**いずれも表示演出であり合否判定には一切影響しない**（判定は常に内部スコアvsライン）。知名度など人気値との相関は付けない
- コメントは `スコア帯 × 審査員` の基本文からランダムに1つ＋**条件フック**（該当時に差し替え/追加）
- スコア帯は5段階: **E**(〜49) / **D**(50〜69) / **C**(70〜84) / **B**(85〜94) / **A**(95〜)
- 条件フック（優先度順・最大1つ発火）:
  - `体力<30`: 後半の失速に言及
  - `メンタル<40`: 緊張・目泳ぎに言及
  - `相性≥18`: 掛け合いの呼吸を称賛
  - `華≥90`: 存在感・スター性に言及
  - `敗者復活経由`: 昼からの2本目に言及
  - `同回戦3年連続`: 常連への激励（審査員が覚えている演出）

## 8. 予選（1〜3回戦・準々）の簡易講評

予選は審査員非表示・紙の講評1行のみ（実在の予選も採点表の短評文化がある）。通過/敗退×汎用でプールを用意し、状態フックを優先適用。

- 通過: 「つかみ良し。次はネタの後半に山をもう一つ。」「声が会場に合っていた。この調子で。」「設定の発明あり。磨けば武器になる。」
- 敗退: 「4分の配分に難。前半に詰め込みすぎ。」「ボケの手数不足。テンポは良い。」「二人の関係がネタに乗っていない。」
- フック例（敗退×体力低）: 「後半の失速が惜しい。コンディション管理も芸のうち。」

## 9. 実装メモ

- コメント本文はGameCoreに置かず、リソース（JSON等）でUI層が持つ。GameCoreは `(帯, 発火フック, 審査員ID)` の選定結果だけ返す（ルール1の分離を維持）
- 同一審査員・同一帯の連続重複を避けるため、直近使用コメントIDを状態に持つ
- 決勝の審査員は7人固定（年による入れ替え演出は将来課題）。表示点の合計で「優勝ライン到達感」を演出するが、**判定はあくまで内部スコアvsライン**（表示はフレーバー）

## 10. v0.1 追記（リサーチ反映とコメント増量）

### 10-A. 実在予選との対応関係（設計の意図）

追加リサーチより: 実在の1回戦は**審査員3名×100点の合計が基準を超えれば合格**——本作の「内部スコアvsライン」の絶対評価方式は予選段階の実物と同型であることが確認できた。一方、実在の落選者には**理由のフィードバックがほぼ無く**、これが若手最大の痛点になっている。本作の予選「紙講評」（§8）は現実には無い優しさを意図的に足したもの＝**「フィードバックのある賞レース」という願望充足**がこのゲームの隠れた価値。講評は必ず「直せる一点」を含む文にする（性格批判・才能否定の文は作らない）。
