# 関係役柄のアーキタイプ変種 v0 — 後輩・先輩・同期を毎周「別の型」にする

- 日付: 2026-07-05
- 位置づけ: `character_archetypes_v0.md §5`「後輩/先輩/同期に runSeed でアーキタイプを割り当てて周回ごとに性格が変わる」を実テキスト化。`relationship_champions_v0.md §1-3`（絆イベントの骨＝何が起きるか）は**不変**のまま、**口調と数カ所の署名ビートをアーキタイプで上書きする overlay 方式**。
- **口調に応じた能力アップ**（オーナー要望）: 各アーキタイプの得意能力（`character_archetypes §3`）に対応して、絆が育つと**プレイヤーに小さな稽古ボーナス**を供給（§4・全て【仮】・sim対象）。
- **原則継承**: 性別は各型どちらもあり（`champion_selection §0-2c`・性別を煽り装置にしない）。**関西弁は既定でなく一変種だけ**（`relationship_champions §0-2b` 改訂）。優勝時セリフの禁止2語ルール・ノーボケ決意も継承。
- 判定ロジック・golden に非干渉（演出/イベント層・表示分岐フラグのみ）。数値は全て【仮】。

---

## 0. overlay の仕組み（骨は不変・口調と署名だけ差し替え）

1. 周回開始時、`runSeed` で各関係役柄（同期`.douki`／後輩`.kohai`／先輩`.senpaiGenius`）に**アーキタイプを1つ決定的に割り当てる**（名前生成と同じ層・`champion_selection §3` に相乗り・非バランス）。
2. `relationship_champions §1-3` の絆イベント（D1-8/K1-8/S1-8）は**構造（場面・展開・絆の深まり）を保持**し、**セリフの口調**を割当アーキタイプの役割語に、**1〜2箇所の"署名ビート"**をその型固有の一言に差し替える。
3. 優勝時（`relationship_champions §4`）は、関係色×優勝経験の骨に**アーキタイプ・フレーバー1行**を重ねる（下記各§の「優勝flavor」）。**禁止2語ルール（優勝済は先を越された/先に行かれた禁止）・ノーボケは厳守**。
4. 割当アーキタイプの**得意能力**が、その周回の絆ボーナスの対象になる（§4）。

> 既定変種（`relationship_champions §1-3` に既に書いた版）は各役柄の「基準の声」＝同期=委員長寄り／後輩=育てる素直系／先輩=苦労人(関西弁)。以下はそのバリエーション。

## 4. 口調に応じた能力アップ（絆ボーナスの供給・全て【仮】）

- 各周回、割当アーキタイプの**得意能力**が絆ボーナスの対象になる。絆イベントの節目（絆2/絆4等）で**プレイヤーに小さな稽古ボーナス**を渡す:
  - **先輩**: 「教わる」＝先輩の得意能力の**稽古効率+一時的**（例: 職人先輩→表現／博士先輩→センス）。`relationship_champions §3` の「間を半拍足せ」＝表現の実地レッスン。
  - **後輩**: 「教える」＝自分の**その能力が整理されて+**（人に教えると伸びる）。後輩の得意能力方向に、プレイヤーが微増。
  - **同期**: 「競う」＝同期の得意能力で**張り合ってプレイヤーも微増**（ネタ数競争＝センス等）。
- 効果は【仮】・最小・`EVENT_FIRE_CAP`（`dialogue_batch3 §8`）予算内。`character_archetypes §2` の得意能力ボーナスと同じく **enabled=off 既定・sim検証（sideways）を通すまでバランス非適用**（§6同書）。→ golden・現行バランス不変。
- **暴騰回避**: 絆ボーナスは周回1相手あたり総量上限を設ける（`canonical §2-B` のメタ再価格の教訓＝小さな一時ボーナスの積み上げが優勝率を跳ねさせない）。最終値は Fable の sim 較正待ち。

## 5. 実装メモ

1. **アーキタイプ選択**: `champion_selection §3` の名前割当と同じ層で、役柄ごとに `champHash(runSeed, 0, Salt.archetype+roleIndex) % 変種数` で決定的に選ぶ（セーブ再現・非バランス・新規RNG非消費）。消費順は正典（`name_generator §5-2` に相乗り）。
2. **口調 overlay**: 絆イベント（`relationship_champions §1-3`）のセリフは「骨テキスト＋役割語ID」で持ち、割当アーキタイプの役割語IDで表層を生成。署名ビートのみ本書の型別1行に差し替え。
3. **優勝flavor**: `relationship_champions §4` の関係色×優勝経験マトリクスの**フレーバー行**に、本書の型別1行を重ねる（反応・袖・禁止2語ルールは §4 のまま不変）。
4. **性別**: 各型に男女どちらも割当可（立ち絵で見せる・テキストは性別分岐しない・`champion_selection §0-2c`）。
5. **既定変種との整合**: `relationship_champions §1-3/§4` に既に書いた版＝各役柄の「【既定】」行（同期=委員長／後輩=育てる素直系／先輩=苦労人）。runSeedで既定が選ばれた周回は既存テキストそのまま。

## 6. 未決事項（オーナー/Fable）

1. 各役柄の変種数（現案: 同期5/後輩6/先輩5）。増やすほど周回の新鮮さ↑・制作量↑。
2. 絆ボーナスの方式・数値（§4・【仮】）。§得意能力ボーナス（`character_archetypes §2/§6`）と同じ sim 較正枠で sideways を保証してから on。
3. 同期にも「教える/競う」以外の絆ボーナスを持たせるか（現案: 競争でセンス微増のみ）。
4. アーキタイプを立ち絵・ボイスまで作り分けるコスト（MVPは口調テキストのみ・立ち絵は数枚のバリエーション）。
