# MVP更新 指示文 v1 — プレイテストfeedback反映・CLI(Mac)向け・コピペ即用

> 使い方: Mac CLIで `git pull --rebase` 後、下の三連バッククォート内を貼る。前提: 既に1年MVPは動いている（commit f763e97）。今回はプレイテストで出た指摘の反映＝**更新**。見た目の正典は更新済みの `docs/ui_mockup_pawapuro_v2.html`（v3内容）。

```
あなたはMac上のClaude Code。既に動いている1年MVP（漫才師育成SLG）を、プレイテストで出た指摘に沿って【更新】する。新規実装でなく既存の改修。まず git pull --rebase origin claude/manzai-slg-foundation-wprqlu。

■最初に読む
1. docs/ui_mockup_pawapuro_v2.html … 見た目の正典（v3・更新済み）。open して確認。SCREEN01の「押す→変種を選ぶ」パネル挙動が今回の目玉。
2. docs/dialogue_design_v0.md（会話トーン）／既存の ManzaiGame/Sources（改修対象）。

■絶対ルール（変わらず）
- GameCore(純Swift)とUIの分離を崩さない。golden/数式/乱数消費順に触らない（触る必要が出たら止めて相談・規律A）。新規Swiftファイルは xcodegen generate。固有名は架空・数値は【仮】。UIは swift test で検証不可＝simulatorでビルド→起動→通しプレイ目視までで「完了」。1コミット1目的＋Co-Authoredトレーラ。

■今回入れる変更（＝直し8個＋操作構造。この順で・各段階ビルド目視）
1. 【改行】育成メイン上部を整理：「2年目 14週」は1行、目標も1行に収める（折り返しの気持ち悪さ解消）。
2. 【所持金】上部に常時表示（体力バーの隣に ¥12,000 のように）。体力とお金＝毎週やりくりする2大リソースを両方トップに固定。
3. 【会話＝俺の心の声】現状の谷口の掛け合いを常時出すのをやめ、"主人公(俺)の内心モノローグ"に置換。状態駆動：大会前=助言（「そろそろ準々決勝か。表現あげとくか」）／低体力=「さすがに体が重い」／金欠=「今月きついな、バイト挟むか」／連敗=「何が足りないんだ…」。相方(谷口)や先輩後輩の会話は"レアイベント"でだけ出す（毎週は出さない）。
4. 【コア原則：ゲームは一切ボケない】全テキスト方針。会話は"笑わせる"でなく人間味・助言・ドラマ。ネタ本文は書かない＝笑いは笑い波形・通過敗退・講評で抽象的に見せる。俺の心の声も先輩の会話もボケない。
5. 【コマンド＝押す→中から選ぶ2段階】下のコマンドをグループ表示（けいこ／しごと・休み）にし、タイルを押すと変種パネルがせり上がる→選ぶ→獲得プレビュー→つぎへ。変種（MVPはこの中身で・器を作るのが主眼）：
   - ネタ作り → 自宅(無料・集中切れて効率▽) / 喫茶店(-¥500・安定して効率△)
   - ネタ合わせ → 公園(無料・声出せず▽) / カラオケ(-¥1,200・大声OKで△)
   - バイト → 引越し(+¥15,000・体力-大) / 居酒屋(+¥8,000・-中) / 交通量調査(+¥4,000・-小)
   - ライブ → フリーライブ(無料・華+少) / 事務所ライブ(-¥3,000・華+中)
   - 休む → 自宅(無料・回復中・無風) / サウナ(-¥1,500・回復大・たまに先輩イベント)
   ※パネルは押した時だけ開く＝普段の画面はスッキリ。効率▽△は各能力の伸びの大小に反映。
6. 【能力アップ＝バーが伸びる演出】つぎへ後、対象能力の"バー自体がアニメで伸びる"＋バー上に「+N」がポンと浮く＋軽くフラッシュ。静的な「獲得→表現+6」テキストは補助に格下げ。要所(節目)は全画面演出でもよいが必須でない。
7. 【笑い波形＝結果連動】笑い波形を本番スコアに連動：高スコア/通過→大きく育ってオチで跳ねる（暖色）／低スコア/敗退→フラットで疎ら・盛り上がらない（寒色）。区間別(ツカミ/中盤/オチ)反映は理想だが最低でも全体の振幅をスコア連動に。通過/敗退で色も変える。
8. 【講評の重なり解消】審査員名は左下・通過/敗退スタンプは右下で被らせない。両方読めるように。

■社会イベント（メンタル連動・レア・今回は"最小"で）
- 飲み/打ち上げ/道での偶発など"偶然の出会い"イベントは【低頻度・希少】（1年で数回級）。行く/行かない選択：行く=金減る＋体力若干減る＋メンタル↑＋関係／行かない=無風。金欠で後輩の誘いを断る=メンタル↓（「情けない先輩だ…」＝自己嫌悪。華でなくメンタルに乗せる＝採点主軸を汚さない・成長予算外で安全）。
- 相手はキャリア段階でゲート：1年目/無名は同期のみ（後輩なし・先輩薄い）。※MVPは"同期と道で会って飲むか断るか"1〜2種だけ実装で十分。ノルマ経済/オファーライブ/後輩の本格登場は次フェーズ。

■スコープ（重要・盛りすぎ厳禁）
- 今回やる：直し1〜8 ＋ 2段階パネル ＋ 変種(上記の中身) ＋ 社会イベント最小(同期の飲み1〜2種・メンタル上下)。
- 次フェーズ（今回やらない）：ライブのチケットノルマ経済(赤字/チャージバック)、オファーライブ、後輩の登場と人望web、飲みの全パターン、イベント38種フル。器だけ作って中身は後で足す設計に。

■完了の定義（simulatorで実際に遊んで）
1. ビルド成功・クラッシュなし・1年通しプレイ可。
2. 見た目がmockup v3に一致（所持金上部・俺の心の声・2段階パネル・バー伸び・波形連動・講評分離）。
3. コマンドを押す→変種を選ぶ→バーが伸びる、が気持ちよく回る。
4. 大会でウケ/スベで波形が変わる。
5. スクショ数枚＋commit＋push。

まず mockup を open して SCREEN01 の2段階パネルの挙動を掴み、既存コードを読んでから着手。各段階でビルド目視。時間が足りなければ直し1〜8＋2段階パネル＋変種 を最優先、社会イベントは最後。
```
