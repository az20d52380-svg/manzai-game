# 実績・トロフィー設計 v0 —「才能解放20枠」＋「称号」

> **【失効】本ドキュメントの才能解放部分は `trophy_design_v1.md`（難易度別ポイント制・総枠30pt）に全面改訂された。称号（§2）のみ引き続き有効。**

- 日付: 2026-07-03
- 位置づけ: `docs/endgame_design_v0.md` で確定した「トロフィー1個＝逓減D+1（総枠+20固定）」の中身を埋める。固有名は全て架空。数値は全て【仮】。
- 設計原則:
  1. **本編と別軸の目標を与える**（優勝だけが目標にならないように。負けキャリアにも「今回はこれが取れた」を残す）
  2. **初回プレイで4〜6個が自然に取れる配分**にする → 2周目がD124〜126となり、エンドゲーム検証のM1帯（優勝なし3〜31%・優勝が視界に入る）へ正確に接続する
  3. 取得条件は全てGameCoreの既存パラメータで判定可能なものに限定（新システムを要求しない）
  4. ネガティブ体験（失敗）も実績で回収して「おいしい」に変換する（お笑いの世界観と相性が良い）

- 1周目の自然取得見込み: #1,2,8,10 はほぼ確実、#3,5,11,19 が高確率、#4,13,15 が中確率 → **4〜7個（D124〜127）**。エンドゲームのM1帯（D125前後）に一致。

## 3. 実装メモ

- 判定は全て週次処理の末尾で行うフラグ蓄積（GameCoreに `AchievementTracker` を追加、UI非依存）
- トロフィーのD加算は**周回開始時に一括適用**（キャリア途中の取得は次周から有効）。途中適用にすると同一キャリア内で難易度が動き、検証(シード再現)が壊れるため
- 【設計上の懸念】総枠20は endgame_design_v0.md §3-1 の通り絶対固定。イベント報酬などで追加のD+を配りたくなったら、この表から置き換える（追加はしない）
