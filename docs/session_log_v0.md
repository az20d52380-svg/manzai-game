## 4. 2つのClaudeの分担と衝突回避 ★重要

いま2つのClaude Codeが同じ1本のブランチ（`claude/manzai-slg-foundation-wprqlu`）で動いている。能力が根本的に違うので、**能力で仕事を割る**。

### 能力の違い（手軽さではなく "何ができるか"）

| | **Mac CLI（Terminal）** | **クラウド（デスクトップ/ブラウザ = この画面）** |
|---|---|---|
| 動く場所 | Macの中（ローカル） | クラウドの計算機（リモート） |
| Swift / Xcode | **○ 本物。ビルド・swift test・シミュレータで実際に動かせる** | ✕ 無い（Swiftツールチェーンが無い） |
| Python検証(sim) | △（要セットアップ） | **○ 実績あり・すぐ回せる** |
| docs編集・git | ○ | ○ |
| どこから使えるか | Macの前にいる時だけ | **スマホ・外出先・別PCからでもOK。Macが消えていても動く** |
| 向いてる仕事 | **実装・ビルド・テスト・実機で"動くか"の検証** | **設計・docs・バランス(sim)・Fableの頭脳作業・計画** |

### 分担ルール（これだけ守れば衝突しない）

1. **役割で触るファイルを分ける**（最大の予防）:
   - **Mac CLI** = `GameCore/`・`ManzaiGame/` のSwiftコード、ビルド、swift test
   - **クラウド** = `docs/` の設計、`tools/` のPython検証、バランス
   - 別ファイルなら同時に動いても rebase は一瞬でクリーン（このセッションで何度もそうだった）
2. **push前に必ず取り込む**: どちらも `git pull` してから作業・push。クラウド側（私）は自動でやる。**Mac側には「作業を始める前と push 前に git pull して」と一言指示**すればよい
3. **同じ1ファイルを同時にいじらない**: どうしても両方が同じファイルを触る時だけ、順番に（片方push → もう片方pull → 作業）
4. **push が弾かれても慌てない**: `git pull`（競合が無ければそのまま）→ push。役割分担していれば競合はまず起きない
5. 迷ったら合言葉: **「Mac＝作って動かす／クラウド＝考えて書く」**

### 外出先ワークフロー（デスクトップ/スマホの強み）

- 外出中: スマホからクラウド（この画面）に「◯◯を設計して」「simで◯◯を確認して」と指示 → クラウドがGitHubに保存
- 帰宅後: Macで `git pull` → 続きをビルド・テスト
- ＝ **Macの前にいなくても設計・検証は前に進む**。ただし **"ゲームを実際に動かして見る" のは Mac(CLI) でしか出来ない**
