## 0. 北極星（North Star・v4）

毎週を **「コマンドを押す → その場で実行（段階リビール）→ 次週」** の最短リズムにする（パワプロ サクセス忠実・「つぎへ」なし）。
各コマンドタイルに**効果を常時表示**して押す前に伸びを読ませ、常時見える〈体力ゲージ＋所持金〉〈6軸能力（通過ラインは出さない）〉で
**「コンビを今まさに仕上げている実感」**を作る。**通過ラインは隠し**、通過可否は本番の笑い波形→半紙の紙講評で初めてわかる（＝ドキドキ）。

## 2. 参考2ゲームから借りる骨格（翻案）

- **確定前プレビュー（両ゲームの核）**: コマンド仮選択で「入る量・伸び幅・失敗率%」を実行前に見せる → 実行で回収。本作の2段コマンドの正しい延長。
- **基礎能力の常時固定表示（ウマ娘）**: 6軸を「強くする対象」として画面上に常駐。稽古を開いても消さない。
- **段階リビール（両ゲーム）**: 結果を一度に出さず「バー上昇→数字ポップ→ランク更新」を時間差で。同じ+3でも満足度が上がる。
- **逆算（サクセス査定／ウマ娘の目標レース）**: 「地力 XX / 通過ライン YY・あと Z」を常時見せ、毎週逆算させる。
- **山場だけ握らせる（サクセスのピンチャン）**: 笑い波形の平場は自動、掴み/畳みかけ/オチだけ判定を握らせる。
- **人格ある評価者（両ゲーム）**: 半紙の審査講評で数値を言葉に。ウィットの効いた一行で"次の一点"を指す。
- **キャラ関係＝常時ブースト＋段階イベント＋勝負（サクセスの彼女/ライバル/仲間）**: 将来の先輩/相方/ライバルのレアイベントへ翻案（MVP外・器だけ）。

## 3. 実装できる「正直な純getter」（RNG非消費・golden安全）

> **【設計上の懸念／最重要】** 素朴に作ると**予告が嘘をつく**。理由:
> - `GameEngine.add()` は能力上昇に **成長逓減 ×(1 − 現在値/120)** と**成長予算キャップ**を掛けてから加算する。
>   → `CommandVariant.gains` の生文字列（"+6"等）をそのまま出すと高ステで大嘘（センス100で+3は実際+0.5）。
> - 決勝の実効ライン = `gpFinalLine − fameFinalBonus×(知名度−50)/50`。静的な gpFinalLine では境界で判定とズレる。
> - 「次の関門」は GP進行状態（gpStage/gpAlive/finalist/revival＝WeekRunnerのprivate）に依存。カレンダー読むだけでは出ない。

正しい実装（全て乱数を消費しない＝消費順に触れない＝golden・swift test 緑のまま）:

1. **`GameSession.previewGains(_ action:) -> (gains:[(Ability,Double)], stamina:Double, money:Int)`**
   state の**コピー**に `applyTraining/applyJob/applyRest/applyOffer` を適用して差分を取る。
   add() の逓減・予算を通るので**正直**。**RNGのケガ判定・rollOffer は呼ばない**（＝確定分だけ予告、運は別表示）。
   ※`lastGains`（確定後の実差分）は既に逓減後なので、**確定後の段階リビールは既存データで正直**。

2. **`GameEngine.baseScore(_ s:, config:) -> Double`** ＝ `jitsuryoku(s,config) + s.compat + staminaPenalty(s)`（ブレ・ハマ夜は除く＝"地力"）。純算術。

3. **`GameEngine.blurWidth(mental:)`** は既に public（±バンド幅）。

4. **`GameSession.nextStageLine() -> (name:String, week:Int, line:Double)?`**
   WeekRunner に**読み取り専用アクセサ**（`gpStage/gpAlive/finalist/revival`）を public 追加し、GP path なら次の回戦/決勝ラインを、道中大会は eligible な次週の line を選ぶ。
   決勝は `effLine = gpFinalLine − fameFinalBonus×(fame−50)/50` を計算して返す。
   **【工数注意】** ここはスペック初期見積りより重い（WeekRunner private の露出が要る）。ただし RNG非消費で golden 安全。

5. **`injuryRisk(stamina, action, config) -> Double`** ＝ risky(キツいバイト) && stamina<injuryThreshold(20) ? (20−stamina)×injuryProbPerPoint(0.02) : 0（**表示専用**・rng非消費）。

6. **`staminaGated(action, state, config) -> Bool`** ＝ train && stamina<staminaGate(20)（稽古 disabled の理由表示。既存ゲート規則の可視化）。

> **burst の扱い**: baseScore は burst(+12@10%) を含まない（"地力"としては正直）。ゲージが「届かない」を含意すると稀なburst通過が嘘に見えるので、**帯の外側に薄い上振れ目盛り**で「たまに跳ねる」を明示する余地を残す。

## 4. 画面別スペック（今 → こう変える・なぜ・優先度）

### S1 育成メイン `WeekMainView.swift` — 縦4層固定に再編
`TopHUD → 6軸意識づけ帯（＋地力ゲージ）→ CalendarStrip（畳み可）→ 舞台 → 2段コマンド`。稽古を開いても上2層は縮小固定で消さない。

- **[P0] TopHUD の2ゲージ化**: 体力ゲージを大きく＋警告色（緑→黄<50→赤<20＋脈動）、所持金は負で朱＋帯に薄赤。GameConfig既存値を読むだけ。危機を「盤面の色」で予告。
- **[P0] 6軸「意識づけ帯」常駐化**: 変種パネル展開中も6軸ミニバーを縮小して残す（VStack順を固定）。ウマ娘=基礎能力常時固定表示。
- **[P0] ScoreVsLineGauge 新設**: §3の getter で「地力62 / 通過74・あと12」を常時表示。毎週"あと何点で通過か"を逆算。育成の逆算の実感＝**最大の投資対効果**。
- **[P0] CalendarStrip**: 48週横タイムラインに大会/GP/決勝マーカー＋current-weekピン＋「あとN週で△△」。1件先しか見えない現状を見通しに。CalendarConfig 読み出しのみ。
- **[P1] 立ち絵拡大＋表情差分**: 舞台の主役サイズへ。genki(体力)＋直近結果で表情、心の声は視覚下位の吹き出しへ。**※仮シルエットのヒーロー拡大は逆に未完成感が出るので、本素材が来るまでは"ほどほどの拡大＋表情"に留める（critique指摘）**。
- **[P1] クール分割**: 48週を春予選/夏中規模/冬本戦に色分け、区間初週に心の声で区間目標（標準語）。

### 稽古コマンド／変種パネル — 確定前ゴースト予告（最重要の手触り）
- **[P0] ゴースト予告**: 変種の仮選択で、6軸帯の該当バー末尾に**半透明の増加分ゴースト**（§3-1の正直な予測）＋体力ゲージに半透明の予定減少を先出し。押す前に伸びが読める安心感＝アンティシペーション、実行で回収。**素朴版（生gains表示）は出荷禁止**。
- **[P0] 失敗率%チップ**: キツいバイトのみ、体力<20で `injuryRisk` を%表示。稽古は体力<20で disabled＋「谷口:今日は休め」（既存ゲートの可視化）。
- **[P1] 相性かみ合い予告グロー**: ネタ合わせ/相方と過ごすの縁をスポット色パルス。**数値は一切足さない**＝「相性が本番スコアに直乗り」する既存事実の可視化のみ。boolean getter 1本。
- **[P1] グループタイルの「！」おすすめ**: weakAbility が伸びる変種を含むグループに小さな「！」＋ドット強調。第1段で予告→第2段で確定の二段高揚。
- **[P0] 大きな実行ボタン**: 右下にドンと構えた「つぎへ ▶」（今回の指摘）。仮選択で点灯、押下で沈み込み。選ぶ→予告→**大ボタンで確定**の起伏を作る。

### 週送りのリズム（選ぶ→予告→確定→結果→次の5フェーズ）

## 6. やってはいけない（critique＋規律）

- **golden を壊さない**: 数式・乱数消費順・GameConfig・判定に触れない。UIは表示・演出・情報配置＋RNG非消費 getter のみ。
- **中間経験点プールを新設しない**（6軸直接上昇の仕様＝新設は golden 破壊）。伸び幅プレビュー＋段階リビールに振り切る。
- **発火/おすすめで獲得量・倍率を増やさない**（相性グロー等は"既存効果の可視化"のみ）。
- **ゴースト表示のために乱数を先読み消費しない**（消費順ズレ＝3年ビット一致崩壊・規律A-2）。運は幅（最小〜最大）表示に留める。
- **ボケ（オチ/ネタ本文）を書かない**（笑いは波形＋SE＋講評＋心の声の筆致で担保）。
- **パクリにしない**（構造は借り、名称・世界観・数値・配色は漫才ドメイン＝寄席の暖色へ翻案。固有名は全て架空）。
- **素材沼に落ちない**（仮立ち絵のまま大きさ・配置・表情差分・微モーションで存在感。高精細前提の実装をしない）。
- **情報過密にしない**（常時固定は2リソース＋6軸帯まで。詳細は稽古選択時に退避。横スクロールは CalendarStrip 1本）。
- **UIは swift test で検証不可**＝ simulator でビルド→起動→通しプレイ目視まで確認して完了（規律D-10）。

## 7. critique が指摘した「構造的な弱点」（正直な自覚・MVP外の宿題）

このジャンル最大の高揚を、現在の数式では**構造的に持てない**点を直視する（今すぐ直さない・器の設計として記録）:

1. **友情トレの絆ビルドアップ→虹発火→複数ステ大量獲得が無い**。相性は+1ずつ・上限20でフラットに乗るだけ。相性グローはその不在の"化粧"。→ 将来、相方/サポート格の関係システムを本編で検討（今は可視化に留め、盛り過ぎない）。
2. **やる気が独立ステでない**（体力からの導出ラベル）。5段表情は見た目だけ。→ 将来やる気を機構化するか、"表情は演出のみ"と割り切る。
3. **経験点振り分けのドキドキを放棄**（規律上は正しい）。残る意思決定は「5稽古のどれ」と薄い。→ ゴースト予告＋段階リビール＋逆算ゲージで代替。
4. **所持金は攻めでなく防御リソース**（破産回避）。体力と同格の巨大ゲージに昇格させ過ぎない（glanceability圧迫）。→ 体力を主、所持金を副の見せ方に。
5. **対話型の選択肢イベントが手触りから脱落**（ウマの周回の飽き止めの主柱）。→ MVP外だが、器（特殊コマンド枠）だけ P2 で用意。
