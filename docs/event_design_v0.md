# 突発イベント設計 v0 — 単発38種＋連鎖アーク3本＋季節催事

> **【正典v2移行 2026-07-05】** 本書のライン・優勝率・D（才能解放）等の数値は**旧正典**の実測値です。
> 現行の正典は `canonical_v2_spec.md`（年齢カーブ上限・ハマった夜・生活ルール・準決74/決勝80・トロフィー=上限加算）。
> 本書の**設計結論・構造・演出**は有効。数値だけ読み替えること。


- 日付: 2026-07-03
- 位置づけ: 実装順「突発イベント」段階のマスターデータ案。効果は全て既存パラメータ（所持金/体力/知名度/能力5種/相性/オファー率）の増減で表現でき、新システムを要求しない。固有名は全て架空、数値は全て【仮】。
- 設計原則:
  1. **週次選択の結果に「物語の顔」を付ける**（数値変動をイベント文で包む）
  2. 発生条件を状態に紐付け、「自分のプレイが世界に反映されている」実感を作る（低所持金なら貧乏イベント、稽古続きなら稽古イベント）
  3. 選択肢イベントは「正解のない二択」を基本にする（リソース交換であって上位互換を作らない）
  4. ネガイベントは必ず**小さな救い**か**回収の余地**を残す（理不尽で終わらせない）
- 頻度設計【仮】: 週次イベント判定は大会週を除く週に12%。同一イベントは同一キャリアで原則1回（※印は再発可）。

## 3. 連鎖アーク（複数週の物語）

### アーク① 解散危機（全4週・条件: 相性8未満が6週継続 or #36でB選択）

> リサーチ根拠: 実在の解散理由の筆頭は「方向性の違い」「温度差」「楽屋で話さなくなる」。これを4段階で追体験させ、**プレイヤーの行動選択で回避可能**にする。

1. **兆候**: 「楽屋で相方がイヤホンを外さなくなった。」（相性成長停止）
2. **衝突**: 選択—「話し合う」（メンタル-3・次週へ）／「ネタで語る」（稽古強制・次週へ）
3. **岐路**: 相方の一言「……俺たち、何のために組んだんだっけ」。ここまでの2週で相性+があれば和解ルート、なければ最終週へ
4. **決着**: 和解「結成の日のファミレスで、また6時間話した」（相性+5・以後相性成長率1.5倍）／破局回避失敗でも解散はさせず「冷戦」（その年の相性成長なし）——**MVPでは解散させない**（キャラ交代システムがないため）。【設計上の懸念】解散を実装する場合は相方ガチャと同時に

2. 夏週: 同期・後輩からの寄せ書き（メンタル+5）
3. 大会週の演出強化: 審査員コメントに「最後の年」フックを追加（judge_design_v0.md と連動）

## 4-B. 【2026-07-04追記】イベント層のsim組み込み検証（§5の宿題の回答）

効果つきイベント18種の代表近似＋変な仕事＋季節催事を `sim_career.py`（EVENTS_ONフラグ・既定OFF）に実装して実測:

1. **「同一イベントはキャリア1回」の規約はバランスの生命線**——無制限再抽選だと優勝率が0.9%→7.8%に暴騰する（イベントの能力効果が実質無限稽古になる）。1回制で2.45〜2.75%まで収束
2. それでも正のEVが残るため、**「イベント込み」を正典バランスとしてラインを再調整**: 準決85→**86**・決勝93→**94**（人気補正1.5込みで実効92.5）→ **初回優勝1.5%・決勝到達25〜35%**で目標帯に着地
3. 設計則: 今後イベントを追加するときは (a)キャリア1回制を守る (b)恒久能力の正効果は控えめに（+1〜2） (c)追加のたびに EVENTS_ON で優勝率1〜2%帯の維持を再計測
4. 【2026-07-04追記2】(c)の恒久解: **効果発火はキャリア通算15回まで**の総量予算を採用（`sim_career.EVENT_FIRE_CAP=15`・検証は `tools/exp_events.py`）。プールが33種に増えても優勝率1.4〜1.7%を維持——詳細は dialogue_batch3_v0.md §8

## 5. 実装メモ

- イベントはGameCoreでは `(id, 効果パッチ, 選択肢)` の純データ。テキストはUI層リソース
- 乱数はイベント抽選専用のストリームを分離（シード再現が大会結果と干渉しないように）
- シミュレータ同期: 効果が数値である以上 `tools/sim_career.py` に「イベント有効」フラグを足して分布への影響を再計測すること（【設計上の懸念】平均効果がプラスに寄るとエンドゲームの優勝なし率が動く。実装時に必ず回す）
