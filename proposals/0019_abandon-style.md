# 提案 v0 — 「型を捨てる相談」連敗の底（lossStreak>=3）の路線変更イベント

- 日付: 2026-07-06 ／ 数値は全て【仮】／ 既存ファイルは読むだけ ／ トーン＝ドラマ主軸（proposals/0009 確定）
- 対象読者: イベント機構（event_design §2 二択）実装担当・DialogueData / GameSession / WeekMainView の書き手
- 位置づけ: 単発イベント相当。既存 `lossStreak` フックの**未使用の段（≥3）**に載せる。event_design が『lossStreak を段階的に使う路線変更の岐路が未設計』な点を埋める1枚。0010（前夜の一本）と同じ「未実装の二択機構に載る1件のデータ」の扱い。

## 狙い

`lossStreak` は現状 `>=2` の一段（`DialogueData.swift:29,50-53` の `losing` プール＝内心の一言）でしか使われていない。**「連敗が深い＝底」でプレイヤーが握れる意思決定が無い**。負けが込むほど「今の型を作り替えるか、信じて通すか」という下積みの芯の岐路が立つのに、ゲーム上そこは通り過ぎるだけになっている。

やることは一つ。**`lossStreak>=3` の底に「型を作り替えるか、通して磨くか」の二択を確定発火で置く**。要点は三つ。

1. **弱軸の賭け＝「発想を取りに行くか／表現を積み増すか」の非対称交換**（主眼）。A＝作り替えは、積み上げた**表現 −1** と**相性 −2** を払って **発想 +2** の新機軸を取る。B＝堅持は**表現 +2・相性 +1** を積むが**メンタル −1**の不安を抱える。ポイント総和は A が net −1（一点集中の賭け）・B が net +2（安定・純増）で、**盤面（弱軸・現相性帯・次の大会まで）次第で最適が入れ替わる**。
2. **相性を"能動的に削る"唯一の枠**（副次だが重要）。現状、相性を動かすのは `ネタ合わせ`(+1)/`相方と過ごす`(+1) の**加算のみ**（`GameEngine.swift:40-41` の `compatGrows` 経路・減算パスはゲーム内に一つも無い）。A の相性 −2 が**プレイヤーの意思で相性を賭ける初めての手**になる。ただし**トレードの成立は相性 −2 に依存させない**（→レッドA-1・下限クランプ対策）。
3. **連敗の底のドラマを相方依存の声で背負わせる**（0009）。「同じ負け方をした」という事実の底で、崩すか通すかを相方との一往復に載せる。谷口＝関西弁、谷口以外＝標準語（声は相方IDで差し替え）。

## 発火条件（実フック）

- **一次条件**: `GameSession.lossStreak >= 3`（`GameSession.swift:29` の `private(set)` フィールド。更新は `pump()` 内 `:149-151`＝通過で `lossStreak=0` リセット／敗退で `+1`）。既存 `losing`(≥2) の**一段深い未使用の段**にちょうど載る。
- **発火位置の厳密化（原案の曖昧さを是正）**: lossStreak が 3 に達するのは**S3結果画面を処理する `pump()` の中**。イベントの二択は結果画面ではなく、**その次の `freeAction` 週（育成 monoBox）で `lossStreak>=3 && !styleTalkDone` のとき確定発火**する（＝プレイヤーが行動を選べる画面に載せる）。「達した週に出す」ではなく「達した後の最初の育成週で出す」。
- **確定発火（RNGを避ける）**: `lossStreak>=3` の育成週に**確定で出す**（12%の週次ランダム枠に入れない）。ランダム枠に差すと乱数消費順に触れて golden を壊し得る（→実装(3)）。連敗の底は "彩り" でなく**必ず立つ節目**（確定イベント側）に置くのが構造的に正しい。
- **反復制御（必須・一発化）**: `lossStreak>=3` は**次に通過するまで毎週真**。毎週発火すると相性を毎週削れて予定調和＋崩壊するので、**同一連敗中は一度だけ**。`GameSession` に `styleTalkDone: Bool`（【仮】）を持ち、発火時に true、**`lossStreak=0` にする `pump()`（`:150`）と同一トランザクションで false に戻す**（0003 の失効の手筋と同型・更新点と失効点を必ず揃える）。
- **MVP 発火の可否は要sim**: 1年48週内で3連敗が実際に到達するかは**未計測**（原案は断定するが、年内の stage/GP 敗退回数と配置に依存）。**発火するか本編用かは sim_career の実測で確定**（→未解決）。本文・機構は MVP から載る形で書くが、頻度は要確認。

## イベント本文（場面のセットアップ＋二択）

> トーン＝ドラマ主軸。感情語（悲しい/悔しい/惨め）を地の文から抜き、**"この稽古場・この相方に固有の少しズレた物"**（＝谷口がタイトルをカタカナで書く癖）に温度を背負わせる。俺＝標準語・計算屋・自己憐憫なし。**会話は話者ごとに1セリフずつ（"／"で詰めない）**。ボケ（オチ/ネタ本文）は書かない。数値は全て【仮】。

```

稽古場の白板に、同じネタのタイトルが三回、書いては消してある。
谷口はタイトルを、漢字でも必ずカタカナで書く。消しあとのカタカナが、うっすら三つ重なっている。

```

```
俺「三回、同じ負け方をした。」                             [技法=事実の並置]
谷口「同じ負け方は、同じネタのせいか、同じ俺らのせいか、どっちや。」  [技法=観測/問い返し・関西弁]
```

---

**選択肢A：新機軸に作り替える**

**【効果】** 発想 +2（新しい角度）／ **表現 −1**（磨いた型を一度崩す）／ **相性 −2**（噛み合わせを組み直す軋み・下限0でクランプ）。数値は全て【仮】。

> レッドA-1 反映：**A の下振れを "表現 −1" で床から独立させた**。原案の「相性 −2 のみ」だと連敗の底で相性が既に低い場合クランプ（`GameEngine.swift:41` の `clamp(0, cap)`）で 0 に張り付き、A が実質ノーコストの発想 +2＝**支配戦略化**する（＝トレードオフ崩壊）。表現 −1 は相性帯に関係なく必ず効くので、相性が高い帯ほど「噛み合いを賭ける」重みが増し、低い帯でも「磨きを捨てる」重みが残る＝**全相性帯で二択が成立**する。

会話（谷口版・話者ごと1セリフ）:
```
俺「作り替える。今の型は、いったん崩す。」                        [技法=決断・計算屋]
谷口「崩すいうことは、噛み合うてるとこも一回ばらすことやで。……ええんやな。」 [技法=コストの言語化・関西弁]
俺「同じ負け方を、四回目はしたくない。」                          [技法=白板〔四回目〕への接続]
```

**選択肢B：型を通して磨き込む**

**【効果】** 表現 +2・相性 +1（迷わず組んだ分だけ噛む）／ **メンタル −1**（変えないという不安）。数値は全て【仮】。

会話（谷口版・話者ごと1セリフ）:
```
俺「変えない。この型のまま、精度だけ上げる。」                    [技法=決断・計算屋]
谷口「……せやな。俺は、この形の俺らが好きや。」                  [技法=愛着を物〔この形〕で置く・関西弁]
俺「好きなだけじゃ、四回目も同じところで消える。」                [技法=白板〔書いては消してある〕への静かな接続]
```

（B の俺の締め「消える」＝地の文の白板への接続。メンタル −1 の不安を感情語でなく事実で置く。）

**相方依存バリエーション（谷口以外の相方＝標準語ベース・声は相方IDで差し替え）**

```
             消しあとの小さな字が、うっすら三つ重なっている。
A・相方「崩すってことは、噛み合ってた分もほどくってことだよ。……それでいいんだね。」
B・相方「……いいと思う。僕は、この形の僕らが好きだ。」
```
（相性帯で温度を差せる：0-7 他人行儀＝同意を薄く短く／15-20 ツーカー＝先回りの一言。※compat上限20＝谷口に阿吽21+は出ない。谷口以外に関西弁を混ぜない＝相方依存の声・0009。）

## レッドチーム（A/B）

**A. 実装妥当性**

1. **【是正・最重要】支配戦略の穴を塞いだ**。原案 A＝相性 −2 のみは、連敗の底で相性が既に低い場合クランプ（`GameEngine.swift:41`）でコストが消え、A が実質ノーコストの 発想 +2＝**明白な最適解**になる（event_design §2 違反）。→ A に**床から独立する 表現 −1** を追加。全相性帯で二択が成立する。
