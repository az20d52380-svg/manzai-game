# 提案 v0 — 大会週の「温度」を心の声に差す（緊張帯／緩和帯の二段設計）

- 日付: 2026-07-05 ／ 数値は全て【仮】／ 既存ファイルは未編集（読むだけ）
- 対象読者: DialogueData / WeekMainView 実装担当・dialogue_design / writing_craft の書き手

## 狙い

大会週は本作で最大の感情ビートなのに、常時表示の「俺の心の声」は現状 `そろそろ\(m.name)か。\(weakAbility)、あと少し上げておきたい。` という weakAbility 穴埋め1テンプレしかない。感情の最高点を「稽古ToDoリマインダ」という**最も乾く実務文**で埋めている(§why_weak)。

やることは二つ。
1. **大会前2週を「緊張帯」に切り替える**。実務リマインダは大会3週以上前・平常帯へ後退させ、前2週に入った瞬間から感情文を優先する。
2. **温度を状態で差す**。メンタル値で3帯（余裕／そわそわ／過緊張）、大会格で振れ幅、直後は勝敗2値でなく「高揚／薄い手応え／落胆」の3帯。緊張は心の声が張り、緩和はリザルトと谷口(§4)が抜く——**役割分担で"緊張の緩和"を作る**。

writing_craft の基調（過小申告＋温度差実況）を全帯で守り、誇張（例2）と擬人化（例7）は**決勝×過緊張の"ここぞ"1枚だけ**に集中させて §6/§7 の低頻度・天丼規律と両立させる。

## どこから学んだか（技術＋出典URL）

- **緊張の緩和（桂枝雀）** — 笑いは張った空気が緩む瞬間に出る。心の声＝張る役、リザルト/谷口の些末＝緩める役、と分けるのが構造的に正しい。 https://waraitext.com/post-17/17/ ／ https://utakahiro.hatenablog.com/entry/2016/02/07/044327
- **逆U字理論（ヤーキーズ・ドッドソン）** — 緊張ゼロも過緊張も両方パフォーマンスを落とし中程度が最高。「緊張してる/してない」でなく**最適点からの距離**で3帯に書き分ける根拠。 https://www.css.nihon-u.ac.jp/about/publicity/columns/post-1/
- **ルーティン＝心のスイッチ（アンカリング）** — 決まった手順の反復で緊張を鎮める。俺の固定確認動作を天丼で置く根拠（3回目＝決勝が最大）。 https://d-d-webtantou.com/sports-routine/
- **決勝の現場温度** — 堂々と見えて膝はガクガク、大爆笑中でも震える。高揚と緊張は**同時に来る**＝直後を「勝った＝晴れやか」と単純化しない根拠。 https://diamond.jp/articles/-/319910
- **「さざ波の笑い＝全国の前でスベってる」恐怖** — 半端なウケが最も怖い。justPassed に**薄い手応え帯**を足す根拠。 https://books.bunshun.jp/articles/-/7620
- **スベる主因＝力み・空回り・狙いすぎ** — 感情の最高点を事務文で埋めるのが最も乾く失敗、力まず過小申告で流すのが最も外れにくい。 https://owa-rai.com/suberu/
- **パワプロ サクセスのセクション（必発）＋ランダムの二層** — 大会前2週を**必発の確定枠**として平常ローテと分離。分岐は乱数でなく状態（メンタル・格・手応え）で決める＝golden 安全。 https://xn--odkm0eg.gamewith.jp/article/show/46318
- 自作 writing_craft §2 例2/6/7・§6/§7、dialogue_design §4（既存正典）。

## 実装イメージ（そのまま貼れるスニペット＝既存データ構造に載る形）

`innerVoice` の優先度に**帯を割り込ませる提案**。実務文（weakAbility）は最下段へ後退。呼び出し側から `tier`(予選/準決/決勝)・`weeksLeft`(0=前夜)・`lastResultTone`(.high/.thin/.low) を追加で渡す想定。

```swift
// 優先度(提案): 低体力 > 金欠 > 連敗 > 大会直後(帯) > 大会前2週(帯) > 平常(weakAbility実務文はここへ後退)

// --- 大会前2週・緊張帯（メンタル×格でスケール。weeksLeft==0が前夜=最大） ---
if let m = nextMilestone, m.weeksLeft <= 2 {
    let eve = (m.weeksLeft == 0)
    let big = (tier == .final)              // 決勝のみ誇張1枚を許可
    switch state.メンタル {
    case 60...:   return Advice(name:"俺", text: pick(eve ? preHighEve : preHigh, salt: Int(state.fame)))
    case 40..<60: return Advice(name:"俺", text: pick(eve ? preMidEve  : preMid,  salt: Int(state.stamina)))
    default:      return Advice(name:"俺", text: pick(big ? preLowFinal : preLow,  salt: Int(state.メンタル)))
    }
}
```

**大会直後・緩和帯（passedLines を tone 3帯へ拡張する提案）**

```swift
// L54-57 の justPassed 2行を、勝敗2値でなく手応え帯へ
switch lastResultTone {
case .high:  // 通過・手応えあり＝谷口の些末で緊張を抜く

case .thin:  // 通過したが薄い＝最も苦い中間（さざ波の恐怖）

}
```
