# 提案 v0 — 大会週の「温度」を心の声に差す（緊張帯／緩和帯の二段設計）

- 日付: 2026-07-05 ／ 数値は全て【仮】／ 既存ファイルは未編集（読むだけ）／ レッドチーム（A/B）反映済み
- 対象読者: DialogueData / WeekMainView 実装担当・dialogue_design / writing_craft の書き手

## 狙い

大会週は本作で最大の感情ビートなのに、常時表示の「俺の心の声」は現状 `そろそろ\(m.name)か。\(weakAbility)、あと少し上げておきたい。`（DialogueData.swift L35-36）という weakAbility 穴埋め1テンプレしかない。感情の最高点を「稽古ToDoリマインダ」という**最も乾く実務文**で埋めている。

やることは二つ。

1. **大会前2週を「緊張帯」に切り替える**。実務リマインダは大会3週以上前・平常帯へ後退させ、前2週に入った瞬間から感情文を優先する。
2. **温度を状態で差す**。メンタル値で3帯（余裕／そわそわ／過緊張）、格（GP決勝か否か）で振れ幅を変える。直後は「勝敗2値」でなく「高揚／薄い手応え／落胆」の3帯を**目標**とし、MVPでは実装可能な帯から段階投入する（後述の信号依存）。緊張は心の声が張り、緩和はリザルトと谷口（§4）が抜く——**役割分担で"緊張の緩和"を作る**。

writing_craft の基調（過小申告＋温度差実況）を全帯で守り、誇張（例2）と擬人化（例7）は**GP決勝×過緊張の"ここぞ"1枚だけ**に集中させて §6/§7 の低頻度・天丼規律と両立させる。

## どこから学んだか（技術＋出典URL）

- **緊張の緩和（桂枝雀）** — 笑いは張った空気が緩む瞬間に出る。心の声＝張る役、リザルト/谷口の些末＝緩める役、と分けるのが構造的に正しい。 https://waraitext.com/post-17/17/ ／ https://utakahiro.hatenablog.com/entry/2016/02/07/044327
- **逆U字理論（ヤーキーズ・ドッドソン）** — 緊張ゼロも過緊張も両方パフォーマンスを落とし中程度が最高。「緊張してる/してない」でなく**最適点からの距離**で3帯に書き分ける根拠。 https://www.css.nihon-u.ac.jp/about/publicity/columns/post-1/
- **ルーティン＝心のスイッチ（アンカリング）** — 決まった手順の反復で緊張を鎮める。俺の固定確認動作をメンタル中帯に置く根拠（＝preMid の「三回めくる」）。 https://d-d-webtantou.com/sports-routine/
- **決勝の現場温度** — 堂々と見えて膝はガクガク、大爆笑中でも震える。高揚と緊張は**同時に来る**＝直後を「勝った＝晴れやか」と単純化しない根拠。 https://diamond.jp/articles/-/319910
- **「さざ波の笑い＝全国の前でスベってる」恐怖** — 半端なウケが最も怖い。直後に**薄い手応え帯（.thin）**を足したい根拠（ただし実装は margin 信号待ち・後述）。 https://books.bunshun.jp/articles/-/7620
- **スベる主因＝力み・空回り・狙いすぎ** — 感情の最高点を事務文で埋めるのが最も乾く失敗、力まず過小申告で流すのが最も外れにくい。 https://owa-rai.com/suberu/
- **パワプロ サクセスのセクション（必発）＋ランダムの二層** — 大会前2週を**必発の確定枠**として平常ローテと分離。分岐は乱数でなく状態（メンタル・格）で決める＝golden 安全。 https://xn--odkm0eg.gamewith.jp/article/show/46318
- 自作 writing_craft §2 例2/6/7・§6/§7、dialogue_design §4（既存正典）。

## 実装イメージ（そのまま貼れるスニペット＝既存データ構造に載る形）

`innerVoice` の優先度に**帯を割り込ませる提案**。実務文（weakAbility）は最下段へ後退。呼び出し側から `isFinal`（次のマイルストンが GP決勝か）を追加で渡す想定。**tier は既存の `nextMilestone()` が返す `name` を `config.calendar` の GP決勝ラベルと突き合わせて UI 側で導出できる**（新フィールド不要）。

```swift
// 優先度(提案): 低体力 > 金欠 > 連敗 > 大会前2週(緊張帯) > 大会直後(緩和帯) > 平常
//   ・連敗(lossStreak>=2)は既存どおり心の急所として据え置き（最優先群）。
//   ・緊張帯を直後帯より"上"に置くのが要点。GP回戦は週39/41/43/45/47 と近接するため、
//     「勝った直後」と「次戦が目前」が同じ週に両立する。afterglow より次戦の張りを勝たせる。

// --- 大会前2週・緊張帯（メンタル×格でスケール） ---
if let m = nextMilestone, m.weeksLeft <= 2 {
    // 育成monoBoxで到達しうる weeksLeft の最小値は 1（大会週そのものは別画面＝tournamentDecision）。
    // ゆえに weeksLeft==0 は monoBox に出ない → 「前夜」に相当するのは weeksLeft==1。
    let eve = (m.weeksLeft == 1)
    switch state.メンタル {
    case 60...:   return Advice(name:"俺", text: pick(eve ? preHighEve : preHigh, salt: Int(state.fame)))
    case 40..<60: return Advice(name:"俺", text: pick(preMid, salt: Int(state.stamina)))   // MVPはeve変種を持たず2週共通
    default:      return Advice(name:"俺", text: pick(isFinal ? preLowFinal : preLow, salt: Int(state.メンタル)))
    }
}

// --- 大会直後・緩和帯（通過のみ・MVP） ---
// 次戦が目前(weeksLeft<=1)なら上の緊張帯が既に勝っているので、ここが出るのは次戦が3週以上先の時だけ。
if justPassed {
    return Advice(name:"俺", text: pick(passedHigh, salt: Int(state.fame)))
}
```

## リスク・注意

- **【設計上の懸念】状態配線が未実装**。`isFinal`（および将来の直後トーン信号）を innerVoice へ渡す経路が要追加（WeekMainView L154-156）。これは "未実装なだけ" で欠陥ではない（MVP 実装途中）。**DialogueData/monoBox は乱数を一切消費しない純 View getter＝golden 不変**（salt は state から取るだけ）。数式・乱数消費順には触れない。ただし .thin のために StageResult に margin を足すなら、それは GameCore の構造体変更＝compile 影響と `swift test`（golden はビット不変・消費順不変）を1コミットで揃えて確認する。
- **weeksLeft==0 は monoBox に出ない**。大会週そのものは `tournamentDecision` フェーズ＝別画面で、育成 monoBox は出ない。ゆえに「前夜」に相当するのは **weeksLeft==1**（前週）。元案の `eve=(weeksLeft==0)` は monoBox では常に false＝死に枝だったため修正済み。
- **緊張帯 > 直後帯の順序**。GP回戦は週が近接（39/41/43/45/47）するため、「勝った直後」と「次戦が目前」が同週に両立する。直後帯を上に置くと**クライマックスで afterglow が緊張を隠す**逆転が起きる。緊張帯を上に置き、直後帯は次戦が3週以上先の時だけ出す。
- **閾値 60/40 は【仮】**。sim で「大会前2週に各帯が適切頻度で出るか」実測してから詰める（体感を数値で直さない・§7-B）。
- **決勝アクセント（preLowFinal）の頻度**は §6/§7 の低頻度・天丼規律に従い GameConfig 管理【仮】。誇張・擬人化を予選から出すと寒くなる＝格（isFinal）でゲート。
- **§4 との重複回避**。心の声は「谷口が言う前の俺の頭の中」に限定し、§4 の谷口会話と同じオチ（「腹減ったな」等）や同じ言い回し（「緊張って割れんのよ」等）を心の声側で使わない。口癖の登場回数は §6（1年2回まで）を侵さない。
- **反復回避**。preHigh 等の小プールは salt が state 由来で週間ほぼ不変＝2週連続で同一行が出うる。proposal 0001 の直近回避（状態別 recent・上限付き）を同じ innerVoice に効かせるのが本筋。
- **UI は swift test で検証不可**。書いたら simulator でビルド→大会2週前まで送って目視（§10）まで確認して「完了」とする。
- **1テキスト＝技法1つ**（§92 安全弁1）。上の文例も盛らない。

## レッドチーム（A/B）

**A. 実装妥当性**

1. **【修正済】前夜フラグが死に枝**。元案 `eve=(m.weeksLeft==0)` は、大会週が `tournamentDecision`（別画面）で育成 monoBox が出ないため monoBox 側では**常に false**。到達しうる最小 weeksLeft は 1。→ `eve=(m.weeksLeft==1)` に修正し、前夜相当を前週へ寄せた。
2. **【修正済】直後帯 > 緊張帯の優先で climax が逆転**。GP回戦は週39/41/43/45/47 と近接し、勝った直後の週に次戦が weeksLeft==1 で両立する。元の優先順（直後を上）だと最終盤で afterglow が張りを隠す。→ 緊張帯を上に置き、直後帯は次戦3週以上先のみ。
3. **【修正済/降格】直後の3帯（.high/.thin/.low）が単一信号 justPassed に乗らない**。(a) `.low`（敗退直後）は justPassed が敗退時 false のため**この枠で発火不能**＝別信号 justLost（proposal 0003）が前提。0003 未採用時は既存 losing 経由に留める。(b) `.thin`（薄い手応え）は StageResult に**通過マージンが無く**表現不能＝GameCore に margin 露出（golden非対象のデータ追加）が要る。→ MVP は `.high` のみ出荷、.thin/.low は依存を明示して後続へ降格。
4. **【確認済】golden/乱数消費順は無関係**。innerVoice/monoBox は純 View getter で RNG を一切消費しない。salt は state 由来。元案の「消費順が変わらない想定だが要検証」は過剰＝**確定で golden 不変**と明記（margin 露出時のみ構造体 compile 影響を注記）。
5. **【確認済】tier は新フィールド不要**。既存 `nextMilestone()` の name を GP決勝ラベルと突合して `isFinal` を UI 導出できる。参照した L 番号（DialogueData L21-39/L35-36/L54-57、WeekMainView L154-156/L402-415）は実コードと一致。メンタル/fame/stamina 各フィールドも GameState に実在（型も一致）。
6. **【注記】天丼エスカレーションは未設計**。出典のルーティン・アンカリング（3回目＝決勝が最大）は preMid の一行に留まり、大会横断の段階天丼までは実装していない。将来課題として分離。

**B. 会話の中身**

1. **【修正済】谷口セリフが §4 の近似コピペ**。元案の `谷口「緊張、こっちに半分よこせや」／俺「緊張は割り勘できる物じゃない」` は §4 既存正典 `谷口「…俺はしてる。半分持って」／俺「緊張って割れんのよ」` の言い換え＝自己重複。提案自身の「§4 と重複しない」原則に反する。→ **谷口セリフを心の声側に足す案ごと削除**。心の声は「俺の頭の中」だけを担い、§4 谷口前夜は正典のまま。
2. **【修正済】技法タグが2本＝盛り**。`preHigh` の「…俺の胃はもう店じまいを始めている」は温度差実況＋擬人化の二重。1本=技法1つ（§92）に反する。→ 擬人化を外し「俺の茶碗は、半分で箸が止まっていた」＝温度差実況の単一に。
3. **【修正済】決勝アクセントがキザ＆二重技法**。「十年分の稽古が…俺の胃の中で同窓会を始めた」は誇張＋擬人化で、かつ"同窓会"が狙いすぎ（キザ／笑わせにいっている）。→ 「十年やってきた稽古の重さが、今夜だけ、体重計に乗り切らない気がする」＝誇張単一・具体物・内心観測に。
4. **【修正済/整理】tag の複記を単一化**。preMid の「ルーティン+自虐」→ 自虐、passedHigh の「温度差実況」等、各行1技法へ揃えた。三回…三回の反復は自虐内のアナフォラ（口癖天丼ではない）として許容。
5. **【確認済】口調ブレ無し**。俺の全行は標準語・計算屋トーン。谷口は削除したため関西弁の混入リスク自体を消した。ボケ（オチ/ネタ本文）は無く、全て反応・観測・過小申告。既存作品（火花/べしゃり暮らし/M-1 実在ネタ）との一致は各行を照合し無し。固有名は架空、数値は【仮】。
6. **【確認済】§4 の "決勝の照明は思っていたより熱かった" 等の既存モノローグを心の声側で再利用していない**（層の重複回避）。
