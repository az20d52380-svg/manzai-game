# 現状から「遊べるデモ」への批評的監査を頼みたい

manzai-game は iOS/SwiftUI の「漫才師コンビ育成SLG」MVP。この数日で機能が次々に積み上がった——1年48週の週ループ、稽古、道中大会/グランプリ、リザルト、そして今まさに実装中の「経験点／能力アップ（割り振り）」。個々は作り込まれているが、**1本の"遊べるデモ"として噛み合っているか**を誰も俯瞰していない。あなた（Fable）の product 判断で、そこを冷静に棚卸ししてほしい。

## あなたに求めること（判断・監査＝あなたの本領）
現状を読み取り専用で調べ、正直に判定せよ:

1. **いま実際に遊べるもの（end-to-end）**: タイトル→コンビ結成→週を進める→稽古/バイト/休む→大会→リザルト、の一連が本当に通しで成立しているか。どこで途切れる・破綻するか。
2. **"遊べるデモ"に足りない/粗い/順序がおかしいもの（クリティカルパス）**: コンパイルではなく**体験として**。優先順位つきで。「無いと démo が成立しない必須」と「あると良い」を分ける。
3. **面白さ・感情の核はどこにあり、どこで削がれているか**: このゲームが「もう1週やりたい」と思わせる瞬間はどこか。逆に、作り込みが本筋（下積み→大会の悔しさ→成長の実感）から逸れて散らかっている箇所はないか（過剰実装・寄り道の指摘も歓迎）。
4. **経験点機能の直後に、次に作るべきもの**とその理由。1つに絞って推奨し、反証条件を添える。
