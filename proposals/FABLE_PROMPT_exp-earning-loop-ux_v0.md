# 経験点「稼ぐ→貯まる→注ぐ→伸びる」ループの体験設計を頼みたい

manzai-game のパワプロ式「経験点／能力アップ」機構は、機構確定と「注ぐ側」実装が済んでいる。あなたに頼むのは、**まだ設計されていない「稼ぐ側〜ループ全体の体験」を、構造・演出・情報配置の判断として設計すること**（実装はMac側がやる。あなたはコードを納品しない＝上のコメント厳守）。

## 確定済みの前提（動かない・この上に設計せよ）
- **機構＝二区画中間**：稽古で得る経験点は「粒」。粒の**75%は同色ロック**（その能力にしか注げない）、**25%は共通枠**。共通枠は稽古の系に重ねた2グループ——**「ネタ」＝センス/発想**、**「舞台」＝表現/華**（各2能力限定）。**メンタルは同色のみ**。数値は【仮】。
- **注ぐ側は完成済み**：`AllocationView`（能力アップ画面「のばす」）でプレイヤーが貯めた粒を能力へ注ぐ。同色のみ・η（色変換税）は無い・**余った粒は捨てる（余剰OK）**。注入は「+1タップで1段」文法・確定リビール・器ゲージ転生済み。
- **会計＝割り振り時予算**：予算（総成長の器）は「注ぐ瞬間」に効く。稼ぐ時は予算非消費で粒がただ貯まる。総成長量は器が固定＝バランス安全（sim実証済み）。
- **稼ぐ側は今Mac側が実装中**：稽古を「能力に直接+N」から「粒を稼ぐ（同色75%＋グループ共通25%）」に置き換える会計移設。**この稼ぐ側のプレイヤー体験（見せ方）がまだ空白**で、そこがあなたの仕事。
- 既存の週メイン画面：下部に稽古/回復/バイト等のカテゴリ→変種カード（タップ即実行・1週進む）。現在カードは実行前に「+N」能力プレビューを出している。能力5色＝センス青/発想紫/表現橙/華桃/メンタル緑、確定朱`verm`・見せ場金`gold`。

## 設計してほしいこと（判断＝あなたの本領）
1. **稼ぐ瞬間の見せ方**：稽古カードは実行前に何を予告すべきか（現状「+N能力」→「どの粒がいくつ入るか」？ 同色分と共通枠分をどう描き分けるか）。実行後の「稼いだ」フィードバックの一拍。**粒は5能力色＋2グループ（ネタ/舞台）という異なる粒度が混在する**——これを1目で分からせる情報設計。
2. **ループのペースと週リズムへの溶かし込み**：「稼ぐ（週の行動）」と「注ぐ（別画面AllocationView）」が別リズムになる。毎週注ぐのか、貯めて時々注ぐのか、プレイヤーが自然に取る所作は何か。摩擦にならないテンポ。
3. **未割り振り粒への気づかせ方**：貯まった粒があることをどう気づかせるか（AllocationView入口のバッジ等）。「注ぎ忘れ」を責めず、でも見落とさせない誠実なトーン。
4. **余剰（捨てる）の見せ方**：器を使い切ると以降の粒は無駄になる。これを「損」でなく「その年は満ちた・別の遊びへ」と前向きに見せる（既存の器の文言「この年の器は、満ちた。」の思想を稼ぎ側にも）。
5. **UXリスクの指摘**：この二区画中間＋別画面注ぎのループで、プレイヤーが混乱・退屈・見落としを起こしうる箇所を正直に洗い出し、対処案を添える。
