# 突発イベント「行けない飲み会」提案 v0（2026-07-06）

金欠帯（低所持金）の世界観に、`event_design_v0` §D「コンビの機微」と §A「貧乏と飯」の交点＝**"付き合い費 vs 稽古場代"の正解なき二択**を1枚足す。効果は既存 `GameState` フィールド（`money/compat/mental/発想`）の増減のみで表現でき、新フィールド不要。数値は全て【仮】、既存ファイルは読むだけ。

> **採番注記**: 起票番号は **0011**。`0010` は既存（`proposals/0010_pre-tournament-eve-choice.md`「前夜の一本」）で衝突するため。ファイル名は `proposals/0011_broke-drinking-invite.md`。

## 狙い

`event_design_v0` の貧乏イベント（#1〜5）は**通知型（自動発生・選択なし）**に寄っており、「金欠を、プレイヤー自身の手で選ばせる」枠が薄い。本案は残高そのものをリソースにした二択で、この穴を塞ぐ。

- **付き合い（相性・息抜き）も、稽古場代（創作原資）も、どちらを取っても損が出る**。§原則3「正解のない二択（上位互換を作らない）」を金欠帯で最も切実な形に落とす。
- A の得（人脈・メンタル）は"今"効き、A の損（`money-4000`）は**持続コスト**として来週以降の残高バッファを細らせる。B の得（発想）は"今"入り、B の損（メンタル-）も"今"効く。**軸も効くタイミングも別**にして即断を難しくする。
  - **【正直な訂正】**「稽古場代がさらに細る"隠れコスト"」という**別建ての将来ペナルティは実装しない**（そんな機構は無い）。A のコストは `money-4000` 一本で、それが金欠帯では次週以降のバッファ減として持続的に効く、という意味に限定する。ここを誇張しない。
- 感情語（惨め/情けない）を一切書かず、**残高の数字に温度を背負わせる**（proposals/0009 の芯）。固有の物として**稽古場の"百円入れると一時間つく照明"**を置き、金額を「明かり◯時間ぶん」に翻訳する（**換算式 100円＝1時間を全セリフで厳守**）。

## 発火条件（実フック）

| 項目 | 実フック | 値【仮】 |
|---|---|---|
| 所持金帯 | `GameState.money` | `< 50_000`（低所持金帯。既存参照は event_design_v0 #2「百円パスタ」が `<5万`、`DialogueData.lowMoney` が `<3万`＝より深い帯。**#2 と同じ `<5万` に合わせる**） |
| 週 | 大会週を除く（`Calendar` の大会週集合の外） | 非大会週 |
| クールダウン | 同一キャリアで連週再発を防ぐ永続フラグ（例 `last飲み会Week`、`weeksSince >= N`） | N=8週【仮】 |
| 頻度 | 下記「発火の実装」参照（golden 影響が方式で分岐） | ― |

**【設計上の懸念・原案から要修正】原案の「bankruptLine近傍」は事実誤り**。現行 `GameConfig.bankruptLine = -1_000_000`（-100万＝夜逃げライン、GameEngine.swift:146 で参照）で、5万とは百万円以上離れている。本案が狙うのは夜逃げ寸前ではなく**MVPで最も長く滞在する"日常の金欠帯"**。文言は「低所持金帯（`< 5万`）」で起票する（本表に反映済み）。

**【設計上の懸念・balance】`compat` 上限20で A の相性+1 が頭打ち**。`compatCap = 20.0`（GameConfig:83、GameEngine:41 でクランプ）。相性が上限に達した終盤で本イベントが出ると、A の相性+1 は無効化され、A は「メンタル+1 のために4000円払う」だけの弱い選択に痩せる＝ドミナント崩れの温床。**対策（要判断）**：(a) 発火を `compat < compatCap` でゲートする、または (b) 相性が上限のときは A の主報酬をメンタル側へフォールバックさせる。実装時にどちらかを明示（既定は(a)＝データ側の `fireable` に条件を1つ足すだけで golden 非依存）。なお `compatGrows` は現状 `[TBD]`（GameConfig:84）＝相性成長自体が未確定である点も前提に注意。

## イベント本文

**場面（セットアップ）**

谷口の携帯が、鳴って、切れて、また鳴った。
稽古場の照明は、壁の箱に百円玉を入れると一時間だけつく。今夜の残りは、その百円玉で数えられる程度だ。

谷口「今日、同期が集まるらしいわ。会費、ひとり四千円やて」

俺「今月の稽古場代、残りがちょうど四千円なんだよな」

**選択肢**

**A｜行く（同期の輪に顔を出す）**
- 【効果】所持金 **-4,000**（会費）【仮】／相性 **+1**・メンタル **+1**【仮】（人脈と息抜き）
- 選んだ後の会話（話者ごと1セリフ・ボケない）

  谷口「久しぶりに、みんなの顔見たわ」

  俺「その分、来週は箱に入れる百円玉を、一枚ずつ数えることになる」

**B｜断って稽古場に残る**
- 【効果】所持金 **±0**（維持）／発想 **+1**（一人で書けた）・メンタル **-1**【仮】（外の輪から一歩下がる）
- 選んだ後の会話

  谷口「……なら、俺も残るわ」

  俺「払わずに済んだ会費が、二人で八千円。箱の照明なら、八十時間ぶんだ」（※デッドパンの一滴・低頻度）

いずれも感情語ゼロ・オチなし。俺は標準語・計算屋（金額を時間/明かりに換算する）で、自己憐憫を書かない。谷口は関西弁で、Bの「俺も残るわ」は**相方依存の声**＝谷口が自分の息抜きも黙って差し出す一言（悔しさは言語化しない）。固有の物＝「百円で一時間の照明」は、この稽古場・この二人にしか無い、残高と創作原資を1本の綱でつなぐ物。**換算は 100円＝1時間で全セリフ統一**（Bの「八千円＝八十時間」もこの式に一致させた）。

## 実装イメージ（既存機構に載る形・コードは"提案"）

**前提の事実**：現状、**選択肢つき突発イベントの実行エンジンは未実装**。`event_design_v0` の選択肢イベント群は"正典データ"として設計済みだが、実装済みの会話機構は `DialogueData.innerVoice`（`ManzaiGame/Sources/DialogueData.swift`、俺＝標準語のみ・DialogueData.swift:41）＝**UI側・非RNGの通知/独白のみ**で、選択肢分岐は持たない。よって本案は「実装順『突発イベント』段で入る**選択肢イベント基盤**に載る最初のデータの1枚」として提案する（基盤自体の設計は別票）。効果対象はいずれも既存 `GameState` フィールドで、新フィールド不要。

**発火の実装は3案。既定は(2)。**

- **(1) 状態駆動・完全決定（golden安全だが"突発"感が薄い）**：`money<5万 && 非大会週 && cooldown` だけで決定的に発火。`WeekRunner` の `RandomSource` を一切呼ばず golden 不変。ただし**毎プレイ同じ条件で必ず出る＝"突発"の意外性が消え、event_design_v0 の12%確率モデルとも噛み合わない**。
- **(2) UI局所の非golden乱数で確率発火（推奨・新規）**：発火抽選を **`runYear`/`WeekRunner` とは別系列の UI側 RandomSource**（golden に食い込まない独立シード）で引く。これで「12%の突発」らしさを出しつつ **golden・乱数消費順・`tools/*.py` に一切触れない**。効果適用（`money-=4000` 等）も UI 側 `GameState` 反映で `runYear` の乱数列に割り込まない。**golden 安全 かつ 確率的**。
- **(3) runYear内で抽選（案原文どおり12%）**：週ループ内で `WeekRunner`/`runYear` の乱数を引く＝**乱数消費順・golden・正典（`tools/*.py`）に触れる**。**この場合は必ず「止めて相談」**（CLAUDE §A：`tools`→`GameCore`→`gen_golden.py`→`swift test` を1コミットで揃える手順が要る。テキストだけの変更では済まない）。

**提案スニペット（未適用・基盤に載る想定のデータ形）**：
```swift
// 【提案・未実装基盤に載るデータ／効果値は全て【仮】】
// 選択肢イベントの最小形。効果は既存GameStateフィールドの増減のみ。
struct ChoiceEvent {
    let id: String                 // "broke_drinking_invite"
    let fireable: (GameState) -> Bool   // 案(2): money<50_000 && 非大会週 && cooldown && compat<compatCap
    let choices: [EventChoice]
}
struct EventChoice {
    let label: String
    let apply: (inout GameState) -> Void  // A: money-=4000; compat+=1; mental+=1
    let afterLines: [Advice]              // 話者ごと1セリフ（谷口=関西弁, 俺=標準語）
}
```
効果適用は上限/下限クランプ（`compat` は20上限＝GameEngine の既存ヘルパ）に委譲し二重管理しない。**このデータ自体は golden の乱数を引かない**（発火方式(1)/(2)前提）。

## リスク・注意

- **【最重要・balance】発火するのは"金が最も無い時"＝その瞬間に4000円払うAが最も割高**。B は無コストで発想+1＝金欠帯の文脈では **B が軟らかく上位互換化しうる**（金を守りつつ能力が増える）。対抗はAの相性+メンタル vs Bのメンタル-のみ。**"どちらも選ばれるか"を sim で必ず確認**してから会費/+値を確定（体感を数値でなく演出で解く／CLAUDE §B-5 は保つ）。ここが本案の生死。
- **【最重要・golden】発火方式で golden 影響が分岐**。(1)(2)＝golden 不変で安全、既定は(2)（確率の突発感＋golden安全）。(3)＝真ランダム週は乱数消費順に触れる＝**止めて相談**。
- **【要修正・済】「bankruptLine近傍」は事実誤り**（bankruptLine=-100万）。低所持金帯（`<5万`）へ修正済（発火条件節）。
- **【要判断・compat上限】** 終盤 `compat=20` で A の相性+1 が死ぬ。`fireable` に `compat<compatCap` を足す（既定）か、メンタルへフォールバック。
- **【要修正・済】B俺セリフの算術**。原案「八千円で二時間だけ明かりがついた」は換算式（100円＝1時間）と矛盾（8000円＝80時間）で、計算屋の声を壊す。**「八千円＝八十時間ぶん」に修正済**。計算屋のセリフは算術が合って初めてデッドパンになる。
- **選択肢イベント基盤が未実装**。本案はデータ1枚で、基盤（選択肢シートUI・効果適用・クールダウン永続化）は別工程。UI追加を伴うため完成判定は simulator 目視まで（CLAUDE §D-10）。
- **クールダウン必須**。金欠帯はMVPで長く滞在＝無制限だと連週で出て希釈する。`weeksSince >= N`【仮】で間隔管理。`last飲み会Week` はセーブに載せる永続フラグ。
- **声の規律**。俺＝全行標準語・金額を明かり/時間に換算する計算屋・自己憐憫ゼロ。谷口＝関西弁一貫（Bの「俺も残るわ」に本音を1つだけ）。**谷口以外の相方に関西弁を移さない**（partner_characters_v0：東雲ら他相方は全員標準語）。デッドパンの一滴はB俺の1行のみ。ボケ本文ゼロ。
- **固有の物の一意性**。「百円で一時間の照明」は本作固有プロップ。他イベントで別の意味に流用しない（定番プロップ化＝予定調和で沁みなくなる／proposals/0009）。既存作品のセリフ・設定のコピペは無し。

## レッドテスト（A/B）

**A. 実装妥当性**
- **採番衝突（既存0010）→ 0011 に修正。** 原案 `0010_broke-drinking-invite` は `0010_pre-tournament-eve-choice`（採用済）と衝突。
- **発火条件の事実誤り（bankruptLine近傍）→ 低所持金帯 `<5万` に修正。** `bankruptLine=-1_000_000`（GameEngine:146）で百万単位のズレ。既存 #2 の `<5万` に整合。
- **golden 安全性の分岐を3案に整理し(2)を新規推奨。** 原案の(1)決定発火は"突発"感を失い12%モデルと不整合、(3)は消費順に触れ止めて相談。**UI局所の非golden乱数**なら確率＋golden不変を両立＝原案の二者択一を1段改善。
- **ドミナント懸念を明文化。** 発火が"最も金が無い時"＝Aが最も割高＝**Bの軟上位互換化リスク**が構造的に残る。これは"正解のない選択"要件の核心で、**sim検証前は未決**（→verdict）。
- **compat上限で A の相性+1 が死ぬ穴を発見・対策提示**（`compat<compatCap` ゲート or メンタルフォールバック）。`compatGrows=[TBD]` も前提注記。
- **架空の"隠れコスト"を削除。** 別建ての将来ペナルティ機構は無い。コストは `money-4000` 一本と正直化。
- 効果対象は全て既存 `GameState` フィールド＝新システム不要。MVP(1年)で `<5万` は #2 同帯＝発火し得る（本編待ちでない）。

**B. 会話の中身**
- **算術バグを修正（最大の指摘）。** B俺「八千円で二時間」は 100円＝1時間 と矛盾（正しくは80時間）。計算屋の声が壊れるため「八千円＝八十時間ぶん」に書き換え、全セリフを換算式に統一。
- **セットアップ俺セリフの曖昧さを修正。**「あと四千円で足りるんだよな」→「残りがちょうど四千円なんだよな」（会費4000との等価デッドパンを明確化）。
- **方言・話者規律 OK。** 谷口=関西弁一貫、俺=標準語一貫、他相方に関西弁を移さない旨を明記。話者ごと1セリフ（"／"詰めなし）に整形。
- **感情語ゼロ／ボケ・オチ無し／固有プロップ（百円照明）一意 を確認。** 既存作品のコピペ・酷似なし（全文オリジナル）。デッドパンの一滴はB俺1行のみで乱発なし。
- **キザ／サムさ**：会費4000＝稽古場代残4000 の一致はやや作為的だが、それ自体が本イベントの芯（等価の対置）なので許容。

**残課題**：(1) sim で「AもBも選ばれるか（B軟上位互換化しないか）」の実測、(2) 発火方式(2)採用と compat ゲート方式のオーナー/実装判断、(3) 選択肢イベント基盤（UI・効果適用・クールダウン永続化）は別票で未実装。
