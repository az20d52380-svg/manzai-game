# 提案 v0 — 「前夜の一本」大会前夜(weeksLeft==1)の選択イベント（#28眠れない夜の選択版）

- 日付: 2026-07-06 ／ 数値は全て【仮】／ 既存ファイルは読むだけ ／ トーン＝ドラマ主軸（proposals/0009 確定）
- 対象読者: イベント機構（event_design §2 二択）実装担当・DialogueData / WeekMainView の書き手
- 位置づけ: 単発イベント #39 相当。既存 **#28 眠れない夜（自動・メンタル±2）** の**選択版**。**#38 賞レース前の温泉ロケ**（2週前・知名度ゲート）に頼っていた「大会前調整」の穴を、無ゲートの前夜1枚で埋める。

## 狙い

大会週は本作最大の感情ビートなのに、その直前週（前夜相当）で**プレイヤーが握れるハンドルが薄い**。現状ここに立つのは #28（自動・メンタル±2の運任せ）だけで、**「詰めるか／整えるか」という前夜固有の意思決定**が存在しない。#38 温泉ロケは "休養と収入 vs 調整" の別軸で、しかも知名度30ゲート＝終盤の大会密集期（GP各回戦が近接）に**毎回は出ない**。結果、**大会が近づくほど前調整の選択肢が枯れる**（event_design が指摘する「終盤の供給不足」）。

やることは一つ。**前夜（weeksLeft==1）に「最後まで詰めるか、整えて寝るか」の二択を置く**。要点は三つ。

1. **正解のないトレードオフ**（event_design §2/§設計原則3）。A=詰めは**上振れ／下振れの両方**を持つ不完全情報（プレイヤーに確率を見せない）、B=温存は**安定だが上積みなし**。期待値が計算で割り切れない＝明白な最適解を作らない。
2. **#28 との棲み分け**（温度事故と重複発火の停止）。前夜は #28 の "運任せの±2" でなく**プレイヤーの手**で温度を作る。#28 は自動枠から後退（→ 実装イメージ(3)）。
3. **相方依存の声で前夜のドラマを出す**（0009）。詰めの是非を相方との一往復で背負わせる。谷口＝関西弁、谷口以外＝標準語（声は相方IDで差し替え）。

## 発火条件（実フック）

- **一次条件**: `nextMilestone()`（`WeekMainView.swift:407-415`）が返す `weeksLeft == 1`。**育成 monoBox で到達しうる weeksLeft の最小値は 1**（大会週そのもの weeksLeft==0 は `tournamentDecision`＝別画面で、育成画面のイベント判定は走らない。proposal 0004 で確認済みの死に枝回避）。＝ここが実装上の「前夜」。
- **二次ゲート（反復制御・【仮】）**: **格の高い大会の前夜のみ確定発火**（新人賞級・準決・GP決勝）。GP各回戦のような近接・多数の前夜まで確定にすると、48週の中で同型イベントが乱れ撃ちになり予定調和化する。**回戦級の前夜は従来どおり #28（自動・軽い揺らぎ）に委ねる**。格の判定は `nextMilestone()` の `name` を `config.calendar` のラベルと突合して UI 側で導出（新フィールド不要・0004 と同じ手筋）。
- **再発**: 大会ごとに再発可（※）。ただし本文プールを回し（proposal 0001 の直近回避）、固定文の反復を避ける。
- **MVP 発火可**: 春新人賞 W12 等、1年内の格の高い大会前週で実働する（多年前提でない）。

## イベント本文（場面のセットアップ＋二択）

> トーン＝ドラマ主軸。感情語（悲しい/悔しい/惨め）を地の文から抜き、**"この稽古場・この相方に固有の少しズレた物"**（＝谷口が持ち込んで直さない、四分進んだ置き時計）に温度を背負わせる。俺＝標準語・計算屋・自己憐憫なし。**会話は話者ごとに1セリフずつ（"／"で詰めない）**。ボケ（オチ/ネタ本文）は書かない。数値は全て【仮】。

稽古場の置き時計は、谷口が持ち込んだ日から四分進んでいる。直そうと言うと、
谷口は「本番、四分早う終わる気分でちょうどええ」と言って、直さない。

**【効果】** 表現 +1（詰めの上積み・確定） ／ **内部判定（不完全情報・確率はプレイヤー非表示）**: 上振れ＝メンタル +1 ／ 低確率で 体力 −（数値【仮】）＝ **本番メンタルに −（大会入場時に発現・遅延ペナルティ）**。数値は全て【仮】。

**【効果】** 体力 +10 ／ メンタル +1。**詰めの一手は打たない**（表現の上積みなし・上振れも下振れも無い安定）。数値は全て【仮】。

## どこから学んだか（構造＋出典URL）

学んだのは**構造・型のみ**。既存作品のセリフ・攻略本文は転載しない。

- **必発セクション＝節目は"必ず起きる"に置く**（サクセスの確定イベント／彩りのランダムと二層）。前夜は "運任せ" でなく**プレイヤーの手が届く確定節目**にすべき。 https://ja.wikipedia.org/wiki/実況パワフルプロ野球_サクセスモード ／ 大会前を必発枠として平常ローテと分離: https://xn--odkm0eg.gamewith.jp/article/show/46318
- **逆U字理論（ヤーキーズ・ドッドソン）** — 緊張ゼロも過緊張も両方パフォーマンスを落とし中程度が最高。**「詰めれば必ず上がる」わけではない＝A に下振れ（詰めすぎて過緊張で本番メンタル−）を入れる根拠**。整えるB が "安定だが上限も低い" 非対称の理論的裏付け。 https://www.css.nihon-u.ac.jp/about/publicity/columns/post-1/
- **不完全情報下の意思決定（期待値が見えない選択）** — 確率を見せないことで「計算で割り切れない・最適解のない」判断になる（event_design §2 の核）。プレイヤーは期待値でなく"性格"で選ぶ。 https://ja.wikipedia.org/wiki/期待値
- **緊張の緩和（役割分担で緊張を作る/抜く）** — 心の声＝張る役、リザルト/相方＝抜く役。前夜の詰め（張り）と温存（抜き）は同じ構造の選択版。 https://waraitext.com/post-17/17/
- **感動を宣言しない（事実と温度差で置く）** — 前夜の重さを地の文で叫ばず、"四分進んだ時計"という物一つに背負わせる（0009 の芯）。 https://ameblo.jp/isawo-t1307/entry-12689913076.html
- 自作 `event_design_v0` §2・§設計原則3（正解のない二択）、`proposals/0004`（前夜＝weeksLeft==1 の確定・格ゲート）、`proposals/0009`（ドラマ主軸・固有の物・相方依存の声）。

## 実装イメージ（既存機構に載る形・コードは"提案"）

**(1) 二択イベントとして event_design §2 の型に載せる。** 効果対象は既存6軸＋体力＋所持金＋メンタルの増減のみ＝新システム不要（event_design 設計原則の範囲）。発火は `weeksLeft==1` ×格ゲート。

```swift
// 【提案・未適用】前夜イベント #39。event_design §2 の二択型に載る想定。
// 発火: nextMilestone().weeksLeft == 1 かつ isHighStakes(格の高い大会) のとき確定。
//       回戦級(!isHighStakes)は発火せず #28(自動)に委ねる。
// 効果は既存パラメータの増減のみ。A の内部判定だけが RNG を消費する（→(4) 要相談）。
struct PreTournamentEve {
    static let 表現UpA = 1          // 【仮】A 確定の上積み
    static let staminaBonusB = 10   // 【仮】B 体力
    static let メンタルUpB = 1       // 【仮】B メンタル
    // A の内部判定（不完全情報）:
    //   上振れ  → メンタル +1        【仮・確率非表示】
    //   下振れ  → 体力 -X, かつ本番メンタル -Y（遅延）  【仮・低確率】
}
```

**(2) A の「本番メンタル−」＝遅延ペナルティの持ち方。** 前夜で下振れを引いたら、その週で即メンタルを削るのでなく、**次の大会入場時に発現**させたい（"前夜に無理した疲れが本番に出る" のドラマ）。最小実装は `GameSession` に `pendingFinalメンタルPenalty: Int = 0`（【仮】）を持たせ、大会処理の直前に一度だけ適用して 0 に戻す。**これは大会判定の "手前" で state を触るだけ＝runner の RNG 消費順の内側には入れない**設計にできれば golden 非対象に保てる（→ 要確認・(4)）。**【設計上の懸念】** 遅延ペナルティの適用タイミングが大会スコア計算のどこに入るかで golden への影響が変わる。**入場前に state を確定させ、その後に runner を回す順序を崩さないこと**（消費順の正典・開発規律A-2）。

**(3) #28 眠れない夜との棲み分け（重複発火の停止）。** 前夜（格の高い大会）は本イベントが確定発火するので、**#28 の "大会前週・自動" 枠はそこから外す**。案の二択:
- (a) #28 を**回戦級の前夜専用**に後退（格の高い大会の前夜は本イベントが独占）。
- (b) #28 を**大会2週前**へずらし、前夜を本イベントに一本化。
→ **(a) を推奨**（回戦級は軽い運任せ #28、格の高い大会はプレイヤーの手＝メリハリが付く）。**これは event_design（既存doc）の #28 発生条件を上書きする**ので、取り込み時に該当doc へ「正典vN移行」バナーで履歴を残す（開発規律C-6）。

**(4) 【止めて相談】A の内部判定は RNG を消費する。** 選択肢A の上振れ/下振れロールは**新規の乱数消費**。イベント判定（週次12%）の RNG 消費順に**サブロールを1回差し込む**＝ **乱数消費順が変わり golden（3年ビット一致）が壊れ得る**。**ここは実装前に必ず止めて相談**（開発規律A-1/A-2）。取り込む場合の正典同期手順: ①`tools/*.py`（正典）に前夜ロールの消費順を先に定義 → ②GameCore へ反映 → ③`gen_golden.py` 再生成 → ④`swift test` green を**1コミットで揃える**。**B（体力/メンタルの確定加算）は RNG 非消費＝golden 不変**なので、MVP第一弾は "A=確定効果のみ・下振れ無し" で出し、下振れロールは相談後に足す段階投入も可（温度事故を出さずに前夜の選択だけ先に入る）。

**(5) 会話プールと反復回避。** A/B の後会話は相方IDで分岐（谷口＝関西弁プール／他相方＝標準語プール）。再発時の固定文反復は proposal 0001 の直近回避（状態別 recent・上限付き）を効かせる。`Advice.name` の話者切替（俺/相方）は dialogue_design §6-7 のコスト論点＝未実装なら相方セリフは俺の後会話に相方引用を挟む形で暫定表現も可（本実装は話者切替の口を待つ）。**UI は swift test で検証不可＝simulator で W11→W12 まで送って前夜イベントを目視**（開発規律D-10）。

## リスク・注意

- **【止めて相談・最重要】A の内部ロール＝RNG 消費順に触れる**（(4)）。golden が壊れ得る。相談前に実装しない。B は golden 不変。
- **【設計上の懸念】遅延ペナルティの適用位置**（(2)）。本番メンタル− を大会スコア計算のどこで引くかで golden 影響が変わる。**入場前 state 確定→runner 実行の消費順を崩さない**。
- **【doc上書き】#28 の発生条件を変える**（(3)）。event_design の既存記述を消さず「正典vN移行」バナーで残す（規律C-6）。
- **反復リスク**。前夜は大会ごとに再発＝格ゲートで頻度を絞っても、固定文だと予定調和で沁みない。**格の高い大会のみ確定・本文プール＋直近回避**で薄める。しきい値・格の線引きは【仮】、sim で「前夜イベントが年内で適切頻度か」実測してから詰める（体感を数値で直さない・ui_design §7-B）。
- **不完全情報の誠実さ**。A の確率を UI に出さない（＝計算不能）のが狙いだが、**下振れが理不尽に感じられない**よう頻度は低く・救い（表現+1 は確定で残る）を残す（event_design 原則4）。
- **口調ブレ**。俺＝全行標準語・計算屋・自己憐憫なし。谷口＝関西弁一貫。**谷口以外の相方に関西弁を混ぜない**（相方依存の声・0009）。ボケ（オチ/ネタ本文）ゼロ＝全て反応・観測。既存作品（火花/べしゃり暮らし/M-1実在ネタ）とのセリフ一致は各行照合し無し。固有名は架空・数値は【仮】。
- **デッドパンの一滴は "四分進んだ時計" の1個だけ**（5-6回に1回の規律）。後会話に二個目のウィットを盛らない（1テキスト＝技法1つ）。

## レッドチーム（A/B）

**A. 実装妥当性**

1. **【確認済】前夜＝weeksLeft==1 が実フックと一致**。`nextMilestone()`（WeekMainView.swift:407-415）は `next.0 - session.week` を返し、育成画面で到達する最小は 1（大会週 weeksLeft==0 は tournamentDecision＝別画面）。0004 と同じ確認。`config.calendar` からの格導出も新フィールド不要で成立。
2. **【要相談・降格】A の下振れロールが golden を壊す**。イベント RNG 消費順にサブロールを差し込むため。→ MVP第一弾は **A=確定効果のみ（下振れ無し）＋B** で出荷可、下振れロールは正典同期手順を1コミットで揃えて相談後に追加、と段階投入へ降格。温度事故は出ない。
3. **【設計上の懸念】遅延ペナルティの state 持ち越し**。`pendingFinalメンタルPenalty` の適用位置次第で golden 影響が変わる。入場前確定・消費順維持を明記。
4. **【doc整合】#28 の条件変更が必要**。前夜の二重発火を避けるため #28 を回戦級専用へ後退（(3a)）。event_design 上書き＝バナー必須。未実装ではなく既存doc数値の読み替え扱い。
5. **【確認済】新システム不要**。効果は6軸＋体力＋所持金＋メンタルの増減のみで、event_design 設計原則（新システムを要求しない）に収まる。B は純加算＝golden 不変。

**B. 会話の中身**

1. **【確認済】ボケ無し・ドラマ主軸**。A/B とも俺＝事実と判断、谷口＝短い同意/観測。オチ・ネタ本文なし。「朝の自分を信用してない」は自嘲でなく計算屋の事実処理＝自己憐憫回避。
2. **【確認済】固有の物が定番プロップでない**。天井の染み・冷めたコーヒーを避け、"谷口が持ち込んで直さない四分進んだ置き時計" ＝この相方・この稽古場固有・少しズレた物1個。デッドパンの一滴も自己言及ゼロ。
3. **【確認済】相方依存の声**。谷口＝「せやな/へんねん/ええ判断やと思うで」で関西弁一貫。谷口以外＝標準語バリエーションを別プールで用意し、関西弁を混ぜない。相性帯（0-7/8-14/15-20）で温度差も出せる（compat上限20＝阿吽21+は出さない）。
4. **【確認済】1セリフずつ**。全会話が話者ごと改行・"／"で詰めていない＝普通のゲームの会話送り。
5. **【確認済】既存作品コピペ無し**。谷口/相方/俺の各行を火花・べしゃり暮らし・M-1実在ネタと照合し一致・酷似無し。時計のくだり・「半拍を明日の勘に預ける」は新規。固有名は架空、数値は【仮】。
6. **【確認済】技法1つ/行・一滴規律**。デッドパンは時計の1個のみ、後会話に二個目のウィットを盛っていない。
