# イベント提案 0011「畳んだコンビの椅子」v0（2026-07-06・レッドチーム済）

> **【採番修正】** 起草時は「提案0010」表記だったが `proposals/0010_pre-tournament-eve-choice.md` が既存＝衝突。**0011へ採番し直し**。ファイル名 `0011_peer-folded-chair.md`・索引は `NIGHT_LOG.md` へ追記。

『同期の脱退』seed のMVP発火版。既存 #30「同期の解散」（`event_design_v0.md` §1-G・3年目自動・メンタル-3/発想+2の通知型）とは**別物**。あちらは多年でしか出ない自動イベント。本案は**1年内・週20以降にランダム発火する選択版**で、去った者の空き枠を「取る／取らない」のトレードオフを載せる。数値は全て【仮】、既存ファイルは読むだけ。

## 狙い

MVP（1年48週）は場数・数値の最適化ゲームに寄りやすく、**「続けている自分」を外から照らす瞬間**が薄い。#30 は3年目以降なのでMVPでは発火しない＝1年プレイヤーは「同期が畳む」世界に一度も触れない。ここを、**週20以降（少し場数を踏み、劇場の勝手が分かってきた頃）に立つ選択イベント**で埋める。

狙いは二つ。
1. **実利と後味の相克で最適解を消す**（event_design §2）。空いた出番枠は場数と知名度の即物的な得。だがそれは「抜けた奴の座っていた場所に座る」ことでもある。得と後味が同じ一つの椅子に同居する。
2. **相方依存で温度を変える**。谷口との相性帯で会話の湿り気だけを変え、数値は変えない（声の較正はテキスト側）。

## 発火条件（実フック）

- **状態フック**: `week >= 20`（`GameSession.week`）。**年次ゲートなし**＝MVPで確実に「発火**帯（＝抽選対象週）**」に入る。
- **抽選**: 大会週を除く自由週のイベント判定（既定12%・`event_design` §頻度）に**キャリア1回制**で乗せる。同一キャリアで**1回だけ**。
- **【是正・元案の overclaim 修正】** 「発火帯に入る」＝「必ず発火する」**ではない**。実際の発火は 12%抽選 × 他イベントとの枠競合 × `EVENT_FIRE_CAP=15`（§4-B）予算内の確率事象で、**1年run内で一度も出ないことがある**。本案の狙い（1年プレイヤーが同期の解散に一度は触れる）を**保証**したいなら pity/確定枠が要り、それは分布に効く＝**実装時に相談**（下記リスク）。保証しないなら「出れば拾える彩り」として割り切る。**元案の「MVPで確実に発火」は誤りにつき撤回**。
- **前提しない状態**: 連敗数・直近敗退・知名度・所持金は発火条件に**しない**（世界の彩りで、プレイの巧拙と無関係に「同期は畳む」を置くため）。ただし会話プールは相性帯（0-7他人行儀／8-14噛み合い／15-20ツーカー）で分岐（下記）。
- **【設計上の懸念・要目視】** 発火帯を `week>=20` の一点にするか、`20<=week<=40`（大会直前の詰め週を避ける）に絞るかは温度判断。simでは測れない＝ simulator 目視で決める（CLAUDE §10）。

## イベント本文

**セットアップ**（発火時・地の文＝俺視点）:

> 同期のコンビが、今日で畳む。片方が辞める。もう一方は、一人でどうするかまだ決めていないらしい。
> 谷口が倉庫から折り畳みのパイプ椅子を一脚提げてきた。あいつがいつも座っていた椅子だ。座面は片側だけ沈んでいて、座り方の癖がそのまま型になって残っている。背もたれには、コンビ名を書いたガムテを剥がした跡が、粘って光っている。
> 枠はひとつ。座れるのは、ひとりずつだ。当たり前のことを、わざわざ確かめた。

谷口「あいつら、今日で解散やって。倉庫のこれ、俺らにくれるって」

俺「あいつの出番枠、来月から空くらしい」

---

### 選択肢A：空き枠を引き継ぐ

**【効果】** 知名度 +2 ／ 体力 −10（出番増）／ メンタル −1（抜けた穴に座る後味）。相性は動かさない。

（A選択後）

俺「枠は、取る。場数がいる」

谷口「取ろか。……座り心地は、あんま良ないやろけどな」

（A選択後・相性15-20＝ツーカー帯のみ差し替え／地の文一行）

> 谷口はパイプ椅子を舞台袖まで運んで、沈んだ側を壁に向けて置いた。俺が座る側が、平らな方だった。

---

### 選択肢B：枠は取らず、送り出しだけする

**【効果】** メンタル +1 ／ 相性 +1（`compat`・**上限20でクランプ**。谷口が既に20なら加算は乗らない＝ツーカー帯では実質メンタル+1のみ。※`config.compatGrows` が false 化されると加算経路ごと無効＝この場合もメンタル+1のみ／リスク欄）。場数は増えない・知名度も動かさない。

（B選択後）

俺「枠はいい。今取る枠じゃない」

谷口「ほな、見送りだけ行こか。あいつ、俺らが来たら椅子の話しかせえへんやろけど」

（B選択後・相性0-7＝他人行儀帯のみ差し替え）

俺「枠はいらない。……理由は、うまく言えない」

谷口「言わんでええ。この椅子、あっちに戻してこよか」

## どこから学んだか（構造＋出典URL）

- **確定発火／ランダム彩りの二層**（節目は確定、世界の質感はランダムで散らす）。本案は「彩り＝ランダム」層に置く同期イベント。パワプロ「サクセス」の二層構造。https://ja.wikipedia.org/wiki/実況パワフルプロ野球_サクセスモード
- **去る者と残る者の非対称**（同じ養成所を出て、片方が辞め片方が続く＝下積みの非対称そのもの）。この構造だけを抽出、声は移植しない。又吉直樹『火花』の世界。https://ja.wikipedia.org/wiki/火花_(小説)
- **「空いた席」というメーター＝物that語る**。人が抜けた事実を、感情でなく「残された物（座面の型・剥がしたガムテ跡）」で置く手法（proposals/0009 の芯＝感情でなく物）。https://toyokeizai.net/articles/-/717910
- **正解のない二択＝上位互換を作らない**（リソース交換であって片方が明白に強くない）。event_design §2。（内部doc `docs/event_design_v0.md` §2・§6）

学んだのは構造・型のみ。既存作品のセリフ・本文は一切引かない（#30 の flavor 文とも非重複を確認）。

## 実装イメージ（提案・既存機構に載る形）

**現状の前提**: `ManzaiGame/Sources/` に**選択肢イベントのランタイムは未実装**（Event系ファイルなし）。`event_design_v0.md` §5 の通り、イベントは GameCore では `(id, 効果パッチ, 選択肢)` の純データ／テキストは UI 層リソース、という設計だけがある。よって本案は**そのデータ骨子の提案**。

想定データ形（純データ・テキストは別リソース）:

```swift
// 【提案・未適用／選択肢イベント機構が入った時の1エントリ】
// 効果パッチは既存 GameState 6軸+体力+知名度の増減のみ。新システム不要。
EventDef(
    id: "peer_folded_chair",
    fire: .random(minWeek: 20, oncePerCareer: true),   // 年次ゲートなし・発火は確率事象
    choices: [
        .init(id: "A_take",  patch: [.fame(+2), .stamina(-10), .mental(-1)]),
        .init(id: "B_seeoff", patch: [.mental(+1), .compat(+1)]),  // compatはclamp(0,20)既存経路（compatGrows前提）
    ]
)
```

**効果の適用経路**: いずれも既存 `GameEngine` の clamp 済み加算経路に載る。compat は `GameEngine.swift:41` の `clamp(s.compat + amount, 0, config.compatCap)` を通る＝上限20は自動。**ただし直上 `GameEngine.swift:40` に `if config.compatGrows` ガードがある**＝compat加算はこのフラグ依存（既定 true・【TBD】）。知名度・体力・メンタルは既存フィールド（`GameState.fame/stamina/メンタル`）の増減のみ。

**会話の相性分岐**: 発火時の相性帯（`state.compat` を 0-7／8-14／15-20 で判定）で会話プールを差し替えるだけ。話者名（俺／谷口）は `Advice.name` 相当の枠に載る。

**RNG・golden・乱数消費順**: 本案は**イベント抽選ストリーム**（`event_design` §5 で大会結果と分離を明記）を引く。どのストリームから・どの順で引くかは**乱数消費順の正典に触れる**。**→ 実装時に必ず止めて相談**（`tools/*.py` 正典 → GameCore → gen_golden → swift test の1コミット同期／CLAUDE §A）。効果パッチ自体は数式・消費順を変えないが、**「いつ何回発火するか」は分布に効く**ため `sim_career.py` の EVENTS_ON で優勝率1〜2%帯維持を再計測（§4-B(c)・本案は恒久能力+0で安全側）。

## リスク・注意

- **【要相談・正典】発火抽選の乱数消費順**。抽選を引く位置は golden に効く＝「止めて相談」対象（CLAUDE §A-2）。
- **【新規・A軸の要注意】EVENT_FIRE_CAP との相互作用で"偽の選択"化**。§4-B の総量予算は**超過分は「フレーバー表示のみ・効果は適用しない」**運用（`sim_career.py:505`）。**選択イベントが予算超過時に発火すると、A/B どちらを選んでも数値パッチが不発＝選択が無意味化**する。回避策: (a) 本イベントを予算カウント**対象外の"彩り枠"**として明示予約する（総量15とは別勘定）、(b) 予算超過時はこのイベント自体を抽選から落とす（選ばせて空振りさせない）。**実装時に(a)/(b)いずれか要決定**。
- **【発火保証なし】** 狙い達成（1年で一度は触れる）を保証するには pity/確定枠が要り、それは分布に効く＝相談対象。保証しないなら「出れば拾える彩り」と割り切る。オーナー判断待ち。
- **#30 との棲み分け**。#30（3年目自動・通知型・メンタル-3/発想+2）と本案（1年・選択型）は別イベントとして両立。本編（多年）では**同一プレイで両方出る**可能性＝「同期が畳む」が二度。天丼化するなら本編側で相互排他フラグ（片方出たら他方の重み減）を検討。MVP単体では #30 が出ないので衝突しない。
- **後味 −1 の押し売り回避**。A のメンタル−1 は翻訳。数値で悲哀を押し付けると感動ポルノ化する（0009・体感は演出で解く）。−1 は**小さく保ち**、効かせるのは会話（座面の型・平らな側を空ける谷口）側。
- **相性クランプ／compatGrows の説明責任**。B の compat+1 はツーカー帯（20）で乗らない＝上級者には実質メンタル+1のみ。「阿吽の谷口とは、もう送り出しで距離は縮まない」の翻訳として**意図的**。加えて `compatGrows`【TBD】が false 化されると全プレイヤーで加算が消える＝この依存を**実装コメントに残す**（バグでなく仕様）。
- **バランス再計測**。イベント追加＝`sim_career.py` EVENTS_ON で分布再測（§4-B・§5）。本案は恒久能力効果+0（fame/体力/メンタル/相性のみ）で「無限稽古」化リスクは低いが、総量予算15の扱い（上記(a)/(b)）確定後に再測。
- **固有名・数値**: 畳む同期コンビ名は本文で名指ししない（架空・匿名で温度を保つ）。全数値【仮】。既存作品セリフ流用なし・ボケ本文ゼロ。

**トーン自己点検**: 感情語（悲しい/悔しい/惨め）不使用。感情は物＝「片側だけ沈んだ座面／剥がしたガムテの粘る跡／平らな側を空けて置く谷口／あっちに戻す椅子」に背負わせた。固有プロップ＝定番（天井の染み・冷めたコーヒー）でなく「この同期の・この椅子固有の、座り方の型が残った折り畳み椅子」1つ。俺＝標準語・計算屋（「場数がいる」「今取る枠じゃない」）・自己憐憫なし。デッドパンの一滴＝「枠はひとつ。座れるのは、ひとりずつだ。当たり前のことを、わざわざ確かめた。」を1箇所のみ。谷口＝関西弁一貫（相方依存の声）。会話は話者ごとに1セリフずつ・「／」詰めなし。ボケ本文なし。

## レッドテーム（A/B）

### A. 実装妥当性

- **[修正済・採番衝突]** 起草時「0010」だが `0010_pre-tournament-eve-choice.md` が既存。**0011へ採番修正**（本文冒頭バナー）。放置すればファイル上書き事故。
- **[修正済・overclaim]** 「年次ゲートなし＝MVPで確実に発火」は**誤り**。week>=20 は抽選"対象週"に入るだけで、12%×枠競合×総量予算の確率事象＝1年で0回もある。**発火帯／発火保証を分離**し撤回。狙い（一度は触れる）を保証したいなら pity＝分布に効く＝相談、と明記に格上げ。
- **[新規・最重要]** **EVENT_FIRE_CAP 超過時の"フレーバーのみ"運用と選択イベントの相性が悪い**（`sim_career.py:505`）。予算超過で発火すると A/B どちらも数値不発＝**選択が空振り＝「正解のない二択」以前に「意味のない二択」**。彩り枠を予算別勘定にする(a) or 超過時は抽選除外する(b)、を実装前に決める必要。→ リスク欄に追加済。
- **[検証済・OK]** 効果パッチは既存 clamp 加算経路に全て載る（`GameEngine.swift:38/41/44/46`）。新システム不要は正しい。ただし compat 加算は **line40 `if config.compatGrows`【TBD】ガード依存**＝この条件を本文・コメントに明示（追記済）。
- **[OK]** RNG消費順を「止めて相談」対象と明記＝CLAUDE §A 準拠。効果自体は数式・消費順不変（既存フィールド増減のみ）という切り分けも妥当。
- **[破綻ルート確認]** 低体力で大会直前週に発火→A(-10)でrun崩壊、は**選択で回避可（B）**＝バグでなく設計上のジレンマ。連敗/知名度を発火条件にしない判断も彩りイベントとして一貫。特定ルートでの一方的破綻は無し。
- **[MVP/本編判定・OK]** 本案=MVP発火（week>=20・1年内）、#30=本編（3年目）と明確に別。棲み分け記述あり。

### B. 会話の中身

- **[修正済・register漂流]** 元案 谷口「今日は、そういう日や」は"その日の空気"を**名指し**＝感情の翻訳を怠り抽象に逃げた（0009「感情を書くな・物を書け」違反寄り）。**「言わんでええ。この椅子、あっちに戻してこよか」に差し替え**。物（椅子を戻す＝枠を取らない選択の身体化）で処理、A（椅子を残し平らな側に座る）と鏡構造になり物語が締まる。
- **[良・維持]** A の谷口「座り心地は、あんま良ないやろけどな」＝デッドパンで後味を背負う一滴。A 15-20帯の地の文「沈んだ側を壁に向けて置いた。俺が座る側が、平らな方だった」＝谷口の無言の気遣いを**説明せず物で**。0009 の理想形。改変不要。
- **[良]** 俺=標準語・計算屋（「場数がいる」「今取る枠じゃない」）／谷口=関西弁一貫。相方依存の声=谷口のみ関西、を遵守。話者ごと1セリフ・「／」詰めなし＝通常の会話送り準拠。
- **[許容]** 俺「理由は、うまく言えない」は感情近接だが"言語化の失敗"として抑制的＝自己憐憫には落ちていない。谷口が物で受けるので浮かない。可。
- **[確認済・コピペ無し]** #30 flavor（「打ち上げで泣きながら笑った」等）とも、火花本文とも非重複。火花の"僕・関西の笑い・思弁"は移植せず世界だけ抽出＝0009 の一線を守れている。ボケ/オチ本文ゼロ。キザ・サムさ無し（比喩の連射なし・粘るガムテ跡は具体で下品でない）。
- **[明白な最適解チェック]** A(知名度+2/体力-10/メンタル-1) と B(メンタル+1/相性+1) は相互に非支配＝どちらも strictly better でない。ただし**MVPでは知名度が出番・オファーのゲート通貨**として効くため、min-max視点で A が"やや強く見える"傾向はある。数値で締め直すのは 0009/§7-B（体感を数値で直さない）に反するので**据え置き**、A の後味は会話（沈んだ座面・戻す椅子）で担わせる方針を維持。

**総評**: 構造は既存機構に無理なく載り、トーンはドラマ主軸の理想に近い（特にA15-20帯とB他人行儀帯の物-描写）。**採番衝突・発火overclaim・FIRE_CAP空振りの3点は要修正→本版で是正済**。残る要相談は「RNG消費順」「発火保証の要否」「予算別勘定(a)/(b)」の3点＝いずれも実装着手時のオーナー/正典判断。
