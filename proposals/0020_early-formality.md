# 提案 v0(採番0020) — 「まだ敬語の残る間」相性0-7(他人行儀帯)×結成初期の選択イベント

- 位置づけ: 単発選択イベント #40 相当。**相性帯（0-7 他人行儀）で分岐する初の選択枠**。既存の相性イベントは「相方と過ごす」（GameConfig の回復＋相性 action）しか無く、**相性帯そのものを条件フックにした"選択"は現状ゼロ**。その gap を、無ゲート・序盤で自然に立つ1枚で埋める。

## 狙い

相性は行動（ネタ合わせ／相方と過ごす）で黙って積む数値でしかなく、`state.compat`（GameState.swift:13・init 5.0／cap 20.0）が上がっても**プレイヤーがその距離感に触る瞬間が無い**。結成直後の **0-7＝他人行儀帯**（compat 初期値5はここに入る＝序盤で必ず通る帯）は探り合いの時期なのに、そこ固有のドラマも選択も存在しない。

やることは一つ。**他人行儀帯の結成初期に「踏み込むか、間合いを保つか」の二択を置く**。要点は三つ。

1. **正解のないトレードオフ**（event_design §2）。A＝踏み込む＝**永続の前進を"実コスト"で買う**（相性↑・体力↓）、B＝間合いを保つ＝**消耗せず整える（が距離は動かない）**（体力↑・相性据え置き）。**A の相性は本イベント唯一の相性源＝"ただの得"ではなく体力という実弾で払う**。B は次週の稽古の燃料（体力）を蓄える別ルート。前進と整えが等価。
2. **相性帯を初めて"条件フック"に使う**。既存の JudgeData（compat>=18 で審査コメント分岐・ManzaiGame/Sources/JudgeData.swift:57）は帯を**結果表示**に使うが、**入力＝選択の発火条件**には使っていない。ここを相性帯ドリブンの選択の最初の一枚にする。
3. **相方依存の声で"距離"を出す**（0009）。他人行儀＝まだ本音の一歩手前、を相方との一往復で背負わせる。谷口＝関西弁、谷口以外＝標準語（声は相方IDで差し替え）。**本帯（0-7）専用**なので噛み合い/ツーカー版は要らない（compat上限20＝谷口に阿吽21+は出ない）。

## 発火条件（実フック）

- **一次条件（相性帯）**: `state.compat` が **0〜7（他人行儀帯）**。config.compatInit 5.0（GameConfig.swift:82）なので結成直後は確定でこの帯。
- **二次条件（結成初期）**: `session.week`（GameSession.swift:14・`private(set) var week: Int`）が **< 15**（目安・【仮】）。週は 0 始まりで pump 経由に進む。帯が空いていても週が深ければ出さない＝"初期の探り合い"というテーマを守る。
- **三次（ランダム）**: 週次の突発イベント判定内で抽選（実装順「突発イベント」枠・event_design §2 の型）。頻度は【仮】、**結成期の1回だけ立てば十分**（→ 実装イメージ(4) で一度きりフラグ）。
- **MVP 発火は"機構次第の条件付き"**: compat 初期5・週1〜14 は 1年内の序盤そのものだが、**選択イベント dispatch と話者切替UIが未実装**（→ リスク・注意）。機構が MVP に載れば発火可、素の現状（innerVoice 単発・俺のみ）では出せない。
- **【設計上の懸念】発火機構そのものが未実装**。CLAUDE 実装順で「突発イベント」は大会の後。選択イベントの共通ディスパッチが無い段階なら、本件は**その第一号として機構ごと相談**（0010 と同じ前提）。

## どこから学んだか（構造＋出典URL）

学んだのは**構造・型のみ**。既存作品のセリフ・攻略本文は転載しない。

- **タックマンのチーム発達段階（forming→storming→norming）** — 「混乱期（storming＝ぶつかる）」を通らないと「統一期（norming）」に行けない。**A＝踏み込む＝storming（消耗だが前進）／B＝保つ＝formingに留まる（消耗せず整えるが距離は動かない）**の非対称の理論的裏付け。 https://en.wikipedia.org/wiki/Tuckman%27s_stages_of_group_development
- **社会的浸透理論（Altman & Taylor・自己開示で関係が深まる）** — 関係は自己開示の深さ・幅で進む。「半分しか信じてない」と踏み込む＝自己開示のコストを払って距離を縮める行為。据え置き＝開示を保留。 https://en.wikipedia.org/wiki/Social_penetration_theory
- **好感度／親密度パラメータで分岐する関係イベント（構造のみ）** — 恋愛SLGの好感度帯・サクセスの相棒イベントは「関係値の帯」で場面が変わる。本件は相性帯を初めて入力フックに使う設計を、この二層（行動で積む値＋帯で発火する場面）から借りる。 https://ja.wikipedia.org/wiki/ときめきメモリアル ／ https://ja.wikipedia.org/wiki/実況パワフルプロ野球_サクセスモード
- **期待値の見えない選択（不完全情報）** — A の「不発なら据え置き」を将来確率化する場合、確率を見せないことで計算不能＝性格で選ぶ判断になる（event_design §2 の核）。**本改稿では実コストで成立させ、不完全情報は後回しにできる**。 https://ja.wikipedia.org/wiki/期待値
- **感動を宣言しない（事実と温度差で置く）** — 距離の縮まりを地の文で叫ばず、"両手で返すネタ帳・拳ひとつ分の隙間"という物一つに背負わせる（0009 の芯）。 https://ameblo.jp/isawo-t1307/entry-12689913076.html
- 自作 `event_design_v0` §2（正解のない二択）、`docs/canonical_v2_spec`（compat init 5 / cap 20 / スコア＝実力+相性+ブレ）、`proposals/0009`（ドラマ主軸・固有の物・相方依存の声）、`proposals/0010`（選択イベント発火機構の前例）。

## 実装イメージ（既存機構に載る形・コードは"提案"）

**(1) 二択イベントとして event_design §2 の型に載せる。** 効果対象は既存6軸＋体力＋所持金＋相性の増減のみ＝新システム不要。発火は `compat∈[0,7]`×`week<15`×抽選。**A/B とも RNG 非消費＝golden 不変**（下振れロールを入れない限り）。

```swift
// 【提案・未適用】他人行儀帯イベント #40。event_design §2 の二択型に載る想定。
// 発火: 0.0 <= state.compat <= 7.0 かつ session.week < 15 かつ 抽選ヒット かつ 未発火。
// 効果は既存パラメータの増減のみ・RNG非消費（不発ロールは入れない → (4)要相談）。
struct KeigoNoNokoruMa {
    // A: 踏み込む（永続の前進を実コストで買う）
    static let compatUpA   = 2      // 【仮】相性 +2（確定・本イベント唯一の相性源）
    static let staminaDownA = -15   // 【仮】衝突は消耗する（体力に実弾で効く＝トレードオフの要）
    // B: 間合いを保つ（消耗せず整える・距離は動かさない）
    static let staminaUpB  = 10     // 【仮】次週稽古の燃料を蓄える
    static let mentalUpB   = 1      // 【仮】メンタル微増
    // B の相性は +0（据え置き）＝A の「買う」と対にする
}
```

**(2) 相性帯判定は新規に導入が要る（既存は帯を持たない）。** JudgeData.swift:57 は `state.compat >= 18` の**単一閾値**であって 0-7/8-14/15-20 の三帯システムは**現状コードに存在しない**。よって「他人行儀 0-7」の帯境界は **GameConfig に新規で一本化して定義**し（compatCap=20・GameConfig.swift:83 に合わせる）、以後 JudgeData 側もその定数を参照する形に寄せて二重定義を避ける（開発規律B-4）。＝"既存と共有"ではなく"帯定義そのものを足す"のが正しい認識。

**(3) 効果適用は event 専用の apply で clamp を踏襲。** 相性の増減は GameEngine.swift:40-41 の `clamp(compat+amount, 0, compatCap)` と**同じ式**を踏むが、**その行そのもの（`if config.compatGrows` でゲートされた training 経路）に載せてはいけない**。イベント効果は compatGrows フラグ（training用・【TBD】）と独立に確定適用すべき。体力は `clamp(stamina+amount, 0, 100)`（GameEngine.swift:44）と同経路。**いずれも RNG 非消費＝golden 不変**。

**(4) 【止めて相談・入れる場合のみ】A の「不発なら据え置き」は RNG を消費する。** 不発ロールは**新規の乱数消費**で、突発イベント判定の RNG 消費順に**サブロールを1回差し込む**＝**乱数消費順が変わり golden（3年ビット一致）が壊れ得る**。**実装前に必ず止めて相談**（開発規律A-1/A-2）。取り込む場合の正典同期: ①`tools/*.py`（正典）にロール消費順を先に定義 → ②GameCore へ反映 → ③`cd tools && python3 gen_golden.py` 再生成 → ④`cd GameCore && swift test` green を**1コミットで揃える**。**本改稿はコスト（体力−15）で既にトレードオフが成立するので、不発ロールは"任意の後付け"に降格**（無くても選択の等価性は保てる）。

**(5) 一度きり発火の管理（golden 対象外）。** 結成期の探り合いは何度も来ると予定調和。`GameSession` に `didSeeKeigoMa: Bool`（【仮】）を持たせ発火後 true。**これは UI 層の状態で runner の RNG を呼ばない**＝golden 非対象。前例は**既に出荷済みの `justPassedStage`（GameSession.swift:32）・`lossStreak`（同:29）**＝UI 層で持つ非RNGフラグと同型（0003 の justLost は未配線なので、生きた前例はこちら）。セーブ永続化は本編工程（MVP は1周内で足りる）。

**(6) 会話プールと話者切替（MVP の最大の穴）。** 現状 `Advice.name` は「俺」ほぼ固定で、コメントに「将来のレアイベントで谷口等に切替」（DialogueData.swift:11）とある通り**話者切替の口は未実装**。かつ innerVoice は週1件の俺モノローグを返すだけで、**複数話者の会話送りUIも選択後会話UIも無い**。本イベントは谷口の複数セリフを要するので、**dispatch＋話者切替UIが載るまでは素の現状で出せない**。MVP最小で先に出すなら、選択後の相方セリフを**俺の心の声に引用として畳む暫定形**（0010(5) と同手筋）。**UI は swift test で検証不可＝simulator で結成直後（週1〜14・compat 5前後）まで送って目視**（開発規律D-10）。

## リスク・注意

- **【最重要・トレードオフ】相性を"ただの得"にしない**。**スコア＝実力値＋相性＋ブレ（GameEngine.swift:132）で相性は永続・1:1 でスコア平均に効く**一方、**メンタルは weight=0（同:18）でスコアに直接乗らず、ブレ幅を僅かに絞るだけ**（blurWidth=5+15(1−メンタル/100)・同:122）。旧案（A 相性+2/メンタル−2 vs B 相性+1/メンタル+1）は**A がスコア期待値で B を確定支配**＝§2破綻だった。本改稿は A のコストを**体力**へ移し（体力は staminaPenalties でスコアに実損・稽古の staminaGate も効く＝実弾）、B の相性を +0 に。**A=永続の前進を実弾で買う／B=消耗せず整える**の等価に組み替えた。**しきい値・増減幅は【仮】、sim で「序盤に自然な頻度・A/B が拮抗するか」実測してから詰める**（体感を数値で直さない・ui_design §7-B）。
- **【設計上の懸念】選択イベント発火機構＋話者切替UIが未実装**（(6)）。共通ディスパッチが無ければ本件は第一号として機構ごと相談（0010 同様）。MVP発火はこの機構次第の条件付き。
- **【帯導入】0-7 帯定数はコードに未実在**（(2)）。JudgeData は >=18 単一閾値のみ＝帯システムを新規に足す前提。GameConfig 一本に置き散らさない（開発規律B-4）。
- **一度きり制御**（(5)）。結成期に何度も出すと探り合いが予定調和化。`didSeeKeigoMa` で1回に絞る（UI層・非RNG）。
- **口調ブレ**。俺＝全行標準語・計算屋・自己憐憫なし。**"仕舞っとく"のような関西縮約を俺に混ぜない**（→"仕舞っておく"に修正済）。谷口＝関西弁一貫。**谷口以外の相方に関西弁を混ぜない**（相方依存の声・0009）。ボケ（オチ/ネタ本文）ゼロ＝全て反応・観測。既存作品（火花/べしゃり暮らし/M-1実在ネタ）とのセリフ一致は各行照合し無し。固有名は架空・数値は【仮】。
- **デッドパンの一滴は "今、初めて片手でネタ帳を寄越したな" の1個だけ**（5-6回に1回の規律）。地の文と後会話に二個目のウィットを盛らない（1テキスト＝技法1つ）。距離を地の文で"まだ縮まらない"と宣言しない（→削除済・物だけに背負わせる）。

## レッドチーム（A/B）
