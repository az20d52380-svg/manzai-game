# イベント提案 0025「楽屋で無言の十分」v0（2026-07-06・レッドチーム済）

`docs/event_design_v0.md` §1「#18 楽屋で無言」（相性8未満・敗退後・**通知型**・flavor一行のみ）の**会話化**版。ただし**別物**として置く。#18 は「不仲の入口（相性8未満）」を一行通知で示す既存イベント。本案は**その一つ上の帯＝相性8-14（噛み合い始め）で、スベった夜に立つ非選択の一場面会話**。選択肢は無い（proposals/0010-0023 のパワプロ型＝A/B効果つきとは別カテゴリ）。数値は全て【仮】、既存ファイルは読むだけ。

> **【レッドチーム是正・最重要】** 起草時、棲み分けの対象を #18（敗退後）だけに絞っていたが、**台帳 0014「終電までの反省会」（`fable_kickoff_drama_allin_v1.md` §5-E）が「フリーライブ直後」で発火＝本案とフックが重なる**。0014 は選択イベント（A敗因洗う／B畳んで飯・知名度<20ゲート）、本案は非選択・compat 8-14ゲート・効果なし。**同一週に両方が発火候補になり得る**ため、下記に相互排他／優先度と、フリーライブの「低評価」判定フックの実在確認を追記した（元案は 0014 との衝突を見落とし）。

## 狙い（どの薄いスロットを埋めるか）

薄いのは二か所で、両方を一場面で埋める。

1. **#18 が会話化されていない**。`event_design_v0.md` §1 の #18 はflavor一行（通知型）で、タップ送りの会話としては存在しない。fable_kickoff v1 §5「地の文・相方会話（C/D軸）」が求める"相方会話"の在庫として、この機微を会話に開く。
2. **相性8-14帯の書き分けサンプルが週次1場面しか無い**。`dialogue_design_v0.md` §2 で相性帯の書き分け見本があるのは**「稽古終わり」1場面だけ**（0-7／8-14／15-20／21+）。8-14帯の**別状況＝スベった夜の楽屋**が無い。ここは「噛み合い始めたばかりだから、沈黙を段取りで埋めるしかない」が最も効く帯で、書き分けの穴。**この"8-14帯×楽屋"が本案の唯一の新規貢献**であり、0014（フリーライブ直後の掘る/畳む選択）とも #18（敗退後の通知）とも重ならない一点。

狙いの芯は、**沈黙をセリフで説明しないこと**。「気まずい」「間がもたない」と地の文でも会話でも言わず、二人が交わすのは**次の現場の段取り（撤収・楽屋を空ける時刻・電車・次の入り時間）だけ**。沈黙は地の文の「十分、どちらも喋らなかった」一行に置き、会話本体は無味の実務で平板に保つ。これは fable_kickoff v1 §6「見せ場は2割・残り8割はこの平板さが正解」の実践で、**この一場面は意図的に自嘲もデッドパンも入れない**（乾いた一滴 1/6 の"入れない側"の1本）。

## 発火条件（実フック）

- **状態フック**: **低評価のフリーライブ結果の直後**（`t_フリーライブ` の週の結果が低評価だった週）。**大会敗退（justLost）ではなくフリーライブの低評価**を起点にする（#18 の"敗退後"とずらす）。
  - **【A軸・要確認】** justLost は既存フラグだが、**「フリーライブが低評価だった」という判定フックが GameCore に実在するか未確認**。`ManzaiGame/Sources/DialogueData.swift` の `reaction` は `t_フリーライブ`＝「客は少ないけど、場数だ。」の固定一言で、**"低評価だった"の分岐は現状無い**。event_design §1 の #6「スベった夜」も発火は"稽古2週連続"であって低評価ライブではない。→ **低評価判定を新設するなら GameCore ロジック追加**（既存の結果値に閾値を当てるだけなら RNG は引かないが、"新規ロジック"である以上 §A の同期対象になり得る＝実装時に確認）。既存に相乗りできるフックが無ければ、**発火起点を「フリーライブを選んだ週」まで緩めて compat 8-14 だけで絞る**のが安全側。
- **相性帯ゲート**: `state.compat` が **8〜14**（噛み合い始め）のときのみ。0-7 は #18（不仲の入口）の担当、15-20/21+ は沈黙が段取りでなく阿吽で埋まる帯なので**この会話は出さない**（帯を8-14に固定＝薄いスロットの一点狙い）。
- **0014 との棲み分け（同週競合の解決）**: 0014「終電までの反省会」も**フリーライブ直後**で発火する。棲み分けは (a) 0014 は**知名度<20**ゲート＝結成初期の選択イベント、本案は**知名度ゲートなし・compat 8-14**の非選択フレーバー、(b) **同一週に両方候補なら選択イベント（0014）を優先**し本案を落とす（"選ばせる"イベントを"眺めるだけ"のフレーバーより上に置く）、の二段で切る。**この相互排他は実装時に明示予約**（下記リスク）。
- **週**: 大会週を除く自由週。**「週ランダム」を素直に採ると抽選RNGを1回引く＝乱数消費順に触れる**（下記リスク・実装で止めて相談）。RNGを避けたいなら「低評価ライブ直後×8-14帯」を満たした週に**条件確定発火**（抽選しない）にできる。どちらを採るかは実装判断。
- **前提しない状態**: 体力・所持金・連敗・年次は発火条件に**しない**（帯と直近結果だけで決まる世界の彩り）。頻度過多なら「キャリアで数回まで」の上限を別途。
- **【設計上の懸念・要目視】** #18（相性8未満・通知）と本案（8-14・会話）が**同一プレイで近接週に連続すると「楽屋の無言」が二度**。天丼化するなら帯境界（7→8）を跨いだ直後は片方の重みを落とす。simでは測れず simulator 目視（CLAUDE §10）。

## イベント会話本文

**セットアップ（発火時・地の文＝俺視点／2〜4行、固有プロップ1個＝共有ドーラン）**

> ライブがはねて、楽屋に戻った。四分の出番で、客席が動いたのは頭の三十秒だけだった。
> 共有のドーランが、机の上で蓋を開けたまま転がっている。俺の指が当たる側だけ角が深く凹んで、次に買い足すのがいつになるかは、決めていない。
> 十分、どちらも喋らなかった。先に段取りの話をしたのが、どちらだったかは覚えていない。

> **【レッドチーム・B軸修正】** 元案は「二人分の顔が一度に映りきらない一枚鏡」と「共有ドーラン」の**2プロップ**を置いていた。fable §3-A「物は1つだけ選ぶ」に反し、かつ鏡の「二人分が映ったことは、まだ一度もない」は"二人がまだ一つに収まらない"を物で言い直した**意味ありげ＝感情語の言い換え**（§7-A-10チェックリスト該当）。**鏡を削り、ドーラン一つに集約**。ドーランは会話の「ドーラン、閉めとく」↔セットアップの「蓋を開けたまま」で**所作の背骨**を担い、置換不能（共有＝二人で一本／俺の指側だけ凹む）＝定番プロップ（天井の染み・冷めたコーヒー）ではない。

谷口「ほな、先出るわ」

## 実装イメージ（既存機構への載せ方）

**現状**: `ManzaiGame/Sources/DialogueData.swift` は「俺の心の声（状態駆動 innerVoice・優先度カスケード＝低体力>金欠>連敗>直近通過>大会前>平常）＋選んだ変種への一言 reaction」まで。**谷口との複数行会話のランタイムは未実装**（ヘッダに「谷口/先輩の会話は将来レアイベントで（毎週は出さない）」と明記・確認済）。選択肢イベント機構も未実装。本案は**その"レアイベント会話"層の1エントリの提案**で、UI層テキストのみ。

想定の載せ方（`dialogue_design_v0.md` §5 のテーブル選択に準拠）:

- **選択キー** = `(場面ID: "greenroom_silence", 相性帯: 8-14, 条件フック: 低評価フリーライブ直後)`。GameCore は state と 場面ID を渡すだけ、会話列（話者＋セリフの並び）は UI 層リソース。
- **話者プール分岐** = 相方が谷口なら関西弁プール、谷口以外なら標準語プール（上記2プール）。`Advice.name` 相当の枠に話者名を載せ、**話者ごとに1セリフずつ別送**（`／`詰め禁止＝タップ送りの間を殺さない）。
- **効果パッチ** = 推奨は**なし**。これなら **GameState・数式・golden・乱数消費順に一切触れない**＝安全。純テキスト追加のみ。

**【止めて相談】乱数消費順に触れる二点**:
1. **「週ランダム」抽選**を入れると、イベント抽選ストリームを1回引く＝**乱数消費順の正典に触れる**。3年ビット一致（golden）が壊れる。→ **実装時に必ず止めて相談**（CLAUDE §A：`tools/*.py`正典 → GameCore → `gen_golden.py` → `swift test` green を1コミット同期）。回避策＝「低評価ライブ直後×8-14帯」の**条件確定発火**にすればRNGを引かず golden 不変。
2. **効果を【仮】微少で載せる場合**（翌週回復わずか↑等）、状態を触るなら分布に効く＝`sim_career.py` 側の再計測が要る（event_design §4-B・§5）。→ これも止めて相談。**効果なし版なら不要**。

まとめ: **効果なし＋条件確定発火**なら golden も分布も無傷でUI追加だけ。RNG抽選や効果を足す瞬間に §A の1コミット同期＝止めて相談。

## リスク・注意

- **【要確認・A軸】低評価フリーライブ判定の実在**: 発火起点の"低評価だった"フックが GameCore に無い可能性（reaction は `t_フリーライブ` 固定一言のみ）。新設するなら GameCore ロジック追加＝実装時に §A 対象か確認。無ければ「フリーライブを選んだ週×8-14帯」まで条件を緩める安全側で。
- **【要決定・A軸】0014 との同週競合**: 0014「終電までの反省会」（フリーライブ直後・知名度<20・選択型）と本案（フリーライブ低評価直後・compat 8-14・非選択）は**フックが重なる**。棲み分けは (a) 0014=知名度<20の初期選択／本案=知名度ゲートなしのフレーバー、(b) **同一週は選択イベント優先で本案を落とす**、を実装で明示予約。放置すると同週に二重発火。
- **【要相談・正典】RNG消費順**: 「週ランダム」抽選は golden に効く＝止めて相談（上記実装イメージ）。安全側は条件確定発火。
- **#18 との棲み分け・天丼**: #18（相性8未満・通知・効果 相性-1→翌週相方過ごす2倍）と本案（8-14・会話・効果なし〜微少）は別イベント。相性が7→8を跨いだ直後の連続発火で「楽屋の無言」が二度出ると天丼＝重み調整（要目視）。効果を載せるなら #18 と**加算経路を分ける／片方のみ**を明示（二重取り禁止）。
- **無味の会話が"手抜き"に見えるリスク**: 段取りだけの会話は、演出（間・タップ送り・BGM無音・地の文の沈黙一行）が伴わないと単なる素っ気ない台詞に見える。**体感は演出で解く**（0009・fable §7-B）。沈黙は地の文「十分、どちらも喋らなかった」と、ドーランの物描写に背負わせ、会話は平板のまま置く。バランス数値では触らない。
- **固有プロップの単一化（是正済）**: 採ったのは「共有ドーランの、俺の指側だけ凹んだ減り」1個のみ（鏡は削除）。定番（天井の染み・冷めたコーヒー・小銭）を避け、**この二人・この楽屋にしか成立しない物**（共有＝二人で一本／指側だけ凹む）に限定。1場面1個の原則（fable §3-A）を満たす。
- **俺の声の自己点検**: 俺＝標準語・計算屋（「二十分で開けてくれって」「電車、一本早めるか。乗り換えで詰まる」＝段取りを数える）。自己憐憫なし・警句/アフォリズムで締めない・地の文に方言漏れなし。「ドーラン、閉めとく」＝谷口の去った後に俺が蓋を閉める所作で、気遣いを説明せず物で置く（セットアップの「蓋を開けたまま」と対）。
- **トーン自己点検**: 感情語（気まずい/惨め/悔しい/寂しい）不使用。感情は物＝「頭の三十秒だけ動いた客席／凹んだ共有ドーラン／閉めとく蓋」に背負わせた。**この一場面は意図的にユーモアゼロ**（自嘲もデッドパンも入れない＝連続で引く並びの中の"入れない側"の平板な1本）。谷口＝関西弁一貫／相方版＝標準語（関西弁を移さない）。会話は話者ごと1セリフ・`／`詰めなし。ボケ本文ゼロ。固有名は畳む相手や箱の名を出さず匿名。数値【仮】。既存作品・#18 flavor の流用なし。

## レッドチーム（A/B）

### A. 実装妥当性

- **[新規・最重要／是正済]** **台帳 0014「終電までの反省会」もフリーライブ直後で発火**（`fable_kickoff_drama_allin_v1.md` §5-E）。元案は棲み分け対象を #18 のみに絞り、**0014 との同フック衝突を見落とし**。同週二重発火の恐れ→ 発火条件欄・リスク欄に (a)知名度ゲート差／(b)選択イベント優先の相互排他を追記して是正。
- **[新規・要確認／格上げ済]** **「低評価フリーライブ」判定フックの実在が未確認**。`DialogueData.reaction` の `t_フリーライブ` は固定一言で"低評価だった"分岐は無く、event_design #6 も発火は稽古2週連続。元案は「reaction の'スベった'判定に相乗り」と書いたが、**相乗り先が存在しない可能性**。→ 新設ロジックの要否と §A 該当性を実装時確認、無ければ発火を緩める安全側を明記。
- **[OK]** RNG消費順を「止めて相談」対象と明記＝CLAUDE §A 準拠。**効果なし＋条件確定発火なら golden・分布・消費順に一切触れない**という切り分けは妥当（0015 と同水準の安全側）。
- **[OK]** 採番 0025 は 0024 が最新既存＝衝突なし。#18 flavor（55行目）・8-14既存サンプル（dialogue_design 34-37行目）の参照は実在を確認。
- **[OK・MVP/本編判定]** 本案は MVP でも本編でも「compat 8-14×フリーライブ低評価」の帯彩り。#18（本編寄り・敗退後）・0014（初期・選択）と別レイヤーで両立。
- **[破綻ルート]** 非選択・効果なしのため特定ルートで数値が崩れる経路は無い（効果を載せた瞬間に分布再測が要る点だけ相談対象）。

### B. 会話の中身

- **[修正済・1プロップ違反]** 元案は鏡＋ドーランの**2プロップ**（fable §3-A「物は1つだけ」違反）。かつ鏡「二人分が一度に映ったことは、まだ一度もない」は**意味ありげ＝感情語の言い換え**（テーマの物語り）。**鏡を削除しドーラン一本に集約**。ドーランは会話「ドーラン、閉めとく」↔setup「蓋を開けたまま」で所作の背骨を担い、置換不能（共有＝二人で一本／指側だけ凹む）＝定番プロップでない。集約でむしろ焦点が締まる。
- **[良・維持]** 会話は**段取りの実務のみ**（幕・平台・楽屋二十分・二十時入り・電車一本・先出る・ドーラン閉める）＝感情語ゼロ・ボケゼロ・数値/軸名を口に出さない。沈黙は地の文「十分、どちらも喋らなかった」に一行で背負わせ、会話は無味で平板＝fable §6「8割は平板が正解」の実践。
