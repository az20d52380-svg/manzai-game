# 提案 v0 —「正社員の話」バイト二段gapの選択イベント（金欠×バイトhook・段階投入でMVP射程）

- 日付: 2026-07-06 ／ 数値は全て【仮】／ 既存ファイルは読むだけ ／ トーン＝ドラマ主軸（proposals/0009 確定）
- 対象読者: イベント機構（event_design §2 二択）実装担当・DialogueData / WeekMainView の書き手
- 位置づけ: 単発「選択型」バイトドラマの新規枠。event_design §1-C「バイトと生活」（#13〜#16＝**自動・通知型・未実装**）とは別に、低所持金帯へ初めて置く**二択**。既存 #13〜#16 の効果とは重複させない。
- **レッドチーム後の最大の是正**: 下書きは「バイト実行カウンタを流用」「稽古上限−の新field」を既存前提のように書いていたが、**実コードにはバイトカウンタも二択イベント機構も稽古上限debuff用のfieldも存在しない**。稽古上限は既存 `growthBudget`（正典v2・Python `s._yg` 同期）が唯一の器。よって本イベントは「既存フックに軽く載る一枚」ではなく、**発火土台ごと新設が要る＝段階投入前提**に格下げした（下記A/B・§実装イメージ）。

## 狙い

バイトは本作で唯一「芸人の時間を金に替える」行動なのに、今そこにあるのは**結果を受け取るだけの自動枠**（設計上）で、**プレイヤーが"どこまで芸人でいるか"を握る一手が無い**。金欠帯（`money < 10万`）は世界観テキストが厚い一方、**その裏で進む生活の意思決定**が空いている。

やることは一つ。**バイトを一定回数積んだ金欠のプレイヤーに、店長が「正社員登用」を持ちかける二択を置く**。要点は三つ。

1. **バイト二段gap**。バイト行動＝「時間を売る単純作業」。このイベントは**その延長で"もっと売れ＝正社員"を突きつける**。バイトhook（回数・所持金）から生えるのが肝で、平常ローテとは別温度。
2. **正解のない等価トレード**（event_design §2/原則3）。A＝受ける＝**まとまった金と引き換えに体力・メンタルを差し出す**（＋段階2で"芸の伸びしろの天井"＝growthBudget 減算・要正典同期）。B＝断る＝**金は一円も動かず**、メンタルと相性が上がるが、**金欠圧（破産リスク）はそのまま次週へ**。どちらにも痛みが残る設計を狙う。
3. **相方依存の声で生活ドラマを背負わせる**（0009）。安定か芸か、を相方との一往復で背負う。谷口＝関西弁、店長＝標準語（第三者に関西弁を移さない＝谷口の声を汚さない）。

## 発火条件（実フック）

- **一次条件（バイトhook・新設が要る）**: 直近のバイト実行回数が一定以上。**コードにこのカウンタは存在しない**（`GameState` は money/stamina/fame/5能力/compat/growthBudget系のみ）。#13/#16 も設計doc止まりで実装されていない。＝**バイト系イベント共有の軽量カウンタを一本新設**（散らさない）。
- **二次条件（金欠hook）**: `state.money < 100_000`（【仮】）。`money` は既存 `Int` フィールド（GameState.swift:5）。
- **三次（反復制御）**: 週次イベント判定枠（大会週を除く週の12%【仮】＝**この抽選機構自体が未実装**）のランダム発火。**同一キャリア原則1回**（正社員登用の重さ＝反復させない）。
- **MVP 射程だが単体では載らない**: 概念は「キツいバイトを重ねた金欠の中盤」で1年内に十分立つ（多年前提でない）。ただし発火には**(i)バイト実行カウンタ (ii)週次イベント抽選パイプライン (iii)二択イベント機構**の3つが未実装＝いずれもRNG消費順/golden同期を要する新規土台。**このイベント1枚を単独で差せる状態ではない**（→ §実装イメージ(2)(3)）。

## イベント本文（場面のセットアップ＋二択）

> トーン＝ドラマ主軸。感情語（悲しい/悔しい/惨め）を地の文から抜き、**"このバイト先・この店長固有の少しズレた物"**（＝店長手書きのシフト表。正社員は黒ペン、バイトは鉛筆）に温度を背負わせる。俺＝標準語・計算屋・自己憐憫なし。**会話は話者ごとに1セリフずつ（"／"で詰めない）**。ボケ（オチ/ネタ本文）は書かない。数値は全て【仮】。

**セットアップ（地の文・俺視点）**

```
バックヤードのシフト表は、店長の手書きだ。正社員の名前は黒ペン、
バイトの名前は鉛筆で書いてある。鉛筆の名前は消しても跡が残る。
俺の名前の下に、先月辞めた誰かの名前が、薄く残ったままだ。
```
```
店長「悪い話じゃない。……いつまでも、若い体力に甘えてらんないよ。黒で、書き直してやる。」
俺「ありがとうございます。少し、考えさせてください。」
谷口「店長、こいつ今、シフト表の自分の名前見てますわ。……見てるうちは、まだ決めてへん。」
```
（技法: デッドパンの一滴＝"鉛筆の名前は消しても跡が残る"・自己言及ゼロ・定番プロップでない固有の物。黒ペン＝確定＝正社員／鉛筆＝消せる＝芸人、をシフト表一つに背負わせる。**谷口の"見てる／まだ決めてへん"は結論を予言しない中立の観測**＝どちらのルートを選んでも会話が崩れない。B会話で"見てから決めた"を回収する。全て架空・数値【仮】。）

**【効果・段階1＝MVP第一弾・RNG非消費/golden不変】** 所持金 `+30,000`（【仮】＝これまでの働き分の確定反映。**engineに月次/継続収入モデルは無い＝一時金でしか表現できない**）／ 体力 `−10`（シフト拘束増・【仮】）／ メンタル `−1`（【仮】）。純加算/減算のみ＝乱数非消費。
**【効果・段階2＝要正典同期・後日追加】** 以後の稽古の伸びしろに天井（恒久）。**新fieldは作らず既存 `growthBudget` を X 減算**で表現（下記実装イメージ(4)）。**これは正典v2の成長予算＝Python `s._yg` と同期＝止めて相談・golden再生成が必須**。段階1では入れない。

## どこから学んだか（構造＋出典URL）

学んだのは**構造・型のみ**。既存作品のセリフ・攻略本文は転載しない。

- **育成SLGの「進路/副業が練習可能回数を圧迫する選択」型**（サクセスの二層＝確定節目＋彩りのランダム、その彩り側で"時間の使い道を縛る選択肢"を出す）。本イベントは彩り側で稽古の伸びしろを縛る。 https://ja.wikipedia.org/wiki/実況パワフルプロ野球_サクセスモード
- **機会費用**——「正社員の安定」を取ると失うのは金では測れない"芸人でいる時間"。A/B の非対称（金 vs 伸びしろ）を等価トレードとして成立させる裏付け。 https://ja.wikipedia.org/wiki/機会費用
- **不完全情報下の意思決定**——A の"伸びしろ−"がどれだけ将来を削るかを数値で見せない＝プレイヤーは期待値でなく"性格"で選ぶ（event_design §2 の核）。 https://ja.wikipedia.org/wiki/期待値
- **下積みの手触り・売れる側/沈む側の非対称**（世界の型のみ・声は移植しない）。バイト先＝生活の圧、芸＝不安定、を"店長の親切"越しに置く。 https://ja.wikipedia.org/wiki/火花_(小説)
- **感動を宣言しない（事実と物の温度差で置く）**——登用の重さを地の文で叫ばず、"黒ペンと鉛筆のシフト表"一つに背負わせる（0009 の芯）。 https://ameblo.jp/isawo-t1307/entry-12689913076.html
- 自作 `event_design_v0` §2・原則3/4、同 §1-C（#13〜#16＝設計のみ）、`proposals/0009`（ドラマ主軸・固有の物・相方依存の声）、`proposals/0010`（RNG/goldenに触れる効果の段階投入手筋＝本提案もこれに倣う）。

## 実装イメージ（既存機構に載る形・コードは"提案"・golden影響は要相談）

**(1) 二択イベントとして event_design §2 の型に載せる。** 選択肢→効果（ステ/関係/金）＋短い会話。効果対象は6軸＋体力＋所持金の増減。**段階1のA/Bは純加算/減算のみ＝RNG非消費＝golden不変。**

```swift
// 【提案・未適用】正社員の話。event_design §2 の二択型に載る想定。
// 発火: recentArbeitCount >= N かつ state.money < 100_000 かつ (週次イベント判定に当選)。同一キャリア1回。
struct RegularEmploymentOffer {
    // A・段階1（確定効果のみ・純加算/減算＝RNG非消費）
    static let moneyGainA   = 30_000   // 【仮】一時金（engineに継続収入モデルは無い）
    static let staminaCostA  = -10.0   // 【仮】拘束増
    static let mentalCostA   = -1.0    // 【仮】
    // A・段階2（伸びしろ天井）は新fieldを作らず growthBudget を減算 → (4)・要正典同期
    // B（純加算・RNG非消費・救済なし）
    static let mentalGainB   = 2.0     // 【仮】
    static let compatGainB   = 1.0     // 【仮】compat上限20
}
```

**(2) バイトhook（発火の器）＝新設が要る。** **コードにバイト実行カウンタは無い**（`GameState` 未所持・#13/#16 も未実装）。バイト系イベント共有の軽量カウンタ（例 `recentArbeitCount`）を `GameState` に一本新設。**カウンタを state に足す＝Codable/初期化/Python同期の対象**になり得る点に注意（純加算なら golden 不変だが、追加位置は消費順の外に置く）。

**(3) 【止めて相談・最重要】週次イベント抽選パイプライン自体が未実装。** GameSession/WeekRunner の現行 Phase に「週次12%でイベントを引く」機構は無い。**プールに新イベントを足すどころか、抽選の器を新設する＝乱数消費順に新規ロールが入る＝golden（3年ビット一致）が壊れる**。正典同期手順は **①`tools/*.py`（正典）にイベント判定順を先に定義 → ②GameCore 反映 → ③`cd tools && python3 gen_golden.py` 再生成 → ④`cd GameCore && swift test` green を1コミットで揃える**（規律A-1/A-2）。**実装前に必ず止めて相談。**

**(4) 段階2「伸びしろ天井」は growthBudget 減算で正典化（新field禁止）。** 既存の稽古上限は `GameEngine.applyTraining` が `growthBudget − growthUsed` の残予算で頭打ちにする方式（GameEngine.swift:23-37、`s._yg`/`s._inj` とPython同期）。**下書きの `practiceCapPenalty` 新fieldは"第二の並行cap"になり Python正典に存在しない＝golden乖離を必ず生む**。よって天井を下げたいなら **`growthBudget` を X 減算**（または `growthUsed` へ加算）＝**既存の唯一のcap系に集約**する。ただしこれは**正典v2の成長予算そのものを触る＝Python正典→Swift→gen_golden→swift test を1コミット同期する重い変更**。**clamp は決定的でRNG非消費だが、state軌道が変わり下流判定が動く。止めて相談。** 段階1では入れず、0010 と同じく"確定効果のみ先行→天井は相談後に段階2追加"で出す。

**(5) 会話プールと反復回避・UI検証。** A/B 後会話は相方IDで分岐（谷口＝関西弁プール／他相方＝標準語プール）。話者切替（俺/店長/相方）は dialogue_design §6-7 の未実装論点＝口が無ければ暫定で俺の後会話に相方引用を挟む形も可。**UIは swift test で検証不可＝simulator で金欠×バイト反復の状況を作って発火を目視**（規律D-10）。新規Swiftファイルを足したら `cd ManzaiGame && xcodegen generate`（規律D-9）。

## リスク・注意

- **【止めて相談・最重要】RNG/golden**（(3)(4)）。週次抽選機構の新設＝乱数消費順に触れる／段階2の growthBudget 減算＝正典v2の成長予算に触れる。**相談・正典同期前に実装しない**。段階1のA/B効果本体は純加算/減算＝RNG非消費。
- **【設計上の懸念】"伸びしろ天井"の持ち方**（(4)）。**新field(practiceCapPenalty)を作らない**。既存 `growthBudget` に集約しないと Python正典と二重管理になり golden が必ず乖離する。
- **【等価トレードの検証・B の穴】** 下書きのBは「金が動かないだけ」で**機構的コストがゼロ**（メンタル+2・相性+1は純プラス）。これが等価トレードになるのは**破産圧（`GameState.bankrupt`＝夜逃げ）が実在して"金を取らない"が本当の犠牲になる時だけ**。金欠ゲート下で破産到達が現実的頻度か **sim で実測してから数値確定**（体感を数値で直さない・ui_design §7-B＝まず頻度/演出）。破産圧が薄いと B が支配戦略化する。
- **【A のfiction/機構の継ぎ目】** "正社員＝安定した継続収入"のfictionを、engineは一時金（+30,000）でしか表現できない。安定の実感は**段階3として"金欠系ネガイベントの頻度減"で回収**する余地（event_design 原則4の救い）＝ただし抽選機構実装後。段階1は一時金＋拘束コストの正直な範囲に留める。
- **重複回避**。効果もテキストも将来の #13〜#16 と被らせない。同一キャリア1回で重く置く。
- **口調・トーン**。俺＝全行標準語・計算屋・自己憐憫なし。谷口＝関西弁一貫。**店長・谷口以外の相方に関西弁を混ぜない**（0009）。ボケ・オチ・ネタ本文ゼロ。デッドパンの一滴は"鉛筆の名前の跡"1個に絞り、Bで二個目のウィット（旧"消しゴム"）は盛らない。既存作品（火花/べしゃり暮らし/M-1実在ネタ）とのセリフ一致を各行照合し無し。固有名は架空・数値は全て【仮】。

## レッドチーム（A/B）

**A. 実装妥当性**

1. **【是正・降格】バイト実行カウンタは存在しない**。`GameState`（GameState.swift:4-23）は money/stamina/fame/5能力/compat/growthBudget/growthUsed/recoveryWeeks/bankrupt のみ。#13/#16 は event_design_v0 の設計表で**コード未実装**。下書きの「流用」は誇張＝**カウンタは新設**（(2)）と明記に修正。
2. **【是正・最重要】週次イベント抽選機構が未実装**。GameSession の Phase（tournamentDecision/freeAction/gpRound/gpRevival/gpFinal/weekDone/yearDone）に「週次12%でイベントを引く」器は無い。＝このイベント1枚を単独では差せない。**発火土台ごと新設＝RNG消費順に触れ golden を壊す＝止めて相談**（(3)）。MVP「発火可」は"概念は1年射程"に格下げし、土台3点の未実装を明記。
3. **【是正・設計懸念】稽古上限debuff は新fieldでなく既存 growthBudget へ**。下書きの `practiceCapPenalty` は Python正典に無い第二cap＝golden乖離必至。既存の成長予算（`s._yg` 同期・GameEngine.swift:23-37）を減算する形へ変更し、正典同期必須と明記（(4)）。段階2へ後ろ倒し。
4. **【是正・トレード】B に機構的コストが無い**（純プラス）＝破産圧が実在しないと支配戦略化。Bの唯一のコスト＝金欠持ち越し（破産リスク）である旨を本文とリスクに明記し、sim実測を条件化。
5. **【是正・fiction継ぎ目】** money は `Int`・継続収入モデル無し＝"正社員の安定給料"は一時金でしか出せない。A効果を一時金＋拘束コストの正直な範囲に限定し、"安定の実感"は抽選機構実装後の段階3（金欠イベント頻度減）へ分離。
6. **【確認】段階1のA/B効果は6軸＋体力＋所持金の純増減のみ＝event_design 設計原則（新システム不要）の範囲・RNG非消費・golden不変**。危険はすべて土台側（抽選・成長予算）に局在。

**B. 会話の中身**

1. **【是正・ルート崩壊】セットアップの谷口が結論を予言していた**。旧「こいつ今、断る顔しとりますわ」は**A（受ける）を選ぶと矛盾**＝会話が崩れる。中立の観測「シフト表の自分の名前見てますわ／見てるうちは、まだ決めてへん」に差し替え、どちらのルートでも成立させた。B回収も"迷う顔"→"見てから決めた"に接続し直し。
2. **【是正・二個目のウィット】** B旧「店長、消しゴムだけ返してきた」はセットアップに無い小道具＝意味不明＋二個目のプロップ・ウィット（自作リスク欄の禁則に自分で抵触）。「黒のペンは出さなかった」に変更し、プロップを黒ペン/鉛筆の一系統へ統一＝デッドパンの一滴は"鉛筆の跡"1個のみ。
3. **【是正・語感】** 俺「返事、少しください」は不自然＝「少し、考えさせてください」へ。俺＝全行標準語・計算屋・自己憐憫なしを維持。
4. **【確認】相方依存の声**。谷口＝「しとりますわ/へん/ほな/やんけ/わ」で関西弁一貫。谷口以外＝標準語バリエーションを別プールで用意し関西弁を混ぜない。相性帯（0-7/8-14/15-20）で温度差可（compat上限20＝阿吽21+は出さない）。
5. **【確認】1セリフずつ・ボケ無し・ドラマ主軸**。全会話が話者ごと改行（"／"で詰めていない）。オチ/ネタ本文なし＝全て反応・観測。感情語は地の文から排し、シフト表の物に温度を背負わせている。
6. **【確認】既存作品コピペ無し**。俺/店長/谷口/相方の各行を火花・べしゃり暮らし・M-1実在ネタと照合し一致・酷似無し。シフト表/黒ペン・鉛筆のくだりは新規。固有名は架空、数値は【仮】。
