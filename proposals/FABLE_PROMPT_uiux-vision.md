# Fable投げ用プロンプト：完成版UI/UXビジョン設計 v2

v1をOpus 4.8で2軸（①完成版としての網羅性 ②プロンプトエンジニアリングとしての実効性）から赤チームし、「要修正」判定を受けて全面改訂した。主な変更：①「完成版」の範囲を1年MVPの内装に固定（ライブサービス系画面は対象外と明言）、②既存実装（`WeekMainView.swift`）を正確に接地（v1は実態より古い説明をしていた）、③確定パレットの実値を本文に埋め込み、④対象7画面・分量規約・優先順位・出力フォーマットを明記、⑤理想と実現可能性を2層に分離。

## ここからプロンプト本文（コピペ用）

あなたにはこの漫才師コンビ育成SLGアプリの「完成版UI/UX」を、実際にリリースされている人気スマホゲーム（パワプロ サクセス for スマホ、ウマ娘 プリティーダービー）レベルのクオリティで想像し、具体的に設計してほしい。

### この依頼における「完成版」の定義（スコープ固定・最優先で読むこと）

「完成版」とは**単一キャリア（1年48週）を最後まで遊ぶ体験の、演出・内装が完成した状態**を指す。以下は今回の対象外とし、想像・設計しないこと：

- **対象外A**：ホーム画面/ロビー、ガチャ（相方抽選）画面とその演出、図鑑・コレクション、ランキング/リーダーボード、フレンド、お知らせ・イベント告知バナー
- **対象外B**：課金・ショップ・スタミナ制・ログインボーナス等のマネタイズ/再エンゲージメント導線
- **対象外C**：2周目以降・王者編・多年キャリアの画面

（理由：本作は1年完結の育成体験の質を最優先で作る方針。参照点にウマ娘/パワプロを挙げるのは演出の"質感"を借りるためであり、それらのゲームが持つ機能一式を全部持ち込む意図ではない。もし「1年ループの体験を最高にするために対象外A〜Cのどれかが構造上どうしても要る」と判断した場合のみ、本編の最後に**「将来拡張メモ」として3行以内**で触れてよい——本文の主成果物には混ぜないこと）

### 現状の到達点（ここはゼロではない。この画面文法を土台に深化させること。破壊する提案は不可）

週メイン画面は既に稼働しており、以下の文法が確定済み：

- **画面構成**：上部＝立ち絵シーン、下部＝コマンドゾーン、最下部＝固定の帯（状態表示）の3段構成。
- **立ち絵シーン**：右下に相方2人のシルエット（現状は色付き幾何形のプレースホルダ、本イラスト差し替え待ち）。左上に**6軸の暗色ピル**（センス/発想/表現/華/メンタル/相性）を常時表示。稽古等を実行した直後だけ、伸びたピルに「+N」がオレンジで一瞬（約1.2秒）浮かぶ。カテゴリ未選択時は左下に「心の声」の吹き出し（俺の内心モノローグ、状態に応じて切り替わる）。
- **コマンドゾーン**：「カテゴリのアイコン列」と「選んだカテゴリの変種カード横スクロール列」を**画面の同じ場所で切り替える**（別画面には遷移しない）。戻るボタンは右上のみ。
- **決定ボタンは無い**：変種カードを**タップした瞬間に即実行・1週進む**（「つぎへ」ボタンで確定する二段階操作ではない）。実行できない場合（体力不足・所持金不足）はカードをグレー表示＋タップ時に短いトースト（「体力が足りない。今日は休もう。」等）を出し、実行はしない。
- **最下部の帯**：「年目・週」「次の大会までN週」「体力ゲージ（3色：緑50以上/黄50未満/赤20未満＝この閾値は本番ペナルティと一致した"警告"の機能色）」「所持金（負債は赤字表示）」を常時表示。
- **足りていないのは「内装の質」**：本イラスト・表情差分・BGM/SE・大会本番や成長の"見せ場"の演出の厚み・画面間トランジションの気持ちよさ。ここを理想形まで引き上げてほしい。

### 確定済みパレット・トークン（意味論は変更不可。深化はこの内側で）

- 地色：クリーム系グラデーション（`bgTop` #FFF7EC → `bg2` #FFE6CF → `bgBottom` #FFDDBE）／カード：白・`card2` #FFF9F0
- アクセント：朱`verm` #E8402C（相方の枠・強調・相性）／金`gold` #F6B301（見せ場・マイルストン）
- 文字：濃紫`ink` #2C2740、淡色`inkDim` #8E86A0、さらに淡い`inkFaint` #B9B2C6
- **能力5色（意味が固定・入れ替え禁止）**：センス=青`cSense` #3B8BFF／発想=紫`cIdea` #8B5CF6／表現=橙`cExpr` #FF7A2F／華=桃`cChara` #FF5C93／メンタル=緑`cMental` #22B07A／相性=朱`cCompat` #E8402C
- **体力ゲージ3段（色=機能的な警告の意味。配色刷新の対象外）**：緑（50以上・好調）→黄`staminaWarn` #F2A93B（50未満）→赤`staminaCrit` #E8402C（20未満・本番ペナルティ閾値と一致）
- フォント：丸ゴシック（system rounded、見出し・数値用）。数値は等幅。
- 実行時の伸び表示は`gainOrange` #FF8A1E、立ち絵上の暗いピル/トースト地色は`pillDark`（黒82%不透明の紫）、最下部帯は`botbarDark`（濃紺紫）

「深化」とは、この色相・意味論を保ったまま**質感**（紙の温かみ・淡い発光・影の付け方・トランジションの気持ちよさ）を上げることを指す。**色そのものの置換、能力色・体力3段の意味変更は提案しないこと**。グロウ/光沢を提案する場合も既存の暖色域の内側で（寒色追加・彩度全面upはしない）。

**能力体系の内訳**（第6の能力を発明しないための注記）：見た目は6軸だが、実体は「演技系4種（センス/発想/表現/華）＋メンタル＋相性」で**系統が異なる**。演技系4種のみ成長に予算上限があり、メンタルと相性はそれとは別の伸び方をする。視覚的にもこの系統差が伝わるとよい。

### 対象画面（7つ。過不足なく全て扱い、勝手に画面を増減しないこと）

1. **S1** タイトル／コンビ結成（初回：回想カット→コンビ名入力／2回目以降：セーブがあれば直行）
2. **S2** 週メイン（上記の現状描写の画面。ゲーム時間の9割を占める本丸）
3. **S3** 大会・GP本番（会場入り→本番演出→合否→紙講評）
4. **S3-決勝** 頂GP決勝（年間の最大の見せ場。S3と分けて厚く）
5. **S4** 年間カレンダー（48週の俯瞰・読み取り専用）
6. **S5** ネタ帳（能力5種・相性・記録を見る、能力の"伸びしろ"を確認する画面）
7. **S6** 年次リザルト（到達段階・年間サマリ→タイトルへ）

### 優先順位（このボリューム配分で書くこと。均等配分は不可）

**最優先＝S2（週メインの緩急）とS3-決勝（年間最大の見せ場）**。この2つが本作の"飽きない/効いてくる"の核心。次点＝S1のアートディレクション方針。S4〜S6・コンポーネント共通仕様・アセット要件は、S2/S3-決勝を実現する手段として簡潔にまとめてよい。**全項目を均等に薄く埋めるより、核心2つを設計書レベルまで掘ることを優先せよ**。

### 見せ場の演出（ゲーム名でなく"効く要素"で発注する）

パワプロ・ウマ娘の名前をそのまま参照すると再現の精度が保証されないため、狙いたい**要素を分解して**発注する（ゲーム名は答え合わせ用のラベルとして末尾に添えるだけでよい）：

- **溜め→開放の時間設計**：結果が出る直前に0.3〜1秒の「静止・暗転・音の抜き」を挟み、直後に情報と音を一気に解放する（合否・大逆転の瞬間）。
- **画面全体のフラッシュ/シェイクは稀少イベント限定**（決勝・大化け等）。日常の週送りには使わない＝安売り禁止。
- **数値エフェクトより、キャラの反応を主役にする**（表情変化・カットインで「効いた」を伝える）。
- **段階的な情報開示**：合否スタンプ→講評1行→賞金/知名度差分、と時間差で出す（一度に出さない）。

これらを本作の題材（漫才・下積み・感情語を書かない美学）に翻案し、"派手さ"でなく"効いてくる"方向で設計すること。

### 依頼したいこと（優先順位に従って文量配分すること）

1. **アートディレクション方針**：キャラクター表現の方向性（フルイラスト立ち絵／半身の表情差分アニメーション／現行のシルエット幾何形を上質化する、等の選択肢とそれぞれの効果・コスト感）、キービジュアル1枚のイメージ記述（タイトル画面用）。
2. **S2・S3-決勝を中心に、画面ごとの詳細UI/UX仕様**：各画面につき必ず次の4小見出しを埋める（薄い画面は「MVPでは無し」と明記してよいが見出し自体は立てること）：
   - レイアウト
   - トランジション（画面・要素の出入り）
   - マイクロインタラクション（数値が伸びる瞬間・合否が出る瞬間等の気持ちよさ）
   - 音・ハプティクスのキュー
3. **UIコンポーネントシステム**：ボタン・カード・ピル・プログレスバー等の再利用可能なビジュアル言語（既存の「押すと少し沈む」ボタン挙動を土台に統一感を持たせる）。
4. **演出のペース・情報量**：週送りのサクサク感と大会本番の緩急をどう作るか。年間を通した「育っている実感」の継続的な見せ方（成長の軌跡・通算成績・絆の可視化など）。
5. **アセット要件**：必須ティア／推奨ティア／贅沢ティアの3段で提示（絶対的な曲数・枚数の見積もりではなく、段階を示すこと）。既存MVPの最小音響構成（SE4種：行動決定/通過/敗退/週送り、ハプティクスは大会結果と週送りのみ）を土台に、「増やすなら何を最優先か」で書く。
6. **実装ブリッジ**（各画面仕様の末尾に必ず付ける）：この画面で新規に必要なアセット/実装要素を「既存で流用できるもの／新規に要るもの」に分けて1〜3行。実装難度を低/中/高で自己申告。※あなたの出力はそのままSwiftコードにはならず、実装者が既存のPhase駆動構造に変換する中間設計書として使われる。画面はS1〜S6の記号で参照し、動きは秒数で書くこと。

### 提案の書き方（理想と実現可能性を混ぜない）

各項目は**必ず2層**で書くこと：
- **(A) 予算無制限の理想形**を書き切る（ここで実現可能性を理由に妥協しない）
- **(B) 個人〜小規模チームで作る場合の最小到達版**を1〜2行で添える

(A)を最初から(B)側に丸めて「無難な中間案」にすることは禁止。判断材料として両方を並べる。

### 出力フォーマット（厳守）

- 依頼1〜6の順にH2見出しを立てる。各画面（依頼2）はH3で画面名（S1〜S6）を立て、4小見出しを埋める。
- 動きは**秒数の目安**（例：0.3s）と**イージングの方向**（キツく入って緩く抜ける、等）まで書く。色は上記のTheme トークン名（`verm`/`gold`/`cSense`等）で指名する。
- アセット要件（依頼5）は**表**（アセット名｜種別｜ティア｜代替の効く簡易版）で出す。
- 各画面はA4半ページ相当を上限に。「かっこよく」「リッチに」等の抽象語は禁止、それが指す具体挙動に置換すること。

### 文体の使い分け

設計の説明・根拠は明快なプレゼン文体でよい（読み手はSwift開発者）。ただし**ゲーム内に表示される文言（台詞・見出し・トースト文）を例示するときだけ**、本作のトーン——感情語を書かず物・所作で見せる／警句で締めない——を守ること。ゲーム内文言例は「」で括って区別する。

### 制約（禁止事項）

- 実装はSwiftUI（iOS 17+）。技術的に非現実的な提案（フルCGムービー多用等）は避け、実際に個人〜小規模チームで作れる範囲を(B)側で示すこと。
- 既存のゲームロジック（週の進行順序・乱数消費順・大会の判定式）には一切触れない。UI/UXだけを設計する。
- 縦持ち固定・iPhone専用（横画面・iPad対応は無し）。

## 補足（このプロンプトの位置づけ・投げ方）

- 一発で全部投げてもよいが、より高精度を狙うなら**2段階に分けて投げる**選択肢もある：
  - **第1便**：「アートディレクション方針」（依頼1）と「見せ場の演出」節だけを深く聞き、キャラ表現の方向性を2〜3案出させて先に選ぶ。
  - **第2便**：選んだ方針を前提として貼った上で、依頼2〜6を詳細化する。
  - 一発で投げる場合は、上記の優先順位節がFableの文量配分を制御する。
- Fableの出力はそのまま実装できるSwiftコードにはならない想定。実装に落とす際は、既存の`Theme.swift`・`WeekMainView.swift`・`RootView.swift`のPhase駆動構造と整合させる変換作業が別途必要（Mac/CLI側の仕事）。
- Fableの出力を受け取ったら、実装可能性・コスト感でもう一段整理してから`docs/`か`proposals/`に反映するのが安全（Fableの提案を無条件に採用しない）。

## 赤チーム記録（v1→v2の変更根拠）

- **赤チームA（完成版としての網羅性・Opus4.8）**：「完成版」の範囲が二重にブレていた（MVP1年の内装なのか商用ライブサービスなのか）→ スコープ固定節を新設。対象画面が`ui_design_v0.md`のS1〜S6と語彙がズレ、S5ネタ帳が抜けていた→7画面リストと分量規約を追加。確定パレットがファイル名参照のみで実体が無く「変えるな」と「理想は高く」が衝突しうる→パレットの実値を本文に埋め込み。
- **赤チームB（実効性・具体性・Opus4.8）**：「現状」の説明が実装（`WeekMainView.swift`）より2〜3段古く、Fableが存在しないゼロ地点から設計し既存の確定文法（つぎへ廃止・同一場所切替・6軸ピル）と矛盾する提案をするリスクがあった→現状描写を実装から起こして正確化。ゲーム名だけの参照（フラッシュ演出・覚醒エフェクト）はFableが正確な知識を持つ保証がない→"効く要素"に分解して発注。出力フォーマット・優先順位・理想/実現可能性の層分けが無かった→全て追加。
