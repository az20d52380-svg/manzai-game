# イベント会話提案 0025「三行目を一度で」v0（2026-07-06）

`event_design_v0.md` §1-F #27「謎の絶好調」（体力80以上・全能力+1・通知型／docs 行74）の**会話化**。#27 は現状フレーバー地の文1行のみで会話が無い。ここに俺＋谷口の一場面を載せ、「理由の分からない上向き＝下積みの気味悪さ」をドラマとして置く。**数値は #27 の既存効果（全能力+1・【仮】）を一切いじらない**。会話は既存フレーバーへの差し替え（テキストリソース）で、新規効果パッチ・新規乱数ドローを持たない。既存ファイルは読むだけ。

**実装区分＝本編用（post-MVP）。** 自動イベント抽選ランタイム自体が `ManzaiGame/Sources/` に未実装（#27 は doc とサイクル近似 `sim_career.py` のみに存在）。本案は「#27 が本編でランタイム化されたとき、その1行フレーバーの器を会話プールに差し替える」提案であって、**MVP発火ではない**。器（イベント抽選・効果パッチ）の新設は本案の外。

## 狙い（どの薄いスロットを埋めるか）

薄いスロットは二つ。

1. **「好調」の心情面が数値バフだけで会話が無い。** #27 は「全能力+1」という御褒美が本体で、体験としては通知が一瞬光って消えるだけ。`DialogueData.innerVoice` の `passedLines`（`justPassed` 分岐＝直近通過の余韻）は"通過の余韻"を扱うが、**大会結果と無関係に湧く「理由の分からない上向き」**を扱うスロットは既存に無い。ここが空白。
2. **上向きを「叫ばない」見本の不足。** 好調は演出を間違えると最も寒くなる（テンション上げ・自画自賛）。`passedLines` の自制（高揚を叫ばず事実で置く／fable_kickoff §5）と同じ抑制を、**大会文脈の外**でもう一枚作りたい。上向きを"事実で置く"パターンの増補。

固有プロップ＝**ネタ帳の三行目**。いつも詰まって消しゴム跡が濃く、紙が薄くなって光にかざすと三行目だけ透ける箇所。それが今日は消し跡なく一度で通る。上向きを「調子がいい」と言わず、**この二人のネタ帳にしか無い一点**の事実で置く（fable_kickoff §3-A の置換不能性）。ここに12本中1滴のデッドパンを配置する（谷口「こういう日は、たいてい次でこける」＝真顔の事実の並置・自己言及ゼロ・感情の谷でなく平地）。俺は好調の理由を勘定しようとして、数えきれない。

## 発火条件（実フック）

#27 の生条件（体力80以上）だけだと「敗退直後の高体力」や「決勝前週のピリピリ」にも乗ってしまい、"気味悪い凪"の温度が濁る。**会話版は #27 の発火帯を絞る**。

- **状態フック（AND）**:
  - `stamina >= 80`（#27 既存条件・`GameState.stamina`／実在）
  - `lossStreak == 0`（`GameSession.lossStreak`・実在・行29）＝連敗中でない。上向きが"直近の負けの反動"に読まれないため。
  - `2 <= (nextMilestone.weeksLeft) <= 数週`（大会が視界に入る好調帯・数値は【仮】）＝大会前週のピリつき（`DialogueData.innerVoice` の `nextMilestone.weeksLeft <= 2` 帯）とは分離。※`weeksLeft` は `WeekMainView.nextMilestone()` 由来の導出値で GameSession の直フィールドではない。参照はできるが、器のランタイム側からどう見せるかは実装時確認。
- **抽選**: `event_design` §頻度（既定12%）の**イベント抽選ストリーム**に乗る（＝本編で器が実装された前提）。**キャリア低確率**で意図的に薄く出す（毎年出ると"気味悪さ"が制度化して薄まる／頻度は【仮】）。
- **前提しない状態**: `fame`／所持金／相性帯は発火条件にしない（好調は誰にでも理由なく来る、という彩りのため）。ただし会話プールは**相方が谷口か否か**で分岐（下記・谷口=関西弁／他相方=標準語）。※現状アプリは相方＝谷口固定（相方判定フラグ未実装）。分岐は相方スワップ系の実装後に有効化する前提。
- **【設計上の懸念・要目視】** `weeksLeft` の上限（"数週"を2〜4のどこで切るか）と「キャリア低確率」の実数は sim では測れない体感値。simulator 目視で決める（CLAUDE §10）。**発火帯を絞る＝#27 の発火回数分布に触れる**ため、バランス側は下記リスク欄。

## イベント会話本文

**セットアップ**（発火時・地の文＝俺視点）:

> ネタ帳を開いて、いつもの三行目で手が止まるのを待った。あそこはいつも通らない。消しゴムの跡が濃くて、紙が薄くなって、光にかざすと三行目だけ向こうが透ける。
> 今日は、消さなかった。一度書いて、そのまま次の行へ進んでいた。
> 何が変わったのか、勘定してみる。体は軽い。声も出る。客の入りは先週と同じだ。足しても引いても、余りが合わない。

俺「三行目、一度で通ったな」

谷口「こういう日は、たいてい次でこけるねん」

> 谷口はそれだけ言って、台本に目を戻した。俺は、なぜ通ったのかをまだ数えていた。数えきれる気は、しなかった。

---

**相方が谷口以外のとき（標準語プール・谷口版と差し替え／相方スワップ実装後）**

谷口の一滴（デッドパン）だけを標準語で持つ。地の文・俺のセリフは共通。

相方「こういう日、だいたい次でこけるんだよな」

- **効果**: #27 の既存効果（全能力+1・【仮】）を**据え置き**。この会話は効果を持たない（数値をセリフで言わない・地の文でも軸名／数値を口にしない）。

## どこから学んだか（型の出典・任意URL）

学んだのは**構造・型のみ**。既存作品のセリフ・本文は一切引かない。

- **氷山理論／言い落とし（志賀直哉・カーヴァー系）**: 「好調だ」と心情を名指しせず、"いつも消す三行目を今日は消さなかった"という**一つの物の所作**に上向きを背負わせる。因果を説明せず、事実（消えた跡がない・体が軽い・客入りは同じ）を接続詞なしで並置し、余りが合わない感覚を読者に渡す（fable_kickoff §4-1）。
- **ルシア・ベルリン（自己憐憫ゼロの乾いた一滴・位置と長さ）**: デッドパンを**感情の谷ではなく「好調という平地」**に置く。谷口「たいてい次でこけるねん」は最長一文・自己言及ゼロ・種明かしなし。滴らせた直後、地の文を平常（俺が数えている）へ戻して即引く（fable_kickoff §4-2）。 https://ja.wikipedia.org/wiki/Lucia_Berlin
- **下積みの非対称の"世界の事実の積み方"（型のみ・声は移植しない）**: 好調を感想でなく、**勘定が合わない**という下積み特有の不安として置く（"貧しい"でなく数える）。関西の思弁・比喩の連射・自分語りは移植しない（fable_kickoff §4-5）。
- **「メーター＝物that語る」（proposals/0009 の芯）**: 上向きという不可視の状態を、感情語でなく「三行目の紙の薄さ／消えていない跡」という残された物で可視化する。

## 実装イメージ（本編で器がランタイム化された後の載せ方）

**前提**: `event_design_v0.md` §1 の自動イベントは `(id, 効果パッチ, フレーバーテキスト)` の通知型として**設計されている（doc/sim のみ・アプリ未実装）**。#27 が本編でこの形でランタイム化されたとき、本案は**その"テキスト部を1行フレーバーから会話プールへ差し替える"提案**であって、効果パッチ・発火機構そのものは新設しない。

想定の載せ方（純データは不変・テキストリソースのみ拡張）:

```text
# 【提案・未適用／本編で #27 がランタイム化された後】
# id: "lucky_week"   fire: 体力>=80 の自動抽選（器は本編で新設）
# patch: 全能力+1（既存設計・【仮】・不変）
#
# 変更はUI層テキストのみ:
#   - 従来: flavor 1行
#   - 本案: setup地の文3行 + 俺1セリフ + 谷口(or相方)1セリフ + 地の文1行
#           話者名は Advice.name 相当の枠に載る（俺／谷口／相方その他）
#   - 相方分岐: state の相方が谷口か否かで谷口版(関西)／標準語版を選ぶ
#              ※現状アプリは谷口固定。相方スワップ系の実装後に有効化
```

**発火帯の絞り込みについて（`lossStreak==0` ／ `weeksLeft` 上限の追加）**:

- これは **#27 の"効果を発火させる条件"を狭める**変更で、テキスト差し替えとは別レイヤー。**イベント抽選の合否（＝いつ何回 #27 が発火するか）が変わる＝分布に効く。**
- **★乱数消費順・golden**: 条件を AND で足すだけなら**抽選ストリームからの引き自体は増減しない**想定だが、`weeksLeft` バンド化や相方分岐で**新たに乱数を引く実装にすると消費順の正典に触れる**。ここは **→ 実装時に必ず止めて相談**（`tools/*.py` 正典 → GameCore → `gen_golden` → `swift test` の1コミット同期／CLAUDE §A-1,2）。**テキスト差し替え＋既存フラグ参照だけに留める限りは golden 不変**、という切り分けを実装者に明記する。
- **バランス再計測**: 発火帯を絞ると #27（全能力+1＝恒久能力バフ）の**発火回数が減る**方向＝優勝率にはむしろ安全側だが、`sim_career.py` の `EVENTS_ON` で優勝率1.4〜1.7%帯の維持を再測（event_design §137）。`EVENT_FIRE_CAP=15` の総量予算内での扱いも要確認。

## リスク・注意

- **【要相談・正典】発火帯の絞り込みと乱数**: `weeksLeft` バンド化／相方分岐で乱数を引くなら消費順の正典に触れる＝「止めて相談」対象（CLAUDE §A-2）。テキスト差し替え＋既存フラグ参照に留めれば golden 不変。この境界を実装コメントに残す。
- **【器が未実装】本案は本編用**: #27 のイベント抽選ランタイム自体がアプリ未実装。器の新設は本案の外で、本案は器ができた後の"テキスト器の中身"を先に用意しておくもの。MVPには積まない。
- **【相方分岐は相方系依存】**: 標準語プールは相方スワップ系（未実装）が要る。谷口固定の現状では谷口(関西)版のみが有効。標準語版は前倒しで書いておくが、有効化は相方系の実装後。
- **【FIRE_CAP との相互作用】**: #27 は恒久能力バフのため `EVENT_FIRE_CAP=15` の予算対象。予算超過時は「フレーバー表示のみ・効果不適用」運用だが、本案は**元々フレーバーが主役**なので"効果不発でも会話は成立"する（選択肢イベントと違い空振り感が出ない）＝この点は選択肢型より安全。
- **デッドパンの多重化回避**: 一場面に乾いた一滴は**谷口「たいてい次でこけるねん」の1つだけ**。締めの地の文「数えきれる気は、しなかった」を"うまいこと言った"警句に寄せると二滴目になり寒くなる＝**あくまで俺が勘定に失敗している事実**に留める（延ばして救わない）。
- **上向きの押し売り回避**: 「録画しておきたい」（#27 原文フレーバー・docs 行74）のような"好調を愛でる"温度は入れない。本案は**好調を喜ばず、気味悪がって数える**方向で、下積みの不穏さを主にした。原フレーバーと会話版のどちらを出すか（または会話版で置換）は温度判断＝オーナー/目視。
- **相方依存の声**: 谷口＝関西弁一貫（「こけるねん」の "ねん" で軽く関西を刻む・デッドパンは維持）／谷口以外の相方＝標準語（関西弁を移さない・fable_kickoff §3-B）。地の文・俺の一人称は常に標準語。
- **固有名・数値**: 大会名・客数等は本文で名指ししない。全数値【仮】。#27 の効果（全能力+1）は据え置きで本案では触れない。既存作品セリフ流用なし・ボケ／オチ／ネタ本文ゼロ（笑いは地の文の乾いた一滴のみ）。

**トーン自己点検**: 感情語（嬉しい／気味悪い）不使用。上向きは物＝「いつも消す三行目を今日は消さなかった／紙が薄く三行目だけ透ける／余りが合わない勘定」に背負わせた。固有プロップ＝定番（天井の染み・冷めたコーヒー・小銭）でなく「この二人のネタ帳の、消しゴム跡で薄くなった三行目」1つ。俺＝標準語・計算屋（「勘定してみる」「余りが合わない」「まだ数えていた」）・自己憐憫なし・警句で締めない。デッドパンの一滴＝谷口「こういう日は、たいてい次でこけるねん」を1箇所のみ（真顔の事実・自己言及ゼロ・平地に配置・"ねん"で関西を軽く刻む）。谷口＝関西弁一貫、相方版のみ標準語別プール。会話は話者ごとに1セリフずつ・「／」詰めなし＝通常の会話送り。

## レッドチーム（A/B）

### A. 実装妥当性
- **[修正・区分]** 本案は**本編用（post-MVP）**と明記した。自動イベント抽選ランタイムは `ManzaiGame/Sources/` に未実装で、#27 は `docs/event_design_v0.md` 行74 と `sim_career.py` 近似のみに存在。器（抽選・効果パッチ）の新設は本案の外＝MVP発火ではない、と切り分けた。
- **[修正・相方分岐]** 「既存の相方判定を流用・新フラグ不要」は現状誤り（アプリは谷口固定・相方判定フラグ無し）。標準語プールは相方スワップ系（partner_gacha 系・未実装）依存に訂正。谷口(関西)版のみが現状の有効プールと明記。
- **[修正・シンボル名]** `preTournament` は実シンボルでない。実体は `DialogueData.innerVoice` の `nextMilestone.weeksLeft <= 2` 帯。`weeksLeft` は `WeekMainView.nextMilestone()` 由来の導出値（GameSession 直フィールドでない）と訂正。`stamina`（GameState）・`lossStreak`（GameSession 行29）は実在フックで OK。
- **[OK・非選択スコープ]** 選択肢イベント（0010-0023）とは別レイヤー。新効果パッチ無し＝テキストリソース拡張のみ。既存 dialogue 系（0001-0009）とも別スロット（#27 好調＝大会外の上向き）で重複なし。
- **[要注意・分布／golden]** 発火帯の絞り込みは "効果の発火回数" を変える＝ `sim_career.py` 再測対象（恒久バフの**減**方向＝優勝率は安全側）。golden は「乱数を追加で引かない」限り不変。`weeksLeft` バンド化・相方分岐で乱数を引くなら**消費順の正典に触れる＝止めて相談**を本文に明記済（CLAUDE §A-1,2）。

### B. 会話の中身
- **[修正・方言反転]** 初稿は谷口(関西)が標準語寄りの「転ぶ」、標準語相方が関西寄りの「こける」と反転していた。谷口＝「こけるねん」（"ねん" で軽く関西・デッドパン維持）、標準語相方＝「こけるんだよな」に是正。
- **[良]** 上向きを叫ばず「三行目が一度で通った」一点の事実で置く＝ `passedLines` の自制を大会外へ延伸。固有プロップの置換不能性（このネタ帳の三行目）を満たす。コピペ酷似の疑い行なし（火花等の既存本文と一致する像・語は無し・型のみ抽出）。
- **[良]** 谷口のデッドパンが感情の谷でなく好調の平地に置かれ、自己言及ゼロ・種明かしなし・最長一文。直後に地の文を平常（俺の勘定）へ戻して即引く＝ベルリン型の位置と長さを遵守。ボケ（フリ／オチ／パンチライン）ゼロ。
- **[要監視]** 締めの「数えきれる気は、しなかった」は感情近接。**認知の失敗（勘定が閉じない）に留める限り可**だが、実機で湿って読めたら「まだ数えていた。」で止めて1行削る、を代替案として保持（目視判断）。
