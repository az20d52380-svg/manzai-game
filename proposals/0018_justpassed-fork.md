# 提案 0018 — 「通った日の分かれ道」justPassed 二択（勝ち側の別状態・justLost の対）

- 日付: 2026-07-06 ／ 数値は全て【仮】／ 既存ファイルは読むだけ ／ トーン＝ドラマ主軸（proposals/0009 確定）
- 位置づけ: 単発**選択**イベント。0003 の `justLost`（負け側・未配線）と対をなす**勝ち側**の節目。`justPassed` は**MVP で実配線済**（下記フック）。

## 狙い

`justPassedStage`（直近通過フラグ）には現状**選択の顔が無い**。この帯を拾うのは `innerVoice` の `passedLines`（余韻2行・`DialogueData.swift:54-57`）だけで、「通った勢いをどう扱うか」がプレイヤーの手に渡っていない。通過は数少ない上向きの節目だが、上向きだからこそ「乗るか・締めるか」で迷える。ここに `event_design_v0` §設計原則3「正解のない二択」を1枚差す。

トレードは二方向で左右対称に置く。**勢いの換金**（露出で知名度を得るが、体力を削り**気が緩む**）と、**地固め**（実力・相性を1つ積むが、勢いを知名度に換えない）。どちらも上位互換にしない。自制の難しさは相方の一言に背負わせ、数値でなく人間味で落とす。

## 発火条件（実フック）

- **主フック**＝`GameSession.justPassedStage`（`GameSession.swift:31`。`pump()` で `r.passed==true` の週に立つ＝`:150`）。`monoBox` が既に同フラグを読む（`WeekMainView.swift:154-156`）＝**新規の表示フックを足さずに載る**。
- **MVP発火可**：GP各回戦の通過後に `justPassedStage` が立つ＝1年48週の枠内で複数回実働。**`justLost`（0003・未配線）に依存しない**（justPassed は単独で live）。
- **ゲート（【仮】）**：
  - `justPassedStage == true` の**自由行動週（`.freeAction`）**でのみ提示（`monoBox` が出るのはこの画面のみ＝「大会週でない」は画面種で自然に担保）。
  - 次のマイルストンまで**余白がある**（≒`weeksLeft >= 3`【仮】）。次戦直前だと A の機会損失が理不尽になるため。**この判定は View 層の `WeekMainView.nextMilestone()`（`:407-415`）で導出**（GameSession 側に該当プロパティは無い＝下記【A-1】参照）。
  - **低体力ガード（【仮】）**：`state.stamina` が A の消費（-15【仮】）を割り込む水準なら**発火しない**。高揚の場面に「体が動かない」状態が重なると、A が事実上の罠になり演出も齟齬する（優先度は `innerVoice` でも 低体力 > 直近通過）。
- **確定発火寄り**：通過は数少ない節目なので週次12%抽選でなく、上記ゲートを満たす最初の自由週に**確定で1回**。同一通過での多重発火はしない（`choose()` が `justPassedStage=false` に失効させる＝`:64` に相乗り）。**確定発火＝抽選RNGを一切引かない**（golden 非対象・下記実装）。
- **相性帯フレーバー（任意）**：`compat` 15-20（ツーカー帯）なら相方の前振りを一段砕けた変種に差し替え可（上限20＝阿吽21+は出さない、既存規約どおり）。


**【効果】** 知名度 **+3** ／ 体力 **−15** ／ **メンタル −1**（気の緩み・慢心の表現）。相性・能力の上積みなし。すべて【仮】・**すべて既存6軸＋体力への決定的 delta（RNG不使用・新システム不要）**。

## 実装イメージ（既存機構に載る形／コードは"提案"・golden影響は要相談）

- **選択イベント engine は本体未実装**（`event_design_v0` は突発イベントのマスターデータ案＝設計。アプリに選択イベントの実行系はまだ無い）。本案は**将来の突発イベント engine（選択肢→効果＋短い会話 の型）に載せる前提**。
- **発火フック**：`GameSession.justPassedStage`（`:31`）を判定側から**参照するだけ**（新規の永続状態は不要）。**ゲートは全て View 層で導出**：`.freeAction` 画面であること（`monoBox` と同条件）＋`WeekMainView.nextMilestone()`（`:407-415`）の `weeksLeft>=3`＋`state.stamina` の低体力ガード。
- **効果適用**：効果はすべて**既存6軸＋体力＋知名度への決定的 delta**。ただし GameSession は現状 `state` を `private(set)` で持ち、`runner.resolve*` 経由でしか変異しない＝**イベント delta 用の決定的 apply 経路を新設する必要がある**（既存の稽古/バイト適用とは別口）。**この経路は `runner` のRNG（injury抽選・rollOffer）を一切呼ばない**ので golden 不変（`GameSession.swift:78-82` の「previewは runner を呼ばない」規律と同型）。
- **B の「センス or 発想 +1」**：`WeekMainView.weakAbility()`（`:402-405`）と**同型の決定的 min 判定**で低い方の1軸へ加算（RNG不使用）。※`weakAbility()` は5軸の最小を返すので、本イベント用に**センス/発想の2軸に限定した決定的 min** を別途置く（流用ではなく同型の新関数）。
- **golden・乱数**：確定発火＋決定的 delta＝**`WeekRunner`/RandomSource を一切呼ばない**＝**golden 不変・正典同期の対象外**。
- **【止めて相談】**：もし発火ゲートの抽選や効果を **GameCore 側のシード付き RNG（週送りの乱数消費列）**に載せる実装にすると、**乱数消費順が1箇所ズレて3年ビット一致（golden）が壊れる**（CLAUDE §A-2）。その時は**手を止めてオーナー/実装と相談**。推奨は「発火＝確定（抽選なし）／効果＝決定的 delta」に閉じ、runner の消費列に一切触れないこと。

```swift
// 【提案・未適用／将来の選択イベント engine に載せる形の擬似コード】
// 判定は View 層（WeekMainView）で。GameSession/runner に isTournamentWeek 等は存在しない。
// 発火: 通過翌週の自由週・次戦まで余白・体力に余裕。runner の RNG は呼ばない＝golden不変。
if session.justPassedStage,                       // GameSession.swift:31（実在）
   phaseIsFreeAction,                             // monoBox と同じ画面種＝「大会週でない」を担保
   (nextMilestone()?.weeksLeft ?? 0) >= 3,        // View 層の既存ヘルパ WeekMainView.swift:407
   session.state.stamina >= staminaFloorForA {    // 低体力ガード（A を罠にしない）【仮】
    presentChoiceEvent(.justPassedCrossroads)     // 選択肢→確定delta＋短い会話
}
// A: fame += 3; stamina -= 15; mental -= 1                      // 【仮】全て決定的・既存軸
// B: bump(weakerOf: .sense, .idea, by: 1); compat = min(20, compat + 1)  // 【仮】決定的分岐
// いずれも「イベント用の決定的 apply 経路」で適用（runner.resolveAction は呼ばない）
```

- **アンカー**：`GameSession.swift:31`（`justPassedStage`）／`:64`（`choose()` の失効点＝多重発火防止に相乗り）／`:150`（立つ点）／`WeekMainView.swift:154-156`（`monoBox` の読み口）・`:407-415`（`nextMilestone()`）・`:402-405`（`weakAbility()`）／`DialogueData.swift:54-57`（対になる `passedLines`）／`docs/event_design_v0.md` §設計原則3・§4-B。

## リスク・注意

- **【要相談・balance】§4-B の総量予算に必ず載せる**。本イベントは**GP各回戦の通過ごとに再発しうる**＝年内で複数回発火する反復正イベント。恒久能力(+1)・相性(+1)・知名度(+3)は `EVENT_FIRE_CAP=15` と優勝率1〜2%帯の生命線に効く。**取り込み前に `sim_career.py`（EVENTS_ON）で優勝率帯の維持を再計測**（§4-B rule c）。恒久能力の正効果は控えめ(+1)に留める設計則も遵守。
- **【設計上の懸念】A の重さと「翌週」ズレ**。`justPassedStage` は**行動するまで失効しない**（`:64`）ので、通過→（大会週が連続）→数週後の自由週で初めて提示される場合がある。`weeksLeft>=3` ゲートで大半は吸収するが、地の文「次の戦いまで、まだ日がある」は両ケースで齟齬しない文にしてある。engine 実装時に「提示週＝通過の何週後か」を目視確認。
- **明白な最適解の防止**：知名度の価値（オファー率・収入への波及）が高すぎると A 一択、低すぎると B 一択になる。**A/B の重みは `GameConfig` の【仮】値で調整し sim 実測を根拠に**（CLAUDE §B-4）。体感が偏っても**まず頻度・テキストで解き、数値は最後**（§B-5）。
- **B の据え置きケース**：`compat==20` では B の相性+1 がクランプで消え、B は能力+1のみになる。ツーカー帯では B の魅力が一段下がる＝相性帯フレーバー（前振り差し替え）で温度を補い、数値で無理に埋めない。
- **正解のない二択を保つ**：中間の第3択（両取り）は置かない（2択で対称トレードを維持）。
- **声の規律**：俺＝標準語・事実ベース・自己憐憫なし（「今日のは、ただの一日で終わる」は計算屋の損得提示＝嘆きにしない）。谷口＝関西弁一貫、相方≠谷口は関西弁にしない。**ボケ（オチ/ネタ本文）ゼロ**。デッドパンの一滴は地の文「三枚目は、少し曲がっていた」の1箇所のみ（会話に二個目を盛らない）。
- **既存作品コピペ無し**：セリフ・プロップ（フライヤーを一枚だけ貼る癖）は全て新規。諺は verbatim 引用せず谷口が噛み砕く。固有名は架空。
- **golden 非対象だが更新は同トランザクションで**：効果適用・`justPassedStage` 失効を既存の結果処理と**同一トランザクション**で扱う（処理順のズレは表示バグの温床＝0003 と同じ注意）。

## レッドチーム（A/B）

**A. 実装妥当性**

1. **【修正済】存在しない GameSession API 依存を除去**。初稿は `session.isTournamentWeek` と `session.nextMilestone?.weeksLeft` を参照していたが、**どちらも実在しない**（`isTournamentWeek` はコード全体に無し／`nextMilestone()` は `WeekMainView` の private ヘルパ `:407-415` で非Optionalタプルを返す）。ゲート判定は**View 層で導出**に修正。「大会週でない」は `.freeAction` 画面種（`monoBox` の出る画面）で担保。
2. **【修正済】「稽古枠-1」＝新規スケジューリング機構への依存を撤去**。本エンジンに「稽古枠」概念は無く（自由週は毎回 train/job/rest を自由選択）、`consumeOneUpcomingPracticeSlot()` は**新規の永続状態＋稽古カテゴリのUIグレーアウト**を要する＝初稿の「新規状態不要」と自己矛盾。**A の代償を既存軸のメンタル-1（気の緩み）へ置換**し、新システムを排除しつつ「慢心」テーマを強化。トレードは非支配のまま維持。
3. **【確認済/精緻化】golden 非対象**。効果は決定的 delta・確定発火（抽選なし）で `runner` の RNG を一切引かない。ただし GameSession は `state` を `runner.resolve*` 経由でしか変異しないため、**イベント用の決定的 apply 経路の新設が必要**（「既存のステ適用と同経路」は誤り＝events engine 未実装）。新設経路は runner RNG 非依存＝golden 不変。
4. **【要相談・balance】§4-B 予算未計上**。反復正イベントなのに fire-cap(15)/優勝率1〜2%帯の再計測に触れていなかった。**取り込み条件として sim 再計測を必須化**（リスク欄）。
5. **【確認済】MVP 発火・justLost 非依存**。`justPassedStage` は `:150` で live に立つ＝1年内の GP 通過で実働。0003（未配線）に依存しない。多重発火は `:64` の失効で防止。
6. **【追加ガード】低体力時の罠回避**。高揚イベントが低体力週に重なると A が事実上の罰になる＝発火に低体力ガードを追加（`innerVoice` の優先度 低体力>直近通過 とも整合）。

**B. 会話の中身**

1. **【修正済】選択肢の復唱を除去**。初稿の俺「締めるか。それとも、乗るか」は二択メニューの音読で"メニュー臭"＝俺「分かってるなら、気を抜くなよ。」へ差し替え、二択はボタンに委ねる。谷口の本音行「分かってるのと、やめられるのは、別やねんな」は良行として維持。
2. **【確認済】ボケ無し・ドラマ主軸**。全行が反応/観測/損得判断。オチ・ネタ本文なし。「今日のは、ただの一日で終わる」は計算屋の事実提示＝自己憐憫回避。
3. **【確認済】固有プロップ＋一滴規律**。「フライヤーを一枚だけ貼る癖／三枚目が曲がっていた」＝定番でない固有の物。デッドパンは地の文1箇所のみ、会話に二個目を盛らない＝1テキスト1技法。
4. **【確認済】相方依存の声**。谷口＝関西弁一貫（せやろ/やろ/やねんな/ええな）。谷口以外＝標準語別プール、関西弁を混ぜない。俺は全行標準語。相性帯で温度差可（compat上限20＝阿吽21+は出さない）。
5. **【確認済】1セリフずつ・コピペ無し**。全会話が話者ごと改行（"／"詰めなし）＝普通の会話送り。各行を火花/べしゃり暮らし/M-1実在ネタと照合し一致・酷似なし。諺は verbatim 引用せず谷口が噛み砕く。固有名は架空・数値は【仮】。
