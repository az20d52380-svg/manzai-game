# 突発イベント提案: 「終電までの反省会」 v0.1（2026-07-06）

フリーライブ稽古の直後・低知名度帯に差す、**敗因を今掘る（成長と消耗）か・畳んで二人で立て直す（相性と回復）か**のトレードオフ二択。トーンは proposals/0009 確定の**ドラマ主軸**（感情語を書かず"物"を書く／俺＝標準語・計算屋・自己憐憫禁物／谷口＝関西弁／会話は話者ごと1セリフ・ボケない）。数値は全て【仮】。既存ファイルは読むだけ。

## 狙い

『客ゼロに近い帰り道』の seed を、**最も人数が薄い時期にだけ立つ節目**として置く。知名度<20 帯は客席がまばらで笑いも薄く、`event_design_v0` の設計原則2（自分のプレイが世界に反映される＝低知名度なら客の薄い世界が出る）に直結する。フリーライブ（＝華+3/表現+1・体力-30 の身を削る稽古・`GameConfig.swift:134`）を選んだ直後にだけ差すことで、「行動→世界の手応え」を一本の線でつなぐ。

トレードは `event_design_v0` 設計原則3（正解のない二択・上位互換を作らない）を守る:
- **A 今すぐ敗因を洗う** … 恒久能力（センス or 発想 +2）を取る代わりに、既に-30で削れた体力とメンタルをさらに払う。
- **B 畳んで二人で立て直す** … 体力・メンタルを回復し、飯を挟んで**相性（恒久・別軸）を1上げる**代わりに、今日のネタの敗因は明日送り。

**※ v0→v0.1 の主変更（レッドA軸指摘）**: 前稿の B は「体力＋メンタル回復のみ（恒久ゲインなし）」で、A の恒久能力+2 に対し**構造的に見劣り＝明白な最適解がAに寄る**恐れがあった（回復リソースは稽古で取り戻せるが、恒久能力は取り戻せないため、SLG一般則としてAが支配的になりやすい）。そこで B にも**恒久で別軸の good（相性+1）**を持たせ、「ネタの実力（A）」対「二人の呼吸＝相性（B）」の**同格・別軸トレード**に組み替えた。これで event_design_v0 の #20「ボケとツッコミ交換（発想+2 or センス+2）」と同型の"どちらも恒久・どちらも得・でも別物"という真の割れる二択になる。相性の伸びは既存の `相方と過ごす`（体力回復+相性+1・`GameConfig.swift:148`）と同機構で、飯を挟む文脈と噛み合う。

## 発火条件（実フック・行番号は検証済み）

- **行動フック**: `WeekAction.train(.フリーライブ)` を解決した直後（`WeekRunner.resolveAction`・`WeekRunner.swift:214`／`.フリーライブ` spec は `GameConfig.swift:28,134`）。「フリーライブ稽古を選んだ週」に限定。
- **状態フック**: `state.fame < 20`（fame＝知名度・`GameState.swift:7`・初期値 `initFame = 3.0`＝`GameConfig.swift:80`。客が薄い帯）。
- **ランダム**: 上記2条件を満たす週に低確率で抽選（頻度は【仮】。`event_design_v0` の週次12%より低め推奨＝節目感を薄めない）。
- **再発**: 同一キャリアで原則1回（`event_design_v0` §頻度規律。恒久能力を扱うイベントは1回制が優勝率の生命線＝同§検証1）。
- **MVP発火可**: フリーライブ＋fame<20 は1年目内で普通に起きる（初期 fame=3、フリーライブは常時選べる）。年またぎ不要＝MVP検証で実働する。

> 【止めて相談】「ランダム抽選」は `RandomSource.nextUniform()` を1つ引く＝**乱数消費順に触れる＝golden破壊リスク**。しかも `resolveAction` は**既に**この行動フックの直後で体調ダウン抽選（`WeekRunner.swift:219` の `rng.nextUniform()`）を消費している。イベント抽選のドローをこの既存ドローの前か後か、どの分岐条件下で引くかで3年ビット一致が変わる。実装前に消費順を `runYear`/`WeekRunner` と揃える設計を確定させること（詳細は実装イメージ・リスク）。

## イベント本文

**場面（俺の視点・標準語・感情語なし）**

> フリーライブが終わった。客席は八人。数え直しても八人で、そのうち一人は、開演前にうちのチラシを配っていたスタッフだった。
> 谷口のリュックからは、配りきれなかったチラシの束がまだはみ出している。三十枚は残っている。
> 駅のホームに二人。終電まで、二十分ある。
>
> 谷口「実質七人やな。……七人ぶん、どこがあかんかったか、今やっとくか？」

（※固有の少しズレた物＝"配りきれず谷口のリュックに残った三十枚のチラシ"。空席を「惨め」と言わず物で置く。「実質七人」＝八人中一人は身内、はデッドパン＝事実の並置の一滴。デッドパンは本イベント1回まで。）

---

### 選択肢A ── その場で敗因を洗う

**【効果】**（数値【仮】）
- メンタル -2 ／ 体力 -5
- センス **または** 発想 +2（どこを直すかは二人とも分かっている＝**プレイヤーが1軸を選ぶ**。ランダムにしない＝効果値にRNG不使用）
- 知名度・所持金・相性：変化なし

**選んだ後（話者ごと1セリフ）**
> 俺「二十分ある。その二十分で、洗えるだけ洗う。」
> 谷口「ええな。……立ったまま行こ。座ったら、明日になる。」

### 選択肢B ── 今日は畳んで二人で立て直す

**【効果】**（数値【仮】）
- 体力 +5 ／ メンタル +1
- 相性 +1（飯を挟んで呼吸を合わせ直す＝恒久・別軸。`相方と過ごす` の +1 と同機構・`compatCap = 20`＝`GameConfig.swift:83`）
- 所持金 -少額（飯代・【仮】）
- 能力：変化なし（ネタの敗因の洗い出しは明日に回す）

**選んだ後（話者ごと1セリフ）**
> 俺「今日は畳む。反省は、腹が減ってると精度が落ちる。」
> 谷口「せやな。……その七人の話は、明日、湯気の立っとる方でやろ。」

（俺＝計算屋の乾いた処理・自己憐憫なし。谷口＝関西弁で本音を"物（湯気）"に預ける・悔しさは言語化しない。両者ボケなし・ドラマ主軸。）

## リスク・注意

- **【止めて相談】乱数消費順（再掲）**: ランダム発火の1ドローを runYear/WeekRunner の消費順のどこに差すかが未確定。特に `resolveAction:219` の体調ダウン抽選との前後関係を決めずに書くと golden 全崩れ。抽選導入は消費順設計とセットでのみ。
- **体力ゲートとの干渉（実測で確認）**: 初期体力 `initStamina=100`、フリーライブ -30、`staminaGate=20`（`GameConfig.swift:106`）、`injuryThreshold=20`（同`:108`）。満タン起点なら フリーライブ後70→A後65 でゲートには当たらない。ただしフリーライブを連続で選び体力を削ってきた局面では、A の追加-5 が**翌週の train 行動を staminaGate で強制休養化**（`resolveAction:218` の remap は"次に train を選んだ週"の頭で効く）または injury 抽選確率を押し上げ得る。A が実質「罰ゲーム」化しないか、削り済み局面での -5 を sim で確認（バランスは最後・体感は演出で／CLAUDE §B-5）。
- **明白な最適解の検証（A軸の主論点・v0.1 で構造を修正済みだが要実測）**: v0 の「A=恒久能力 vs B=回復のみ」は構造的にAへ寄る恐れがあったため、B に恒久・別軸の相性+1 を足し「ネタの実力 vs 二人の呼吸」の同格トレードに組み替えた。ただし**相性+1 を足したことで優勝率が動く**可能性がある（event_design_v0 §検証1・3: 恒久正効果は+1〜2に抑え、追加のたびに優勝率1〜2%帯の維持を EVENTS_ON で再計測）。相性が伸びると `ネタ合わせ` の sub や finals の相性寄与に波及し得るため、**採用前に sim で「A/B が拮抗し、かつ優勝率が想定帯を割らない」ことを確認**。寄っていたら**まず発火頻度・文面**で、最後に効果量で調整。
- **話者切替の未実装**: 谷口セリフの表示は UI 側に話者の口が要る（0003・`dialogue_design` §6-§7 のコスト論点）。未実装＝欠陥ではないが、本案は「谷口が計3行喋る」前提なので工程はその後。
- **再発制御**: MVP発火可だが `event_design_v0` の「同一キャリア原則1回」に従い、多重発火で希釈しないフラグ管理を（永続化はセーブ設計と同時）。恒久効果イベントの無制限再抽選は優勝率を暴騰させる（同§検証1）ため1回制は必須。
- **トーン規律**: 俺＝標準語・計算屋・自己憐憫禁止／谷口＝関西弁を厳守。デッドパンの一滴（"実質七人"）は本イベントで1回まで＝5-6回に1回の密度を超えない。ボケ本文ゼロ。固有名は架空・数値は全て【仮】・既存作品のセリフ流用なし。

## レッドチーム（A/B）

### A. 実装妥当性

- **【要修正→修正済み・主指摘】明白な最適解**: v0 の B は回復リソースのみで恒久ゲインが無く、A の恒久能力+2 に対し SLG 一般則で構造的に劣後＝二択が「実質いつもA」に潰れる恐れ。→ B に**恒久・別軸の相性+1**を付与し、event_design_v0 #20 と同型の"どちらも恒久・別軸"の真のジレンマに改稿。設計原則3（上位互換を作らない）に適合。**ただし相性+1 の優勝率影響は sim 再計測が必須**（未検証・unresolved）。
- **発火条件は既存構造に無理なく載る**: `resolveAction` の `.train(.フリーライブ)` 直後 + `state.fame<20` + 1ドロー。行番号（`WeekRunner.swift:214`/`GameConfig.swift:28,134`/`GameState.swift:7`）は実ファイルで検証済み・整合。
- **【止めて相談・確定】golden/消費順に触れる**: 発火の1ドローが乱数消費順に触れる。さらに同関数`:219`に既存の体調ダウン抽選があり、挿入位置で消費順が動く＝3年ビット一致が壊れる。効果値自体は決定的でRNG不使用。tools正典→Swift→golden再生成→swift test を1コミットで揃える手順を明記済み。**この確定前に実装しない**。
- **MVP/本編の別**: フリーライブ＋fame<20 は1年目で普通に成立＝**MVP発火可**。年次ラダー不要で MVP 検証に実働する。
- **付番の誤り（修正済み）**: 原稿は自身を 0010 と付番していたが proposals/0010「前夜の一本」が既採用。本稿を **0011** に採番。

### B. 会話の中身

- **ドラマ主軸（物で語る）＝合格**: 感情語ゼロ。空席を「三十枚の配りきれないチラシ」「実質七人（八人中一人は身内）」という固有の物・事実で置いている。定番プロップ回避。

- **方言＝合格**: 俺は全編標準語・計算屋の乾き（"反省は、腹が減ってると精度が落ちる"＝処理として扱う）で自己憐憫なし。谷口は関西弁（"やっとくか""ええな""せやな""湯気の立っとる方でやろ"）が自然。谷口以外の相方は登場せず＝方言の取り違えなし。

- **キザ/サムさ＝可**: "湯気の立っとる方で" は詩的だが物（＝温かい飯）に接地しキザ止まり手前。許容。
- **既存作品の流用＝なし**: 火花等は「下積みの世界の手触り／構造」のみ抽出、セリフ本文の転載・酷似なし。本文は原作。
- **微修正（実施）**: 俺のA後セリフが地の文の"終電まで二十分"と語をエコーしていたため、"二十分ある。その二十分で、洗えるだけ洗う。"に整え、計算屋の"数える"モチーフは残しつつ重複を除去。

**総評**: 会話（B軸）は概ね合格・微修正で確定。実装（A軸）は①乱数消費順の【止めて相談】と②相性+1 追加後の優勝率 sim 再計測が未クリア。文面は採用可、実装は前提2件クリアまで保留。
