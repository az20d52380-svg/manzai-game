# イベント提案：谷口の耳寄りな話 v1（2026-07-06）

- 種別: 選択肢イベント（二択・リソース交換／`event_design_v0.md` §2 系）。単発、同一キャリア原則1回。
- seed: 「相方が別の稼ぎ話を持ってくる」。hook＝所持金／相性。二律背反＝**金を取れば合わせの時間が消え、時間を取れば金が細る**。
- トーン: ドラマ主軸（0009）。俺＝標準語・計算屋・自己憐憫禁止。乾いた一滴は谷口が差す（俺の計算屋の自己防衛を、谷口が事実で1つ上書きする）。谷口＝関西弁。数値は全て【仮】。

## 狙い

金欠帯（所持金<10万）で一番出やすいのは焦りで、放っておくとバイト連打の作業感に落ちる。ここに**相方発の分岐**を一枚差し、金の圧を「二人の時間をどう使うか」という関係の問いに翻訳する。

肝は**上位互換を作らないこと**。受ければ当座の金は片づくが、コンビの核である合わせの時間が三週間まるごと消える（相性の成長が止まる）。断れば同じ舟に乗っている確認は取れるが、通帳の桁は増えない。どちらも損を1つ抱える設計にして、`event_design` §設計原則3「正解のない二択（リソース交換であって上位互換を作らない）」を守る。

副題は**計算屋の自己防衛を相方が崩す**こと。俺は反射で損得を弾く（そろばん）。谷口はそれを責めず、見えていたと一言置くだけで、俺の自己憐憫が立ち上がる前に折る。相方依存の声＝この一言は谷口以外には書けない。

## 発火条件（実フック）

既存の状態から**RNG非消費で導出できる条件のみ**で組む（state＝セッション層で判定、`WeekRunner` の乱数経路には入れない）。

- `state.money < 100000`（所持金<10万・金欠帯／`money` は `GameState` の `Int`、Ability enum 外。セッションは state 直読み）。
- `state.compat >= 8`（相性8以上＝噛み合い帯以上。他人行儀帯 0–7 では谷口はこの話を持ちかけない＝相方として一歩踏み込む信頼の線）。`compat` は `GameState` の `Double`。debug で `s.compat=19` を想定。この **compat>=8 ゲートは解散危機アーク①（相性8未満が6週継続）と発火帯が重ならない**ので、両者の温度事故を構造的に避けられる（副次的な整合性）。
- 大会週・大会2週前は除外（合わせの時間を焦点化する回なので、直近に大会があると意味が濁る）。**さらに本案では「今後3週以内に大会が無いこと」も除外条件に足す**（選択肢Aの3週停止が、プレイヤーの見えないところで直近大会の追い込みを食い潰す“隠れ罠”になるのを防ぐ。レッドチームA-1の是正）。
- ランダム枠（彩り側イベント。確定発火ではない）。**MVP発火可**：条件が所持金／相性／週という多年非依存の状態だけで立つため、1年48週の中でも成立する（同名大会の再会に依存する多年専用イベントと違う）。

> 【止めて相談】「ランダム枠」の抽選をどこで引くかは**乱数消費順に触れる**。イベント判定を週ループ内で `roll` すると、`WeekRunner` の消費順（`injury` 判定の `rng.nextUniform()` → `rollOffer`）に割り込み、3年ビット一致goldenが壊れる（規律A-2）。**この抽選の実装は手を止めてオーナー相談**。当座は条件成立＝確定発火（ランダム性オフ）で先に本文だけ検証するのが安全。

## イベント本文

**セットアップ**

稽古場に谷口が先に来ていた。求人票を四つ折りにして持っている。折り目が、給料の桁をちょうど横切っている。

> 谷口「割のええ短期の仕事、二人分おさえてきた。三週間、まとまるで」
>
> 俺「三週間、ネタ合わせが飛ぶな」
>
> 谷口「飛ぶな。……お前が今、そろばん弾いた顔したん、見えてたで」

ホワイトボードには、先週二人で書いたネタの順番がまだ消えずに残っている。三週間、たぶんこのままだ。

---

**選択肢A：二人で受ける**

【効果】所持金 **＋大**（+15万【仮】）／体力 **−中**（−15【仮】・拘束の疲れ）／相性 **その3週間の成長停止＋相性 −1**【仮】（合わせの時間が消える）／メンタル ほぼ不動（金の当座が片づく分だけ相殺）／知名度 変化なし。

> 谷口「ほな、三週間な。……ホワイトボード、消さんといたるわ」
>
> 俺「合わせは戻ってから、まとめて取り返す。段取りは組んである」
>
> 谷口「お前のその段取り、たまに好きやで。……たまにな」

---

**選択肢B：断って、稽古を選ぶ**

【効果】所持金 **変化なし**（金の不安は残る）／メンタル **−1**【仮】（桁は増えないまま）／相性 **＋1**【仮】（同じ舟に乗っている確認）／体力 変化なし／知名度 変化なし。

> 俺「金は、バイトで少しずつ詰める。三週間まとめて抜けたら、順番が崩れる」
>
> 谷口「そっちか。……ええよ、そっちで。求人票は、鞄の底で腐らせとくわ」
>
> 俺「腐らせとけ。順番が崩れるよりは、安い」

## どこから学んだか（構造＋出典URL）

学んだのは**型と構造だけ**。既存作品の本文・台詞は一切引かない。

- **選択肢イベントの二層構造（確定＝節目／ランダム＝彩り）＋パラメータ交換の二択**。パワプロ「サクセス」の選択イベントは上位互換を作らず、得た物の裏で別の物を失わせる。本案の「金↑と引き換えに時間↓」はこの交換設計の踏襲。 https://ja.wikipedia.org/wiki/実況パワフルプロ野球_サクセスモード
- **正解のない選択（機会費用＝opportunity cost）**。片方を選ぶ＝もう片方を捨てる、をゲームの選択に落とす原理。明白な最適解を置かない。 https://en.wikipedia.org/wiki/Opportunity_cost
- **道徳/関係の選択に「正解」を用意しない（no-right-answer choice design）**。Disco Elysium 型。プレイヤーに損得でなく「どちらの自分でいるか」を選ばせる。 https://gameplayreflections.wordpress.com/disco-elysium-and-the-meaning-of-failure/
- **相方が稼ぎ話を持ち込む＝下積みの実相**。副業・短期バイトで合わせの時間が消えるのは実在の摩擦。金と稽古の綱引きを世界の手触りとして借りる（seed の出所）。 https://ja.wikipedia.org/wiki/漫才
- **自己憐憫を「相方の一滴」で折る／感情を書かず物で置く**（乾いた一滴・自己憐憫ゼロ）。社内正典 `proposals/0009` の配合レシピ（ベルリン＝掃除婦のための手引き書の温度）。谷口の茶化しは責めでなく、俺の湿りを事実で上書きする役。 （社内正典 `proposals/0009_tone-references-and-buylist.md`）
- **正解のない選択＝トレードオフ**（社内正典）。 `docs/event_design_v0.md` §設計原則3・§2。

## 実装イメージ（既存機構に載る形・コードは"提案"）

**現状の前提**：本体に**選択肢イベントのエンジンは未実装**。稼働しているのは `DialogueData.swift` の `innerVoice`（週頭の心の声・`Advice(name: "俺")` 固定）だけで、`event_design_v0.md` §2 の二択イベントはマスターデータ提案の段階（＝薄いのは欠陥でなく未実装／規律D）。よって本案は「既存の会話テーブル方式に載る形のデータ定義」として書く。

- **データ形**：`dialogue_design_v0.md` §6 の `(場面ID, 相性帯, 条件フック, 重み)` テーブル選択方式を踏襲し、選択肢と効果デルタの列を足した拡張とする（＝§6スキーマそのものではなく拡張と明記）。場面ID＝`ev_taniguchi_shortjob`。テキストはUI層リソース、GameCore/セッションは場面IDと状態を渡すだけ。
- **効果適用**：デルタ（money/stamina/compat/mental の増減）は既存の state 加算で表現可能（いずれも `GameState` 直フィールド、money=Int・stamina/メンタル/compat=Double）。**「3週間の相性成長停止」**は新しめの機構：`compatFreezeWeeks: Int` のような一時フラグをセッション層に置き、稽古の相性成長を3週間ゲートする案。これは既存に無い状態なので**新設**。ただし `lossStreak`/`justPassedStage` と同じく `WeekSummary` 系から導出・保持できる範囲で、**RNGは消費しない**設計に保つ。
- **話者切替**：`Advice.name` は現状 `"俺"` 固定（コメントに「将来のレアイベントで谷口等に切替」と予約済み・未実装）。谷口を話者として出すには UI 側に話者切替の口が要る（`dialogue_design` §6・既提案 0003 と同じ前提）。未実装＝欠陥ではないが、**本イベントは話者2名なので着手はその口が付いた後**（＝出荷ゲート）。

## リスク・注意

- **【二択のどちらが支配的か＝最大の未解決】** 一見Aが「金＋当座の安心」で強い。だが**金はバイトで後から稼げる資源**であり、Aの相性−1＋3週停止は取り返しにくい。ならば関係を重視する最適化プレイヤーには**B が安全な支配択**に化ける危険がある（＝逆方向の最適解化）。二律背反が成立するのは、**金欠帯（money<10万）では金が非代替的に切迫している**とき（バイトも同じ稽古時間を食い、生活費の支払いが待ってくれない）——だから **money<10万 ゲートが二律背反の生命線**。この均衡はコード上の compat の効き（大会成績への寄与度）に依存し、**sim 未検証**。`event_design` §4-B（イベント込みで優勝率1〜2%帯を再計測／恒久効果は控えめ＋通算15回予算）に従い、+15万の経済インパクトと合わせて実測で詰める。数値は全て【仮】。
- **【最適解化のリスク（金欠帯でA寄り）】** 逆に、compat の効きが弱ければ金欠帯で常にAが正解に寄る。数値で殺すのでなく、**Aの相性−1＋3週停止**を体感で効かせて釣り合わせる（体感の問題は演出・頻度で解く／規律B-5）。上記sim実測を見て最後に調整。
- **【二択維持】** 中間の折衷（例：一週間だけ受ける）は「金も少し・相性も損なわず」の**実質上位互換**に化けやすい。安易に third option を足すと二律背反が緩む。当面は鋭い二択で保つ。
- **【谷口の一滴の希釈】** 「そろばん弾いた顔、見えてたで」「たまにな」は谷口の乾いた一滴。これを他イベントに流用すると口癖天丼で死ぬ（「腹減ったな」年2回則と同原理／`dialogue_design` §6）。このイベント固有に留める。1プレイで露出するのは〈セットアップの一滴＋選んだ枝の一滴〉の2つまで＝密度上限。
- **【俺の声の規律】** 俺は全行標準語・段取り／計算の観測トーン。金欠を「惨め」と嘆かせない（自己憐憫＝寒い／0009）。Bの「順番が崩れるよりは、安い」は自嘲でも泣き言でもなく、事実の指示＝乾いた温度に保つ。
- **【固有プロップ】** 定番（天井の染み・冷めたコーヒー）を避け、**四つ折りの折り目が給料の桁を横切る求人票**と**消えずに残るホワイトボードのネタ順**をこの稽古場・この相方固有の物として使用。増産時に他イベントと重複させない。
- **【未実装前提】** 選択肢イベントエンジンも `compatFreezeWeeks` も話者切替も未実装。本書は提案であり既存ファイルは読取のみ。採用時は `dialogue_batch*` 同様に「笑」不使用・実在コンビ非想起・既存文非重複のセルフチェックを通す。ボケ（オチ/ネタ本文）は本文中に一切書いていない。

## レッドチーム（A/B）

### A. 実装妥当性

- **[A-1 破綻ルート・修正済]** v0は発火除外を「大会週・大会2週前」までしか見ておらず、firing時点で大会が3週先にある場合、選択肢Aの3週停止が**直近大会の追い込みを見えないところで食い潰す**隠れ罠になり得た。→ **除外条件に「今後3週以内に大会が無いこと」を追加**（freeze幅と一致させ、犠牲が不意打ちにならないようにした）。
- **[A-2 正解のない選択・未解決]** v0は「金欠帯でA支配」だけを警戒していたが、**金がバイトで代替可能／Aの相性損は代替困難**なため、逆に**B支配**の危険がある。二択が成立するのは money<10万 ゲートで金が非代替的に切迫している前提に限る。どちらへ倒れるかは compat の大会寄与度に依存し **sim 未検証**＝残課題（リスク節へ明記）。
- **[A-3 golden/乱数・確認]** 効果デルタは加算のみ＝RNG非消費で安全。危険なのは「ランダム抽選の挿入位置」と「compatFreezeWeeks の減算位置」の2点で、`WeekRunner` の消費順（injury `nextUniform` → rollOffer）に割り込むと3年golden破損。→ **2箇所とも【止めて相談】明記を維持**。
- **[A-4 MVP/本編・確認]** 発火条件が所持金・相性・週の多年非依存状態のみ＝**1年MVPで実働**。多年専用ではない。
- **[A-5 事実確認]** `state.money`(Int)・`compat`(Double, debug=19)・`Advice.name="俺"`固定で谷口切替は未実装、選択肢エンジン未実装——いずれもコードと一致。v0の「compat上限20」の断定は未確認だったため「debug で19を想定」に**トーンダウン**（event本体に非依存）。

### B. 会話の中身

- **[B-1 キザ・修正済]** 選択肢Aの谷口「鏡こっち向いても誰も映らんわ」は詩的に寄りすぎ（キザリスク／0009禁則）。→ 実務屋の谷口らしい**デッドパンで具体物**「ホワイトボード、消さんといたるわ」に差し替え（セットアップの“消えずに残るネタ順”へのコールバックにもなる）。
- **[B-2 ロジック不整合・修正済]** 選択肢B俺「今週飛ばすと」は3週間の仕事を1週に矮小化していた。→「三週間まとめて抜けたら、順番が崩れる」に修正。
- **[B-3 二択を溶かす台詞・修正済]** v0のB末尾「腐る前に一回、二人分の飯代くらいには使え」は、断ったはずの仕事を**部分受注する第三択を再導入**しており、自らのリスク節（third option禁止）と矛盾。かつ求人票を“飯代に使う”は物として非整合。→「腐らせとけ。順番が崩れるよりは、安い」に差し替え（二択を締める・「安い」は計算屋の語彙・求人票コールバック維持）。
- **[B-4 方言・話者・確認]** 谷口＝関西弁を全行で維持、他相方不在ゆえ相方依存の声の違反なし。会話は話者ごと1セリフ（"／"詰めなし）＝通常の会話送り。俺＝全行標準語・計算屋。
- **[B-5 コピペ/酷似・確認]** 火花ほか既存作品の台詞・言い回しの流用なし。ボケ（オチ/ネタ本文）なし。
- **[B-6 一滴の密度・確認]** 谷口の乾いた一滴は1プレイあたり〈セットアップ＋選んだ枝〉の2つに収まる（5-6回に1回則の範囲・固有留め）。
