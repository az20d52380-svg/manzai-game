# イベント会話提案「香盤表の一番上」v0（2026-07-06）

対バン形式のフリーライブ出演の直後に発火する、**選択肢のない一場面の相方会話**。`docs/event_design_v0.md` §B「舞台と劇場」#6〜12 の**会話化**にあたる。#6-12 は地の文＋数値パッチ中心で、相方の生きた声が薄い帯。ここに、この興行にしか無い固有プロップ＝**対バン楽屋の壁の香盤表**を1つ据え、俺と谷口の1セリフずつで「場数の底辺にいる手触り」を物に背負わせる。数値は全て【仮】、既存ファイルは読むだけ。

## 位置づけ（MVP か本編か・先に確定）

**本編用（機構前提）**。理由は §実装イメージ に詳述するが、**話者を切り替えて送る一場面会話のランタイムは現状未実装**（`DialogueData.reaction()` は「俺」単独の内心1文しか返さない）。よって本案は「素の現状に即載り」ではなく、**選択イベントの共通ディスパッチ／話者送りUIと同じ機構を前提とする**（`0010`・`0020` と同じ土俵）。MVP（1年48週・現ビルド）へ即投入できるという前提では**書かない**。

## 狙い（どの薄いスロットを埋めるか）

- **薄いスロット**: `event_design_v0.md` §B #6-12（舞台と劇場）。ここは「テキスト例＝地の文一行＋効果パッチ」で構成され、**相方の会話がほぼ無い**。劇場・舞台まわりの手触りが、数値通知に痩せている。
- **埋め方**: #6-12 と同じ「舞台と劇場」の質感を、**相方の会話**で補填する。近接コードキーは `reaction:t_フリーライブ`（実体は `ManzaiGame/Sources/DialogueData.swift:74`）。既存 #7「伝説の出待ち」のような**上向きの舞台イベントの裏**に、**上向きでも下向きでもない、ただ格が可視化されるだけの無味な一場面**を1本置く。
- **軸**: 出番順が可視化する『格』。対バンの香盤表で**俺たちの名前が一番上＝出番最初＝前座＝客席がまだ温まる前**。紙の一番上が、梯子の一番下。この反転を、**定番プロップ（客席の染み・冷めたコーヒー）でなく、この興行にしか存在しない手書きの順番表**に背負わせる（置換不能性・fable_kickoff §3-A）。
- **谷口の処理**: 順番の**意味**を一言も口にしない。出番順という事実は段取りの前提として口にするが、「前座」「格下」「一番下」といった**格の言語化はしない**。しゃがんで機材を先にまとめる**段取りだけ**を喋る（本音を言語化しない・§3-B）。俺は差引を数える標準語だが、**警句化しない**（「上から数えて〜」的なうまいこと言いを禁じる）。感情語ゼロ・**乾いた一滴も置かない無味ドラマ**（この一本は薬味を抜いて平常帯の平板さで通す・§6[平常/非見せ場]の水準）。

## 発火条件（実フック）

- **近接フック**: `reaction:t_フリーライブ`＝**フリーライブ出演の直後**。実体は行動選択 `t_フリーライブ` の解決後（`DialogueData.swift:74`）。
- **【要確認・母集合】対バン形式限定**: 香盤表は**複数組が並ぶ対バン興行に固有**。かつ現行の `t_フリーライブ` 内心は「客は少ないけど、場数だ。」＝**小箱・少人数**を含意する。本案の香盤表・「客席は埋まっていく」はこれと素直に噛み合う（少人数の対バンで、後の組ほど客が入る小箱）ので**言い回しは既存内心と整合させる**が、単独ライブ（対バンでない回）では物が浮く。**対バン形式のフリーライブ回に発火母集合を絞る**（フラグ or 会場種別）ことを実装時に確定。
- **状態フック**: **体力低め**（`stamina < 閾値【仮】`。`lowStamina` 判定で参照される `state.stamina` と同経路）。前座は入りが早く動きっぱなし＝疲労で発火帯に入る、という体感の物化。ハードゲートでなく発火重みに使う想定。
- **頻度・抽選**: 大会週を除く自由週の**既存イベント判定に相乗り**（`event_design` §頻度＝既定12%）。**新規の抽選ストリームは足さない**。`oncePerCareer` ではなく**キャリア数回まで**（cap = 2〜3【仮】）。世界の彩りとして散らす。
  - **【是正・overclaim 回避】** 「相乗り」でも、**新イベントをプールに1枚足せば、同じ乱数値で選ばれる当選イベントが変わり得る**＝下流state・golden に影響し得る（`0015` §同旨・`EVENT_FIRE_CAP` 予算内の枠競合）。**イベント枠を1つ増やす＝分布に触る可能性**として実装時に相談扱い（下記リスク）。
- **紐付けない状態（明示）**: 連敗（`lossStreak`）・直近敗退（`justLost`）・直近通過（`justPassed`）には**紐付けない**。勝ち負けと無関係に「香盤の一番上にいる時期がある」を世界の質感として置くため。
  - **【重複回避・要確認】** `event_design_v0.md §B #10「対バンの化け物」`（大会敗退の翌週・同じ対バン楽屋が舞台）と**設定が近接**。本案は #10 と違い**勝敗に紐付けず・悔しさ→創作の変換もしない**（格の可視化のみ・無味）ことで差別化する。実装時に**同一週で #10 と二重発火しない**よう排他を確認。
- **効果**: **基本なし（フレーバーのみ）を推奨**。付けるなら【仮】メンタル微少に留め、**数値はセリフで言わない**。連敗・justに紐付けない世界の彩り＝恒久能力+0で分布に効かせない（安全側）。
- **【設計上の懸念・要目視】** 「前座＝一番上」は**低知名度期に最も映える**。`fame` を発火条件にしていないため、`fame` が伸びた中盤以降にも出ると「もう前座じゃないのに一番上」の齟齬が出る。対策候補: (a) 低fame帯に発火重みを寄せる、(b) キャリア早期にcapを寄せる、(c) 本編では香盤位置が下がる演出に将来拡張（今回スコープ外）。simでは測れない＝ simulator 目視で決める（CLAUDE §10）。

## どこから学んだか（型の出典）

- **紙の位置＝格のメーター（物that語る）**: 順番・席次という「並び順の物」に格差を背負わせ、感情語で説明しない手法。人が抜けた事実を残された物で置く `0015` の芯（座面の型・剥がしたガムテ跡）と同系（fable_kickoff §3-A / proposals/0009）。
- **下積みの非対称＝二つの事実の並置**: 「一番上の俺たち」と「一番下＝先週テレビの組」を感想なしで隣に置き、差は受け手に測らせる（下積み群像を扱う既存小説等の**世界の構造のみ**抽出・**声も本文も一切移植しない**／fable_kickoff §4-5）。※参照作品の本文・セリフは転載せず、URL 等の直接リンクも張らない（型の抽出のみ）。
- **相方は本音を言語化せず所作で置く**: 谷口は格の意味を口にせず「機材を先に巻く段取り」だけ喋る＝向田型「所作だけ書く・副詞で湿らせない」（fable_kickoff §4-3 / §3-B）。
- **無味で通す（見せ場の8割は平板が正解）**: 乾いた一滴を敢えて置かず、平常帯の平板さで通す（fable_kickoff §6[平常/非見せ場]・§3-Aの1/6頻度の"分母を作る側"）。

学んだのは構造・型のみ。既存作品の本文・セリフは一切引かない。ボケ/フリ/オチ本文ゼロ。

## 実装イメージ（既存の会話機構への載せ方）

- **【是正・現状前提】** `DialogueData.reaction(variantID:)` は **「俺」単独の内心1文**を返すだけ（`DialogueData.swift:66-77`）。`Advice.name` は常に「俺」で、**話者を切り替えて送る汎用ランタイムは存在しない**（谷口のセリフは `WinFinaleView`／`YearResultView` 等に**個別ハードコード**されているのみ）。選択イベントのランタイムも未実装（`0020` 確認済）。**したがって本案の "俺→谷口→俺" の一場面会話は、データだけでは載らない。話者送りUI＝新規機構が要る。**（元下書きの「新システム不要・機構を待たずに載る」は**撤回**）。
- **載せ方（推奨）**: 選択イベント機構が入るタイミングで、その**話者送りフレームを非選択（分岐なし）で流用**する。構成は「セットアップ地の文（俺視点）＋話者2名の1セリフ」だけ＝選択肢を持たない最小ケース。テキストは全て UI 層リソース（`DialogueData.swift` と同じ層）。
- **相性分岐**: 発火時の `compat` を 0-7／8-14／15-20 で判定し、ツーカー帯のみ地の文一行を差し替える（`0015` と同型・データ差し替えのみ）。
- **相方分岐**: 谷口版（関西弁）／他相方版（標準語）を別プールに持ち、話者に応じて選択（fable_kickoff §5-E の別プール運用）。
- **効果**: フレーバーのみ推奨＝**効果パッチなし**。付けるなら既存 `GameEngine` の clamp 済み加算経路（メンタル微少・【仮】）のみで、新経路不要。
- **【RNG・golden・乱数消費順＝止めて相談】** 発火を**新規の抽選ストリームで引く**なら、どのストリームから・どの順で引くかは**乱数消費順の正典に触れる**＝ golden が壊れる。**必ず止めて相談**（`tools/*.py` 正典 → GameCore → `gen_golden.py` → `swift test` green の1コミット同期／CLAUDE §A-1,2）。**回避策の推奨**: 新規抽選を足さず、(a) `t_フリーライブ` 解決時に**既に消費している乱数/状態から決定的に**分岐させる、または (b) 既存のイベント抽選ストリームに相乗りして**新規drawを増やさない**。ただし (b) でも**イベント枠を1枚増やすと当選イベントが変わり得る**（頻度欄の是正）＝**枠追加自体を相談**。決定的に載せられれば消費順不変＝ golden 無傷。効果パッチを付けない限り数式・`sim_career.py` 分布にも効かない。

## リスク・注意

- **【要相談・正典】** 発火に新規乱数drawを足すと消費順が変わり golden 破壊＝止めて相談。既存ストリーム相乗りでも**イベント枠追加＝分布に触る可能性**あり（決定的分岐が最も安全）。
- **【要相談・話者機構】** 二話者送りのランタイムは**未実装**。本案は選択イベント機構と同じ土俵＝**機構ごと相談**（`0010`/`0020` と同前提）。素の現状ビルドには載らない。
- **【対バン固有プロップの成立条件】** 香盤表は対バン興行固有。単独ライブ回に出すと物が浮く＝**対バン形式回に限定**し、現行内心「客は少ない」と整合させる。実装時に発火母集合を確定。
- **【#10 との重複】** `event_design_v0.md §B #10「対バンの化け物」`と同じ対バン楽屋が舞台。**勝敗非紐付け・格の可視化のみ**で差別化するが、同一週の二重発火排他を確認。
- **【前座設定の賞味期限】** `fame` を条件にしないため中盤以降に出ると「もう前座じゃないのに一番上」の齟齬。低fame帯weight or 早期capで回避（発火条件の設計懸念）。
- **【一滴なし＝無味に振った意図の確認】** 本案は狙い通り**乾いた一滴を置かない**。寒くはないが**平板すぎて素通りされないか**は体感依存＝ simulator 目視で判定（fable_kickoff §6の"見せ場2割/平板8割"の後者側として意図的）。効かないと感じたら**数値でなくセットアップ地の文の固有具体（一番下＝先週テレビの組）**で温度を足し、薬味は最後（0009・§7-B）。
- **谷口が段取りだけ喋る設計の徹底**: 谷口・相方とも**格の意味を口にしない**。出番順は段取りの前提としてのみ触れる。改稿時に「谷口が前座/格下を言語化する」方向へ流すと本音の言語化＝設計違反（§3-B）。
- **俺の警句化禁止**: 俺のセリフ「一番上だ。〜客席は埋まっていく」は事実の並置で格を出す。改稿で「上から一番、下から最後」的な**うまいこと言い（キザ）**へ寄せない（§3-B / チェックリスト）。
- **固有名・数値・コピペ**: 主催名・他の出演コンビ名は**名指ししない**（架空・匿名で温度を保つ）。数値は全て【仮】・**セリフで数値を言わない**。ボケ/フリ/オチ本文ゼロ。既存作品セリフ流用なし（下積み群像の世界構造のみ抽出・声は非移植・直接リンクも張らない）。

**トーン自己点検**: 感情語（悲しい/悔しい/惨め/嬉しい）不使用。感情は物＝「一番上に並ぶ手書きの名前／下へ行くほど見覚えのある名前／先週テレビの組／段取りでケーブルを巻く谷口」に背負わせた。固有プロップ＝定番（染み・コーヒー・小銭）でなく**この対バン興行の手書き香盤表**1つ。俺＝標準語・差引（「一番上だ」「客席は埋まっていく」）・自己憐憫もキザもなし。谷口＝関西弁一貫・段取りのみ・格の意味は非言語化。相方版＝標準語（関西弁を移さない）。**乾いた一滴なし**（無味ドラマとして意図的）。会話は話者ごと1セリフ・「／」詰めなし＝通常の会話送り準拠。

## レッドチーム(A/B)

### A. 実装妥当性
- **A-1【重大・是正済】話者フレーム overclaim**: 下書きは「既存の話者フレームに載る・新システム不要・機構を待たずに載る」と主張。実体は `DialogueData.reaction()` が**「俺」単独の内心1文**を返すのみで、`Advice.name` は常に「俺」。谷口のセリフは各 View に個別ハードコードされているだけで、**汎用の二話者送りランタイムは無い**（sibling `0020` も「話者切替UI未実装」と明記）。→ 位置づけを**本編用（機構前提）**へ格下げし、実装イメージ冒頭に是正を明記。
- **A-2【是正済】ランダム発火 vs 新規draw禁止の内部矛盾**: 「週ランダム・cap 2-3」は本質的にRNG drawを要し、下書き自身の「新規drawを足すな」と衝突。→ **既存イベント判定（§頻度12%）へ相乗り**に統一。さらに「相乗りでも**イベント枠追加は当選イベントを変え得る＝分布に触る可能性**」を `0015` の honesty に倣って明記（overclaim 回避）。
- **A-3【是正済】#10 との近接**: `event_design_v0.md §B #10「対バンの化け物」`が同じ対バン楽屋。→ 勝敗非紐付け・格の可視化のみで差別化する旨と、**同一週二重発火の排他**をリスクに追加。
- **A-4【是正済】フリーライブ=対バんか＋客数の食い違い**: `t_フリーライブ` 内心「客は少ない」と下書きの「客席は埋まっていく」の整合を取り、**対バン形式回に母集合を限定**する要確認を強化。
- **A-5【問題なし】golden/数式/乱数消費順**: 下書きは「新規draw＝止めて相談」「決定的分岐 or 既存相乗り」を正しく明記。数式・`sim_career.py` にも効果パッチ無しなら不干渉。この骨子は維持。

### B. 会話の中身
- **B-1【問題なし・維持】ドラマ主軸/物で語る**: 感情語ゼロ。格差を「手書き香盤表・一番上/一番下・先週テレビの組」の物と事実の並置に転写。固有プロップは定番回避で置換不能。
- **B-2【要監視・注記追加】谷口の格・言語化**: 「俺ら先やからな」は出番順を**段取りの前提**として口にするのみで、「前座/格下」の意味は言わない＝§3-B 合格。ただし改稿で意味言語化へ流れやすい境界線につき、狙い・リスク両方に「格の意味は非言語化」を明示追記。
- **B-3【問題なし】俺の声**: 「一番上だ。俺たちが終わってから、客席は埋まっていく」はデッドパンの事実並置。自己憐憫でも警句（キザ）でもない。「紙の一番上＝梯子の一番下」の警句は**地の文にも出さず**受け手に測らせる設計を維持。
