# 年次リザルト独白の細分 v0 — 4〜7年目「停滞レジーム」に固有の声を与える提案

- 日付: 2026-07-05（レッドチームA/B反映: 2026-07-06） / 提案のみ（既存ファイル未編集・読取のみ）
- 対象: `dialogue_design_v0.md` §5「年次リザルトのモノローグ」（L99-104）を**置換でなく内包拡張**。`fun_flow_review_v0.md` §1/§2/§3 が名指しした谷（4〜7年目・停滞層＝分散63%/のんびり74%が9年目まで決勝未到達）に、テキストだけで階段を刻む（規律B-5＝数値は動かさない）。
- 声: 主人公「俺」＝標準語・計算屋・過小申告。基調はデッドパン（＝英国式 deadpan／過小申告）＋温度差実況。谷口は接続点で事実1-2個だけ。
- **スコープ明示**: 本書は**本編（10年キャリア）**の機能提案。MVP は1年完結（`GameSession.year = 1`・S6リザルト→タイトル）で年次リザルト独白の画面枠自体が未実装＝**薄いのは欠陥でなく未実装**（規律D・MVP前提）。着手は多年キャリア基盤の後。

## 狙い

現状 §5 の独白は成績のみの4分岐（躍進/停滞/貧乏/9年目＝L101-104の4行）で、**4年目の停滞と7年目の停滞が同じ「何もなかった年と書くのは簡単だ」文（L102）に落ちる**。年に一度の感情的総括が、最も回数を踏まれる分岐（停滞）で最も薄い。

解は新イベントでなく「同じ暦の再解釈」。カレンダーは3年目で暗記済みだからこそ、独白は「今年何が起きたか」ではなく、**去年より一年ぶんだけ深くなった俺が、同じ壁をどう名付け直すか**を語る。停滞に「意味の階段」（naive→命名→家具化→予兆）を持たせ、年替わりフレーバー🎯を独白側で解消する。

## 対象箇所

- 主軸: **年数band（2-3 / 4-5 / 6-7 / 8-9）× 成績（躍進/停滞/貧乏・後退）**。中だるみの主戦場は 4-5／6-7 band。
- 修飾: **同一壁連敗数（2連 / 3連=事件 / 4+連）** を1行suffixで溶接（`fun_flow_review` §3の最終年ラダー方式を流用し、在庫を掛け算でなく足し算に抑える）。
- 分岐は乱数でなく**状態キー**で引く（`dialogue_design` §6のテーブル選択方式＝パワプロ流の状態分岐を踏襲）。
- 実効本数（実カウント）: **停滞コア6 ＋ 連敗suffix3 ＋ 躍進3 ＋ 貧乏2 ＋ 後退1 ＋ 接続2 ＝ 独白17行 ＋ 谷口接続1行**。コア×suffixの溶接で実効は数十パターン。`fun_flow` §3の250行枠に十分収まる。

### 【実装依存】必要な新規状態（本編・全て要新設）

**現行 `GameSession.lossStreak` を流用してはならない。** 現行の `lossStreak` は `GameSession.pump`（L143-157）で `summary.results` から**汎用の連続敗退数**として算出され、**どの回戦であっても通過1回でリセット**される（L150 `if r.passed { lossStreak = 0 }`）。本提案の「同一壁◯連敗」とは意味が違う。着手時に以下3種を新設する:

1. **同一壁連敗カウンタ**＝「直近で跳ね返された回戦ID」＋「その回戦での連続敗退数」（`lossStreak` とは別カウンタ）。
2. **最高到達回戦**（自己ベスト・high-water mark。band内の純/更新/後退の判定に使う）。
3. **年数band**（現在年 → 2-3/4-5/6-7/8-9/最終年）。

いずれも `WeekSummary.results`（`isStage`/`passed`/回戦名）から**導出可能で、新規の乱数抽選を伴わない**。現行 `lossStreak` と同様に**UI/セッション層でRNG非消費に保持**すること。**WeekRunner の RNG 消費経路には絶対に入れない**（injury抽選・rollOffer の順序に割り込むと3年ビット一致 golden が壊れる＝規律A-2）。ここに触れる設計変更が要るなら**手を止めてオーナーに相談**。

## どこから学んだか（技術＋出典URL）

- **反復の階段化（same but different）**: 反復ビートは内容が同じでも「質が逓増しない」と読者は忍耐を失う。各失敗はキャラの何かを露呈させる。→ 4年目と7年目が同一文なのはこの原則の教科書的違反。 https://www.septembercfawkes.com/2025/12/writing-try-fail-cycles-strengths-vs-weaknesses.html
- **たるむ中盤（sagging middle）はプロは「新事件」でなく「同じ事件の意味の更新」で埋める**。→ 4-7年目はまさに物語論のsagging middle。 http://scribemeetsworld.com/2016/writing-tips/midpoint-solution/
- **天丼＝反復で感情を増幅（間をあける天丼／最大3回／4回目で反転／3回後は"すかし"で外す）**。→ 様式文（「〜と書くのは簡単だ」）は天丼扱いで10年に最大3回・状況を変えて。3連敗年に様式を裏切る一行。 https://owa-rai.com/tendon/ ／ https://humorsense.s-teem.com/2012/03/post_78.html
- **デッドパン／言葉の経済**: 感情語を足さず事実1-2個を過小申告で置くほど効く。谷の底（6-7年目）は最も削る。 https://satire.info/deadpan-delivery-the-art-of-the-unflinching-satirical-face/ ／ https://www.thewriteflourish.com.au/posts/31-humour-techniques-part-3-exaggeration-and-understatement
- **1テキスト＝技法1つ（最大の禁物＝盛り）**: `writing_craft_and_appeal_v0` §2 運用ルール1。複数技法を盛ると「面白がってる感」で寒くなる。本改稿は各行を単一技法タグに正した。 （社内正典）
- **自虐サムさの境界**: 貧乏年を「金がなくて惨め」と直接嘆くと凍る。財産の言い換えに留める既存方針が正しい。 https://gakumado.mynavi.jp/freshers/articles/12335
- **題材の実相**: 同じ壁への連敗は実在の共感軸（準決の壁に10年）。敗因は単一でなく複合＝独白を「断定」でなく「言語化できない一歩」に。 https://ja.wikipedia.org/wiki/M-1グランプリ ／ https://note.com/laughlovem1/n/nd32ee07f3b92

## リスク・注意

- **様式文の天丼上限**: 現行の決め型「〜と書くのは簡単だ」（L102）は天丼3回則に従い**10年で最大3回・状況を変えて**。谷口の口癖「腹減ったな」（§6・年2回まで）と同じ原理で、独白の決め型も希釈で死ぬ。上表は様式文を意図的に避けて再構成してある。
- **説明しすぎ＝サムさ／規律B-5**: 「GP3回戦で〜点足りず」等、数値・固有名を独白に持ち込んだ瞬間に内輪化して凍る。事実1-2個の過小申告に固定する。
- **suffix合成の破綻回避**: コア本文とsuffixが二重に「壁を名付ける」と天丼過多になる。本改稿では**連敗suffix3本を全て非比喩（過小申告/自嘲/すかし直喩）に固定**し、擬人化コア（停滞_4-5_純／6-7_純）に二層目のメタファーを積まない設計にした。連敗_3（すかし）を差す年は、コア側の擬人化を弱める分岐制御を推奨（同一場面の直近使用ID回避＝§6の審査員コメント機構を共用）。
- **成績キーの一意分類**: 躍進／停滞_更新／後退が曖昧に重ならないよう、`(今年最高到達 − 自己ベスト)` と `(今年 − 去年)` の符号で**年1キーに一意分類**（後退＜0 → 後退年／更新＞0 → 躍進 or 停滞_更新／横ばい → 停滞_純）。分類後にband分岐。1年につき独白コアは必ず1本。
- **接続の整合**: 最終年_接続・9年目_既存維持は `lastyear_calibration` §6「二種の奇跡」と発火条件が競合しないか要確認（オーナー判断待ちの最終年ラダー着火点・START_HERE該当節）。ここは勝手に確定しない。
- **未実装前提**: §5は現状4行のみ＝薄いのは欠陥でなく未実装。本書は提案であり、採用時は `dialogue_batch*` と同様に「笑」の字不使用・実在コンビ非想起・既存文非重複のセルフチェックを通すこと。

## レッドチーム（A/B）

辛口の自己攻撃と、本文に反映した対処。**［修正済］**は本改稿で直したもの、**［保留］**は仕様・オーナー判断待ち。

### A. 実装妥当性

- **A-1［修正済・重要］同一壁連敗を既存 `lossStreak` と混同していた**。`GameSession.lossStreak` は `pump`（L143-157）で `summary.results` から算出される**汎用連続敗退数**で、**どの回戦でも通過1回でリセット**（L150）。本提案の「同一壁◯連敗」とは別物。→「必要な新規状態」節を新設し、①同一壁連敗カウンタ（回戦ID＋連続数）②最高到達回戦③年数band を**要新設**と明記。既存 `lossStreak` 流用は禁止と釘刺し。
- **A-2［修正済］golden・乱数消費順への波及リスクを未言及だった**。新規3状態は `WeekSummary.results` から**RNG非消費で導出可能**（現行 `lossStreak` と同じくセッション層算出）。→「WeekRunner の RNG 経路に入れない・触るなら止めて相談」（規律A-2）を明記。数式・golden には**触れない**提案であることを確定。
- **A-3［修正済］§6スキーマの誤引用**。旧稿は §6 を `(場面ID, 状態キー, 条件フック, 技法タグ, 本文)` と記載したが、実体（L108）は `(場面ID, 相性帯, 条件フック, 重み)`。→「選定方式を踏襲した**拡張**（相性帯→状態キー、技法タグ・本文列を追加）」と正した。正典の記述精度を守る（規律C-6/7）。
- **A-4［修正済］場面の帰属が曖昧だった**。`year_result_mono` は年末（S4相当）の**年1回場面**で、`DialogueData.innerVoice` の週頭優先度ラダーとは別枠＝競合しないと明記。
- **A-5［修正済］自己申告の本数が実カウントと不一致**（旧「コア8＋…＝約20行」）。実測は**コア6＋suffix3＋躍進3＋貧乏2＋後退1＋接続2＝独白17＋谷口1**。実数に訂正。
- **A-6［修正済］成績キーの分類基準が未定義**で躍進/停滞_更新/後退が重なり得た。→ 差分符号による一意分類ルールを追記。
- **A-7［保留］最終年_接続 × `lastyear_calibration` §6「二種の奇跡」の発火競合**。オーナー判断待ち（START_HERE該当節）＝セッションで確定しない。unresolved に送る。

### B. 会話の中身

- **B-1［修正済・重要］谷口接続が既存 `dialogue_design` §4 L84 のほぼ複製だった**。旧「壁の方が俺らの顔覚えた／覚えられてるうちに越える」は §4 予選敗退の掛け合い（「壁の方が俺らに慣れてきてんちゃう？／なら、慣れられてるうちに越えるぞ」）と酷似＝**社内既存文の自己コピペ**。→ 別角度（谷口が舞台裏の段差を暗記＝悔しさを言語化しない温度差実況）に全面書き直し。
- **B-2［修正済］躍進_早期が既存 §5 L101 のほぼ複製**（旧「空気が薄い。……悪くない薄さだ」≒ L101「空気が薄い。……いい薄さだ」）。加えて「空気が薄い」が旧稿で躍進_早期・停滞_6-7_更新・既存L101の**3箇所に重複**＝意図せぬ天丼（本提案が戒める same-but-different 違反）。→ 躍進_早期から「空気が薄い」を除去し別イメージ（高所の足元の不安定さ）に。停滞_6-7_更新からも除去。「薄さ」モチーフは躍進系の署名として1箇所に集約。**移行注記**: 旧 L101（躍進年）は band分割に吸収され退役（正典v移行の履歴として残す前提）。
- **B-3［修正済］「1テキスト＝技法1つ」（writing_craft §2 運用ルール1＝最大の禁物）違反**。旧稿は多くの行が2技法タグ（「擬人化/ズレた冷静さ」等）で内容も2イメージを盛っていた。→ 全行を**単一技法**に再タグ＆本文を剪定（例: 停滞_4-5_純 は末尾「……それはたぶん、良くない」を削り擬人化1本に）。連敗suffixは全て非比喩に統一しコアとの二層メタファーを禁じた。
- **B-4［修正済］「谷の底＝最も削る」ラベルと本文が矛盾**。旧 停滞_6-7_純 は4文・2メタファー（家具化＋名前がない）で**batch中もっとも長く重い**行だった（デッドパン/言葉の経済の原則に自己違反）。→ 2文・擬人化1本に剪定し、実際に最もスパースな行にした。
