# イベント会話提案「所属欄は空」v1（2026-07-06・非選択／予選通過直後スロット増補）

予選通過直後の相方会話は、現状 **汎用1種のみ**（`docs/dialogue_design_v0.md` §予選通過（汎用）＝谷口「名前あった！ 俺らの名前あった！」＋手震え）。全プレイヤーが毎年必ず引く最頻スロットなのに、知名度帯による書き分けが無い。本案はここに **知名度ゼロ帯（fame<20）専用の一場面** を差す。効果は無し（会話フレーバーのみ）。既存ファイルは読むだけ・数値は全て【仮】。

これは **選択肢イベント（0010-0023・パワプロ型 A/B 効果）ではない**。判断を迫らず、大会会場の余韻に留める非選択の会話。0018（`justPassed` の勢い換金・翌週の選択イベント）とは別物（棲み分けはリスク欄）。

> **【出典の是正・v1】** 初稿は薄さの根拠を `content_inventory_plan_v0.md` §3-2 に帰していたが誤り。§3-2 は「日常雑談（帯別）」で本スロットと無関係。同書が「回戦別**通過**プール未解消」と言うのは **register #41／§6-1 の紙講評（審査員の書き講評）**であり、**谷口の相方会話スロットとは別物**（`content_inventory` L31・L167）。本案は紙講評の在庫欠落を埋める提案では**ない**。埋めるのは `dialogue_design_v0` §予選通過（汎用）の**汎用1種しかない**という薄さ。混同を撤回する。

## 狙い（どの薄いスロットを埋めるか）

- **薄いスロット**＝予選通過直後の**相方会話**（`dialogue_design_v0` §予選通過（汎用））。既存は汎用1種＝谷口の高揚（「名前あった！」）＋俺のツッコミ（手震え指摘）。**温度が「勝った側の弾み」一色**で、知名度帯による書き分けが無い。
- **埋める温度**＝「通ったが、誰も俺らを知らない」。通過は事実、しかし場数の底での通過だから、高揚を叫ぶより **会場に残された物** に温度を預けるほうが沁みる。汎用版の高揚と非重複の、静かな一枚を fame<20 帯に置く。
- **固有プロップ＝通過者一覧の紙の「所属」欄**。俺たちの名前の隣、所属欄だけ白い（他の組の行には事務所名が刷ってある）。空席（0015）・空欄（0009 サンプル）と同型の「物that語る」だが、**このプロップは他コンビでは成立しない**（＝知名度ゼロで通過した、この二人のこの紙にしか無い空白）。§3-A の置換不能性を満たす。定番プロップ（天井の染み・冷めたコーヒー・小銭）に逃げていない。

## 発火条件（実フック）

- **状態フック（概念スロット）**：予選通過が確定した直後の相方会話スロット。**コード上の直接の識別子は無い**（`verdict.pass` は本案の仮称）。対応する実フックは **`GameSession.justPassedStage == true`**（`GameSession.swift:31`。`pump()` で `r.passed==true` の週に立つ＝`:150`）を読む **S3 結果画面側**。× **`fame < 20`**（知名度ゼロ帯）。
- **ステージ限定**：**予選ステージのみ**（3回戦〜準々の通過も可、準決通過＝決勝進出は別スロット・別温度）。【要確認】下記リスク欄参照＝`justPassedStage` は bool で**回戦種別を持たない**可能性。予選限定ゲートは結果画面側が回戦/ステージ種別を参照できることが前提。
- **相性帯**（任意・湿り気だけ差し替え）：0-7 他人行儀／8-14／15-20 ツーカー。数値は動かさず、下記プールの語尾トーンのみ分岐可（seed では核1本＋補1本で足りる）。
- **棲み分け（重要）**：
  - **`passedLines`**（`DialogueData.swift:54-57`＝俺の週次 innerVoice・`justPassed` ゲートで**翌週**に出る余韻・独白）とは別物。あちらは俺の独白、これは **結果直後の相方会話**（category②）。
  - fame **≥20** では既存の汎用高揚版（「名前あった！」）を、**<20 でこの静かな版**を出す（同一スロットの fame 排他）。**ただし両版とも現状コード未実装**＝排他は「相方会話機構が入った時に fame ゲートで2プールを分ける」設計（下記【機構前提】）。
- **【機構前提の注意】**：現状 `DialogueData` は **俺の独白のみ実装**（ヘッダ明記＝「谷口/先輩の会話は将来レアイベントで」）。谷口相方会話を結果直後に差す機構は **未実装**。汎用高揚版（「名前あった！」）も `dialogue_design_v0` の設計行に留まりコード未載。本案はその機構が入った時に載せる1プールの提案（実装イメージ参照）。

## どこから学んだか（型の出典・URL）

- **感情を物へ転写（氷山理論・言い落とし）**：通過の重みを「嬉しい」と書かず、**折らないように運ぶ所作・白い罫線** に預ける。カーヴァー／志賀直哉の型（`0009` の芯「感情を書くな、場所の物を書け」）。https://ja.wikipedia.org/wiki/城の崎にて
- **「空いた欄」＝メーターとしての物**：人・肩書きの不在を、感情でなく残された空白で置く。0015「空いた席」・0009 サンプル「宛名は、まだ書いていない」と同型だが、**プロップは別**（通過者一覧の所属欄）。https://toyokeizai.net/articles/-/717910
- **下積みの非対称を事実の並置で**：売れている側（事務所名が刷られた行）と俺たち（白い行）を、感想なしで隣に並べて差を読者に測らせる。火花の**世界だけ**抽出、声（「僕」・関西の思弁・比喩の連射）は移植しない。https://ja.wikipedia.org/wiki/火花_(小説)
- **乾いた一滴の位置**：痛みの直後でなく「谷を通り過ぎた平地」に置く（ルシア・ベルリン）。本案の一滴＝「数えるほど暇じゃなかったけど、数えた」を、空欄を嘆く位置でなく **鞄に入れる谷口の横** に置いて自嘲へ着地。

## 実装イメージ（既存機構への載せ方）

- **載せ先**：予選通過結果（S3 結果画面）の **相方会話プール**。現状 `DialogueData` は俺の独白のみ（`innerVoice` の `passedLines` 等）で、谷口の相方会話は「将来レアイベント」として未実装（ファイル冒頭コメント）。本案は **その相方会話機構が入った時の1プール**（`予選通過 × fame<20` 分岐）。
- **fame 分岐の下地は既にある**：既存 `passedLines` は `pick(passedLines, salt: Int(state.fame))`（`DialogueData.swift:33`）と **fame を salt に使っている**。よって「fame でプールを差し替える」動線は自然で、`fame < 20 ? fame0Pool : genericPool` の分岐を足すだけ（相方会話側も同型の fame ゲートで分岐可）。
- **効果**：**無し**（純フレーバー）。GameState の増減パッチを持たない。
- **golden／乱数消費順**：本案は **数式も乱数消費も変えない**。会話 `pick` は既存の決定的 salt-pick（`DialogueData.swift:84-86`／新 RNG ストリーム不要）で、golden に影響しない見込み。
  - **ただし**「予選通過の結果直後に相方会話を1枚差し込む」演出フックを `WeekRunner`／`runYear` の進行に **新規に挿入** する場合、そのフックが乱数を引く位置次第で消費順（＝正典）に触れる可能性がある。**その場合は必ず止めて相談**（`tools/*.py` 正典 → GameCore → `gen_golden.py` → `swift test` green を1コミット同期／CLAUDE §A-1,2）。会話 pick を既存の salt（決定的・非RNG）で済ませる限りは触れない。
- **相性帯分岐**：発火時 `state.compat` を 0-7／8-14／15-20 で判定してプール差し替えのみ（数値は動かさない）。

## リスク・注意

- **【要確認・回戦種別】予選限定ゲートの実現可否**。`justPassedStage` は bool で回戦（1回戦／準々／準決）を持たない可能性が高い。「予選のみ／準決通過は別スロット」を実装するには、結果画面が **回戦・ステージ種別を参照できること**が前提。参照経路が無ければ、ゲートは「通過全般」に広がる＝準決通過（決勝進出）でも本案の静かな版が出て温度が齟齬する。**機構実装時に回戦種別の可視性を先に確認**。
- **【天丼・温度衝突】汎用「名前あった！」版との二重発火**。同一スロットに高揚版と静穏版が両方乗ると温度がぶつかる。**fame で排他**（≥20＝高揚版／<20＝本案）を実装時に明示。両版とも現状未実装なので、排他は「2プールを同時に実装して fame でゲート」する形。fame 境界（20）は仮＝simでは測れない体感値なので simulator 目視で調整（CLAUDE §10）。
- **【0018 との棲み分け】**。0018「通った日の分かれ道」（`justPassed` の勢い換金・A/B 選択）と本案は **別カテゴリ**。0018 は**翌週**に判断を迫る選択イベント、本案は**結果直後**の判断を迫らない相方会話。同一週に両方出さない（本案＝S3結果画面側、0018＝翌週の自由週イベント抽選側で自然に分離するが、実装時に発火タイミングの重複が無いか確認）。
- **【未実装機構への提案】**。谷口相方会話を結果直後に差す機構自体が未実装。本案採用には、その差し込み動線（S3結果画面に相方会話プールを1枚出す）の実装判断が先。動線に RNG を噛ませるなら消費順＝要相談（上記）。
- **固有名・数値**：所属欄に入る他組の事務所名は **本文で名指ししない**（「事務所の名前が刷ってある」までで匿名・架空を保つ）。効果は無し・数値は全て【仮】。会話でゲーム内軸名（華/センス等）も数値も口に出さない。
- **register 監視**：他相方版の俺「うちの下の行だ」は安堵の暗示だが物（紙の位置）へ逃がしてあり、感情語なし。実機で湿れば谷口版の deadpan（「数えた」）側へ寄せる。

**トーン自己点検**：感情語（嬉しい/惨め等）不使用（初稿の「ほっとした」を v1 で除去）。感情は物＝「白いまま通った罫線／折らないように運ぶ谷口／たった一枚の予備／同じ空欄がうちの下の行にある」に背負わせた。固有プロップ＝定番でなく「知名度ゼロで通過した、この二人のこの紙の所属欄」1つのみ。谷口＝関西弁一貫・**事務所名を茶化さず紙の折り目と予備だけ気にする**（本音を言語化せず所作＝§3-B）。俺＝標準語・計算屋・ツッコミ体質（「数えた」）で自己憐憫もキザも無し。乾いた一滴＝「数えるほど暇じゃなかったけど、数えた」を **1プール1滴**に抑制（初稿の補2・第2滴を除去）。会話は話者ごと1セリフ・「／」詰めなし＝通常のタップ送り。ボケ（フリ/オチ/パンチライン）本文ゼロ、笑いは地の文の乾き（空欄・一枚の予備）にのみ。
