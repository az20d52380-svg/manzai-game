# イベント会話提案 0025「名前の無い予約票」v0（2026-07-06）

event_design_v0.md #19『相方の隠れ稽古』（設計テーブル上の効果＝相性+2・表現+1／通知文「終電後の公園で、相方が一人で立ち稽古をしていた。声はかけずに帰った。」）を、**ドラマ主軸で"会話化"**する。#19 は設計テーブルの一行として存在するだけで、**状態フック駆動の谷口フレーバー会話としては未実装**（Event/会話シーンのランタイムが `ManzaiGame/Sources/` に無い＝0015・0017 の指摘と同じ前提）。ここを、固有プロップ＝稽古場の予約票を軸にした**突発の一場面**で埋める。数値は全て【仮】、既存ファイルは読むだけ・コード改変なし。

## 狙い（どの薄いスロットを埋めるか）

- **薄いスロット＝「状態フック駆動の谷口フレーバー会話」がほぼゼロ**。`DialogueData.swift`（確認済）は現状ほぼ全て「俺」の独白（`innerVoice` ラダー＋`reaction`）で、`Advice.name` に谷口を入れる口はあるが実データは無く、谷口の発話は将来のレアイベント送り（ファイル冒頭コメント2-3行目で明言）。#19 はその題材の筆頭なのに設計テーブルの一行のまま。
- **#19 は『谷口の本音を言語化しない』原則（fable_kickoff §3-B）に最も嵌る題材**。谷口が一人でやっている痕跡＝見せない繊細さ。これを**事実一つ**で置く。
- **なぜ weeksLeft 2-5 か**：大会が近い"詰め期"だが前夜（weeksLeft≤2 の preTournament ラダー）ではない帯。谷口が一人で枠を取る動機（間に合わせたい）が最も自然に立ち、かつ週頭独白と衝突しにくい。
- **谷口所有時のみ**にゲートする理由：この繊細さは谷口の関西弁の声込みで成立する。標準語の別相方に同じ台詞を移すと register が汚染される（fable_kickoff §2-5）。＝**声のロックを守るための発火ゲート**でもある。

## 発火条件（実フック）

全て**決定的フラグ**で判定（＝新規RNGを引かない。0024ブリーフ「なぜgoldenがビット不変か」に準拠）。

- `state.compat >= 8`（相性8以上・噛み合い/ツーカー帯。`compatInit=5.0`・`compatCap=20.0` 確認済＝初期5から数週で到達する帯。0-7他人行儀帯では出さない＝この距離感の会話が成立しない）。
- `nextMilestone.weeksLeft` が **2〜5**（`2...5`。`innerVoice` に渡る `nextMilestone:(name,weeksLeft)` タプル確認済＝実フックあり。大会が近い詰め期・ただし前夜ではない）。
- **谷口所有時のみ**。ただし **【要注意・実装】現状の `GameState` に `partner` フィールドは無い**（MVPは谷口固定）。よってこのゲートは**現状は真値固定（no-op）で実装**し、絶対にシーンを塞がないこと。実体を持つのは本編で標準語相方が入る時＝**関西弁台詞が漏れないための必須ガード**（下記リスク）。擬似コードの `partner: .taniguchi` は**未存在モデルへの前方参照**である点を取り込み時に明記。
- **キャリア1回**（`oncePerCareer` フラグ・1runで一度だけ）。
- **前提にしない状態**：連敗数・直近敗退/通過・所持金・知名度・体力は条件にしない（世界の彩り＝プレイの巧拙と無関係に「谷口も一人でやっている」を置く）。
- **【設計上の懸念・要目視】** 「発火帯に入る」＝「必ず出す」なら**確定発火**（帯に入った最初の適格週で1回点火＝抽選しない）を推奨（golden安全・0024スコープ内）。逆に彩りとしてランダム抽選プールに載せるなら**乱数消費順に触れる＝止めて相談**（下記リスク）。どちらにするかはオーナー判断。

## イベント会話本文

話者＝**俺（標準語・地の文と俺のセリフ）／谷口（関西弁）**。谷口所有時のみ発火＝**標準語相方の別プールは不要**（本編で標準語版が要る時は別途起草＝リスク欄）。会話は話者ごと1セリフずつ（「／」詰めなし＝通常の会話送り）。効果は**基本無し**（推奨）／or【仮】微少。数値はセリフで言わない。

**セットアップ（地の文・俺視点）:**

> 来週の稽古場を押さえに、受付の予約票をめくった。
> うちの枠は、決まって二段で書く。上に俺、下に谷口。二人で一枠。
> 前の週の欄に、平日の朝いちが埋まっていた。上の段に、谷口の名前だけ。下の段は空いている。時刻の「9」の、丸の閉じ方が甘い。あいつの字だ。

谷口「あー、それな。あの日、たまたま場所空いとったから、ちょっと横になってただけや」

俺「そうか。来週は、いつもの土曜で押さえとく。」

**締め（地の文・俺視点）:**

> 予約票を閉じた。
> 朝いちの枠は、半額になる時間だ。横になるだけなら、もっと安い場所がいくらでもある。
> 来週の土曜の欄に、二人分の名前を、上から順に書いた。

**（任意・相性15-20＝ツーカー帯のみ、セットアップ末尾一行を差し替え）**

> めくる前から、その字だと分かっていた。

- **固有プロップ＝1つ**：稽古場の**予約票**（「二段で書くうちの取り方」＝二人で一枠、という**この相方に固有の書き方**／谷口の**癖字**＝「9」の丸の閉じ方の甘さ／朝いち＝半額枠）。天井の染み・冷めたコーヒー・小銭のような定番プロップではない（fable_kickoff §3-A の固有=置換不能の例「相方が予約表に書く時の癖字」に直接対応）。
- **谷口の本音は言語化しない**：谷口は**段取りの嘘**（「寝てただけや」）だけをつく。一人稽古の理由も気持ちも喋らない（§3-B）。
- **俺は追及しない**：予約票を閉じ、来週の欄に二人分の名前を並べて書く＝**事実一つで繊細さを置く**。
- **デッドパンの一滴＝1本のみ**：締めの「横になるだけなら、もっと安い場所がいくらでもある。」＝半額の稽古枠に金を払った事実の並置＝谷口の嘘が透けるが、**俺は谷口に言わない**（嘘をそのまま立たせておく）＝自己言及ゼロ・警句化しない。自己憐憫・キザ無し。
- **ボケ/オチ/ネタ本文＝ゼロ**。笑いは地の文の乾いた一滴のみ。

## どこから学んだか（型の出典）

- **氷山理論・言い落とし**（心情語を消し、手/物が何をしていたかを1つ書く／2つの事実を接続詞なしで並置し判断を受け手に渡す）＝谷口の嘘を立たせたまま「半額の朝枠」という事実だけを隣に置く。志賀直哉『城の崎にて』／カーヴァー『大聖堂』。
- **自己憐憫ゼロの乾いた一滴・非追及**（つらさを説明せず次の行動へカメラを回す＝俺は予約票を閉じて名前を書く動作へ逃がす）。ルシア・ベルリン『掃除婦のための手引き書』。
- **生活の物を所作だけで処理する**（副詞で湿らせない・来歴を語らせない＝予約票の書き方と癖字を、感傷なしの所作で）。向田邦子『父の詫び状』。
- **構造の出典**＝内部doc `docs/event_design_v0.md` #19（相方も生きている演出）／固有プロップ「予約表の癖字」は `proposals/fable_kickoff_drama_allin_v1.md` §3-A の置換不能プロップ例そのもの。会話の型見本は `proposals/0015_peer-folded-chair.md`（物-描写で無言の機微を置く鏡構造）。
- 学ぶのは型・間・物の選び方のみ。既存作品の本文・セリフは一切引かない（#19 通知文とも非重複＝別の物・別の場面に置換）。

## 実装イメージ（既存の会話/イベント機構への載せ方）

**現状**（確認済）：`DialogueData.Advice { name: String?; text: String }` があり、`name` は「俺」独白が基本で「将来のレアイベントで谷口等に切替」とコメント済＝**話者切替の口は既にある**。ただし**複数行の一場面（地の文→谷口→俺→地の文）を順送りする"シーンランタイム"は未実装**（0015 が選択肢ランタイム未実装と指摘したのと同様、非選択の会話シーンも器がまだ無い）。

- **本案が要求する器＝「順送りの非選択シーン」**（選択肢なし）。選択肢イベント（0024）より単純：`[Advice]` を順に送るだけ。
  ```swift
  // 【提案・未適用／非選択イベント会話の1エントリ】
  // 地の文= name:nil、谷口= name:"谷口"、俺= name:"俺" として Advice を順送り。
  EventScene(
      id: "nameless_reservation",
      // partner: .taniguchi は未存在モデルへの前方参照。MVPでは真値固定(no-op)で実装。
      fire: .deterministic(compatMin: 8, weeksLeftRange: 2...5,
                           partner: .taniguchi, oncePerCareer: true), // 抽選しない＝新規RNG無し
      lines: [
          Advice(name: nil,    text: "来週の稽古場を押さえに、…あいつの字だ。"),
          Advice(name: "谷口", text: "あー、それな。…ちょっと横になってただけや"),
          Advice(name: "俺",   text: "そうか。来週は、いつもの土曜で押さえとく。"),
          Advice(name: nil,    text: "予約票を閉じた。…上から順に書いた。"),
      ],
      effect: nil   // ★推奨=効果無し（フレーバー）。相性帯15-20は最終行のみ差し替え。
  )
  ```
- **なぜ golden がビット不変か（正確な根拠）**：第一に **golden の auto-sim（`GameCareer.runYear` + `GoldenPolicy`）は「イベント層＝golden対象外・既定OFF」**（`CareerGoldenTests.swift` の正典コメントに明記・確認済）。＝**シーンも効果も golden シミュレーションでは一切発火しない**。第二の保険として、発火は決定的フラグのみ（compat / weeksLeft / partner / career-once）で**インタラクティブ session の乱数列も1ビットも動かさない**（0024＝抽選しない・効果でサイコロを振らない）。よって `tools/*.py` 同期不要・golden再生成不要・`swift test` 緑のまま。
- **効果の扱い**：**推奨は効果無し**（会話が主・ドラマ主軸）。もし#19の名残で【仮】相性+1 を載せる場合も、（a）イベント層がgolden-OFFである限り golden 値は不変、（b）0024ピース1の**決定的パッチ seam**（`RandomSource` を呼ばない `(inout GameState)->Void`・適用後 `state.clampAll()`）で適用＝RNG非消費。**ただし** `clampAll` の compat 実挙動（0-20 に丸めるか）は適用前に要確認（`compatCap=20.0` は成長ロジック側の上限。seam の clamp と一致するか目視）。**セリフでは数値を言わない**（「相性+1」等と喋らせない）。
- **相性帯分岐**：`state.compat` を 8-14／15-20 で判定し**最終セットアップ行のみ差し替え**（15-20＝「めくる前から、その字だと分かっていた。」）。プール分岐だけで新システム不要。
- **【止めて相談】** もし「ランダム抽選プールに載せる」「効果で上振れ/下振れをロールする」に変えた瞬間、**イベント抽選ストリーム＝乱数消費順の正典**に触れる（イベント層を将来sim-ONにした時に Python 正典と消費順が合う必要がある）。その時だけ ①`tools/*.py`→②GameCore→③`gen_golden.py`再生成→④`swift test`緑 を**1コミットで揃える**（CLAUDE §A-1/A-2）。**本案の推奨（確定発火・効果無し）では触らない。**

## リスク・注意

- **【要相談・正典】発火をランダム抽選にするなら乱数消費順に触れる**＝正典改変＝「止めて相談」（CLAUDE §A-2）。**確定発火＋効果無し**で入れる限り golden はビット不変（イベント層 golden-OFF・0024準拠）。まずはこちらを推奨。
- **【要確認・重複／新規catch】0017『負けた日の稽古場』との"稽古場-料金デッドパン"の被り**。0017 のB分岐で既に **稽古場の予約＋料金**の乾いた一滴（「稽古場の予約は、まだ一時間残ってる。料金は、負けた日も、勝った日も、同じだ」）を使用済。本案の締め「朝いちの枠は、半額になる時間だ。…もっと安い場所がいくらでもある」も**同じ"稽古場の料金"を軸にしたデッドパン系統**。物（褪せたカラーコーン三番 vs 予約票の癖字）と場面（敗退翌週の選択 vs 突発一場面）は別だが、**"稽古場の料金で温度を出す"デッドパン装置が2作で反復＝コーパス単位で手癖化するリスク**。低頻度（本案 oncePerCareer・0017 は justLost 節目）かつ框組みが別（同一料金 vs 半額枠）なので即NGではないが、**取り込み時に「料金デッドパンは稽古場イベント全体で最大1系統に絞る」か、本案の乾いた一滴を料金以外の物（例：癖字そのもの／二段の並べ直し）へ寄せるか**をオーナー判断。＝下書きが#18/#19としか照合していなかった穴。
- **【必須】谷口所有ゲート＝声のロック**。標準語の別相方が所有する状態でこのシーンが出ると関西弁台詞が漏れ register 汚染（fable_kickoff §2-5）。ガードは**バグ回避でなく声のロック**。ただし前述の通り**現状 `GameState` に `partner` 無し＝MVPでは no-op（常時真）実装**。本編で標準語相方版が要るなら、**繊細さ（予約票・非追及・半額の朝枠）はそのまま／声だけ標準語**の**別プール**を新規起草する（谷口版のコピペで方言を消すのは不可＝間・語尾が谷口固有）。
- **weeksLeft 2-5 と preTournament ラダーの棲み分け**。これは**突発シーン**で、`innerVoice` の週頭独白ラダー（weeksLeft≤2＝preTournament／`DialogueData:35`）とは別チャネル。**その週の週頭独白を潰さない/上書きしない**よう、シーンは行動選択とは別のタイミングで一度だけ出す（実装時に発火位置を目視確認・CLAUDE §D-10）。
- **効果の押し売り回避**。繊細さは**物（二段の書き方・癖字・半額の朝枠・二人分を並べ直す所作）**が背負う。数値で悲哀を押し付けない（0009・§7-B「体感を数値で直さない」）＝**効果無し推奨**の理由。
- **#19（設計テーブル一行）／#18（楽屋で無言・敗退後）との重複回避**。#19 は「公園の立ち稽古・声をかけず帰る」＝別の物/場面。本案は「予約票の癖字・半額の朝枠」＝**物も場面も非重複**に置換済。本編で#19通知型も残すなら、片方出たら他方の重み減（相互排他寄り）を検討。
- **谷口の嘘の温度**。「寝てただけや」は**段取りの嘘だけ**＝本音の言語化はしない（§3-B）。ここに「ほんまはな…」と続けさせた瞬間に繊細さが崩れる＝**続けさせない**。
- **register自己点検**：谷口＝関西弁一貫／俺の地の文・セリフ＝標準語（方言漏れなし）。締めの「もっと安い場所がいくらでもある。」は**事実の並置＝デッドパン**であって人生訓/警句ではない（キザ・自己憐憫チェック済）。デッドパンの一滴は本文中**1本のみ**。任意の相性15-20行「めくる前から、その字だと分かっていた」は**別レジスタ（親密の観測）**で、二個目のデッドパンではない（差し替え・非加算）。
- **固有名・数値・コピペ**：稽古場名・大会名は名指ししない（架空・匿名で温度を保つ）。全数値【仮】。既存作品セリフ流用なし・ボケ本文ゼロ。

**トーン自己点検**：感情語（悲しい/悔しい/惨め/嬉しい）不使用＝感情は物に転写（谷口だけの朝いち枠・甘い丸の癖字・半額の時間・二人分を並べ直す所作）。固有プロップ1つ＝定番でなくこの相方/稽古場固有の予約票。俺＝標準語・計算屋（枠と半額を数える目）だが自己憐憫なし・警句で締めない。谷口＝関西弁一貫・本音は言語化せず段取りの嘘のみ。会話は話者ごと1セリフ・「／」詰めなし。ボケ/オチ本文ゼロ＝笑いは地の文の乾いた一滴のみ。

## レッドチーム(A/B)

### A. 実装妥当性
1. **実フックに載るか＝概ね載る（要注意1点）**。`Advice{name:String?}` の話者切替口・`nextMilestone.weeksLeft`・`state.compat`（init5/cap20）は全て実在を確認。`partner` フィールドは**存在しない**ため谷口ゲートは**MVPでは no-op（常時真）実装が必須**——下書きの `partner: .taniguchi` は未存在モデルへの前方参照である旨を本文に明記済（誤って発火を塞がない補強）。順送り非選択シーンのランタイムは未実装＝本案は器の新規提案（0015/0017 と同一前提）。
