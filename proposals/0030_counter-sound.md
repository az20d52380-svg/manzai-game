# イベント提案 0025「数取器の音」v1（2026-07-06・非選択イベント会話）

> **【スコープ明示】** 本案は proposals/0010-0023 の**選択肢イベント（パワプロ型・A/B効果つき）とは別物**。選択肢を持たない**突発イベント（自動発生・短い一場面の会話）**＝`docs/event_design_v0.md` §C「バイトと生活」#13-16 を**ドラマ主軸で会話化・再トーン**した非選択フレーバー。判断を一切迫らない純フレーバーで、**効果は無し（推奨）**。数値は全て【仮】、既存ファイルは読むだけ。
>
> **【採番】** ファイル名 `0025_counter-sound.md`。既存は 0024 まで＝衝突なし。索引（`NIGHT_LOG.md` / `docs/START_HERE.md`）へ1行追記想定。

> **【MVP/本編の別】** 発火条件は所持金・週・楽バイト直後の**多年非依存の状態のみ**＝**1年48週MVPで実働**。多年専用（同名大会の再会等）ではない。

## 狙い（どの薄いスロットを埋めるか）

**金・バイトフックの「非選択・谷口会話」が薄い。** 既存を棚卸しすると：

- `event_design_v0.md` §C #13-16 は**一行フレーバー（通知型・地の文のみ）**で、掛け合いの「間」がない（#16 だけ相方の軽口が一言あるが会話送りではない）。
- `proposals/0012`（谷口の耳寄りな話）と `proposals/0023`（正社員の話）は**選択版**＝金 vs 合わせの時間の判断を迫る。本案はそこと**棲み分け**、判断ゼロの純フレーバー側を埋める。
- `dialogue_batch2_v0.md` R1「カウンターの向こう」は交通量調査＝数える題材を既に持つが、**「数えると物語になる」という上向き（＝発想+2の効能書き）**で締めており、金欠の手触りではない。本案は**効能でなく所作**で置く。

埋めるのは一点：**「金欠を、残高の数字でなく所作で見せる、谷口固有の一場面」**。俺は今月の残りを指で数え、谷口は二人でやった交通量調査の数取器（カウンター）を、家に帰ってもなお無意識に親指で押している。**生活（俺）と場数（谷口）が、同じ「数える」という所作で並ぶ。** 感情語・判断・効能書きなし。

固有プロップ＝**その週の交通量調査で谷口が任された数取器**。定番の「小銭・財布・家賃・封筒の残高表示」でなく、**この楽バイト固有の道具**（`dialogue_design_v0.md`：谷口＝バイト掛け持ちの達人／居酒屋・引越し・交通量調査）。他コンビ・他バイトの物語では成立しない置換不能の物（§0009-3則②準拠）。

## 発火条件（実フック・実装確認済み）

- **状態フック**: `s.money < 50_000`（金欠帯・`GameState.money` は `Int`／【仮】5万相当）**かつ** **その週のアクションが楽バイト＝`.job(.楽)`だった週の結果画面**。
  - **【実フック確認】** 「交通量調査」は `.job(.楽)` の**表示名**（`ManzaiGame/Sources/CommandData.swift`：`(.楽, "交通量調査", …)`）。ただし `.楽` は**試験監督系も内包**するカテゴリ（`dialogue_batch2_v0.md` R1 見出し「楽（交通量調査・試験監督系）」）。よって本案の実フックは厳密には `.job(.楽)`＝「楽バイトを選んだ週」であり、表示上たまたま交通量調査面が出る週に会話が一番なじむ。**同一週内で解決**するため「直近選択」は結果画面で自明＝新規の永続状態は不要。
- **相方ゲート（MVPでは実質no-op・正直版）**: 数取器は谷口の交通量調査に紐づく固有プロップ。**現行MVPは谷口が唯一の相方でハードコード**（`ManzaiGame/Sources/` に partner enum / 相方ガチャは未実装。`GameSession`・各Viewの相方参照は全て「谷口」直書き）。したがって **`partnerIsTaniguchi` ゲートは MVP では常に真＝実質空条件**。多相方（`partner_gacha_design_v0.md`）は**設計段階**で、実装が入った時点で初めて「谷口以外では発火抑止」が意味を持つ。**MVP実装では相方分岐コードを書かず、多相方導入時に固有性ガードを足す**（本案は「谷口以外には移植しない」を将来ガードとして予約するだけ）。
- **抽選**: 自由週の突発イベント判定に**低頻度**で乗せる。**週ランダム**（節目に固定しない＝世界の彩り層）。キャリア内の再発は**天丼レジストリ**（`callback_ledger`／0005系）で最短再登場間隔を管理（近すぎ寒い・3回目が最大）。
- **前提しない状態**: 連敗・知名度・大会まで週数は発火条件に**しない**（生活の彩りで、プレイの巧拙と無関係に置く）。
- **【overclaim回避】** 「金欠×楽バイト直後」に入る＝**必ず出る**ではない。実発火は低頻度抽選×他イベント枠競合の確率事象で、**1年 run で0回もある**。保証したいなら pity/確定枠が要り＝分布に効く＝相談対象（下記）。

## どこから学んだか（型の出典）

学んだのは**型と構造だけ**。既存作品の本文・台詞は一切引かない。

- **感情でなく「場所の物・所作」で置く**（残高を数字で押さず、指の動きに背負わせる）＝氷山理論／言い落とし。志賀直哉『城の崎にて』・カーヴァー『大聖堂』の型のみ抽出。https://ja.wikipedia.org/wiki/城の崎にて
- **自己憐憫ゼロの乾いた一滴を「痛みの直後でなく通り過ぎた平地」に置く**（谷口が「車走ってへんのにな」と気づく＝谷口版のデッドパン一滴、5-6本に1本の分母を守る低頻度）＝ルシア・ベルリン『掃除婦のための手引き書』の型のみ。https://ja.wikipedia.org/wiki/ルシア・ベルリン
- **生活の物を副詞で湿らせず所作だけで処理**（「大事に」「寂しく」を禁じ、押す・戻す・鳴るだけ）＝向田邦子『父の詫び状』の呼吸。
- **短文・句点・改行余白で「間」**（カチ、／また、カチ、の反復音を沈黙で効かせる）＝山下澄人『しんせかい』の文体エンジン。
- 内部正典：`proposals/0009`（感情を書くな・場所の物を書け）／`proposals/fable_kickoff_drama_allin_v1.md`（声のロック・乾いた一滴1/6・固有プロップ1個）／型見本 `proposals/0012`・`0015`。既存作品の本文は一切転載せず、構造・型のみ。

## 実装イメージ（既存機構への載せ方）

**現状の前提**: 突発イベント（自動発生・非選択）のランタイムは `ManzaiGame/Sources/` に**未実装**（Event系ファイルなし）。稼働しているのは `DialogueData.swift` の `innerVoice`（週頭の独白・`Advice(name:"俺")` 固定／谷口切替はコメントで予約済み・未実装）だけ。`event_design_v0.md` §1・§C は**マスターデータ案**の段階で、`(id, フック, 本文プール)` という設計だけがある。本案は**そのデータ骨子＋会話プール**の提案。

想定データ形（純データ・テキストは別リソース／会話は話者付き配列）:

FlavorEventDef(

    fire: .random(
        require: [.moneyBelow(50_000), .afterJob(.楽)],   // 楽=表示名"交通量調査"。谷口固定ゆえ相方ゲート省略
        weight: .low, oncePerCallbackWindow: true         // 天丼レジストリで再登場間隔管理
    ),
    effect: .none,                                          // 効果なし（推奨）
    lines: [
        .narration("その週の交通量調査は二人だった…"),        // セットアップ地の文
        .speaker("谷口", "あの交差点な、朝の二時間だけ…"),
        .speaker("俺",   "その音、さっきから鳴ってるぞ"),
        .speaker("谷口", "……あ。今も押してたわ、俺。…"),

- **話者切替の前提**: 現行 `Advice.name` は `"俺"` 固定。谷口を話者として出すには UI 側に話者切替の口が要る（`dialogue_design` §6・既提案 0003/0012 と同じ前提）。**本イベントは話者2名なので着手はその口が付いた後**（＝出荷ゲート）。

## リスク・注意

- **【要相談・正典】抽選の乱数消費順**。効果なしでも**「発火するか」を決める抽選の1ドローが乱数ストリームを消費する**。どのストリームから・どの順で引くかは**乱数消費順の正典**（`tools/*.py` → GameCore → `gen_golden.py` → `swift test` の1コミット同期／CLAUDE §A-2）に触れる。`WeekRunner` の消費順（injury `nextUniform()` → `rollOffer`）に割り込むと3年ビット一致goldenが壊れる。**実装時に必ず止めて相談。** 本文（テキスト）追加だけなら golden 不変＝**まず既存の突発イベント枠に相乗りして抽選を新設しない形を最優先検討**。抽選ドローを新設する瞬間が分水嶺。
- **【R1 との整合＝オーナー判断】** `dialogue_batch2_v0.md` R1「カウンターの向こう」は交通量調査を**俺ソロの職務フレーバー（発想+2・「谷口に話したら30分ネタが伸びた」）**として置く。本案は同じ楽バイトを**二人でやった週の谷口の所作**に置き直し、**主体を「俺の内心（R1）／谷口の所作（本案）」で棲み分け**て衝突を避けた（v0の「谷口が交通量調査から帰ってきて」は谷口ソロと読め、俺が同じ週に楽バイトを選んだ機構と齟齬した＝レッドチームA-1で是正）。ただし**「数える人」を俺/谷口どちらに一本化するか**は固有プロップ帰属＝キャラ定義に関わる＝**オーナー判断待ち**。
- **【効能書き化の回避】** R1 の「数えると物語になる」に引きずられて**上向き/警句で締めない**（谷口版のオチ「車走ってへんのにな」は達観でなく**事実の気づき＝デッドパン**に留める）。俺の締め地の文も「今月はまだ半分ある」等の**残高の言い直し／教訓に落とさない**（数値をセリフ・地の文で言い直さない）。

## レッドチーム（A/B）

### A. 実装妥当性

- **[A-1 破綻ルート・修正済]** v0のセットアップ「谷口が交通量調査から帰ってきて」は**谷口ソロ**と読め、俺が同じ週に楽バイト（`.job(.楽)`）を選んだからこそ発火する機構と齟齬した（俺も現場にいたはず）。→ **「その週の交通量調査は二人だった。谷口が数取器を任され、俺が用紙に正の字を付けた」に是正**。楽バイトを二人でやった週＝機構と一致させ、数取器が谷口の手にある必然も同時に立てた。R1（俺ソロ内心）とも主体を分けられる。
- **[A-2 実フック・確認/是正]** v0の `.afterJob(.trafficSurvey)` は**架空の粒度**。実コードに `trafficSurvey` enum は無く、「交通量調査」は `Job.楽` の**表示名**（`CommandData.swift`）。`.楽` は試験監督系も内包。→ **実フックを `.job(.楽)` に是正**し、同一週結果画面で解決＝永続状態新設不要と明記。
- **[A-3 相方ゲート・是正]** v0は `partnerIsTaniguchi` を能動的な抽選条件として重く書いていたが、**現行MVPは谷口ハードコード・partner enum/ガチャ未実装**（`Sources/` 実査）＝ゲートは**常に真の空条件**。→ **MVPでは相方分岐コードを書かない**、固有性ガードは多相方導入時に足す予約、へトーンダウン。過剰実装を回避。
- **[A-4 事実誤り・修正済]** v0は「`sim_career.EVENT_FIRE_CAP=15`」と断定。→ **現状コードは `EVENT_FIRE_CAP = None`（`sim_career.py:110`）**。15は `event_design §4-B` の**設計目標**であって未適用。「効果を足すなら予算勘定＋分布再測」の条件付き記述に是正。
- **[A-5 golden/乱数・維持]** 本文追加のみなら golden 不変。危険は「発火抽選ドローの新設位置」の1点で、`WeekRunner` 消費順（injury `nextUniform` → `rollOffer`）に割り込むと3年golden破損。→ **【止めて相談】＋「既存突発枠へ相乗りで消費順を動かさない形を最優先」を維持・強調**。
- **[A-6 重複・確認]** 0012（谷口×金欠の選択版）・0011（付き合い費の選択版）・0023（正社員の選択版）・R1（俺の交通量調査内心）と**種別（非選択フレーバー）か主体（谷口所作）で棲み分け成立**。固有プロップも全て別。近接発火濃度だけリスク節に明記。

### B. 会話の中身

- **[B-1 警句化・修正済]** v0の俺「こっちも数えてる。数える中身が違うだけだ」は、**地の文が既に見せている並置（俺=封筒／谷口=数取器）をセリフで言い切る＝うまいこと言う警句化**（0009/fable v1 禁則）。→ **俺「その音、さっきから鳴ってるぞ」**に差し替え。俺は観測者・ツッコミ体質のまま、谷口が「まだ押していた」ことに気づかせる**デッドパンの事実指摘**に留め、並置は地の文（セットアップの俺の数え／締めの「また一台ぶん鳴った」）だけに背負わせた＝氷山の規律。
- **[B-2 デッドパン一滴・確認]** 谷口「今も押してたわ、俺。ここ、車走ってへんのにな」は達観でなく事実の気づき＝**乾いた一滴1箇所のみ**（5-6本に1本則）。自己言及の湿りゼロ。
- **[B-3 方言・話者・確認]** 谷口＝全行関西弁、俺＝全行標準語・計算屋。会話は話者ごと1セリフ（"／"詰めなし）＝通常の会話送り。MVPは谷口固定ゆえ相方依存の声の違反なし。
- **[B-4 コピペ/酷似・確認]** 火花／べしゃり暮らし／M-1実在ネタ等の台詞・言い回しの流用なし。数取器・右折レーンの減り・用紙の正の字は本作固有の物。ボケ（オチ/フリ/ネタ本文）なし＝笑いは地の文（谷口が無意識に押し続ける所作）にのみ。
- **[B-5 固有プロップ・確認]** 定番（天井の染み・冷めたコーヒー・小銭）を避け、**その週の交通量調査で谷口が任された数取器**1つに集約。他バイト・他コンビでは成立しない置換不能の物。
- **[B-6 軸名/数値の口出し・確認]** セリフ・地の文とも華/センス/発想等の軸名も具体数値も口に出していない（「今月あと何日」は生活の描写であって game stat ではない）。
