# 選択肢イベント案:「先輩の名刺」v1（2026-07-06・レッドチーム反映）

対象: 突発イベント（event_design §2 二択・リソース交換）の補充1件。数値は全て【仮】、固有名は架空、既存ファイルは読むだけ。トーン＝ドラマ主軸（proposals/0009確定）。

> **【スラッグ訂正】** 起草時想定の `0010_senpai-meishi-choice.md` は**衝突**（`0010_pre-tournament-eve-choice.md` が既存）。本案は **0011** に確定。

## 狙い

event_design §2 の選択肢イベントは現状5種（#34〜#38）で、**華を直接動かす選択が一つも無い**。華は career_report が指す「最大gap（伸ばしにくい・イベント供給が薄い）」帯で、そこへ触れる選択を一枚足す。既存 #4「先輩の焼肉」（所持金<20万・自動通知・体力+20/相性+1）は"貰って終わる"温かい自動イベントだが、本案は**同じ「奢られる帯」を舞台にしながら、貰ったあとに残る"借り"を選択に変える**。近道（紹介＝華・知名度を先に取る／借りを背負う）と自力（飯だけ貰って自分の番を待つ／機会は逃す）が損得の別軸で拮抗し、明白な最適解を作らない（§2 原則3・「正解のない選択」）。縦社会の恩送りを、恩恵の受領ではなく**受領のコスト**として設計に翻訳する。

## 発火条件（実フック）

- 状態フック（実装済／近い）: **所持金<20万**（奢られる帯・#4と共有）**かつ 知名度<50**（まだ繋がれる側）**かつ ランダム**。
- 大会週は除外（§0 頻度設計）。**キャリア1回制**（§4-B の生命線・再抽選で恒久効果が無限稽古化するのを防ぐ）。**EVENT_FIRE_CAP=15 の総量予算内**。
- #4（自動通知）との棲み分け: #4は通知のみ・こちらは**選択版**。同一キャリアで両方発火してよい（別id `evt_senpai_meishi`）。ただし同一週での二重発火は禁止（抽選側で1週1イベント上限を確認）。
- MVP発火可（1年48週内で条件成立しうる。年1序盤は所持金<20万・知名度<50が常態＝早期に出る枠）。

## イベント本文

> トーン＝ドラマ主軸。感情語（悲しい/悔しい/惨め）を地の文から抜き、**"この場面固有の少しズレた物"**（＝焼き網の熱で反っていく名刺）に温度を背負わせる。俺＝標準語・計算屋・自己憐憫なし。谷口＝関西弁だが**核心はふとした一言で掠めるだけ**（内面を代弁・解説しない）。**会話は話者ごとに1セリフずつ（"／"で詰めない）**。ボケ（オチ/ネタ本文）は書かない。数値は全て【仮】。

**場面のセットアップ（地の文・俺視点）**

先輩が肉を焼き終えるより先に、伝票を裏返した。番組の名前が入った名刺を一枚、焼き網の縁に置く。角が、熱で少しずつ反っていく。

先輩「ここは俺が持つ。……で、一個、顔繋いどこか？」

俺「ありがとうございます。……その話、少し考えさせてください。」

谷口「先輩、こいつ、肉より先に伝票見る男なんですわ。」

---

**選択肢A｜紹介を受ける**

【効果】華+1・知名度+2 ／ メンタル-1（借りの重さ）〔全て【仮】〕

先輩「ええ度胸や。番号、今日中に鳴らせ。」

俺「はい。……この名刺、反ってますね。伸ばして持っときます。」

谷口「反ったやつ伸ばして使うの、こいつの得意技なんですわ。」

---

**選択肢B｜飯だけ頂き、紹介は辞退する**

【効果】体力+15・メンタル+1 ／（相性+1・上限未満なら）／ 機会は逃す（華・知名度の獲得なし）〔全て【仮】〕

先輩「そうか。……まあ、飯は残さず食え。」

俺「はい。いただきます。……この一本は、また今度で。」

谷口「すんません、うちのが。……先輩、この反った名刺、もらといてええですか。」

## どこから学んだか（構造＋出典URL）

- **「有意な選択＝トレードオフで、支配的な手を作らない」** — Sid Meier「ゲームは面白い決断の連続」。片方が上位互換だと決断は消える。本案は華/知名度（A）と体力/メンタル（B）を別軸に置き、どちらにも"効く軸と失う軸"を持たせた。構造のみ抽出。
  https://www.gamedeveloper.com/design/gdc-2012-sid-meier-on-how-to-see-games-as-sets-of-interesting-decisions
- **育成SLGの恩恵イベント（後ろ盾／コネを"リスク・リターン付き分岐"にする型）** — 育成SLGの選択イベントは「便益を今取るか、コスト（体力/評価/依存）を背負うか」を二択化する。恩恵を"受けるか否か"でなく"受けた後に何が残るか"で設計する型を借りた。本文・数値は転載せず構造のみ。
  https://ja.wikipedia.org/wiki/実況パワフルプロ野球
- **お笑い縦社会の「奢り＝恩送り」文化** — 先輩が奢り「売れたら下に返せ」で縦に循環する慣習。#4はこれを温かい受領として描いた。本案は同じ慣習の**貸し借りとしての重さ**（近道の代償）を主題化する。世界観の手触りのみ参照（proposals/0009の"世界"レイヤー＝下積みの非対称）。
  https://ja.wikipedia.org/wiki/恩送り

## 実装イメージ

既存の §2 選択肢イベント機構（`(id, 効果パッチ, 選択肢[])` の純データ＋UI層テキスト）にそのまま載る。新システム不要。以下は**提案**（差し込み口の設計）。

```
// GameCore: イベントは純データ。効果パッチは決定的（RNG不使用）。
evt_senpai_meishi:
  trigger:  money < 200_000 && fame < 50 && randomHit(eventRng)   // ← 抽選のみRNG
  once:     true                                                  // キャリア1回制（§4-B）
  budget:   counts against EVENT_FIRE_CAP(=15)
  choices:
    A(紹介を受ける):  { 華:+1, 知名度:+2, メンタル:-1 }   // 恒久+は+1〜2に抑制（§4-B(b)）
    B(飯だけ頂く):    { 体力:+15, メンタル:+1, 相性:+1 }   // 主値は体力（飯）＝capで死なない
  // 相性上限は相方ティア依存（谷口base20/ガチャSR〜SSR22〜24）。上限到達済みなら相性は+0クランプ。
  // ★B非支配性の要: 相性はcapで消えるため"添え物"。Bの拮抗力は体力+メンタル（非cap）で担保する。
// テキスト（セットアップ＋各選択肢の会話）はUI層リソース。地の文は俺=標準語、谷口の引用のみ関西弁。
```

**［止めて相談］乱数消費順／golden**: 効果パッチ自体は決定的で乱数を消費しない。しかし**イベントを抽選プールへ1件追加すること自体がイベントRNGストリームの消費順を動かしうる**（§5「抽選専用ストリーム分離」の前提に触れる）。プール追加＝実装は CLAUDE.md §A-2（消費順は数式と同格の正典）に該当するため、**gen_golden.py 再生成前に一度止めて相談**する。

**sim同期（規律5・§4-B(c)）**: 恒久正効果（華+1・知名度+2）を含むので、`tools/sim_career.py` の `EVENTS_ON` フラグ側に本イベントの近似を足し、**優勝率1〜2%帯の維持を再計測**してから確定。B効果を「相性+1中心」→「体力+15中心」へ振替済みのため、**A/Bの選択率が偏らないか**も併せて実測。EVENT_FIRE_CAP=15 予算内であることも `tools/exp_events.py` で確認。

## リスク・注意

- **【最重要・非支配性の実バグ】B が相性capで死ぬ**: 起草版の B は主値が相性+1 だった。相性は谷口base20（ガチャ22〜24）でクランプし、**ツーカー/阿吽帯（中盤以降）ではB＝実質メンタル+1のみ**に痩せる。一方 A は華+1/知名度+2 と最大gap帯に効き続ける＝**Aが晩期に準支配的**になる。→ B の拮抗力を非capの**体力+15（＝実際に飯を食う物）・メンタル+1**へ移し、相性+1は"添え物"に降格した。これで相性上限でもBが生きる。
- **バランス（数値は最後）**: A の華+知名度は career_report の最大gap帯に直接効くため、優勝率へ効きやすい。数値は【仮】、simで測ってから。体感が薄ければ演出・頻度で先に解き、数値をいじるのは最後（proposals/0009・規律B-5）。
- **華+1の恒久付与**: 華は伸ばしにくい＝+1の相対価値が高い。§4-B(b)の「恒久正+1〜2」内だが、simで優勝率が跳ねたら**華+1→表現+1**へ振替（要相談・未決3）。
- **#4との重複感**: 舞台（焼肉・奢り）が#4と近い。プロップ（**焼き網の熱で反る名刺**）と主題（受領でなく借り）で差別化しているが、同一キャリアで#4→本案が連続すると既読感が出うる。抽選間隔を空けるか、片方発火済みならクールダウンを検討（実装時・未決5）。
- **トーン（起草版からの是正）**: 谷口が俺の内面を代弁・解説していた起草版の2セリフ（"貸し借りの計算しとる"／"返す前提でしか物を貰えん男"）は**主題を口で説明する寒さ**（0009「感情/内面を書くな、物を書け」違反）。谷口は三枚目・楽天で"ふとした一言が核心を掠める"声（dialogue_design §）に戻し、**物（伝票・反った名刺）越しの軽い一言**へ縮めた。俺の B セリフも起草版の"自分の番で拾いたい"は**誇りの吐露**（本タスク禁止）だったため、"また今度で"の実務的な先送り＝損得判断に落とした。乾いた一滴はB末尾「もらといてええですか」（谷口のデッドパン・名刺への無説明の執着）1回のみ。ボケは不使用。
- **先輩の方言**: 先輩は関西弁（"ええ度胸や"等）。**先輩は相方ではないNPC**なので「谷口以外の相方は関西弁にしない」（相方依存の声）には抵触せず、お笑い世界の手触りとして許容。ただし標準語NPCと混在させる場合の方針はオーナー判断（未決6）。
- **固有名**: 先輩は無名の「先輩」表記のまま（架空名を足すなら name_generator 経由）。番組名も架空・伏せ。
- **効果判定は実機**（規律D-10）: 奢られる帯（所持金<20万）を実際に周回し、Bを選んだときに"損した義務感"でなく"自分の番を待つ乾いた納得"として送れるか目視。

## レッドチーム（A/B）

### A. 実装妥当性

- **[修正済/致命] スラッグ衝突**: 0010は既存。0011へ訂正。
- **[修正済/致命] 非支配性が相性capで破綻**: 上記の通りBの主値を非cap（体力）へ移し、A準支配を解消。simで選択率の偏りを最終確認（未決4）。
- **[良] golden/RNG止めて相談**: 効果は決定的だが抽選プール追加＝消費順に触れる、を正しく明記（§A-2）。維持。
- **[良] キャリア1回制＋EVENT_FIRE_CAP=15**: §4-Bの生命線を尊重。恒久正効果+1〜2の枠内。
- **[要相談] 華+1**: 最大gap帯への恒久付与は優勝率に効きやすい。sim実測ゲート（華+1→表現+1の振替を予備に）。
- **[良] MVP発火**: 所持金<20万・知名度<50は年1序盤で常態＝1年内に確実に立つ。死に枝なし。
- **[要実装確認] #4クールダウン**: 同キャラ焼肉2連発の既読感。抽選間隔 or 片方発火済みクールダウン。

### B. 会話の中身

- **[修正済/最大] 谷口の主題説明**: 内面代弁2セリフ→物越しの楽天一言に縮小（"物で語れ"回復）。
- **[修正済] 俺の誇りの吐露**: B「自分の番で拾いたい」→「また今度で」の実務先送りへ（自己憐憫・誇り禁止に整合）。
- **[良/維持] 固有プロップ**: 焼き網で反る名刺＝定番プロップ回避の"この場面固有の物"。デッドパン一滴（B末尾）1回。
- **[良] 話者1セリフ・方言・ボケ無し**: 俺=標準語/谷口=関西弁/先輩=NPC関西弁（世界観）。/で詰めず会話送り準拠。ネタ本文なし。
