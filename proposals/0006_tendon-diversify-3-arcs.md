# 天丼在庫を「腹減ったな」一極から3本に分散する — 俺・審査員側にも三段アークを作る

- 日付: 2026-07-06（夜間リサーチ・レッドチーム反映 2026-07-06）
- 位置づけ: 提案のみ・コード改変なし。数値・配置は全て【仮】。既存ファイルは読取のみ。**数式・乱数消費順・golden には一切触れない**（触れる必要が生じたら実装を止めてオーナー相談）。
- 対象の弱点: `writing_craft_and_appeal_v0 §7` が天丼候補として「いけるわ」「今日の最高記録」を名指ししているのに、実際に複数ビートへ仕込まれ回っているのは谷口の「腹減ったな」だけ（`dialogue_design §4` 決勝進出・優勝の感情最大点に配置）。天丼は「1個を3回」でしか機能しない（1回目=フリ／2回目=変化／3回目=最大）のに、在庫が1本深＝他の候補に三段アークが作られていない。谷口偏重で、俺=ツッコミ語り部側・審査員側の再登場語も薄い。

## 狙い

1. **「今日の最高記録」（俺）・「いけるわ」（谷口×俺の検証）・審査員の紙講評モチーフの計3本を設計する**。各天丼を必ず〈フリ→変化→最大〉の3ビートに割り付けてから配置する（単発で撒くと全部が「またか」の内輪符牒になって寒くなる）。
2. **谷口偏重を、俺=語り部側の再登場語（「今日の最高記録」「通算/的中率」）と審査員側の再登場語で相殺する**。天丼の"言い回し・トーン"は1回目と固定し、変えるのは「どの場面で・何に対して言うか」だけ（同型維持・状況更新）。
3. **感情最大点（§4 決勝進出/優勝）には必ず各天丼の"3回目"を割り当てる。ただし到達できなかった年でも回収できるよう、年末リザルト（§5 独白）を"その年に実際に届いた頂点"としてフォールバック回収先に置く**（初回はほぼ勝てない設計＝`fun_flow §1`。決勝固定だと大半の周でフリだけ撒いて未回収になり、本提案自身が禁じる「スベったまま繰り返す」に陥るため）。ボケ本文は一切書かず、笑いは「観測語の再登場」に逃がす（絶対の一線を守る）。

## どこから学んだか（技術＋出典URL）

- **天丼＝3回で1セット。1回目=フリ／2回目=「来た」と思わせる／3回目=変化を加えて被せる＝最大**。2回目以降は必ず"変化"を足す（単純反復は3回目以降で意外感が減衰）。
  - お笑いと構造 第8回 天丼はどこまで盛れるか: https://satzdachs.hatenablog.com/entry/2020/03/20/033000
  - なぜ天丼は3回までなのか（humorsense）: https://humorsense.s-teem.com/2012/03/post_78.html
- **最強のコールバックは"繰り返し"でなく"再文脈化(recontextualize)"**。同じ語を別の意味・別状況で出し、文脈・言い回し・間のどれかだけ捻る。回収は1回目とトーン・言い回しを揃えると「あのときのあれ」が伝わる。
  - Mastering Callback Humor（comedywriter.info）: https://comedywriter.info/mastering-callback-humor/
  - Writing Callbacks in Your Scripts（Final Draft）: https://www.finaldraft.com/blog/writing-callbacks-in-your-scripts
- **Rule of Three = establish / reinforce / surprise** ／ **SREP = Set-up / Reveal / Escalation / Payoff**。走らせた語は戻るたびに新文脈で大きくなり、最後のPayoffが最大。
  - No Film School: Rule of Three: https://nofilmschool.com/rule-of-the-three
  - BCG Pro: SREP: https://www.comedy.co.uk/pro/inside_track/set-up-reveal-escalation-payoff/
- **「言い切らない／余白を残す」で意外感が増す**（3回目を全部言わずに置くと客が先回りして回収の快感が増す）。同上 Rule of Three。
- **サムさに転ぶ条件＝変化なし反復／内輪符牒化**。天丼候補は"状況説明ゼロでも意味が通る観測語"に限る。
  - 天丼とは（owa-rai お笑い芸人用語集）: https://owa-rai.com/tendon/
- **反復イベントの前イベ/後イベ思想**（同じ素材でもコマンド選択の"前か後か"で意味が変わる／フラグ蓄積で最終回収）。天丼の配置に流用: 1回目=結果ビートの"後"に低温で置き、3回目=最大点ビートの"直前"に置いて回収の呼吸を作る。
  - サクセスモード（Wikipedia）: https://ja.wikipedia.org/wiki/%E5%AE%9F%E6%B3%81%E3%83%91%E3%83%AF%E3%83%95%E3%83%AB%E3%83%97%E3%83%AD%E9%87%8E%E7%90%83_%E3%82%B5%E3%82%AF%E3%82%BB%E3%82%B9%E3%83%A2%E3%83%BC%E3%83%89
- **審査軸を"比較(T3)＋愛ある辛口(T6)"で言語化する型**（`judge_comments_v1 §0`）。同一の査定語を回戦をまたいで返す＝大会構造そのものを天丼の器にできる。
  - M-1グランプリ（Wikipedia・審査軸の参考）: https://ja.wikipedia.org/wiki/M-1%E3%82%B0%E3%83%A9%E3%83%B3%E3%83%97%E3%83%AA

### 天丼A「今日の最高記録」＝俺の"最小反応を計測する"温度差実況（過小申告基調）

同型: 俺が客席の"最小の反応"を「今日の最高記録」として淡々と計測する。**計測する対象だけが状況ごとに変わり**、最大点で計測癖そのものが追いつかなくなって回収。谷口ではなく**俺=語り部**が背負うので、谷口偏重を直接崩す。俺のモノローグ（`DialogueData` の既存チャネル）だけで完結し、谷口レアイベントや審査員表示に依存しない＝**3本のうち最も安全＝先行実装候補**。

| 回 | 配置（§4連動・前/後イベ） | 状況差分 | 「最高記録」の中身（＝変えるところ）／技法 |
|---|---|---|---|
| 1回目(フリ) | 序盤の小ライブ結果の**後イベ**・低温 | 日常・不作 | 舞台上で見えた最小の反応を1つ計測／**過小申告（実況体）** |
| 2回目(変化) | 中盤・敗退ビートの後 | 逆境 | 計測対象が曖昧になっていく（拍手の増減の理由が分からない）／**過小申告＋言い切らない** |
| 3回目(最大=決勝進出/優勝、無ければ年末リザルト) | 感情最大点の**直前 or 直後** | 本番の頂点 | "記録"が本物の歓声になり計測が追いつかない＝言い切らずに着地／**過小申告（最大点）** |

### 天丼B「いけるわ」＝谷口の根拠なき断定 × 俺の"通算成績/的中率"検証（温度差実況の核）

### 天丼C 審査員の紙講評モチーフ「行儀がいい」＝毒→承認へ反転する丁寧な三段オチ

`judge_comments_v1 §0` の **T1指摘型＋T6愛ある辛口**。**同じ査定モチーフ「行儀／折れた角」を予選（紙・非表示）→上位回戦（紙・非表示）→決勝（指名審査員）で返し、3回目で落ちの向きが"毒"から"承認"へ反転**する（丁寧な三段オチの型は保つ）。大会構造そのものが天丼の器になる。

> **【重要・レッドチームA反映】** 予選は**審査員非表示・紙1行**が正典（judge_comments §1）。よって本天丼は「同一人物が3回書いた」とは**主張しない**。stage=1/2 は**紙講評プールの反復モチーフ**（匿名）として置き、stage=3 の指名審査員がそのモチーフを**意図的にコールバック**する演出に留める。「実はあの紙もこの人だった」という**明示的な同一人物リビールはオーナー確認待ち**（伏線回収として魅力的だが、非表示仕様に per-審査員の紙帰属を足す必要があるため）。また**全て単一年内の回戦をまたぐ**構成であり、"多年をまたぐ"ではない（MVPは1年・GameSession）。

| 回 | 配置 | モチーフ（同型） | 状況差分と三段オチの落ち／技法 |
|---|---|---|---|
| 1回目 | 予選（敗退寄り・匿名紙） | 「まだ、行儀がいい」 | 低評価寄り。丁寧→丁寧→軽い毒で締める／**丁寧な三段オチ(毒)** |
| 2回目 | 上位回戦（通過・匿名紙） | 同じ「行儀／角」語 | 中評価。前の回戦との差分に触れ、毒が一段軽くなる／**三段オチ(比較T3)** |
| 3回目(最大=決勝・指名審査員) | 決勝講評 | 同じ「行儀／角」語 | 最上級へ反転。落ちが承認になる＝毒の期待を裏切って着地／**三段オチ(反転・承認)** |

```
// judge_comments_v1 §1/§2 の講評形式。stage=1/2 は匿名紙プールの反復モチーフ、
// stage=3 は決勝の指名審査員(丁寧体キャラ=卯月/目白型)がコールバック。※同一人物リビールはオーナー確認待ち。全て【仮】。
[天丼C/stage=1・予選敗退寄り・匿名紙・T1+T6]
  「上手い。ただ、まだ行儀がいい。教科書の角が、一枚も折れていない。」
[天丼C/stage=2・上位回戦通過・匿名紙・T3比較(単一年内・前の回戦比)]
  「行儀が、少し悪くなった。褒めています。折れた角が、一枚。」
[天丼C/stage=3・決勝最上級・指名審査員・反転(落ちを承認へ)]
  「相変わらず上手い。相変わらず——いや、今日は違う。行儀の話は、もうしません。」
```

### 天丼を"順序固定・1回ずつ"で発火させる器（UI進行値＝既存 lossStreak と同格に置く）

**正典（GameCore・数式・golden）には一切触れない**のが最優先。既存の `GameSession` は既に **`lossStreak`／`justPassedStage`／`winFinale`／`pendingResult`** という「結果ビート由来のUI進行値」を `pump()` の中で更新している（GameState には入れていない）。天丼のステージ・カウンタも**同じ場所・同じ流儀**で持つのが最も安全＝GameCore・Python同期・goldenの全てに無干渉。

```swift
// GameSession に足す想定（UI ViewModel の進行値。lossStreak と同格・数値/命名は【仮】）
// 0=未発火, 1=フリ済, 2=変化済, 3=最大点で回収済（4以上は作らない）
private(set) var tendonRecord = 0   // A「今日の最高記録」
private(set) var tendonIkeru  = 0   // B「いけるわ」
private(set) var tendonGyogi  = 0   // C 審査員「行儀／角」

// 発火はロジック層でなく pump() の結果ビート検知に相乗り（RNGを一切引かない純フラグ）:
//  ・stage は必ず 1→2→3 の順で1回ずつ。飛ばさない・戻さない。
//  ・stage=1 は「結果ビートの"後"（後イベ）」に低温で置く（pendingResult 確定後）。
//  ・stage=3 は「感情最大点の"直前/直後"」= winFinale(優勝) / reachedFinal(決勝進出)、
//    どちらにも届かない年は年末リザルト(outcome確定時)にフォールバック。
//  ・4回目は作らない（緊張と緩和の振れ幅が減衰し寒くなる）。
// テキスト本文は UI 層（DialogueData / 紙講評リソース）に置き、GameSession は「今どのstageか」だけ持つ（既存の役割分担どおり）。
```

```
// 天丼ペーシング【仮】（simと実機目視で詰める）。数式・RNGに触れないため Python同期義務は発生しない。
// 置き場所（GameConfig か UI定数か）はオーナー判断＝未解決:
tendonMinGapWeeks = 8   // 同一天丼の stage 間の最短週間隔（近すぎると寒い）
tendonPerYearCap  = 1   // 「腹減った」以外の天丼を1年内で走らせる本数の上限（内輪化防止）
```

> **【設計メモ】永続化について**: 中断セーブ（`ui_design §5`＝GameState+進行状態のCodable保存、`CodableTests.swift` が GameState 側のラウンドトリップを担保）で天丼進行を復帰させたい場合、上記カウンタは**既存の `lossStreak` 等 UI進行値と同じ「まだ保存経路に載っていない」穴**に属する。これは本提案以前からの別課題であり、UI進行値の永続化をまとめて設計する時に一緒に載せる。**どうしてもセーブ必須なら** GameState に足す選択肢もあり、その場合 `CodableTests` の round-trip と `GoldenTests`（数値フィールド比較のみ＝新フィールド追加に非依存）で無害を確認できる。ただし正典ファイルの面積を増やすため、まずは UI 側（GameSession）を推奨。

## リスク・注意

- **【設計上の懸念】"1回目がスベったまま3回目に行く"共倒れ**。stage=1 は必ず"軽く成立"させること（過小申告で外さない語＝「最高記録」「的中率」「行儀／角」はいずれも状況説明ゼロで意味が通る観測語なので比較的安全）。スベったまま繰り返す運用は禁止。
- **未回収で終わらせない**。決勝固定だと初回はほぼ届かず（`fun_flow §1`）、フリだけ撒いて未回収＝共倒れになる。stage=3 は**その年に実際に届いた頂点**に落とす（決勝進出/優勝が無ければ年末リザルト §5 独白）。
- **谷口の「腹減ったな」と近接させない**。同一週・同一大会の最大点に天丼を2本重ねると密度過多で全部が寒くなる。§6 の「腹減った＝最大点」と、天丼A/Bの stage=3 は**別の最大点/別タイミングに振り分ける**か、最低でも `tendonMinGapWeeks` 以上空ける。`tendonPerYearCap=1` で「腹減った＋他1本」に抑える（＝MVPの1年内で走る新天丼は実質1本。3本は"プレイ間のバリエーション"として効かせる）。
