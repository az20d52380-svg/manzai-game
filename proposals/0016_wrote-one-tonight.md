# イベント案「書けた一本」v0（2026-07-06・レッドチーム反映版）

`event_design_v0` §1「B. 舞台と劇場」の隣に置く、**ネタ作り行動フック**の選択肢イベント。seed=「ネタが1本書けた夜」。低確率『当たり』抽選で発火。勢いで磨く（消耗）か、客観のため寝かせる（温存）か——どちらも一理あり明白な優位なし。借り物の格言と本音の落差で人物を出す。数値は全て【仮】。

## 狙い

1. **ネタ作りに「顔」を付ける**。現状 `ネタ作り`（`GameConfig.swift:130` 発想+3・センス+1・体力-20）は毎週同じ数値が増えるだけで、稽古5種の中で最も「創作している手触り」が薄い。ここに**低頻度の当たり**を差し、"書けた夜"という節目を作る。数値変動をイベント文で包む（`event_design_v0` 設計原則1）。
2. **正解のない二択を守る**（`event_design_v0` §2・原則3）。A＝勢いで即詰め（伸びるが体力を切る＝体調ダウン閾値へ近づく）。B＝寝かせて温存（客観が戻り翌週の合わせが効く）。**体力に余裕がある週／大会が近い週はA、体力が細い週／じっくり週はB**——状態次第で最適が入れ替わる本質ジレンマにする。上位互換を作らない。（※このジレンマ成立には発火窓とのズレという設計上の穴がある＝レッドチームA-1で詳述）
3. **人物を出す**。谷口が本音（今すぐやりたい派）を、借り物の格言（先輩の言葉）で一度ごまかしてから漏らす。俺は結論を出さず選択肢だけ差し出す（計算屋・自己憐憫なし）。

## 発火条件（実フック）

- **行動フック**：その週の行動が `train(.ネタ作り)` で、かつ**払える体力があり体調ダウンに落ちなかった**週（＝`resolveAction` で体力ゲート・体調ダウンロールを通過し `action` が `.ネタ作り` のまま `applyTraining` が `true` を返した週）。
- **抽選**：`ハマった夜`（`canonical_v2_spec` §・`GameConfig.burstP` 本番10%の独立ロール）と確率帯・可視化作法を揃えた**独立低確率ロール**。発火率は【仮】8〜12%。
- **上限**：連発で"書けた夜"の重みが消えないよう、**同一キャリアで多くて2〜3回**。上限カウンタは新規の永続フィールド（下記§実装4）。
- **状態フック（文面の出し分け・任意）**：`次の大会まで`が近い週はAの文に「間に合わせる」含み、体力が細い週（`injuryThreshold`帯）はBを谷口が推す等、既存の状態帯（週／大会前／体力）で味付け可。MVP（1年48週）でも `ネタ作り` は初週から選べるため**MVP発火可**。
- **⚠️ この抽選ロールを run 内に置くと新規の乱数ドローになり、`resolveAction` の消費順（体力ゲート→体調ダウン判定→`applyTraining`→`rollOffer`）に1本割り込む**。golden に影響する（→§実装・§リスクで詳述、"止めて相談"）。

## イベント本文

**セットアップ（地の文・俺視点・標準語）**

> 深夜二時。この稽古場の机は脚が一本だけ短くて、前に使っていたコンビが折り畳んだ大会のフライヤーを噛ませてある。谷口が書く手に力が入るたび、そのフライヤーが一ミリずつ迫り出してくる。
> 一本、まとまった。谷口が最後の一行を、誰もいない客席に向かって小さく声に出して読んだ。読み終えて、ペンを置いた。谷口が声に出して読むのは、できたと思ったときだけだ。

谷口「……できたな、これ。久しぶりに、腹の底が動いた」

俺「今からもう一周するか。それとも、寝かせるか」

谷口「寝かせて腐るネタは、もともと腐っとる。——って先輩が言うてた。俺は今すぐやりたい派やけどな」

【効果】発想+2・表現+1／体力-15（徹夜）【仮】（メンタル増減なし）

> 俺「じゃあ、もう一周だ。フライヤーが机から迫り出しきる前に終わる、で計っておく」

## どこから学んだか（構造の出典）

- **練習後の派生イベント（確定進行の合間に差すランダム彩り）＋"当たり"抽選**：パワプロ「サクセス」の練習後イベント構造。行動に低確率で物語の分岐を付け、同じ練習に表情を持たせる型。
- **「一晩寝かせる」＝インキュベーション効果**（時間を置くと客観が戻り解の質が上がる／即時 vs 保留のトレードオフ）。B選択の「翌週の合わせが効く」の下敷き。
- **独立低確率ロールの前例**：本作正典の`ハマった夜`（本番10%で独立加算・"見える運"）。`canonical_v2_spec` §「ハマった夜」。抽選の置き方・確率帯・可視化の作法をここに揃える（新機構を足さない）。
- **翌週バフの前例**：`event_design_v0` #6「スベり倒した夜→翌週稽古効果1.5倍」。B効果はこの"翌週限定バフ"と同じ既存パターンに載せる（新種の状態を増やさない）。

学んだのは構造・型のみ。既存作品のセリフ・ネタ本文は一切引かない。

## 実装イメージ（既存機構に載る形・コードは"提案"）

**⚠️ 本件は RNG／golden／乱数消費順に触れる。実装前に必ず止めてオーナー／MVP側と相談（CLAUDE §A）。** 以下は貼れる形ではなく設計の骨子。

1. **抽選ロールの位置（＝正典追加）**。発火判定は独立ロール1本。`WeekRunner.resolveAction` の中、**体調ダウン判定の後・`applyTraining` が `true` を返した直後**に置くのが自然（"稽古が成立してから当たりを引く"）。だが**これは新規ドローで、既存の消費順（体調ダウンロール → `applyTraining` → `rollOffer`…）に1本割り込む**＝`golden` が全崩れする。よって手順は`tools`正典（`canon_v2.py`／`sim_career.py`／`gen_golden.py`）を**先に**同じ位置・同じ確率で直す→`GameCore`同期→golden再生成→`swift test` green を**1コミットで揃える**（ルール5）。**この同期を伴わない実装は不可＝止めて相談**。ロールを `action==.ネタ作り` の週だけ引く条件付きにするなら、golden 固定policyに `ネタ作り` 週を必ず含めてカバレッジを確保する（消費順の分岐は golden で踏まないと守れない）。
   - 代替案（golden非改変で先行できる路線）：**抽選も効果もGameCoreの run 内に入れず、`GameSession`（UI層）側で `ネタ作り` を選んだ週にだけ独自RNG（`runner`のRNGを呼ばない別シード）で判定し、効果は選択後に `GameState` へ加算する**。`justLost`提案（0003）と同じ"UI層は`runner`のRNGを消費しない＝golden不変"の型。ただしこの場合**効果のステ変動は sim_career.py の実測（バランス）には乗らない**＝バランス検証外の彩り扱いになる。**どちらを正典に入れるかはオーナー判断**（`event_design` の突発イベントを sim に織り込むか否かの未決に接続）。
2. **効果の適用**。A＝発想+2・表現+1・体力-15、B＝体力+5・メンタル+1＋翌週バフ。ステ加算は既存 `GameEngine.add(...)` を通す。**注意：`add()` は能力加算を `growthBudget`／`growthDecayD` に通す**（`GameEngine.swift:24-` 実測）ため、A/B の能力ボーナスは**名目値であって実効値ではない**（キャリア後半ほど budget/decay で目減りする＝"+2"が体感で薄まる）。sim では実効増分で評価する。メンタルだけは予算・減衰の対象外で名目どおり乗る。
3. **B の翌週バフ**。`event_design_v0` #6 と同型の"翌週限定・稽古効果補正"。既存に汎用の翌週バフ枠が無ければ、`GameState` に `nextWeekTrainingBonus: (Training, Double)?` 相当を1フィールド新設し、`applyTraining` 冒頭で対象一致時に係数を掛けて**適用後クリア**する。永続化（セーブ）と失効タイミングは既存の週送りに合わせる。**数値がsim/goldenに乗るなら正典同期対象**。※対象を `ネタ合わせ` 限定にするか、#6同様「翌週の稽古全般」にするかは balance に効く（レッドチームA-2）。
4. **上限カウンタ**。「同一キャリア2〜3回」を守る発火回数カウンタは新規の永続フィールド。UI層先行案なら `GameSession`（0003の `sameWallStreak` と同じ置き場）に、run内案なら `GameState` に持つ。前者は golden 非対象、後者は正典同期対象。
5. **会話・文面**。イベント本文は既存の会話送り機構（`DialogueData`／`WeekMainView` の表示系）に載せる。選択肢2つ＋選択後の短い2行は `event_design` の「選択肢→効果＋短い会話」形式そのまま。**文面レイヤーだけなら golden 不変**だが、本件は効果が絡むため一体で相談。

## リスク・注意

- **【最重要・止めて相談】golden/乱数消費順**。§実装1の通り、抽選ロールを run 内に置くと消費順が1本ズレて3年ビット一致が壊れる。①`tools`正典→②Swift→③golden再生成→④`swift test` green を1コミットで揃えるか、②のUI層先行案（golden非改変・バランス検証外）を採るか、**オーナー判断**。無同期実装は禁止。
- **【バランス・体感で数値を動かさない／§B・§5】**。A/Bの数値（発想+2等）と発火率8〜12%は全て【仮】。決めるのは体感でなく `sim_career.py` の実測（Aを選び続けた時の体力事故率・Bの翌週バフの寄与）。**まず演出・頻度・テキストで解き、数値は最後**。
- **【正解のない選択を保つ／§2】**。Aの体力-15が軽すぎるとA一択（明白な優位）になり原則崩れ。体調ダウン閾値（`injuryThreshold=20`）との距離で"体力が細い週はBが正解に寄る"よう、-15は閾値を跨がせる幅で調整（simで確認）。逆にBの翌週バフが強すぎてもB一択化。**どちらも一理**を数値で担保する。
- **【頻度・希釈】**。当たりが多発すると"書けた夜"の重みが消える。同一キャリア上限2〜3回・確率は`ハマった夜`同格帯に抑える。
- **【トーン】**。俺に高揚・自己陶酔を持たせない（"腹の底が動いた"の熱は谷口に寄せてある＝声を混ぜない・0009）。谷口の格言は"借り物→本音"の落差1回で、繰り返さない（口癖天丼と別系統）。プロップ「フライヤーを噛ませた机」は本件固有・使い回さない。
- **【固有名・コピペ】**。「先輩」は無名（架空・固有名を出さない）。既存作品のセリフ・実在ネタの引用ゼロ、ボケ本文ゼロを確認済み。

## レッドチーム（A/B）

### A. 実装妥当性

- **A-1【最大の穴・"正解のない選択"が発火窓で崩れうる】**。発火条件は「体調ダウンに落ちなかったネタ作り週」。体調ダウンは `stamina < injuryThreshold(20)` でしか確率ロールされない（`resolveAction` 実測）。つまり**発火はほぼ体力に余裕のある状態に自己選択される**。ところが§2のジレンマは「体力が細い週はB」で成立する設計＝**Bが正解に寄る状態と、イベントが実際に撃てる状態がズレる**。放置するとイベントは常に"体力あり＝A有利"の場でしか出ず、**A一択（明白な最適解）に退化**する恐れ。→ 修正の方向：(i) A の体力-15 を injuryThreshold を跨がせ「今Aを選ぶと翌週の稽古が体調ダウン圏に入る」将来コストとして効かせる、(ii) 発火下限を体力40前後まで下げてB有利帯（体力20〜40）を発火窓に含める、のいずれかを sim で確認。**この確認が取れるまで「二択が拮抗」とは主張しない**。提案本文§2に注記済み。
- **A-2【B効果の対象が狭いと信頼性が落ちる】**。Bの翌週バフを `ネタ合わせ` 限定にすると、翌週に別行動（バイト・休養・大会）を選ぶ週はバフが死ぬ＝Bの期待値が状態依存で目減りし、これも A 寄せの一因。#6 の前例は「翌週の稽古全般」。テーマ的には write→rehearse で `ネタ合わせ` 限定が美しいが、balance 上は「翌週の稽古全般」の方が拮抗を保ちやすい。どちらを採るかは owner/sim。§実装3に注記済み。
- **A-3【能力ボーナスは名目値】**。`add()` が `growthBudget`／`growthDecayD` を通すため「発想+2・表現+1」は後半キャリアで目減りする。sim は名目でなく実効増分で評価すべき。§実装2に追記済み（元案は素の+2として語っており誤読の温床だった）。
- **A-4【消費順の主張は正確】**。「run内ロール＝golden全崩れ」「UI層(GameSession)先行＝runnerのRNG不使用でgolden不変」はいずれも実コード（`resolveAction` の draw 順・0003の先例）と一致。overclaim なし。上限カウンタが新規永続フィールドである点だけ元案が明示していなかったので§実装4を追加。
- **A-5【MVP発火可は正しい】**。`ネタ作り` は初週から選択可・year1で成立。0003 の year8/10 のような多年ゲートは無い。

### B. 会話の中身

- **B-1【一人称"俺"の重複は正典どおり＝OK】**。谷口が「俺は今すぐやりたい派」と言い、地の文の視点も俺。一見衝突に見えるが `dialogue_batch2`（谷口「俺ら…」）`dialogue_design`（谷口「俺は好きやで」）で**谷口の一人称は"俺"が確定**。話者ラベルで区別され混乱しない＝修正不要。
- **B-2【方言・話者分離OK】**。俺＝標準語・計算屋（結論を出さず選択肢を差す）、谷口＝関西弁（腐っとる／言うてた／やけどな／区切るなや／ええわ／かい）。話者ごと1セリフずつの通常送りに整形済み（"／"詰めを解除）。谷口以外の相方は登場せず関西弁の誤用なし。
- **B-3【ドラマ主軸・感情語の回避OK】**。悲しい/悔しい/惨めを地の文に出さず、固有の物（脚の短い机に噛ませた前のコンビのフライヤー／声に出して読むのはできた時だけ）で温度を出す。定番プロップ（天井の染み・冷めたコーヒー）不使用＝0009の3則2に合致。"腹の底が動いた"は感情語でなく身体の場所＝物寄りで許容（batch2が「悔しかった」を薬味に許すのより弱い一滴）。ただし熱は谷口に限定し俺に混ぜない＝自己陶酔回避。
- **B-4【デッドパン一滴・ボケ無しOK】**。乾いた一滴は俺の「フライヤーが迫り出しきる前に終わる、で計っておく」1箇所のみ（真顔の事実並置＝デッドパン、自己言及ゼロ）。谷口のセリフはボケ（オチ/ネタ本文）を出さず earnest に留め、`ボケない` ルールを守る。キザ・サムさなし。
- **B-5【コピペ・酷似なし】**。「寝かせて腐るネタは、もともと腐っとる」は創作の民間知を模した**新作の格言**で、既知の実在セリフの引用・酷似なし。参考作品の本文転載ゼロ（構造・型のみ）。「先輩」は無名の架空。
- **B-6【微調整の余地（任意）】**。谷口の冒頭「できたな、これ」は語尾"な"が谷口の関西語尾（batch: 減ったな／すんねんな）に沿い可。より色を出すなら「できたわ、これ」も可だが過修正になるので現状維持を推奨。
