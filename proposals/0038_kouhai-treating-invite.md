# イベント提案：後輩の店選び v1（2026-07-06起草・2026-07-07 赤チーム統合改訂）

- 位置づけ: 選択肢イベント新設（採番0038・0010〜0023の14本には含まれない新規軸）。**「後輩に奢れるか／奢れないか」を下積み時代に置く**イベント。
- 設計の骨子: **「行く」を選んだとき、その時点の所持金によって結果の"質"が変わる**（十分な金がある→先に会計を通せて面目が立つ／金欠→後輩に先を越されて奢れない＝面目は傷むが、後輩の気遣いに触れて距離が縮む）。「断る」は安全だが小さな引け目が残る。**一択・内部二分岐＋もう一択**の構造。

## 前提（重要）

**本イベントは、現状のMVP範囲では発火できない。** 0017が「単独では発火しない」と明記したのと同じ誠実さで、以下を先に置く（詳細は→ リスク・注意／赤チームA）。

1. **ChoiceEvent機構そのものが未実装**。選択肢イベントをMVPに載せる土台（決定的発火の判定・保留状態・2〜3択オーバーレイUI・決定的パッチのseam）は `proposals/0024` が"実装ブリーフ"として設計しただけで、**MVP側CLIが取り込むまではコードに存在しない**（0024 は「実装指示ではなく地図」と自認）。器がない以上、本イベントも他の0010〜0023も、現状では撃てない。
2. **MVP第1弾に本イベントは含まれない**。0024の発火表の優先順は既存の決定的フラグで撃てる **0018→0019→0017→0010**。本イベント（後輩関係フラグ＋金欠内部分岐）は、後輩絆フラグの配線という**追加の土台**を要するため第2弾以降（0024 の「0011-0016・0020-0023は追加の土台が要る＝第2弾以降」と同じ扱い）。
3. **「既存イベント抽選枠に相乗り」という初稿の前提は成り立たない**。event-rules §3 の一線どおり、**週次ランダム抽選プールそのものが「本編送り（新規RNG消費＝golden破壊）」**として MVP から除外されている。抽選枠は MVP に存在せず、そこへ相乗りする設計は「未実装の機構への依存」だった。→ 本改訂では**発火を決定的フラグに寄せて書き直す**（下記 発火条件）。ただし後輩絆フラグの配線・専用トリガ・新イベント枠の新設そのものは**オーナー相談待ち**（`relationship_champions_v0.md §6-1` の絆しきい値と同じ保留枠・0024の一線）。

**要するに**: 本提案は「器ができた後に載せる1本」の設計図であり、単独で今すぐ動くものではない。実装イメージの見出しも「既存機構に載る形」から実態に合わせて改めた（下記）。

## 狙い

後輩に奢る／奢られる文化は、実際には**売れる前の下積み時代からある**（リサーチで確認：TKO・木本さんは仕事が減っても後輩への奢りを続け借金を抱えた実例。松竹芸能の若手には「先輩は奢るもの」という不文律を経済的に実現できない証言もある）。既存の `relationship_champions_v0.md` は**優勝後**の後輩との関係（K1〜K8）だけを描いており、**下積み中に同じ状況（後輩に誘われる・金がない）が起きたときの選択**が空白だった。本イベントはその空白の一枠を埋める。

- **正解のないトレードオフ**（event-rules §STEP2）: 「行く」を選ぶと、**金がある時は面目が立ち、金欠の時は面目を傷めるが後輩との距離が縮む**——**同じ「行く」でも状態（今の所持金）によって取れるものが変わる**。プレイヤーは"見栄を張れる状態か"を自分で判断してから選ぶ。「断る」は安全だが、先輩らしくできなかった小さな引け目が残る。
- **【初稿からの撤回・赤チームB反映】** 初稿は本イベントを「K3/S8/S2＝恩送りの循環の"原点／種"」と位置づけていた。しかしそれは既存正典の**骨格（恩送りの循環・会計での主導権反転）を再利用する口実**になっていた（B軸1の指摘）。**本改訂ではその位置づけを撤回する**。本イベントは恩送りの循環を描かない・伏線も張らない・谷口にその循環を言わせない。描くのは「下積みの一夜、先輩らしく振る舞えたか／振る舞えなかったか」という**その場かぎりの一場面**に絞る。

## 発火条件（実フック・決定的フラグ寄り）

- **時期**: 非大会週・**週15以降**【仮】（後輩と呼べる存在がいる想定。0015「畳んだコンビの椅子」の"週20以降"と同系の考え方）。
- **トリガ（決定的・抽選しない）**: `relationship_champions_v0.md` の `.kohai`（可愛がってた後輩）の**絆フラグがしきい値を超えている周回**（＝その周回で後輩と絆イベントが起きている）＋週閾値の AND が成立した**最初の非大会自由週に1回だけ確定発火**。**抽選プールに載せない＝新規RNG非消費**（0017の justLost と同じ「確定発火」の規律）。
  - ※後輩絆フラグ・絆しきい値・イベント枠の新設は**オーナー判断待ち**（`relationship_champions_v0.md §6-1`＝絆しきい値はsim後決め・START_HEREのバランス保留枠）。フラグ未配線なら**発火しない**（0017の justLost が未配線フラグなのと同じ前提）。
- **選択肢A（行く）の内部分岐（決定的・乱数を引かない）**: 選択時点の `state.money` を見るだけの if 分岐。
  - **十分な金がある**（`money >= 判定線`【仮】）→ 先に会計を通せる分岐
  - **金欠**（`money < 判定線`【仮】）→ 後輩に先を越されて奢れない分岐
  - この判定は既存state値を読むだけ＝**新規の乱数ロールなし＝golden不変**。0010のA内部ロール（上振れ/下振れ＝**確率的**なので本編送り）とは異なり、本イベントは**確定的**（＝0010より安全側）。
- **選択肢B（断る）**: 分岐なし・単一の結果。

## イベント本文

> トーン＝ドラマ主軸（proposals/0009）。感情語を書かず物・所作に温度を背負わせる。俺＝標準語・計算屋・警句締めなし・熱/誇り/自己憐憫の吐露なし。谷口＝関西弁一貫・本音は言語化しない。後輩は第三者＝標準語・敬語ベース（関西弁を移さない・固有名は架空のコンビ名）。会話は話者ごとに1セリフ（「／」で詰めない）。乾いた一滴は各分岐最大1個。数値・システム語はセリフに出さない。

**セットアップ（地の文・俺視点）**

```
今年入った後輩のコンビから、飲みに誘われた。店は、向こうが決めた。
送られてきた店名に、見覚えがあった。二年前、一度だけ入って、品書きの値段を見て、注文する前に出た店だ。
```

```
谷口「……あの店やんけ。二年前の。」
俺「向こうは、知らずに選んでる。」
```

- **固有プロップ＝「二年前に一度だけ入って、品書きの値段を見て、注文する前に出た店」**（1個）。俺たち固有の履歴に紐づく物で、置換テストを通る（他コンビの物語には移植できない）。**財布・小銭・家賃等の定番プロップは使わない**。後輩がその店を"良かれと思って"選んだことが、俺たちにだけ効く——という構図。

**選択肢A：行く（誘いに乗る）**

**【効果・方向のみ・数値は全て仮】**
- 十分な金がある場合: 所持金↓（先に会計を持つ） ／ メンタル↑（面目が立つ） ／ 相性 ±0
- 金欠の場合: 所持金 ±0（後輩に先を越され、払う機会を失う） ／ メンタル↓↓（奢れなかった引け目＝断るより重い面目の傷） ／ **相性↑（後輩の気遣いに触れ、距離が少し縮む）**

会話（十分な金がある場合・谷口版）:
```
俺「今日は、俺が持つ。」
谷口「……ええんか。」
```
```
（俺は品書きを開かなかった。先に席を立って、会計に回った。）
後輩「ごちそうさまです。」
谷口「よう食うたな、お前ら。」
```
（俺の動機はセリフにしない＝地の文の所作だけで見せる。「品書きを開かなかった」＝二年前に値段を見て出た店で、今日は値段を見ずに払う、という所作の反転。俺に「ケチりたくない」等の誇り／面目の吐露を言わせない＝voice-bible §4 の俺の禁止列。）

会話（金欠の場合・谷口版）:
```
（会計に立ったら、もう済んでいた。後輩が、俺が席を立つより先に、通していた。）
俺「次は、俺に出させてくれ。」
後輩「今日は、俺が誘ったんで。」
```
```
谷口「……座れたな、ここ。」
```
（骨格は初稿から作り替えた。初稿の「会計で俺が言葉を濁す→後輩が"持ちます"と制する→谷口が"すまんな"と詫びる」は、既存 S8「会計で先輩が俺を制する→谷口が沈黙」の**主体を1段ずらした同型シーン**だった＝B軸1。本改訂は「**後輩が先に払い終えていて、俺は払う機会を失う**」＝"制される"のではなく"機会を逃す"別の恥に変えた。後輩「今日は、俺が誘ったんで。」は恩に着せず・気遣いを言語化せず、**事実1つで俺を逃がす**＝E7の救い。谷口の締めは恩送りの循環（「今度はお前らが持て」等）を言わせず、**その場の事実だけ**に絞った——二年前は注文する前に出た店に、今夜は座って食べ終えた、という事実。「座れたな、ここ」がこの分岐の乾いた一滴（デッドパン・自己言及ゼロ）。）

## リスク・注意
