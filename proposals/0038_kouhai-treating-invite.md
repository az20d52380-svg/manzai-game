# イベント提案：後輩の店選び v0（2026-07-06）

- 位置づけ: 選択肢イベント新設（採番0038・0010〜0023の14本には含まれない新規軸）。**「後輩に奢る／奢れない」を下積み時代に置く**イベント。既存の`relationship_champions_v0.md`（K3/S8＝優勝後・後輩に奢る側になった俺）は**後日談**であり、本イベントはその**種**にあたる時系列上の原点。
- オーナー確定の設計：**「行く」を選んでも、その時点の所持金によって結果が変わる**（十分な金がある→奢れて面目が立つ／金欠→後輩に借りる二重の恥）。「断る」は安全だが小さな痛手が残る。三者三様の結果になる一択三分岐構造。
- 数値は全て【仮】。既存ファイルは読むだけ。ボケ（オチ/ネタ本文）は書かない。固有名は架空。

## 狙い

後輩に奢る／奢られる文化は、実際には**売れる前の下積み時代からある**（今回リサーチで確認：TKO・木本さんは仕事が減っても後輩への奢りを続け借金を抱えた実例。松竹芸能の若手には「先輩は奢るもの」という不文律を経済的に実現できない証言もある）。この作品の既存コンテンツ（`relationship_champions_v0.md`）は**優勝後**の後輩との関係だけを描いており、**下積み中に同じ状況（後輩に誘われる・金がない）が起きたときの選択**が空白だった。

- **正解のないトレードオフ**（event-rules §STEP2）: 「行く」を選ぶと**先輩の面目を保てる可能性と、金欠なら二重に恥をかくリスクの両方**を引く。「断る」は安全だが、後輩への申し訳なさと、先輩らしくできなかった小さな引け目が残る。**同じ「行く」でも状態（今の所持金）によって結果の質が変わる**——プレイヤーは"見栄を張れる状態か"を自分で判断してから選ぶ。
- **恩送りの伏線**（リサーチで発見した「バトンを絶やさない使命感」）: 金欠でも無理して行った経験は、後年（優勝後）に後輩へ同じことをする動機の原点になる——ただし本イベントでは**明言しない**（機構的なフラグは作らず、テーマの反響のみ）。

## 発火条件（実フック）

- **時期**: 非大会週・ランダム。**週15以降**【仮】（駆け出し過ぎない時期＝後輩と呼べる存在がいる想定。0015「畳んだコンビの椅子」の"週20以降"と同系の考え方）。
- **選択肢A（行く）の内部分岐（決定的・乱数を引かない）**: 選択時点の`state.money`を見て分岐。
  - **十分な金がある**（`money >= 100,000`【仮】）→ 奢れる分岐
  - **金欠**（`money < 100,000`【仮】）→ 後輩に借りる分岐
  - この判定は既存state値を見るだけの`if`分岐＝**新規の乱数ロールなし＝golden不変**（0010の「上振れ/下振れ」の内部判定と同型だが、0010は確率的、本イベントは**確定的**という違いに注意——本イベントの方がより安全）。
- **選択肢B（断る）**: 分岐なし・単一の結果。
- **MVP発火可**（週15以降なら1年内で到達）。

## イベント本文

> トーン＝ドラマ主軸。感情語を書かず物・所作に温度を背負わせる。俺＝標準語・計算屋・警句締めなし。谷口＝関西弁一貫・本音は言語化しない。後輩は第三者＝標準語（関西弁を移さない・固有名は架空のコンビ名）。会話は話者ごとに1セリフ（「／」で詰めない）。乾いた一滴（自嘲orデッドパン）は各分岐最大1個。数値・システム語はセリフに出さない。

**セットアップ（地の文・俺視点）**

```
今年入った後輩のコンビから、飲みに誘われた。「一回、ゆっくり話したくて」と言っていた。
店は、向こうが決めた。地図で調べたら、俺たちが普段入る店より、ひと声、格が上だった。
```

```
谷口「気い遣わせてもうたな、店選びで。」
俺「向こうも、こっちに気を遣ってる。」
```

（固有プロップ＝「後輩が選んだ、俺たちの普段よりひと声格上の店」。財布・小銭等の定番プロップでなく、この誘いに固有の緊張——後輩の気遣いそのものが圧になる、という構図。）

**選択肢A：行く（誘いに乗る）**

**【効果・方向のみ・数値は全て仮】**
- 十分な金がある場合: 所持金↓（会計を持つ） ／ メンタル↑（先輩の面目・後輩への誇り）
- 金欠の場合: 所持金↓（なけなしを出す） ／ メンタル↓↓（後輩に借りる恥。二重の痛手＝断るより悪い結果）

会話（十分な金がある場合・谷口版）:
```
俺「今日は、俺が持つ。」
谷口「……ええんか。」
俺「後輩の前で、そこはケチりたくない。」
```
```
後輩「ありがとうございます、ごちそうさまです。」
谷口「今度はお前らが持て。……って、まだ早いか。」
```

会話（金欠の場合・谷口版）:
```
俺「（会計を見て、財布の中身を数え直す）……悪い。少し、足りない。」
後輩「大丈夫です、俺、持ちます。」
谷口「……すまんな。」
```
```
俺「今度、返す。」
後輩「いいですよ、そんな。」
```
（乾いた一滴：後輩「大丈夫です、俺、持ちます。」の後、地の文なしで俺の「今度、返す。」を即置く——謝罪や自己弁護の一文を挟まず段取りだけで処理する。後輩は恩に着せる言葉を一切足さない＝E7の「救い」をここに置く。感情語・警句なし。）

---

**選択肢B：断る**

**【効果・方向のみ】** メンタル↓（小・気まずさと申し訳なさ）／所持金 変化なし

会話（谷口版）:
```
俺「今日は行けない。悪いが、また誘ってくれ。」
谷口「ほな、俺も今日はやめとくわ。」
```
```
後輩「わかりました、また今度で。」
```

**相方版（谷口以外・標準語ベース）**: 谷口の行を「わかった。俺もその日は空けとく」等、当該相方の口調（`character-voices.md`§2）へ差し替え。今回は谷口版のみ完成、相方版は後工程で個別化が要る（E6：汎用テンプレの流用不可）。

## 実装イメージ（既存機構に載る形・コードは"提案"・golden影響は要確認だが安全側）

```swift
// 【提案・未適用】event:0038。決定的分岐のみ・乱数非消費。
// 発火: 週15以降・非大会週・ランダム（既存イベント抽選枠に相乗り、他イベントとの枠競合はEVENT_FIRE_CAPの予算内で管理）。
// 選択肢Aの内部分岐は既存state.moneyを読むだけのif文＝新規乱数ロールなし。
struct KouhaiTreatingInvite {
    static let moneyThreshold = 100_000  // 【仮】十分な金の判定線。sim実測で確定。
    // A・十分な金: money -= (会計額【仮】) / mental += (量【仮】)
    // A・金欠: money -= (手持ち全部【仮】・僅少) / mental -= (量【仮】・Bより大きい下げ)
    // B: mental -= (量【仮】・Aの金欠時より小さい下げ)
}
```

- **golden**: 発火は決定的フラグ（週・ランダム抽選枠）、選択肢A内部の分岐も既存`state.money`を読むだけの決定的if＝**新規の乱数ロールなし＝golden不変**。抽選プール自体に本イベントを1枚追加する行為（既存の12%判定枠への追加）は、他イベントとの「同じ乱数値で選ばれるイベントが変わる」可能性がある——これは0011等の既存提案と同じ論点で、**イベント枠の新設自体が要る限り一度止めて相談**（`proposals/0024`の一線と同じ）。
- **バランス**: 「金欠で行く」を「断る」より明確に悪い結果にする設計（E1的には状態依存で最適解が変わる——金がある時はAが有利、金欠の時はBが有利、という状況依存の非対称）。数値確定前に`sim_career.py`で金欠帯の発火頻度・メンタルの動きを実測すること。

## リスク・注意

- **【要確認】選択肢Aが金欠時に一方的に不利＝「トラップ選択」であることの妥当性**。event-rules E1は「上位互換を作らない」だが、本イベントは状態（今の所持金）によって最適解が変わる設計＝**固定状態で見れば金欠時のAは劣化選択**。これは意図的（見栄を張ると悪い目にあう、というテーマの体現）だが、プレイヤーが理不尽と感じないよう、B選択肢の効果を「小さな下げ」に留め、A金欠時との差を十分に開けておくこと（sim実測で確認）。
- **【E7救い】** A金欠分岐の後輩の台詞「大丈夫です、俺、持ちます」「いいですよ、そんな」は、恩に着せる言葉を一切足していない（気遣いの言語化は湿るため、後輩の"何も言わない優しさ"で留める）。
- **相方名の焼き込みなし**（このイベントは`会話.*`/`event:*`系＝谷口版と相方版の両建て前提。心の声プールには一切書かない）。
- **既存`relationship_champions`との整合**：本イベントは後日談の"原点"として設計したが、機構的な連動（フラグを持たせて優勝後イベントの文言を変える等）は作っていない（テーマの反響のみ）。連動させたい場合は別途提案・相談。
- **多エージェント赤チーム未実施**：予算配慮のため単独ドラフト＋自己採点（A表16項目・E表7項目）のみで仕上げている。E1（トレードオフの妥当性）は上記の通り要確認フラグを立てた。他は自己採点で○（感情語なし・警句締めなし・方言ロック正しい・話者ごと1セリフ・技法1つ・数値/システム語非露出・「笑」不使用・既存作品との酷似なし）。**採用前に実機目視で"金欠時の借りる展開"が寒くないか必ず確認すること**。
