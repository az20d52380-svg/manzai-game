## 1. 結論（総合判定）

## 3. golden安全性の評価

- **発行ゲート化の規律A適合**: 適合。「数式/消費順を変えたら tools正典→GameCore→gen_golden再生成→swift test green を1コミット」（規律A）に対し、Fableの「経済導入と乗りを同一再生成サイクルに束ねる」判断は正しい。乗り(10a)を後入れすると golden再生成がもう1周増える（設問7c/10a-iiiの指摘は正しい）ため、**[会計移設＋色台帳＋同色1.0＋乗り]を1コミット（golden1回）を強く推奨**。
- **★「goldenが割り振り方針に依存する」新依存のリスクと対処**: これは**本設計が導入する最も重い新依存**。従来の golden は「稽古PATTERNを踏むと状態が決まる」決定論だったが、割り振りが入ると**期待値の定義に「どう割り振るか」が要る**。対処＝Fable指摘の通り**割り振り行動を決定論的に台本化**する。ただし台本は**3系統に同一手順を差す**必要がある: `gen_golden.py:50` の PATTERN、`CareerGoldenTests` の GoldenPolicy、`WeekRunner` 駆動。この3箇所のドリフトが新たな golden毒源になるため、**割り振り台本を単一の純関数（sim正典＝Python）にして3系統が同じ関数を呼ぶ**構造を4.8に要求せよ。
- **⑥乗りdrawの消費順への影響と挿入位置**: `gen_golden.py` の稽古週は現状**0draw**。乗りは稽古解決冒頭で**必ず1draw**（状態非依存＝perform「必ず2draw」と同じ規律）を挿すため、**以降のSplitMix64消費が全ズレ→3年ビット全崩壊**。挿入位置は「アクション解決→既存の事後ロール（体調ダウン等）」の順が推奨だが、**バイト単位の週内相対順は4.8が既存コードの実順を確認して正典に固定する**こと（設計もそう申し送っている——確定要）。draw数固定の grepゲート（稽古経路の rng 接触が正確に1回）を追加せよ。
- **実装の順序（1コミットに束ねるもの）**: **1コミット目＝会計移設（予算段を全経路から外し発行時へ）＋色台帳＋同色1.0発行/支出＋乗りdraw＋golden全面再生成。** 会計移設は `GameEngine.add` / `balance_sim.add` / `WeekRunner`稽古解決 / `sim_career`稽古実行 / `gen_golden`稽古実行 の**同時改修**を要する（規律A最重量）。η・イベントXPは後続コミットへ分離可（下記§5）。

## 4. 設計論理のリスク（R3が見つけた穴＋Fable自認の破綻条件）

Fableが「正直に書く」と宣言した破綻条件（η≥0.85で単一プール化・器満杯溢れ・状態依存draw）は概ね誠実。**しかし宣言の外に、実コード接地で major級の穴が3つ残る。結論に反映し、無条件断言を撤回させる。**

- **[major] 「裁定インフレは構造的に不成立」は overstated → η=0.5数値依存の不変条件**: 不成立の真因は「往復禁止」ではなく **η ≤ min_over_pairs(w_src/w_tgt) ＝ 0.5（=w華/wセ）というηの数値配置**。査定インフレは1回の購入変換で起こりうる種類のもので、往復禁止は残高→残高ループを塞ぐだけ。**η=0.55に緩めた瞬間、華→センス経路が査定利得を生む。**臨界は設計書が§4で書く0.85ではなく0.5。→ **ブリーフ指示: 「裁定不成立は数値不変条件（η≤各色対のw比の最小値=0.5）による」と明記。ηスイープ上限は厳密に0.5未満に固定。ηを可変にするなら査定中立線をアサーションで守れ。**
- **[major] 「貯めた残高は必ず使える」は overstated → 予算壁限定・cap近傍は価格発散で凍結**: `add()` の逓減は支出時に残り、価格=ceil(10/(1−v/120)) は v→120 で発散（v=110で120粒/pt・v=119で四桁台=約1200粒/pt。有理算では厳密1200、素朴なfloat実装だと丸め誤差で1201に振れる。ここは桁の発散を示す例なので値の一意性は問わない——他の granularity 数値15/20/120/131は実算ビット一致を確認済み）。**予算壁では無効化されないが、cap近傍能力向けの色残高は価格発散で実質凍結する**。唯一の逃げ道はη=0.5変換＝査定損の強制。これは設計書の自認破綻条件（jitter溢れ・器満杯溢れ）の**外にある未カバー端例**で、オーナー未決5（リセット無し恒久確定）と併せると**凍結色は死に資産**になる。→ **「貯めた色が全リーチ能力cap近傍で凍結」時の正直表示（「この色はもう注ぎ先がない」）を器満杯文言と同格でMVP必須に追加。**
- **[major] 「二層は別種の意思決定として立つ」は overstated → 能動プレイヤー限定・同色多数派は一層に縮退**: 退化ケース（全員同色1.0）では「ネタ作りを踏む＝発想を伸ばす」で割り振り層は追加意思決定を生まず1タップの手間だけ増える。分離は銀行/変換/照準を能動使用する層でのみ発生する**条件付き**で構造的ではない。到達率アンカーの多数派（のんびり23.1%は「おすすめ注ぎ」＝自動配分＝同色一直線）にとって二層は意思決定でなくUI摩擦。**下限保証（現状を下回らない）は sound だが、「希薄化を解消した」は主張させない。** → **§7-3への応答は下限保証に限定させる。二層維持の必要条件＝逆算ゲージ/要求帯が同色パスでも「今注ぐか貯めるか」の期日判断を立たせること（さもなくば割り振り画面はのんびり型に純摩擦）。**
- **[major/risky] 「確定額＝見積もり合計と1粒も違わない誠実市場」は未証明（価格粒度未定）**: 逓減サンプル粒度（能力+1点あたり1回のceilか・0.1素点=1粒刻みで10回合算か）が未指定。両者は乖離（v=40で15粒 vs 20粒、v=110で120 vs 131・最大~10%）。**ceilの二重適用（粒ごとceilしてから合算）は必ず単発ceilを上回り、逐次表示の総和>確定額＝「見積もりより安く確定」の逆詐称すら起こりうる。** → **§6ブロッカー: 「価格の逓減サンプル粒度＝能力+1点あたり1回のceil」を tools正典に一意固定。見積もり関数と `add()` の丸め整合はgolden毒なので同一純関数をUI/正典/simで共有。**

**sound（採用可）**: 負帰還内蔵（積み上げ供給ほど自分の価格が上がる自己抑制は実在）・上振れ主因=照準力限定・メンタル一方通行壁の会計論理は verified/sound。ただし2つの但し書き——(a) 「バランス型は割り振りでも上がらない」は体調要因のみ根拠で、割り振り効率の底上げ分を見落とす（「体調管理を解かない範囲で」と限定）。(b) メンタル色は予算外で無制限発行されるため、**器ガードは jitsuryoku(査定)供給の上限を握るがメンタル安定性(勝率寄与)は器の外で青天井**——「総成長≤f(器)」は査定限定で真。メンタル過剰蓄積が41.5%帯を上振れさせないかを sim較正対象に追加せよ。

**Fable自認の破綻条件（再掲・妥当）**: η≥0.85で単一プール化／η≤0.3で変換死・料金所化／振れ幅過大で稽古のくじ引き化／状態依存drawで3年ビット全崩壊／価格側への乱数漏出で確定性喪失／乗り×器クランプの溢れ（稀・正直表示で許容）。
