# manzai-judge-comments — 審査講評を書く

講評は"笑いの質"を言葉で伝える**唯一の窓**であり、「フィードバックのある賞レース」という願望充足がこのゲームの隠れた価値。だから講評だけは例外的に、**「オチ/ボケ/テンポ」を批評語として使ってよい**（ネタ本文の禁止はそのまま——批評はしても、ボケの中身は書かない）。

## 手順

### STEP 0 — 前口上

manzai-drama-voice の基調3行に加え、講評固有の1行を自分の言葉で出力する:
「辛口は場所を指せば愛になり、指さなければ暴言になる。人格・才能・容姿には一生触れない。」

### STEP 1 — 枠を特定する

- **予選紙講評**（審査員名非表示・紙の1行）か、**決勝コメント**（顔付き7審査員・署名あり）か。
- スコア帯: E(〜49)/D(50-69)/C(70-84)/B(85-94)/A(95〜)。`final.B`=好調・褒め主体、`final.D`=厳しめ。**B帯が最大の枯渇ホットスポット**——増やすならまずB。
- 状態フック（最大1つ発火）: 体力<30（後半失速）／メンタル<40（緊張・目泳ぎ）／相性≥18（掛け合い称賛）が実装済み。華≥90／敗者復活経由／同回戦3年連続は**設計のみ・コード未実装**（正本 judge_design_v0）。**依頼にフック指定がなければ、フックなしの一般帯として書く**（勝手に状態を仮定しない）。
- 実装先はUI層（JudgeData.swift のプール）。選定はsalt剰余＝乱数非消費。

### STEP 2 — 審査員を選び、人格を読む

`references/judge-personas.md` で当該審査員の 軸・口調・点数癖・型配分（T1〜T7）・実例 を読む。**講評文は審査員のキャラも同時に立てる**——同じ指摘でも7人で言い方が全部違うのが正解。誰の声か決めずに書き始めない。

### STEP 3 — 書く（講評の型）

1. **辛口（D/E帯・fail）は必ずT1（具体箇所）を含む**: 「どこが」を時刻・位置・構造で指す（「最初の60秒でボケ4つ、うち3つが同じ構造」）。処方は1個・押し付けない。
2. **優しい（B帯・pass）は必ず「次の一歩」か「具体的な褒め所」を含む**。空の全肯定はA帯のみ。
3. 型は 褒め→反転→直せる一点 が基本骨格（人格により崩す）。
4. 身体反応でshowする: 「面白い」と評さず、客席・審査員の身体反応を書く。ただし**意味が一義に立つ反応を選ぶ**——「ペンを置かなかった」単体は感心とも粗探しとも読める両義。両義になるなら意味を確定する一語を添えるか、別の反応にする。
5. 帯と温度を一致させる: 短い辛口コメント＋高得点（またはその逆）は点数との乖離が炎上の火種。

### STEP 4 — 採点ゲート（必須）

manzai-drama-voice `references/self-check.md` のA表とB表（複数本ならB3のGrep照合・証跡付き）を走らせ、さらにJ表を走らせて結果を出力する。×→手筋→再採点。2回で直らなければ没。
**話者依存の読み替え**（self-check「話者による判定差」参照）: A7の1点フィニッシュ・A11のユーモア2種限定は審査員には適用せず、人格の型（T1〜T7）との一致をJ6で裁く。A13の数値禁止は**ゲーム内部数値**（能力値・表示点）のみ——舞台上の尺・秒数・ボケ数（「最初の60秒でボケ4つ」）はT1/T4の道具として可（J8と同義）。

## J表 — 講評固有の○×

| # | チェック | 手筋 |
|---|---|---|
| J1 | 辛口に「直せる一点（具体箇所）」がある | どこが・いつ・何が、を1個足す。足せない辛口は没 |
| J2 | 人格否定・才能否定（「センスがない」）・容姿言及がゼロ | その節を削り、場所の指摘に置き換える |
| J3 | 「笑」の字ゼロ（代替: ウケる/沸く/客席が揺れる。諺・「笑顔」等の例外域は可） | 置換 |
| J4 | 実在審査員チェック: 見た人が特定の実在審査員*その人*を思い浮かべない（口調・経歴・決め台詞の借用なし） | 実在の言い回しを消し、personas.mdの架空人格の型で書き直す |
| J5 | ネタ本文を書いていない（批評語「オチ/ボケ/テンポ」はOK、ボケの中身の引用・再現はNG） | 中身を構造の指摘に抽象化する（「同じ構造が3回」等） |
| J6 | 話者の人格・口調・型配分（T1〜T7）と一致している | personas.mdと突き合わせて口調を直すか、話者を替える |
| J7 | 帯と温度が一致している（B帯なのに説教・D帯なのに絶賛になっていない） | 帯に合わせて温度を平行移動する |
| J8 | 数値・システム名の口出しなし（表示点・能力名の生値を言わせない。能力の「様子」は可: 「声の圧が落ちた」） | 数値を身体反応・時刻・位置の描写に翻訳する |
| J9 | **フック領土の不侵食**: 一般帯（フック非発火）のコメントが、フック専用主題（呼吸・掛け合い=相性／後半失速・声の圧=体力／震え・目線=メンタル／立ち姿の圧=華）を**主軸にしていない**（一般プールがフック主題を先食いすると、フック発火時の特別感が死ぬ） | 主題をその審査員の軸の別の切り口へ移す |
| J10 | **合否の先取り禁止**: 決勝コメントは結果発表前の発話。「来年」「今年は無理」等、**その年の合否・順位を確定させる語がない**（B帯は優勝もあり得る帯——優勝した年に「来年」が出たら破綻する） | 時制を「今日の4分」の中に戻す |
