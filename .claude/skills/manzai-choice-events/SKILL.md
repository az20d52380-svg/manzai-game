# manzai-choice-events — 選択肢イベントを設計する

イベントは「テキスト」と「構造」の二層でできている。テキストの声は **manzai-drama-voice** の管轄（このSkillはそれを前提に呼ぶ）。このSkillが足すのは**構造の手順**——トレードオフの幾何・効果の書き方・golden安全だ。

## 手順

### STEP 0 — 前口上

manzai-drama-voice の STEP 0（基調3行の自分の言葉での要約）を実行し、さらにイベント固有の1行を足す:
「選択肢は正解のないトレードオフ。効果は方向だけ言葉に噛ませ、数値は一切書かない。乱数を引く構造は書かずに止める。」

### STEP 1 — 台帳の確認

`references/event-rules.md` §1 の台帳（0010〜0023・全14本）を確認する。
- **既存イベントの追記**（相方版プール等）なら、その採番の `proposals/00XX_*.md` 原本を読む。成果物は**その原本への追記提案**として出し、勝手に本体コードへ入れない。構造（効果の幾何）は確定済みなので STEP 2 は「継承・変更なし」の確認だけでよい。
- **相方版を書くときは、setup地の文の相方固有要素（プロップ・癖・呼び名・俺の返しの前提）を必ず点検する**。谷口固有の要素が残って矛盾するなら、**相方版setup（地の文＋導入の掛け合い）も同プールの必須成果物**——分岐セリフだけの出荷は不可。
- **新設**なら: 発火条件が既存と被らないか台帳で確認し、採番は次の空き番号を `ls proposals/` で確認。新設は提案（proposals/）として書き、勝手に本体へ入れない。

### STEP 2 — トレードオフの幾何を先に設計する（テキストより先）

1. 各選択肢の効果を「方向」で書く（例: A=知名度↑↑／体力↓・次戦準備減）。数値は書かない【仮】。
2. **上位互換を作らない**: 各選択肢が**別の軸**（能力/体力/メンタル/相性/金/知名度）を取り合うことを確認する。どれかが「全部得」なら設計からやり直す。
3. どの選択肢にも代償があるか。ネガ寄りの分岐には**小さな救いか回収の余地**を残す（理不尽で終わらせない）。
4. 発火は**決定的フラグ**（justPassed・lossStreak・weeksLeft・所持金等）で書く。

### STEP 3 — golden安全の確認（一線）

- 発火判定に**抽選を入れない**・効果で**サイコロを振らない**。この2条件を守る限り乱数列は1ビットも動かない＝golden安全。
- 「弱い能力を上げる」等の対象選択は乱数でなく**決定的min判定**で書く。
- ランダム抽選プール・効果内乱数ロールが必要な設計になったら、**テキストを書かずに止めてオーナーに相談する**（乱数消費順は正典。CLAUDE.md §A）。

### STEP 4 — テキストを書く

書式（`references/event-rules.md` §2）:
- **setup** = 俺視点の地の文2〜4行・固有プロップ1個 ＋ 導入会話（話者ごと1セリフ）。
- **各分岐（.A/.B/.C）** = 選んだ後の会話、俺1セリフ＋相方1セリフ程度。**選んだ側の手触りだけ置き、勝ち負けを断じない**。
- 谷口版（関西弁）と相方版（標準語）は**別プール**で書く。第三者が入るイベントは標準語（関西弁を移さない）。
- 乾いた一滴（自嘲orデッドパン）は各分岐最大1個。
- **禁止**: 選択肢の音読（「締めるか、乗るか」＝メニュー臭。二択はボタンに委ねる）／効果の数値・システム名の口出し。

### STEP 5 — 採点ゲート（必須）

1. manzai-drama-voice の `references/self-check.md` A表を**全行に**走らせる（結果を出力）。
2. 同ファイルの**B表（並び検査）も必ず走らせる**——特にB3（名詞・動詞のGrep照合・証跡貼付）とB5（口癖）。**イベントは反復発火する**（0018はjustPassedごとに年複数回）ので、分岐セリフは「毎年複数回引かれる並び」として一滴・口癖の分母を数える。
3. このSkillのE表（下記）を走らせる。×→手筋→再採点。2回直して残る行は没。

## E表 — イベント固有の○×

| # | チェック | 手筋 |
|---|---|---|
| E1 | 各選択肢は別の軸を取り合い、上位互換がない | 効果の方向を組み替える（0017の幾何=A能力を買う/B体力を戻す/C関係軸、が見本） |
| E2 | 効果は「方向」だけで、数値・式・確率をどこにも書いていない | 数値を消し【仮】に倒す。バランスはsim/GameConfigの管轄 |
| E3 | 発火は決定的フラグのみ・効果に乱数なし | 乱数が要る設計なら**止めてオーナー相談**（書き進めない） |
| E4 | 選択肢の音読がない・分岐会話は手触りだけで勝ち負けを断じない | メニュー復唱を消す。正解臭のする分岐のセリフを等温に均す |
| E5 | 固有プロップはイベント全体で1個・置換テストを通る。**置換テストは「いま書いている相方」を前提に走らせる**（谷口の癖に紐づく物は早乙女版では×） | `../manzai-drama-voice/references/voice-bible.md` §2の手筋。相方版ではsetupごと差し替える |
| E6 | 谷口版/相方版が別プールで、方言ロックが正しい。相方版は当該相方の口癖・芯（`../manzai-drama-voice/references/slots.md` §5）で個別化されている（汎用テンプレの流用で済ませない） | slots.md §5の声プロファイルで直す |
| E7 | **ネガ分岐**（効果が純減・後味残し側に寄る分岐のみ。例: 0015-A「後味↓」。**対称トレードの二択には適用しない**——どちらもネガでないのが「正解のない二択」）に小さな救いか回収の余地がある | 救いは**効果か事実1つ**で置く（説明しない）。**セリフに気遣い・保護の申し出を後付けして満たさない**（気遣いの言語化は湿る） |
