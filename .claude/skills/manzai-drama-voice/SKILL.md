# manzai-drama-voice — 「声」を手順で書く

このSkillは「良い文章を書け」とは言わない。**判断を全部手順に落としてある**。
自分の文章センスを信用するな。センスがあると思った瞬間の一行が一番寒い。
以下のSTEPを順に踏み、STEP 4の採点ゲートを**全部○になるまで**通せ。近道はない。

## 基調（これだけは頭に入れてから始める）

1. **基調はドラマ主軸**。感情語（悲しい/悔しい/惨め/嬉しい）を書かず、感情が起きた「場所の物」に背負わせる。因果を説明せず事実を並置し、判断は読者に渡す。
2. **笑いはネタ本文でなく地の文にのみ宿る**。ゲームはボケ/フリ/オチ/パンチラインを一文字も書かない。ユーモアは乾いた一滴（自嘲 or デッドパン）を5〜6本に1本以下（天井でありノルマではない）。
3. **声のロック**: 俺=標準語・計算屋、ただしキザ（うまいこと言う警句）と自己憐憫は禁物。熱・執念は俺でなく谷口・支配人へ寄せる。谷口=関西弁で本音は言語化しない。谷口以外の相方・第三者=標準語。

## 手順

### STEP 0 — 前口上（省略禁止）

書き始める前に、上の基調3点を**自分の言葉で3行に要約して出力する**（コピペ不可。自分の言葉で言えない規約は運用できない）。要約がずれていたら `references/voice-bible.md` を読み直す。

### STEP 1 — スロット特定

`references/slots.md` を読み、書く対象のスロットを特定する:
- 固定スロット名⇄コードキー（`default` / `lowStamina` / `verdict.pass` / `final.B` / `event:0017.A` …）
- 話者（俺/谷口/他相方/第三者/審査員/中立ナレーター）
- 発火条件（排他カスケード＝金欠中に連敗台詞は出ない。他の高優先が偽のときに出る前提で書く）
- 尺とトーン指定（平常は「物も一滴も無い平板さが正解」。長尺はE軸のみ）

### STEP 2 — 見本の読み込み

`references/voice-bible.md`（規約の正本）と `references/examples.md` の**該当スロットの良い例/悪い例の対**を読む。悪い例の「なぜ寒いか」タグを先に読むこと——書く前に落とし穴の名前を知っていれば落ちない。
**相方・NPC・ライバルのセリフを書くときは `references/character-voices.md`（人物別ボイス仕様シート）が正本**——その人物の解釈フレーム・リズム・口癖の類型を確認してから書く。

### STEP 3 — 書く（感情→物の変換レシピ）

**優先順位: slots.md のスロット別トーン指定は、この一般レシピに優先する。** `default` など「物も一滴も無い平板さが正解」のスロットでは、下の手順2（固有物の挿入）を踏まずに craft の事実だけで書く（物なしはA表で「○(物なし)」と記す）。

1本ごとに必ずこの順で書く:

1. **感情語を一度書く**（下書き。「悔しい」「不安だ」等、そのまま）。
2. **その感情語を消す**。消した跡に「その瞬間、手/目/物が何をしていたか」を**1つだけ**書く。物は置換テスト（→voice-bible §2）を通った固有物に限る。定番物（コーヒー・蛍光灯・小銭・財布・家賃・染み）は没。
3. **因果を説明しない**。2つの事実を接続詞なしで並置し、差は読者に測らせる。「＝つまり緊張していた」の種明かしは氷山を壊す。
4. **感情のピーク直前で「〜た。」で止める**。その先を書きたくなったら、そこが引き際。

ユーモアを入れる場合は先に分母を数える（このバッチで連続で引かれる並びの何本目か）。5〜6本に1本を超えるなら入れない。入れるなら自嘲 or デッドパン、最長二文、感情の谷の直後には置かない。

### STEP 4 — 採点ゲート（必ず走らせる・全部○まで）

1本書くごとに `references/self-check.md` の○×表（A表）を走らせ、**採点結果を明示的に出力する**。
- ×が出たら、同ファイルの**修復手筋**を適用して書き直し、再採点する。
- **2回直しても×が残る1本は、直さず没にして新しく書く**（「即引く」。延ばして救った1本は必ず寒い）。
- ただし**B3（共食い）・B4（同一構文型）の×は書き直し不可＝必ず没にして別の発想から新作する**（骨格を残した書き直しは下位互換を生むだけ）。

### STEP 5 — バッチ検査（複数本書いたとき）

self-check.md のB表（並び検査）を走らせる: 乾いた一滴の頻度／既存正典行との重複・山場の共食い／同一構文型の連続／切り札型口癖の混入。既存テキストとの照合は Grep で実際に検索する（記憶で済ませない）。
**セリフ（相方・NPC・ライバル）を含むバッチは、character-voices.md §3 のV表（名前隠しテスト等）も必ず走らせる**——話者名を伏せて誰か分からないセリフは声がない＝没。

## ハード上限（交渉不可の数字）

| 項目 | 上限 |
|---|---|
| 乾いた一滴（自嘲orデッドパン） | 連続で引く並び5〜6本に1本以下・1バッチ最大1本・イベント各分岐最大1個・最長二文 |
| 固有物（プロップ） | 1テキストに1個 |
| 技法 | 1テキストに1つ（二重盛りは即没） |
| 会話 | 話者ごとに1セリフずつ送る（「／」で詰めない）・イベントは選択肢ごと俺1＋相方1程度 |
| 地の文 | 1画面2〜4行。長尺は決勝敗退・優勝・年次独白（E軸）のみ解禁 |
| 口癖（切り札型のみ） | 年2回まで・うち1回は感情最大点（日常型口癖は通常頻度で可＝voice-bible §8） |
| 見せ場 | 全体の2割。残り8割は平板が正解（本数は天井でありノルマではない。寒くなりそうなら本数を減らせ） |

## 絶対禁止

- ネタ本文（ボケ/フリ/前提/オチ/パンチライン）の本文化。相方の軽口も対象。例外は審査講評が「オチ/ボケ/テンポ」を**批評語**として使う場合のみ。
- 「笑」の字（代替: ウケる/沸く/客席が揺れる。諺・慣用句・「笑顔」等の例外域は voice-bible §9）。
- 実在の芸人・審査員・大会・既存作品（火花/べしゃり暮らし/M-1実在ネタ等）のセリフのコピペ・酷似。固有名は全て架空。
- 数値の言及（「メンタル+1」等をセリフ/地の文で言わない。効果は"方向"を言葉に噛ませるだけ）。
- **数式・GameConfig・golden・乱数消費順に触れる変更**。テキストはUI層のプール追加のみ。乱数を新規に引く実装が必要になったら、書かずに止めてオーナーに相談する。

## 採点モード（既存原稿のレビュー依頼のとき)

STEP 0→1→2 を踏んだ後、各行に self-check.md のA表を適用し、**self-check.md §C の表形式（6列・タグ付き）で返す**。修復案も書いたらそれ自体をA表で再採点してから提出する。

## 自己テスト（このSkillを初めて使うセッションで一度だけ）

本番の量産に入る前に、"勘のないモデルのつもり"で `default`（平常）を**本番に数えない捨て1本**として試走し、STEP 4の採点結果まで出力する。全部○になってから本番へ進む（本番の1本目を試走扱いにしない）。○にならないのに進むのは、このSkillの使い方として誤り。

## references（必要な時に読む）

- `references/slots.md` — スロット地図・コードキー対応・話者名鑑・大会/回戦名・イベント台帳
- `references/character-voices.md` — 人物別ボイス仕様シート・書き分けレバー・口癖三類型・話者判別ゲート（セリフの正本）

- `references/self-check.md` — ○×採点表と修復手筋（このSkillの心臓）
