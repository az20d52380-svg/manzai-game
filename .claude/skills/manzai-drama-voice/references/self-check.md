# self-check — ○×採点表と修復手筋

> これは種・数値は全て【仮】・最終確定は実機目視。

**運用**: 1本書くごとにA表を走らせ、結果を明示的に出力する（頭の中で済ませない）。×→手筋を適用→再採点。**2回直しても×が残る1本は没にして新しく書く**（延ばして救った1本は必ず寒い）。複数本書いたらB表。

**採点の歯止め**: **全項目○・迷いゼロの採点表は、採点失敗のサイン**（過去の試走で「全○・迷いなし」を自己申告したバッチから×2本が出ている）。各行につき「×にできる最有力候補の項目」を1つ挙げ、なぜ×でないかを一言書いてから○を確定する。迷った箇所は隠さずリストにして納品する。

## A表 — 1本ごとの○×（16項目）

| # | チェック（○の条件） | ×が出たときの手筋 |
|---|---|---|
| A1 | ネタのボケ・フリ・前提・オチ・パンチラインを一文字も本文化していない（観客に見せたら笑いを取りにいく台詞になっていない。相方の軽口も対象） | **手筋なし＝その行は没**。反応・観測・言い回しだけで書き直す。講評の「オチ/ボケ/テンポ」批評語は例外で○ |
| A2 | 感情語（悲しい/悔しい/惨め/嬉しい/不安/緊張/気まずい 等）を名指ししていない | 感情語を消し、その瞬間の手/目/物が何をしていたかを1つ書く |
| A3 | 種明かしがない（事実の並置のあとに「つまり」「〜ということだ」等の意味の解説を足していない） | 解説の文を丸ごと削る。並置だけ残す |
| A4 | 物が意味ありげすぎない（「手が震えていた」「ネタ帳を握ったまま」級の感情語の変装・空気の名指しになっていない）。**因果が読者に判読できない演出的所作（意味ありげな不可解行動）も×**——「判断を読者に渡す」と「謎かけ」は別物 | 物を、意味を持たない実務の所作に差し替える（誰がどう扱ったか、だけ） |
| A5 | 物は置換テストを通る固有物か（他コンビの物語に移植して成立しない）。定番（コーヒー・蛍光灯・小銭・財布・家賃・染み・雨）でない。**物が無い行は「○(物なし)」と記して通過**（平板は正解） | その稽古場/相方/客席の履歴が残る物に差し替える。思いつかなければ**物なしの平板**にする |
| A6 | 物は1個以下 | 履歴の濃い方を残して他を消す |
| A7 | 俺の行が警句/教訓/アフォリズム/「うまいこと言った」で締まっていない | 警句を消し、最後の文を平常の事実か段取りに戻す |
| A8 | 俺に自己憐憫・熱・執念・誇りの吐露が漏れていない | 事実処理・勘定に書き換える。熱がどうしても要る場面なら谷口/支配人のセリフへ移管する |
| A9 | 方言ロック: 地の文=標準語／谷口=関西弁／他相方・第三者=標準語（例外は二階堂・支配人・夜行列車型・苦労人先輩型のみ） | 該当話者の声プロファイル（slots.md §5）に合わせて直す |
| A10 | 会話は話者ごとに1セリフ（「／」で詰めていない・1画面に2話者を積んでいない） | 1エントリ=1話者1セリフに分割する |
| A11 | ユーモアが入っている場合: 自嘲orデッドパンである・最長二文・感情の谷の直後でない・自己註釈（「情けない話だが」等）がない。**一滴か平板か迷ったら一滴としてカウントする（安全側）** | 位置を平地に移す／二文に切る／註釈を消す。それでも湿るなら一滴ごと消して平板にする |
| A12 | 技法は1つ（誇張＋擬人化などの二重盛りでない） | 効いている方を残し、もう片方を平常の記述に戻す |
| A13 | 説明台詞・数値の復唱・システム名・選択肢の音読がない（「メンタル+1」「台帳がそう言っている」「締めるか、乗るか」）。**禁止はゲーム内部の数値（能力値・表示点・効果量）。舞台上の尺・秒数・客数など世界内の数字（「最初の60秒でボケ4つ」「客は六人」）は可** | 削除。内部数値は身体反応・時刻・位置の描写に翻訳する。二択はボタンに委ねる |
| A14 | 尺: 地の文2〜4行・講評/心の声は短文。長尺はE軸（決勝敗退・優勝・年次独白）のみ。「笑」の字ゼロ（例外域除く） | 削る。「笑」は ウケる/沸く/客席が揺れる に置換 |
| A15 | 冗長・同義反復がない: 同じ内容を言い換えて二度言っていない。評価語（良かった・無駄がなかった・悪くない等）だけで構成されず、**craftの事実か所作が最低1つある**。体言止め・断片文の連発（電文調・ポエム臭）になっていない | 二文のうち事実側だけを残す。事実が無ければ没。断片文は一箇所だけ普通の文に崩す |
| A16 | slots.md の**該当スロットのトーン指定を満たしている**（losing=計算屋の「何が足りないか」が読める／passedLines=余韻＋気を抜くなの自制が読める／default=平板 等） | トーン指定を読み直して書き直す。指定と合わない良文は、合うスロットへ移すか没 |

**話者による判定差**: A7のキザ判定・A11のユーモア2種限定は**俺・相方・第三者の声**に対する規定。審査員の声では人格の型が優先される——1点フィニッシュ（「その精度は凶器になります」級）や比喩・小噺（神楽坂T7・花園T7）は、judge-personas.md の型配分に一致していれば○（J6で裁く）。俺の地の文で同じことをやったら没。

## B表 — バッチ（並び）検査

**バッチの定義**: 同一の執筆回で書いた全本（スロットをまたいでも1バッチ）。B4の「並び」は同一スロット内で数え、B3はスロット・正典を横断して照合する。反復発火するスロット（イベント分岐等・年に複数回引かれる）は「毎年複数回引かれる並び」として分母を数える。

| # | チェック | 手筋 |
|---|---|---|
| B1 | 乾いた一滴は、連続で引かれる並び5〜6本に1本以下・このバッチに最大1本・イベント各分岐最大1個 | 超過分の一滴を**無味のドラマ地の文に書き直す**（削るのでなく平板で埋める。本数を守るために薄い一滴を足すのは逆＝天井でありノルマではない） |
| B2 | 自嘲/デッドパンがバッチ内で偏っていない（自嘲だけ=泣き言モノトーン／デッドパンだけ=達観演技） | 連続バッチ間で種類を入れ替える |
| B3 | 既存正典との共食いがない。手順は2段: **(1) Grep** — 新作の**名詞と動詞**で DialogueData.swift／JudgeData.swift／docs/／proposals/／**.claude/skills/（Skill本文の見本・examples.mdの○例を含む）**を実際にGrepし、**実行したコマンドとヒット行をB3欄に貼る**（証跡のないB3は未実施とみなす。例: `grep -rn '<語>' ManzaiGame/Sources docs proposals .claude/skills`。ヒット行周辺の確認だけならSkill外ファイルを開いてよい）。**(2) パラフレーズ判定** — Grepヒットゼロでも、新作1本ごとに**最も近い既存行（同一スロットの既存プール行・正典確定サンプル・Skill見本）を1本引用**し、モチーフ・文意の距離を判定する（言い換え・骨格の焼き直しはGrepに掛からない）。×の対象: ①山場・レア行・天丼素材（「いけるわ」「腹減ったな」「なんとかなるやろ」近縁）の複製 ②同一カスケード・同一スロットの既存行/頻出行とのビート被り（動詞＋モチーフの一致。例:「N本書いて処分した」骨格） ③確定サンプル・Skill見本のパラフレーズ（「同じところにいる」→「距離は同じまま」型） ④作品内で意味が確立した語の多義衝突（「畳む」=切り上げ/解散 等） | 該当したら**新作側を捨てる**（既存ビートを希釈しない）。骨格を残した書き直しは不可＝別の発想から新作する |
| B4 | 同一構文型が2本並んでいない。**構文型＝骨格**（反転オチ/メタ抑制/「なぜか覚えている」/「AしたがBだった」等）。表層の動詞一致はB4でなくB3で裁く | 片方を没にする（書き直しで逃げない） |
| B5 | 口癖の天丼: このバッチで口癖を使ったのは年2回以内・うち1回は感情最大点に温存されている | 平時の使用を削る |
| B6 | 発火条件との整合: 各行が**断定する事実**（回戦名・経過年数・「今週も」・直前に大会があったこと・季節・相方名）は、**この発火条件下で常に真か**。排他カスケードとの整合: 各行は「他の高優先スロットが偽」の前提で読める（金欠中に出ない行に金の話を書いていない・敗退翌週に出る行に励ましがない） | 状態から保証されない断定を消して、条件だけから真になる書き方に直す。無理ならスロットを移す |
| B7 | 見せ場の割合: バッチの8割は平板（物も一滴もない行が過半を占める） | 山を削って平板に戻す。全行が「良い文」なバッチは失敗 |

## C — 採点モードの出力形式（既存原稿レビュー時）

```
| # | 本文（先頭15字…） | 判定 | ×項目 | タグ | 修復案 |
|---|---|---|---|---|---|
| 1 | 「谷口は寂しそうに…」 | × | A2,A4 | 【感情語】【言い換え】 | （手筋適用後の書き直し） |
```

- タグは examples.md のタグ一覧から。修復案自体もA表で再採点してから提出する。
- 既存実装に許容済みの行（DialogueData/JudgeDataの現行プール）を×にする場合は「既存・置換提案」と明記し、勝手に消さない。

## D — 自己テスト手順（このセッションで初めてA表を走らせる直前に一度）

1. STEP 0の前口上（基調3行の自分の言葉での要約）を出力する。
2. `default`（平常）を**本番に数えない捨て1本**として書く——**わざと物も一滴も入れない平板で**（本番の1本目を試走扱いにしない）。
3. A表の全項目の採点結果を出力する。
4. 全部○になるまで手筋で直す。○が揃ってから本番のスロットへ進む。
5. 迷いが出たら voice-bible §6 の判定表で断定する。判定表にない迷いは、書かずに本数を減らす方に倒す。
